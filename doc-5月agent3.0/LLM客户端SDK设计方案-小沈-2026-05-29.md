## 一、需求说明

### 1.1 业务背景

Agent 系统中，有 3 个模块需要调用 LLM API（OpenAI / Anthropic 等）：

| 模块 | 调用方式 | 当前问题 |
|------|---------|---------|
| **BaseAIService**（主 LLM 调用） | 长连接，单例 | 客户端生命周期管理缺失，替换时不 close |
| **intent_classifier**（意图分类） | 每次请求新建 | 独立创建客户端，不复用 |
| **capability_detector**（能力探测） | 每次探测新建 | 独立创建客户端，不复用 |

**根本问题**：没有一个统一的 LLM 客户端 SDK。每个模块各自创建 httpx.AsyncClient，各自管理生命周期，各自处理超时和错误。

### 1.2 SDK 定位

**LLM 客户端 SDK**，是一个**基础模块**，被其他模块调用：

```
调用方                          SDK
─────────────                ─────────
BaseAIService      ──────→   LLMClientSDK
intent_classifier  ──────→   LLMClientSDK
capability_detector ─────→   LLMClientSDK
```

**核心职责**：

1. **创建 LLM 客户端** — 统一入口，配置集中
2. **发送请求** — 非流式 / 流式 / 带工具调用
3. **管理生命周期** — 创建、复用、替换、关闭

**明确不包括**：

**一、不处理网络工具的 HTTP 请求。** 那是 Network HTTP SDK 的职责。

**二、不处理 Agent 业务逻辑。** 不管 Agent 如何选择模型、如何处理 prompt、如何解析响应。

**三、不做请求缓存。** 不缓存 LLM 响应。

**四、不做请求限流。** 不限制调用频率。

**五、不做负载均衡。** 本地桌面应用，单 provider。

**六、不处理数据库。** 不涉及 DB schema 变更。

### 1.3 使用场景

| 场景 | 调用方 | 触发条件 | 涉及 API |
|------|--------|---------|---------|
| **SC-01 流式调用** | BaseAIService | ReAct 循环每步调用 LLM | `chat_stream()` |
| **SC-02 工具调用** | BaseAIService | LLM 返回 tools 参数 | `chat(tools=[...])` |
| **SC-03 意图分类** | intent_classifier | 用户输入时分类意图 | `chat()` |
| **SC-04 能力探测** | capability_detector | 首次使用时探测模型 FC 能力 | `chat()` |
| **SC-05 客户端替换** | factory | 用户切换模型 | `close()` + 重新创建 |
| **SC-06 应用关闭** | main.py | 服务停止 | `close()` |

### 1.4 功能需求

#### FR-01 创建 LLM 客户端

| 项目 | 规格 |
|------|------|
| **标识** | FR-01 |
| **名称** | 创建 LLM 客户端 |
| **输入** | provider（提供商）、model（模型名）、api_key（API 密钥）、base_url（API 地址，可选）、timeout（超时配置，可选） |
| **输出** | `LLMClient` 实例 |
| **行为** | 创建 httpx.AsyncClient 长连接池（max_connections=10, max_keepalive=5），配置 API Key Header，配置超时 |
| **异常** | 参数无效 → ValueError |
| **使用场景** | SC-01 ~ SC-04 |

#### FR-02 非流式调用

| 项目 | 规格 |
|------|------|
| **标识** | FR-02 |
| **名称** | 非流式 LLM 调用 |
| **输入** | messages（消息列表）、max_tokens（可选）、temperature（可选）、seed（可选）、tools（可选）、tool_choice（可选） |
| **输出** | LLM 响应 dict |
| **行为** | 构建请求体，POST 到 LLM API，返回完整响应 |
| **异常** | 网络错误 → 重试；API 错误 → 返回错误信息 |
| **使用场景** | SC-03, SC-04 |

#### FR-03 流式调用

| 项目 | 规格 |
|------|------|
| **标识** | FR-03 |
| **名称** | 流式 LLM 调用 |
| **输入** | messages（消息列表）、max_tokens（可选）、temperature（可选）、seed（可选）、tools（可选）、tool_choice（可选） |
| **输出** | 异步生成器，逐块 yield SSE 数据 |
| **行为** | 构建请求体（stream=True），POST 到 LLM API，逐块解析 SSE 流 |
| **异常** | 网络错误 → 重试；API 错误 → yield 错误块 |
| **使用场景** | SC-01 |

#### FR-04 关闭客户端

| 项目 | 规格 |
|------|------|
| **标识** | FR-04 |
| **名称** | 关闭 LLM 客户端 |
| **输入** | 无 |
| **输出** | 无 |
| **行为** | 关闭底层 httpx.AsyncClient，释放连接池 |
| **异常** | 无 |
| **使用场景** | SC-05, SC-06 |

#### FR-05 错误处理

| 项目 | 规格 |
|------|------|
| **标识** | FR-05 |
| **名称** | 统一错误处理 |
| **输入** | HTTP 响应或异常 |
| **输出** | 标准化错误 dict |
| **行为** | HTTP 错误 → 返回 `{"code": "ERR_LLM_XXX", "data": {...}, "message": "..."}` 格式的错误 dict；调用方通过 `code.startswith("ERR_LLM_")` 判断是否失败 |
| **异常** | 无（不抛异常，返回错误 dict） |
| **使用场景** | SC-01 ~ SC-04 |

**复用模式**：与系统 `build_error()` 格式完全一致（`code/data/message` 三段式），LLM SDK 内部实现 `build_llm_error()`，不从 tool 层导入。

**LLM 错误码定义**：

| 错误码 | HTTP 状态码 | 含义 | data 字段 |
|--------|-----------|------|----------|
| `ERR_LLM_RATE_LIMIT` | 429 | 限流 | `{"retry_after": 30}` |
| `ERR_LLM_AUTH` | 401 | 认证失败 | `{"provider": "openai"}` |
| `ERR_LLM_FORBIDDEN` | 403 | 权限不足 | `{"provider": "openai"}` |
| `ERR_LLM_NOT_FOUND` | 404 | 模型不存在 | `{"model": "gpt-4"}` |
| `ERR_LLM_VALIDATION` | 422 | 请求格式错误 | `{"detail": "..."}` |
| `ERR_LLM_TIMEOUT` | 408/超时 | 请求超时 | `{"timeout_sec": 60}` |
| `ERR_LLM_SERVER` | 500/502/503/504 | 服务端错误 | `{"status_code": 500}` |
| `ERR_LLM_NETWORK` | 网络异常 | 网络连接失败 | `{"error": "..."}` |
| `ERR_LLM_UNKNOWN` | 其他 | 未知错误 | `{"status_code": xxx}` |

**错误响应格式**（与系统 `build_error()` 格式一致）：

```python
# 成功
{"code": "SUCCESS", "data": {...}, "message": "..."}

# 错误
{"code": "ERR_LLM_RATE_LIMIT", "data": {"retry_after": 30}, "message": "Rate limited: ..."}

# 调用方判断
if response["code"].startswith("ERR_LLM_"):
    # 处理错误
```

### 1.5 非功能需求

| 需求 | 说明 | 指标 |
|------|------|------|
| **连接池复用** | 长连接池，跨调用复用 | max_connections=10, max_keepalive=5 |
| **超时可控** | 统一超时配置，支持细粒度覆盖 | 默认 connect=30s, read=60s, write=10s, pool=10s |
| **生命周期受控** | 创建者负责关闭，替换时先关旧再开新 | 无连接泄漏 |
| **API Key 安全** | API Key 不暴露在日志或错误信息中 | — |
| **零破坏性** | 调用方只需改导入路径，功能不变 | — |

```python
client = create_llm_client(
```

```python
response = await client.chat(
    messages=[
        {"role": "system", "content": "你是一个助手"},
        {"role": "user", "content": "你好"},

    max_tokens=1000,
    temperature=0.7,
)
```

```python
async for chunk in client.chat_stream(
    messages=[
        {"role": "system", "content": "你是一个助手"},
        {"role": "user", "content": "你好"},
    ],
    max_tokens=1000,
):
    print(chunk)  # 逐块输出
```

```python
response = await client.chat(
    messages=[
        {"role": "user", "content": "今天天气怎么样？"},
    ],
    tools=[
        {
            "type": "function",
            "function": {
                "name": "get_weather",
                "description": "获取天气",
                "parameters": {"type": "object", "properties": {"city": {"type": "string"}}},
            },
        }
```

```python
response = await client.chat(messages=[...])

# 成功
if response["code"] == "SUCCESS":
    data = response["data"]

# 错误
if response["code"].startswith("ERR_LLM_"):
    code = response["code"]
    data = response["data"]
    
    if code == "ERR_LLM_RATE_LIMIT":
        retry_after = data.get("retry_after", 30)
        # 等待后重试
    elif code == "ERR_LLM_AUTH":
        # 检查 API Key
    elif code == "ERR_LLM_TIMEOUT":
        # 超时重试
    elif code == "ERR_LLM_SERVER":
        # 服务端错误，等待后重试
```

### 3.2 client_sdk.py 设计

```python
"""
LLM 客户端 SDK
Author: 小沈 - 2026-05-29

基础模块，被 BaseAIService / intent_classifier / capability_detector 调用。
只支持 OpenAI 兼容格式的 API（/chat/completions 端点）。
"""

import httpx
from typing import Any, AsyncGenerator, Dict, List, Optional


# 集中配置
DEFAULT_CONNECT_TIMEOUT = 30.0
DEFAULT_READ_TIMEOUT = 60.0
DEFAULT_WRITE_TIMEOUT = 10.0
DEFAULT_POOL_TIMEOUT = 10.0
DEFAULT_MAX_CONNECTIONS = 10
DEFAULT_MAX_KEEPALIVE = 5


# ===== LLM 错误码 =====

ERR_LLM_RATE_LIMIT = "ERR_LLM_RATE_LIMIT"
ERR_LLM_AUTH = "ERR_LLM_AUTH"
ERR_LLM_FORBIDDEN = "ERR_LLM_FORBIDDEN"
ERR_LLM_NOT_FOUND = "ERR_LLM_NOT_FOUND"
ERR_LLM_VALIDATION = "ERR_LLM_VALIDATION"
ERR_LLM_TIMEOUT = "ERR_LLM_TIMEOUT"
ERR_LLM_SERVER = "ERR_LLM_SERVER"
ERR_LLM_NETWORK = "ERR_LLM_NETWORK"
ERR_LLM_UNKNOWN = "ERR_LLM_UNKNOWN"
SUCCESS_CODE = "SUCCESS"


# ===== 标准化错误响应（复用系统 build_error 模式）=====

def build_llm_success(data: Any, message: str = "操作成功") -> Dict[str, Any]:
    """构建成功响应"""
    return {"code": SUCCESS_CODE, "data": data, "message": message}


def build_llm_error(code: str, message: str, data: Any = None) -> Dict[str, Any]:
    """
    构建错误响应 — 格式与系统 build_error() 完全一致

    Args:
        code: 错误码，必须符合 ERR_LLM_* 格式
        message: 错误描述
        data: 可选附加错误数据

    Returns:
        {"code": "ERR_LLM_XXX", "data": {...}, "message": "..."}
    """
    return {"code": code, "data": data, "message": message}


# ===== 错误转换 =====

def _handle_http_error(response: httpx.Response) -> Dict[str, Any]:
    """将 HTTP 响应转换为标准化错误 dict"""
    status = response.status_code
    body = response.text

    if status == 429:
        retry_after = None
        if "retry-after" in response.headers:
            try:
                retry_after = float(response.headers["retry-after"])
            except (ValueError, TypeError):
                pass
        return build_llm_error(ERR_LLM_RATE_LIMIT, f"Rate limited: {body}", {"retry_after": retry_after})

    if status == 401:
        return build_llm_error(ERR_LLM_AUTH, f"Authentication failed: {body}")

    if status == 403:
        return build_llm_error(ERR_LLM_FORBIDDEN, f"Forbidden: {body}")

    if status == 404:
        return build_llm_error(ERR_LLM_NOT_FOUND, f"Not found: {body}")

    if status == 422:
        return build_llm_error(ERR_LLM_VALIDATION, f"Validation error: {body}", {"detail": body})

    if status in (500, 502, 503, 504):
        return build_llm_error(ERR_LLM_SERVER, f"Server error {status}: {body}", {"status_code": status})

    return build_llm_error(ERR_LLM_UNKNOWN, f"HTTP error {status}: {body}", {"status_code": status})


def _handle_exception(e: Exception) -> Dict[str, Any]:
    """将异常转换为标准化错误 dict"""
    if isinstance(e, httpx.TimeoutException):
        return build_llm_error(ERR_LLM_TIMEOUT, f"Timeout: {e}", {"timeout_sec": str(e)})
    if isinstance(e, httpx.ConnectError):
        return build_llm_error(ERR_LLM_NETWORK, f"Connection failed: {e}")
    return build_llm_error(ERR_LLM_UNKNOWN, f"Unknown error: {e}")


# ===== 请求体构建（内部函数）=====

def _build_request_body(
    messages: List[Dict],
    model: str,
    max_tokens: Optional[int] = None,
    temperature: Optional[float] = None,
    seed: Optional[int] = None,
    tools: Optional[List[Dict]] = None,
    tool_choice: Optional[str] = None,
    stream: bool = False,
) -> Dict:
    """统一构建 LLM 请求体"""
    body = {"model": model, "messages": messages}
    if max_tokens is not None:
        body["max_tokens"] = max_tokens
    if temperature is not None:
        body["temperature"] = temperature
    if seed is not None:
        body["seed"] = seed
    if stream:
        body["stream"] = True
    if tools:
        body["tools"] = tools
    if tool_choice:
        body["tool_choice"] = tool_choice
    return body


# ===== LLM 客户端 =====

class LLMClient:
    """LLM 客户端实例"""

    def __init__(
        self,
        provider: str,
        model: str,
        api_key: str,
        base_url: Optional[str] = None,
        timeout: Optional[int] = None,
    ):
        self.provider = provider
        self.model = model
        self._api_key = api_key
        self._base_url = base_url or self._default_base_url(provider)

        read_timeout = float(timeout) if timeout else DEFAULT_READ_TIMEOUT
        self._client = httpx.AsyncClient(
            timeout=httpx.Timeout(
                connect=DEFAULT_CONNECT_TIMEOUT,
                read=read_timeout,
                write=DEFAULT_WRITE_TIMEOUT,
                pool=DEFAULT_POOL_TIMEOUT,
            ),
            limits=httpx.Limits(
                max_connections=DEFAULT_MAX_CONNECTIONS,
                max_keepalive_connections=DEFAULT_MAX_KEEPALIVE,
            ),
            headers={"Authorization": f"Bearer {api_key}"},
            base_url=self._base_url,
        )

    def _default_base_url(self, provider: str) -> str:
        """根据 provider 返回默认 API 地址（仅 OpenAI 兼容格式）"""
        urls = {
            "openai": "https://api.openai.com/v1",
            "deepseek": "https://api.deepseek.com",
            "qwen": "https://dashscope.aliyuncs.com/compatible-mode",
            "groq": "https://api.groq.com/openai",
            "ollama": "http://localhost:11434",
        }
        return urls.get(provider, "")

    async def chat(
        self,
        messages: List[Dict],
        max_tokens: Optional[int] = None,
        temperature: Optional[float] = None,
        seed: Optional[int] = None,
        tools: Optional[List[Dict]] = None,
        tool_choice: Optional[str] = None,
    ) -> Dict[str, Any]:
        """
        非流式 LLM 调用（含工具调用）

        成功返回: {"code": "SUCCESS", "data": {...}, "message": "..."}
        失败返回: {"code": "ERR_LLM_XXX", "data": {...}, "message": "..."}
        """
        body = _build_request_body(
            messages=messages,
            model=self.model,
            max_tokens=max_tokens,
            temperature=temperature,
            seed=seed,
            tools=tools,
            tool_choice=tool_choice,
            stream=False,
        )
        try:
            response = await self._client.post("/chat/completions", json=body)
            if response.status_code >= 400:
                return _handle_http_error(response)
            return build_llm_success(response.json())
        except Exception as e:
            return _handle_exception(e)

    async def chat_stream(
        self,
        messages: List[Dict],
        max_tokens: Optional[int] = None,
        temperature: Optional[float] = None,
        seed: Optional[int] = None,
        tools: Optional[List[Dict]] = None,
        tool_choice: Optional[str] = None,
    ) -> AsyncGenerator[str, None]:
        """流式 LLM 调用，逐行 yield SSE data 字符串"""
        body = _build_request_body(
            messages=messages,
            model=self.model,
            max_tokens=max_tokens,
            temperature=temperature,
            seed=seed,
            tools=tools,
            tool_choice=tool_choice,
            stream=True,
        )
        try:
            async with self._client.stream("POST", "/chat/completions", json=body) as response:
                if response.status_code >= 400:
                    error = _handle_http_error(response)
                    yield f"data: {httpx.Request('POST', '/').stream json(error)}\n\n"
                    return
                async for line in response.aiter_lines():
                    if line.startswith("data: "):
                        data = line[6:]
                        if data.strip() == "[DONE]":
                            break
                        yield data
        except Exception as e:
            error = _handle_exception(e)
            yield f"data: {httpx.Request('POST', '/').stream json(error)}\n\n"

    async def close(self):
        """关闭客户端，释放连接池"""
        await self._client.aclose()


def create_llm_client(
    provider: str,
    model: str,
    api_key: str,
    base_url: Optional[str] = None,
    timeout: Optional[int] = None,
) -> LLMClient:
    """
    创建 LLM 客户端 — 唯一入口

    Args:
        provider: 提供商（openai / deepseek / qwen / groq / ollama）
        model: 模型名（gpt-4 / deepseek-chat / qwen-plus 等）
        api_key: API 密钥
        base_url: API 地址（可选，默认根据 provider 自动设置）
        timeout: 超时秒数（可选，默认 60）

    Returns:
        LLMClient 实例

    使用方式：
        client = create_llm_client(provider="openai", model="gpt-4", api_key="sk-xxx")
        response = await client.chat(messages=[...])
        await client.close()
    """
    return LLMClient(
        provider=provider,
        model=model,
        api_key=api_key,
        base_url=base_url,
        timeout=timeout,
    )
```

### 3.3 调用方改造要点

| 调用方 | 改造前 | 改造后 |
|--------|--------|--------|
| **BaseAIService** | 自己创建 httpx.AsyncClient，自己管理生命周期 | 使用 `create_llm_client()`，调用 `client.chat_stream()` / `client.chat(tools=[...])` |
| **intent_classifier** | 每次请求 `async with httpx.AsyncClient(timeout=90)` | 使用 `create_llm_client(timeout=90)` 或复用全局 client |
| **capability_detector** | 每次探测 `async with httpx.AsyncClient(timeout=15)` | 使用 `create_llm_client(timeout=15)` |
| **factory** | 替换时不 close，reset 时不 close | 替换时 `await old_client.close()`，reset 时 `await client.close()` |
