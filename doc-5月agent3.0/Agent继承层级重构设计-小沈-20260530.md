## 一、需求和问题说明

### 1.1 重构背景

本次Agent继承层级重构源于多轮代码审查中发现的一系列架构问题。在持续优化过程中（Problem 1.37截断操作、2.5工具执行链、2.22+2.29重试机制收敛、2.3正则分散、2.10常量分散），不断暴露出Agent子类之间初始化逻辑分散、重复代码堆积、继承关系混乱的根本性缺陷。

### 1.2 核心需求

| 需求 | 说明 |
|------|------|
| **统一初始化入口** | 所有公共初始化（LLM、状态、消息、工具、策略、追踪、候选）由基类一次完成，子类不做重复工作 |
| **清晰的三层继承链** | 建立 `BaseAgent → GenericReactAgent → UniversalReactAgent → DesktopReactAgent` 的单一继承链，消除混乱的多重继承 |
| **减少重复代码** | 消除各个Agent子类中重复的 `_execute_tool()`、`_init_*` 等方法 |
| **解耦Config依赖** | UniversalReactAgent不应强制要求 `AgentConfig`，让DesktopReactAgent可以独立使用 |

### 1.3 现存问题

| 问题 | 严重程度 | 描述 |
|------|---------|------|
| **初始化逻辑分散** | P1 | BaseAgent `__init__` 仅做注册，实际初始化依赖各子类手动调用 `_init_*`，违反SRP和DRY |
| **`_execute_tool()` 重复定义** | P1 | 3个Agent子类各自实现了相同逻辑的 `_execute_tool()`，违反DRY |
| **UniversalReactAgent硬依赖config** | P1 | `__init__` 要求 `config: AgentConfig`，导致DesktopReactAgent必须构造config |
| **继承链不清** | P2 | DesktopReactAgent直接继承BaseAgent，绕过中间层，失去复用机会 |
| **MRO潜在冲突** | P2 | 多重继承（ToolStepMixin, ReactAgentMixin, RollbackMixin）的MRO顺序未显式验证 |

### 1.4 设计目标

```
最终形态（1基类 + 1中间层 + 2具体Agent）：

BaseAgent (base_react.py)           ← 地基：所有公共初始化
    │
    └── GenericReactAgent            ← 第一层：策略式LLM调用（fallback）
            │
            └── UniversalReactAgent  ← 第二层：工具执行+回滚（主力Agent）
                    │
                    └── DesktopReactAgent  ← 第三层：桌面专用prompts
```

## 二、设计原则

```
底层做完所有公共事务，上层只加独有的东西。
上一层不能把底层的事情重做一遍。
```

```
BaseAgent (base_react.py)           ← 地基：ReAct循环+所有公共初始化
    │
    └── GenericReactAgent            ← 第一层：最简单，只调LLM
            │
            └── UniversalReactAgent  ← 第二层：加工具、加回滚
                    │
```

**每层职责**：

| **第三层** | `desktop_react.py` | 加桌面专用prompts | `self.prompts = DesktopPrompts()` |

## 五、Rollback 组合化设计

### 5.1 问题

`RollbackMixin` 作为 `UniversalReactAgent` 的父类，导致 `DesktopReactAgent` 继承后也携带了不需要的 rollback 能力，违反 YAGNI。

| 问题 | 影响 |
|------|------|
| DesktopReactAgent 白拿 rollback 从未使用 | 语义不准确，Desktop "是"一个 RollbackAgent |
| MRO 多一层无关父类 | 增加 MRO 复杂度 |
| RollbackMixin 通过 `self` 访问 agent 属性 | 强耦合到 agent 实例，不适合独立复用 |

### 5.2 方案：组合模式

将 `RollbackMixin` 替换为独立的 `RollbackManager` 类，`UniversalReactAgent` 按需组合。

```
前：  UniversalReactAgent(RollbackMixin, GenericReactAgent)  ← mixin 继承
         └── DesktopReactAgent  ← 被迫携带

后：  UniversalReactAgent(GenericReactAgent)                 ← 无 mixin
         │  └── self._rollback_manager = RollbackManager(self)  ← 按需组合
         └── DesktopReactAgent  ← 干净，没有 rollback
```

### 5.3 RollbackManager 设计

```python
# mixins/rollback_manager.py

class RollbackManager:
    """回滚能力管理器（组合模式，替代 RollbackMixin）"""
    
    def __init__(self, agent):
        self._agent = agent
    
    async def rollback(self, step_number: Optional[int] = None) -> bool:
        """回滚操作，逻辑与原来的 RollbackMixin.rollback() 一致"""
        agent = self._agent
        try:
            if not agent.task_id:
                raise ValueError("Session ID is required for rollback")
            
            if step_number is None:
                result = await agent._execute_tool('rollback_session', {'task_id': agent.task_id})
                success = result.get("status") == "success"
            else:
                steps_to_rollback = [s for s in agent.steps if s.step_number > step_number]
                if not steps_to_rollback:
                    return False
                success = True
                for step in sorted(steps_to_rollback, key=lambda s: s.step_number, reverse=True):
                    observation = step.observation or {}
                    result_data = observation.get("result", {}) if isinstance(observation, dict) else {}
                    operation_id = result_data.get("operation_id")
                    if operation_id:
                        step_result = await agent._execute_tool('rollback_operation', {'operation_id': operation_id})
                        step_success = step_result.get("status") == "success" if isinstance(step_result, dict) else bool(step_result)
                        success = success and step_success
                    else:
                        raise ValueError(f"No operation_id found for step {step.step_number}")
            
            agent.status = AgentStatus.ROLLED_BACK
            return success
        except ValueError:
            raise
        except Exception as e:
            logger.error(f"Rollback failed: {e}")
            return False
```

```python
# universal_react.py — 需要改
class UniversalReactAgent(ToolStepMixin, ReactAgentMixin, GenericReactAgent):
    def __init__(self, llm_client, task_id, config=None, tool_category=None,
                 max_steps=None, candidates=None, **kwargs):
        # 有config用config，没config用传入的参数
        effective_category = tool_category or (config.category if config else None)
        effective_max_steps = max_steps or (config.max_steps if config else DEFAULT_MAX_STEPS)
        rollback_enabled = config.rollback_enabled if config else True

        super().__init__(
            llm_client=llm_client,
            task_id=task_id,
            tool_category=effective_category,
            max_steps=effective_max_steps,
            rollback_enabled=rollback_enabled,
            candidates=candidates,
            **kwargs
        )

        # RollbackManager 按需组合（放 super().__init__() 之后，确保 agent 属性已就绪）
        if rollback_enabled:
            self._rollback_manager = RollbackManager(self)

        # 有config才设置prompts，没有则由子类设置
        if config:
            self.config = config
            self.prompts = config.prompt_class()
```
