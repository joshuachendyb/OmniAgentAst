## 一、设计原则

```
底层做完所有公共事务，上层只加独有的东西。
上一层不能把底层的事情重做一遍。
```

---

## 二、目标层级

```
BaseAgent (base_react.py)           ← 地基：ReAct循环+所有公共初始化
    │
    └── GenericReactAgent            ← 第一层：最简单，只调LLM
            │
            └── UniversalReactAgent  ← 第二层：加工具、加回滚
                    │
                    └── DesktopReactAgent  ← 第三层：加桌面prompts
```

**每层职责**：

| 层 | 文件 | 职责 | 独有代码 |
|---|------|------|---------|
| **地基** | `base_react.py` | ReAct循环+所有公共初始化（LLM、状态、消息、工具、策略、追踪、候选） | 无 |
| **第一层** | `generic_react.py` | 策略式LLM调用（不调工具） | `_get_llm_response()` 用strategy |
| **第二层** | `universal_react.py` | 加工具执行、加回滚能力 | `_get_system_prompt()`, `_get_task_prompt()`, `run()`, `_run_with_task_tracking()` |
| **第三层** | `desktop_react.py` | 加桌面专用prompts | `self.prompts = DesktopPrompts()` |

---

## 三、需要解决的问题

| 问题 | 现状 | 解决方案 |
|------|------|---------|
| UniversalReactAgent的`__init__`需要`config: AgentConfig` | DesktopReactAgent没有config | config改为可选参数，给默认值 |
| DesktopReactAgent需要传`tool_category=DESKTOP` | UniversalReactAgent的init里硬编码了`tool_category = config.category` | 拆分：有config用config，没config用传入的tool_category |
| MRO冲突 | 多重继承可能有顺序问题 | 检查mro()验证 |

## 四、每层的`__init__`设计

### 4.1 地基：BaseAgent

```python
# base_react.py — 不改，已经OK
class BaseAgent(ReActHandlerMixin, ABC):
    def __init__(self, llm_client, task_id, tool_category=None, max_steps=DEFAULT_MAX_STEPS,
                 rollback_enabled=True, candidates=None, **kwargs):
        self._init_llm(llm_client, **kwargs)
        self._init_state(task_id, tool_category, max_steps)
        self._init_messages()
        self._init_tools()
        self._init_llm_strategies()
        self._init_task_tracking(enable=rollback_enabled)
        self._init_candidates(candidates)
```

### 4.2 第一层：GenericReactAgent

```python
# generic_react.py — 不改，已经OK
class GenericReactAgent(BaseAgent):
    def __init__(self, llm_client, task_id, strategy, **kwargs):
        BaseAgent.__init__(self, llm_client=llm_client, task_id=task_id, tool_category=None, **kwargs)
        self._strategy = strategy
```

### 4.3 第二层：UniversalReactAgent

```python
# universal_react.py — 需要改
class UniversalReactAgent(ToolStepMixin, ReactAgentMixin, RollbackMixin, GenericReactAgent):
    def __init__(self, llm_client, task_id, config=None, tool_category=None,
                 max_steps=None, candidates=None, **kwargs):
        # 有config用config，没config用传入的参数
        effective_category = tool_category or (config.category if config else None)
        effective_max_steps = max_steps or (config.max_steps if config else DEFAULT_MAX_STEPS)
        rollback_enabled = config.rollback_enabled if config else True

        super().__init__(
            llm_client=llm_client,
            task_id=task_id,
            tool_category=effective_category,
            max_steps=effective_max_steps,
            rollback_enabled=rollback_enabled,
            candidates=candidates,
            **kwargs
        )

        # 有config才设置prompts，没有则由子类设置
        if config:
            self.config = config
            self.prompts = config.prompt_class()
```

### 4.4 第三层：DesktopReactAgent

```python
# desktop_react.py — 需要改
class DesktopReactAgent(UniversalReactAgent):
    def __init__(self, llm_client, task_id, tool_category=None,
                 max_steps=DEFAULT_MAX_STEPS, candidates=None, **kwargs):
        # 不需要config，直接传tool_category
        super().__init__(
            llm_client=llm_client,
            task_id=task_id,
            config=None,
            tool_category=tool_category or ToolCategory.DESKTOP,
            max_steps=max_steps,
            candidates=candidates,
            **kwargs
        )
        self.prompts = DesktopPrompts()  # 覆盖universal的prompts
```

## 五、MRO验证

预期MRO（方法解析顺序）：

```
DesktopReactAgent
  → UniversalReactAgent
    → ToolStepMixin
    → ReactAgentMixin
      → ToolInitMixin
      → LLMDispatchMixin
      → PromptBuildMixin
    → RollbackMixin
    → GenericReactAgent
      → BaseAgent
        → ReActHandlerMixin
        → ABC
```

**验证方法**：代码改完后执行 `print(DesktopReactAgent.__mro__)` 确认无冲突。

## 六、每层的方法继承关系

| 方法 | 地基(Base) | 第一层(Generic) | 第二层(Universal) | 第三层(Desktop) |
|------|-----------|----------------|-------------------|----------------|
| `run_stream()` | ✅ 定义 | 继承 | 继承 | 继承 |
| `_get_llm_response()` | ✅ 默认`_call_llm()` | **覆盖**（用strategy） | **覆盖**（用`_call_llm()`） | 继承Universal |
| `_get_system_prompt()` | ❌ 抽象 | ✅ 简单prompt | **覆盖**（用config） | 继承Universal |
| `_get_task_prompt()` | ❌ 抽象 | ✅ 直接返回task | **覆盖**（用prompts） | 继承Universal |
| `_execute_tool()` | ✅ 默认`execute_tool_with_unified_retry()` | 继承 | 继承 | 继承 |
| `run()` | ❌ 无 | ❌ 无 | ✅ 定义 | 继承Universal |
| `_run_with_task_tracking()` | ❌ 无 | ❌ 无 | ✅ 定义 | 继承Universal |
| `self.prompts` | 无 | 无 | config.prompt_class() | **覆盖** DesktopPrompts |
