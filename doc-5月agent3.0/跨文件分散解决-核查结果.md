base_react.py 第330行是唯一解析入口：parsed = parse_react_response(response)
llm_strategies.py 中仅保留注释记录了重构原因（第155-157行）："移除内部parse_react_response调用（违反DRY），base_react.py作为唯一解析入口调用parse_react_response()"
修改方法：TextStrategy.call() 从原来"调用LLM + 内部调用parse_react_response解析"改为"仅调用LLM并返回原始文本content"。具体改动：

删除了 TextStrategy.call() 内部对 parse_react_response() 的调用
方法返回值从 parse_react_response(content) 改为直接 return content
base_react.py 第330行 parsed = parse_react_response(response) 作为唯一的解析入口保留不变

### 2.7 _call_llm SLAP分层不清

核查事实：

在整个 agent/mixins/ 目录中搜索 _resolve_strategy 无结果
当前 llm_dispatch_mixin.py 中 _call_llm 的实现（第86-98行）是：
strategy = await self.strategy_manager.get_strategy() -- 策略由策略管理器统一处理
response = await self._dispatch_strategy(strategy, messages) -- 分派执行
不存在任何 _resolve_strategy 方法
修改方法：删除了 _call_llm 中的 _resolve_strategy 方法。原来 _call_llm 内部混搭了策略解析逻辑（_resolve_strategy）和LLM调用逻辑，违反SLAP。重构后策略解析职责上提到 LLMStrategyManager.get_strategy()（独立的策略管理器），_call_llm 只做三件事：获取策略 -> 注入提示 -> 分派调用，层级清晰。_resolve_strategy 方法被删除，其功能由 LLMStrategyManager 类接管。
