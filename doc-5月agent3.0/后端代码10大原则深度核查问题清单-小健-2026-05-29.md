## 二、P0-紧急（运行时崩溃，必须立即修复）

### 问题 P0-001: `data_format_helper.py` 缺少关键导入 ✅ 已修复

| 字段 | 值 |
|------|-----|
| **文件** | `backend/app/services/tools/toolhelper/data_format_helper.py` |
| **行号** | 37, 43, 45, 53, 73, 75, 83, 95, 97 |
| **原则** | DRY / 复用优先 |
| **严重程度** | 🔴 P0-紧急 |

**现象**: `_parse_ini()`、`_parse_xml()`、`_parse_properties()` 三个函数共9处调用 `build_success()` 和 `build_error()`，但这两个函数**从未在该文件中导入**。文件顶部导入块（第21-29行）仅导入标准库模块，底部导入块（第98-112行）仅从 `app.constants` 导入常量。

**根因**: 重构过程中函数从别处迁移，但导入项未同步迁移。

**后果**: 调用这三个函数中任意一个时会抛出 `NameError`，使解析INI/XML/Properties文件的功能彻底崩溃。

**修复方案**:
```python
# 在文件顶部导入块中添加：
from app.services.tools._response import build_success, build_error
```
