## 一、文件目录位置

```
backend/app/services/agent/
├── base_react.py              # 核心ABC基类
├── universal_react.py         # 通用Agent（配置驱动）
├── desktop_react.py           # 桌面操作Agent
├── tool_executor.py           # 工具执行统一入口（函数）
└── mixins/
    ├── rollback_mixin.py      # 回滚能力
    ├── tool_step_mixin.py     # 工具步骤编排
    └── react_agent_mixin.py   # 初始化逻辑
```

---

## 二、继承关系

```
BaseAgent (ABC) + ReActHandlerMixin          ← base_react.py
    ↑ 定义4个抽象方法:
    │  _get_llm_response()
    │  _execute_tool()
    │  _get_system_prompt()
    │  _get_task_prompt()
    │
    ├── ToolStepMixin     ← tool_step_mixin.py (编排一次工具执行)
    ├── ReactAgentMixin   ← react_agent_mixin.py (初始化逻辑)
    └── RollbackMixin     ← rollback_mixin.py (回滚能力)
        ↑
        ├── UniversalReactAgent  ← universal_react.py (通用，混入全部)
        └── DesktopReactAgent    ← desktop_react.py (桌面专用，无回滚)
```

---

## 三、每个文件的职责

| 文件 | 类/函数 | 职责 | 关键代码 |
|------|---------|------|---------|
| **base_react.py** | `BaseAgent` | ReAct循环骨架，定义抽象方法 | `run_stream()` + 4个抽象方法 |
| **universal_react.py** | `UniversalReactAgent` | 配置驱动通用Agent，实现4个抽象方法 | `_execute_tool()` → `execute_tool_with_unified_retry()` |
| **desktop_react.py** | `DesktopReactAgent` | 桌面操作Agent，实现4个抽象方法 | `_execute_tool()` → `execute_tool_with_unified_retry()` |
| **rollback_mixin.py** | `RollbackMixin` | 回滚能力，被UniversalReactAgent混入 | `rollback()` → `execute_tool_with_unified_retry()` |

---

## 四、调用流程

```
Agent.run(task)
    │
    ▼
BaseAgent.run_stream()          ← ReAct循环（定义在base_react.py）
    │
    │ 每轮循环:
    │   1. _get_llm_response()   ← 子类实现（调LLM）
    │   2. parse → action
    │   3. _execute_tool(action)  ← 子类实现（调工具）
    │       │
    │       └──→ tool_executor.py::execute_tool_with_unified_retry()
    │                 │
    │                 └──→ tool_retry_engine.py::execute_tool_with_retry()
    │
    │ 观察结果 → 继续循环/退出
    │
    ▼
AgentResult
```

## 五、本次PATCH的变化

| 修改前 | 修改后 |
|--------|--------|
| `base_react.py` 持有 `self.executor = ToolExecutor()` | 删除 `self.executor = None` |
| `universal_react.py` 调 `self.executor.execute()` | 改为 `from tool_executor import execute_tool_with_unified_retry` |
| `desktop_react.py` 调 `self.executor.execute()` | 同上 |
| `rollback_mixin.py` 调 `self.executor.execute()` | 同上 |

**效果**：4个文件都统一委托给`tool_executor.py`这一个入口，去掉了ToolExecutor这个中间对象层。

## 六、Agent层 vs Tool层

```
┌─────────────────────────────────────────┐
│          Agent层（这4个文件在这里）        │
│                                         │
│  base_react.py        ← ReAct循环骨架   │
│  universal_react.py   ← 通用Agent实现    │
│  desktop_react.py     ← 桌面Agent实现    │
│  rollback_mixin.py    ← Agent回滚能力    │
│                                         │
│  职责：思考→决策→调工具→观察→循环        │
└──────────────┬──────────────────────────┘
               │ _execute_tool()
               ▼
┌─────────────────────────────────────────┐
│          Tool层（另外的文件）              │
│                                         │
│  tool_executor.py     ← 工具执行统一入口 │
│  tool_retry_engine.py ← 工具重试引擎     │
│  各category_tools.py  ← 工具实现         │
│                                         │
│  职责：接收action+params → 执行 → 返回结果│
└─────────────────────────────────────────┘
```

**Agent层**定义"什么时候调工具、调哪个工具"，**Tool层**负责"怎么把工具跑起来"。Agent层不碰工具实现，只通过`_execute_tool()`这个抽象方法向下层委托。
