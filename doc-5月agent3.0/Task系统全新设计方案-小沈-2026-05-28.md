## 一、需求说明

### 1.1 业务背景

当前系统中，Task 追踪存在以下 5 个问题：

**问题一：仅 file 意图有持久化追踪。** 其他 8 个意图（shell / document / network / code_execution / system / desktop / time / meta）使用内存存储，导致服务重启后这些意图的任务记录全部丢失。

**问题二：未按意图分类查询。** tasks 表没有 intent 字段，无法区分"哪些任务是 shell 做的、哪些是 file 做的"。

**问题三：操作表名局限于文件。** `file_ops` 表名只能存文件类操作，shell 做了文件操作（如 `cp`、`rm`）无处可记，或者强行塞进 file_ops。

**问题四：层次冗长。** 4 层调用链路（mixins → TaskExecutionTracker → get_tracker → Service），代码绕，理解成本高。

**问题五：向后兼容残留。** `_deprecated_imports.py` 保留旧名旧路径，违反铁规，代码臃肿。

**根本原因**：原设计错误地认为"只有 file 意图需要持久化"，但实际上所有意图的业务最终都可能落到文件操作或类似的需回滚操作上。

### 1.2 系统定位

**通用任务追踪系统**，服务于 Agent 系统中全部 9 个意图（file / shell / document / code_execution / network / system / desktop / time / meta），提供统一的任务生命期管理和操作明细记录。

**核心职责**：
1. **任务追踪** — 任意意图执行时，创建任务记录，追踪从开始到完成的完整过程
2. **操作明细记录** — 任务执行过程中产生的具体操作（文件移动、Shell 命令、网络下载等），逐条记录，带顺序号，可用于回滚
3. **查单个任务** — 按 task_id 查任务详情
4. **查最近任务** — 按时间倒序查最近 N 条任务列表
5. **按意图筛选** — 查某个意图（如只看 file 或 shell）的所有任务
6. **查操作明细** — 查某个任务的所有操作，按回滚顺序排列

**明确不包括**：

**一、不执行回滚。** 它只标记"该操作已回滚"，不做实际的文件恢复。回滚是文件操作的业务逻辑，归各意图自己执行。

**二、不存 Agent 执行细节。** 不存 LLM 的 prompt、response、token 消耗。那不是 task 的职责，有独立的对话记录系统。

**三、不做实时推送。** 没有 WebSocket，不能"任务完成立刻通知前端"。本地 SQLite，需要时查询即可。

**四、不做任务依赖。** 不支持"任务 B 必须等任务 A 完成才能开始"。Agent 每次执行相互独立。

**五、不做统计报表。** 不做成功率趋势、分析图表。表里有数据，真要分析直接 SQL 查。

**六、不做权限控制。** 无用户认证、无"谁能看哪些任务"。Agent 内部用，不对外开放。

**七、不做分布式。** 本地桌面应用，单进程够用。

**八、不做独立微服务。** 不是 REST API 对外暴露的，是 `services/task/` 模块。

### 1.3 使用场景

| 场景 | 参与者 | 触发条件 | 涉及表 |
|------|--------|---------|--------|
| **SC-01 创建任务** | Agent | 任意意图开始执行 | tasks |
| **SC-02 完成任务-成功** | Agent | 意图执行成功 | tasks |
| **SC-03 完成任务-失败** | Agent | 意图执行抛出异常 | tasks |
| **SC-04 记录操作** | Agent | 执行过程中有文件创建/移动/删除、Shell 命令、网络下载等 | tasks + operations |
| **SC-05 标记操作失败** | Agent | 具体某个操作执行失败 | tasks + operations |
| **SC-06 标记回滚** | Agent / 回滚模块 | 用户或系统发起回滚 | tasks + operations |
| **SC-07 查询最近任务** | API 调用者 / 前端 | 用户查看历史记录 | tasks |
| **SC-08 按意图查询任务** | API 调用者 / 前端 | 筛选特定意图的历史 | tasks |
| **SC-09 查看操作明细** | API 调用者 / 前端 | 用户查看某个任务的详细操作 | operations |
| **SC-10 生成报告** | Agent | 任务完成后，生成操作报告 | tasks |

|------|------|

### 1.5 数据模型需求

#### 1.5.1 状态枚举

**TaskStatus** — 任务生命周期状态：

| 枚举值 | DB 存值 | 含义 | 进入条件 |
|--------|---------|------|---------|
| PENDING | "pending" | 待执行 | 任务刚创建时 |
| EXECUTING | "executing" | 执行中 | FR-01 创建后立即设为 EXECUTING |
| SUCCESS | "success" | 成功完成 | FR-02 正常返回 |
| FAILED | "failed" | 执行失败 | FR-02 捕获到异常 |
| PARTIALLY_ROLLED_BACK | "partially_rolled_back" | 部分回滚 | FR-05 部分操作回滚（op_ids 有值） |
| ROLLED_BACK | "rolled_back" | 全部回滚 | FR-05 所有操作回滚（op_ids 为 None） |

**OperationStatus** — 操作明细状态：

| 枚举值 | DB 存值 | 含义 | 进入条件 |
|--------|---------|------|---------|
| PENDING | "pending" | 待执行 | 操作记录刚创建时 |
| EXECUTING | "executing" | 执行中 | 操作开始执行前（可选） |
| SUCCESS | "success" | 成功完成 | FR-03 插入时即为 SUCCESS |
| FAILED | "failed" | 执行失败 | FR-04 失败标记 |
| ROLLBACK | "rollback" | 已回滚 | FR-05 回滚标记 |

#### 1.5.2 TaskRecord — 任务记录主体

| # | 字段名 | 类型 | 约束 | Pydantic 类型 | 说明 |
|---|--------|------|------|--------------|------|
| 1 | task_id | TEXT | PK, NOT NULL | str | 主键，自动生成 `task-{uuid4().hex}` |
| 2 | intent | TEXT | NOT NULL | str | 创建本任务的意图标识：file / shell / document / code_execution / network / system / desktop / time / meta |
| 3 | agent_id | TEXT | NOT NULL | str | 执行者标识 |
| 4 | task_description | TEXT | NOT NULL | str | 任务描述，由 Agent 传入 |
| 5 | status | TEXT | NOT NULL, DEFAULT 'executing' | TaskStatus | 当前任务状态 |
| 6 | total_operations | INTEGER | DEFAULT 0 | int | 本次任务涉及的操作总数 |
| 7 | success_count | INTEGER | DEFAULT 0 | int | 成功操作数 |
| 8 | failed_count | INTEGER | DEFAULT 0 | int | 失败操作数 |
| 9 | rolled_back_count | INTEGER | DEFAULT 0 | int | 已回滚操作数 |
| 10 | report_generated | INTEGER | DEFAULT 0 | bool | 是否已生成操作报告（0/1） |
| 11 | report_path | TEXT | NULLABLE | Optional[str] | 报告文件存储路径 |
| 12 | created_at | TIMESTAMP | NOT NULL | datetime | 任务创建时间（自动填充） |
| 13 | completed_at | TIMESTAMP | NULLABLE | Optional[datetime] | 任务完成时间（FR-02 写入） |

**索引**：
- `idx_tasks_intent` ON `intent` — 支持按意图过滤
- `idx_tasks_created` ON `created_at` — 支持按时间排序
- **约束**：`intent` 字段使用 CHECK 约束限制为9个有效值

#### 1.5.3 OperationRecord — 操作明细记录

| # | 字段名 | 类型 | 约束 | Pydantic 类型 | 说明 |
|---|--------|------|------|--------------|------|
| 1 | operation_id | TEXT | PK, NOT NULL | str | 主键，自动生成 `op-{uuid4().hex}` |
| 2 | task_id | TEXT | FK → tasks.task_id, NOT NULL | str | 所属任务 |
| 3 | intent | TEXT | NOT NULL | str | 执行此操作的意图标识 |
| 4 | operation_type | TEXT | NOT NULL | str | 操作类型：create / move / copy / delete / run / write / download / 其他 |
| 5 | status | TEXT | NOT NULL, DEFAULT 'pending' | OperationStatus | 操作状态 |
| 6 | source_path | TEXT | NULLABLE | Optional[str] | 来源路径。文件类：源文件路径；网络类：下载 URL；其他：源位置 |
| 7 | destination_path | TEXT | NULLABLE | Optional[str] | 目标路径。文件类：目标文件路径；其他：目标位置 |
| 8 | backup_path | TEXT | NULLABLE | Optional[str] | 备份路径，用于回滚时恢复原始文件 |
| 9 | file_size | INTEGER | DEFAULT 0 | int | 涉及文件大小（字节），非文件操作为 0 |
| 10 | file_hash | TEXT | NULLABLE | Optional[str] | 文件哈希值，用于完整性校验 |
| 11 | sequence_number | INTEGER | NOT NULL | int | 任务内的操作顺序号，从 1 递增，决定回滚顺序（逆序回滚） |
| 12 | details | TEXT | NULLABLE | Optional[str] | JSON 扩展字段（存储为字符串）。Shell 操作用来存 `{"command": "...", "exit_code": 0}`；Network 操作存 `{"url": "...", "status_code": 200}`；其他任意 |
| 13 | error | TEXT | NULLABLE | Optional[str] | 错误信息，失败时记录 |
| 14 | created_at | TIMESTAMP | NOT NULL | datetime | 创建时间（自动填充） |

**外键约束**：`operation_id` → `tasks.task_id`（CASCADE DELETE）

**索引**：
- `idx_operations_task` ON `task_id` — 按任务查询操作
- `idx_operations_seq` ON `(task_id, sequence_number)` — 回滚顺序查询
- `idx_operations_intent` ON `intent` — 按意图查询操作

| 意图 | 典型 operation_type | source_path | destination_path | backup_path | file_size/hash | details 示例 |

| shell | run | — | — | — | ❌ 0/null | `{"command": "cp a b", "exit_code": 0}` |
| document | write | — | 文档路径 | — | ✅ 填充 | `{"format": "md"}` |
| code_execution | write | — | 代码文件路径 | — | ✅ 填充 | `{"language": "python"}` |
| network | download | URL | 本地路径 | — | ✅ 填充 | `{"status_code": 200, "content_type": "json"}` |

### 1.6 非功能需求

| 需求 | 说明 | 指标 |
|------|------|------|
| **持久化** | 所有任务记录持久化到本地 SQLite | 服务重启不丢数据 |
| **存储路径** | DB 文件位置 `~/.omniagent/task_tracker.db` | — |
| **单实例** | 整个进程内只有一个 Tracker 实例 | 避免多实例状态不同步 |
| **连接安全** | 每次操作使用独立的数据库连接，用完释放 | 避免连接泄漏 |
| **并发安全** | SQLite WAL 模式 | 支持多线程并发读写 |
| **性能** | 单次查询 < 50ms（本地 SQLite） | — |

| 项目 | 命名 | 依据 |

| 操作 Pydantic 模型 | `OperationRecord` | — |
| ids | `task-{uuid4().hex}` / `op-{uuid4().hex}` | 可读性好，避免纯数字自增 id |

## 二、核心设计理念

### 2.1 关键认知

**所有 9 个意图的业务最终都可能落到操作上**：

| 意图 | 操作可能性 | 举例 |
|------|-----------|------|
| file | 100% | 移动/复制/删除文件 |
| shell | 高 | `rm`, `cp`, `mv`, `echo > file` |
| document | 高 | 创建文档、写入内容 |
| code_execution | 高 | 写入代码文件 |
| network | 中 | 下载到本地 |
| system | 中 | 写配置文件 |
| desktop/time/etc | 低 | 基本不涉及 |

所以 **不能按 intent_type 硬分类来决定谁用 SQLite、谁用内存**。所有意图统一用持久化存储。

### 2.2 设计原则

| 原则 | 体现在 |
|------|--------|
| **SRP** | TaskTracker 管任务生命周期+操作+回滚+报告（同一领域）；TaskQueries 管查询 |
| **DRY** | 双表设计：task 级字段只存一次在 tasks 表，operations 通过 FK 关联；连接管理一个上下文管理器 |
| **KISS** | 两个类、两个表、没有路由层，所有意图统一处理 |
| **YAGNI** | 不做两套实现，不做多余接口抽象，不做统计缓存；迁移代码只做必要操作 |
| **SLAP** | 业务逻辑（Tracker）与存储细节（db.get_conn）分离 |
| **ISP** | TaskTracker 供 Agent/回滚/报告使用；TaskQueries 供 API/前端使用 |
| 禁止向后兼容 | 全新模块+独立数据库 task_tracker.db，不保留旧名旧路径 |
| **复用优先** | 复用 OperationStatus 枚举 + db.get_conn + safe_parse_json；TaskRecord/OperationRecord 为新建独立模型 |

### 3.1 枚举定义

**新增枚举**（`app/services/task/models.py`）：

```python
class TaskStatus(str, Enum):
    """任务生命周期状态 — 新增，已有枚举无此定义"""
```

### 4.2 表结构

```sql
-- 任务表：task 级信息只存一次（DRY）
CREATE TABLE tasks (
    task_id          TEXT PRIMARY KEY,
    intent           TEXT NOT NULL CHECK(intent IN ('file', 'shell', 'document', 'code_execution', 'network', 'system', 'desktop', 'time', 'meta')),
    agent_id         TEXT NOT NULL,
    task_description TEXT NOT NULL,
    status           TEXT NOT NULL DEFAULT 'executing',
    total_operations INTEGER DEFAULT 0,
    success_count    INTEGER DEFAULT 0,
    failed_count     INTEGER DEFAULT 0,
    rolled_back_count INTEGER DEFAULT 0,
    report_generated INTEGER DEFAULT 0,
    report_path      TEXT,
    created_at       TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at     TIMESTAMP
);

CREATE INDEX idx_tasks_intent   ON tasks(intent);
CREATE INDEX idx_tasks_created  ON tasks(created_at);

-- 操作表：operation 级信息，通过 task_id 关联
CREATE TABLE operations (
    operation_id     TEXT PRIMARY KEY,
    task_id          TEXT NOT NULL REFERENCES tasks(task_id) ON DELETE CASCADE,
    operation_type   TEXT NOT NULL,
    status           TEXT NOT NULL DEFAULT 'pending',
    source_path      TEXT,
    destination_path TEXT,
    backup_path      TEXT,
    file_size        INTEGER DEFAULT 0,
    file_hash        TEXT,
    sequence_number  INTEGER NOT NULL,
    details          TEXT,                -- JSON 扩展字段
    error            TEXT,
    created_at       TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_ops_task   ON operations(task_id);
CREATE INDEX idx_ops_seq    ON operations(task_id, sequence_number);
```

**关键设计决策**：
- 双表：`tasks`（task 级）+ `operations`（operation 级），DRY
- `operations` 通过 `task_id` FK 关联 `tasks`，CASCADE DELETE
- `details` TEXT 存 JSON 字符串
- 不按意图分表，不搞多套存储

### 4.3 旧表迁移策略

现有 `operations.db` 已有 `file_operations` 和 `task_operations` 两张旧表。新数据库 `task_tracker.db` 建好后，执行一次性幂等迁移：

```python
def _migrate_from_old_db(self):
    """从旧 operations.db 迁移数据到新 task_tracker.db（幂等，失败不阻塞启动）"""
    try:
        with self.get_conn("operations") as old_conn:
            # 迁移 file_operations → operations（旧操作数据）
            old_ops = old_conn.execute("SELECT * FROM file_operations").fetchall()
            if old_ops:
                with self.get_conn("task_tracker") as new_conn:
                    for row in old_ops:
                        new_conn.execute(
                            """INSERT OR IGNORE INTO operations
                               (operation_id, task_id, operation_type, status,
                                source_path, destination_path, backup_path,
                                file_size, file_hash, sequence_number, details, error, created_at)
                               VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)""",
                            (row["operation_id"], row["task_id"], row["operation_type"],
                             row["status"], row["source_path"], row["destination_path"],
                             row["backup_path"], row["file_size"], row["file_hash"],
                             row["sequence_number"], row["metadata"], row["error_message"],
                             row["created_at"])
                        )
                    logger.info(f"Migrated {len(old_ops)} operations from old DB")
    except Exception as e:
        logger.warning(f"Migration skipped (old DB may not exist): {e}")
```

**说明**：
- 从旧 `operations.db` 的 `file_operations` 表读取数据
- 写入新 `task_tracker.db` 的 `operations` 表
- `INSERT OR IGNORE` 保证幂等
- 迁移失败只记日志，不阻塞应用启动（旧库可能不存在）

## 五、核心实现

### 5.1 TaskTracker — 任务生命周期 + 操作管理

```python
# task_tracker.py

import json
from uuid import uuid4
from datetime import datetime
from typing import Optional, Dict, Any, List

from app.db import db
from app.utils.logger import logger
from app.db.models.operation_enums import OperationStatus
from .models import TaskStatus


class TaskTracker:
    """
    任务追踪器 — 双表操作：tasks（task 级）+ operations（operation 级）。
    
    调用方：Agent、回滚模块、报告模块
    """
    
    VALID_INTENTS = {"file", "shell", "document", "code_execution", 
                     "network", "system", "desktop", "time", "meta"}
    
    # ===== 任务生命周期 =====
    
    def create_task(self, intent: str, agent_id: str, description: str) -> str:
        """创建任务 → 返回 task_id"""
        if intent not in self.VALID_INTENTS:
            raise ValueError(f"Invalid intent: {intent}. Must be one of: {self.VALID_INTENTS}")
        
        task_id = f"task-{uuid4().hex}"
        with db.get_conn("task_tracker") as conn:
            conn.execute(
                """INSERT INTO tasks 
                   (task_id, intent, agent_id, task_description, status) 
                   VALUES (?, ?, ?, ?, ?)""",
                (task_id, intent, agent_id, description, TaskStatus.EXECUTING.value)
            )
        return task_id
    
    def complete_task(self, task_id: str, success: bool = True) -> None:
        """完成任务 — 更新 tasks 表"""
        status = TaskStatus.SUCCESS.value if success else TaskStatus.FAILED.value
        with db.get_conn("task_tracker") as conn:
            row = conn.execute(
                "SELECT COUNT(*) FROM operations WHERE task_id = ? AND status = ?",
                (task_id, OperationStatus.SUCCESS.value)
            ).fetchone()
            success_count = row[0] if row else 0
            conn.execute(
                """UPDATE tasks SET status = ?, completed_at = ?,
                   success_count = ? WHERE task_id = ?""",
                (status, datetime.now(), success_count, task_id)
            )
    
    # ===== 操作管理 =====
    
    def add_operation(
        self, task_id: str, operation_type: str, *,
        source_path: Optional[str] = None,
        destination_path: Optional[str] = None,
        backup_path: Optional[str] = None,
        file_size: int = 0,
        file_hash: Optional[str] = None,
        details: Optional[Dict[str, Any]] = None,
    ) -> str:
        """记录一次操作 → 插入 operations 表"""
        with db.get_conn("task_tracker") as conn:
            # 验证 task 存在
            task_row = conn.execute(
                "SELECT task_id FROM tasks WHERE task_id = ?", (task_id,)
            ).fetchone()
            if not task_row:
                raise ValueError(f"Task {task_id} not found")
            
            # 获取下一个 sequence_number
            seq_row = conn.execute(
                "SELECT COALESCE(MAX(sequence_number), 0) + 1 FROM operations WHERE task_id = ?",
                (task_id,)
            ).fetchone()
            seq_num = seq_row[0]
            
            operation_id = f"op-{uuid4().hex}"
            conn.execute(
                """INSERT INTO operations 
                   (operation_id, task_id, operation_type, status,
                    source_path, destination_path, backup_path,
                    file_size, file_hash, sequence_number, details)
                   VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)""",
                (operation_id, task_id, operation_type, OperationStatus.SUCCESS.value,
                 source_path, destination_path, backup_path,
                 file_size, file_hash, seq_num,
                 json.dumps(details) if details else None)
            )
            conn.execute(
                "UPDATE tasks SET total_operations = total_operations + 1 WHERE task_id = ?",
                (task_id,)
            )
        return operation_id
    
    # ===== 状态管理 =====
    
    def mark_failed(self, task_id: str, operation_id: str, error: str) -> None:
        """标记某个操作为失败"""
        with db.get_conn("task_tracker") as conn:
            conn.execute(
                "UPDATE operations SET status = ?, error = ? WHERE operation_id = ?",
                (OperationStatus.FAILED.value, error, operation_id)
            )
            conn.execute(
                "UPDATE tasks SET failed_count = failed_count + 1 WHERE task_id = ?",
                (task_id,)
            )
    
    def mark_rolled_back(self, task_id: str, op_ids: Optional[List[str]] = None) -> None:
        """标记操作为已回滚，检查是否全量回滚"""
        with db.get_conn("task_tracker") as conn:
            if op_ids:
                placeholders = ",".join("?" for _ in op_ids)
                conn.execute(
                    f"UPDATE operations SET status = ? WHERE operation_id IN ({placeholders})",
                    [OperationStatus.ROLLBACK.value] + op_ids
                )
                row = conn.execute(
                    "SELECT COUNT(*) FROM operations WHERE task_id = ? AND status != ?",
                    (task_id, OperationStatus.ROLLBACK.value)
                ).fetchone()
                all_rolled_back = (row[0] == 0) if row else False
                task_status = TaskStatus.ROLLED_BACK.value if all_rolled_back else TaskStatus.PARTIALLY_ROLLED_BACK.value
            else:
                conn.execute(
                    "UPDATE operations SET status = ? WHERE task_id = ?",
                    (OperationStatus.ROLLBACK.value, task_id)
                )
                task_status = TaskStatus.ROLLED_BACK.value
            
            conn.execute(
                """UPDATE tasks SET status = ?, 
                   rolled_back_count = (SELECT COUNT(*) FROM operations WHERE task_id = ? AND status = ?) 
                   WHERE task_id = ?""",
                (task_status, task_id, OperationStatus.ROLLBACK.value, task_id)
            )
    
    # ===== 报告管理 =====
    
    def mark_report_generated(self, task_id: str, report_path: str) -> None:
        """标记任务报告已生成"""
        with db.get_conn("task_tracker") as conn:
            conn.execute(
                "UPDATE tasks SET report_generated = 1, report_path = ? WHERE task_id = ?",
                (report_path, task_id)
            )
```

```python
# task_queries.py

from typing import Optional, Dict, Any, List

from app.db import db
from app.utils.data_utils import safe_parse_json


class TaskQueries:
    """
    任务查询服务 — 只负责查询，不修改数据。
    
    调用方：API、前端
    """
    
    def get_task(self, task_id: str) -> Optional[Dict[str, Any]]:
        """获取单个任务"""
        with db.get_conn("task_tracker") as conn:
            row = conn.execute(
                "SELECT * FROM tasks WHERE task_id = ?", (task_id,)
            ).fetchone()
            return dict(row) if row else None
    
    def get_recent_tasks(self, limit: int = 10, intent: Optional[str] = None) -> List[Dict[str, Any]]:
        """最近任务列表，可按意图过滤"""
        with db.get_conn("task_tracker") as conn:
            if intent:
                rows = conn.execute(
                    "SELECT * FROM tasks WHERE intent = ? ORDER BY created_at DESC LIMIT ?",
                    (intent, limit)
                ).fetchall()
            else:
                rows = conn.execute(
                    "SELECT * FROM tasks ORDER BY created_at DESC LIMIT ?", (limit,)
                ).fetchall()
            return [dict(r) for r in rows]
    
    def get_operations(self, task_id: str) -> List[Dict[str, Any]]:
        """获取任务的所有操作（逆序）"""
        with db.get_conn("task_tracker") as conn:
            rows = conn.execute(
                "SELECT * FROM operations WHERE task_id = ? ORDER BY sequence_number DESC",
                (task_id,)
            ).fetchall()
            result = []
            for r in rows:
                d = dict(r)
                if d.get("details"):
                    d["details"] = safe_parse_json(d["details"], label="operation_details")
                result.append(d)
            return result
```

## 六、使用方式

```python
# 单例工厂（线程安全）

import threading
from typing import Optional

_tracker: Optional[TaskTracker] = None
_lock = threading.Lock()

def get_tracker() -> TaskTracker:
    """获取追踪器单例 — 线程安全，所有意图共用"""
    global _tracker
    if _tracker is None:
        with _lock:
            if _tracker is None:  # 双重检查
                _tracker = TaskTracker()
    return _tracker


# Agent 中的使用

class BaseAgent:
    def __init__(self, intent: str, agent_id: str):
        self._intent = intent
        self._agent_id = agent_id
        self._tracker = get_tracker()
        self._current_task_id: Optional[str] = None
    
    def run(self, task: str):
        self._current_task_id = self._tracker.create_task(
            intent=self._intent,
            agent_id=self._agent_id,
            description=task
        )
        try:
            result = self._execute(task)
            self._tracker.complete_task(self._current_task_id, success=True)
            return result
        except Exception:
            self._tracker.complete_task(self._current_task_id, success=False)
            raise
        finally:
            self._current_task_id = None
    
    def _record_operation(self, operation_type: str, **kwargs):
        """任意意图记录操作 — 统一入口"""
        if self._current_task_id:
            self._tracker.add_operation(
                task_id=self._current_task_id,
                operation_type=operation_type,
                **kwargs
            )
```
