## 一、问题背景

### 1.1 原来的4套重试机制

| # | 机制 | 文件 | 调用方 | 流程 |
|---|------|------|--------|------|
| 1 | `_post_with_retry` | `core.py:158` | `BaseAIService.chat()` | 非流式调用，429异常捕获重试 |
| 2 | `_StreamRetryContext` | `core.py:57` | `BaseAIService.stream()` | 流式调用，429状态码判断重试 |
| 3 | `chat_stream_query`重试循环 | `chat_stream_query.py:209` | 流式SSE主循环 | idle_timeout/network_error重试 |
| 4 | `base_react`内联 | `base_react.py` | ReAct循环 | parse_error/empty_response重试 |

### 1.2 核心问题

**_StreamRetryContext存在3个违规点**：
1. **封装违反**：调用`_engine._calculate_delay()`（私有方法）
2. **DRY违反**：手写重试循环，与RetryEngine重复
3. **未用已设计接口**：RetryEngine.execute_context()未使用

## 三、4个重试场景的定位和关系

### 3.1 4个重试场景分别是什么？

| 场景 | 函数 | 所属层 | 功能 | 说明 |
|------|------|--------|------|------|
| **场景1：非流式调用** | `BaseAIService.chat()` | LLM服务层 | 非流式调用 | 发送完整请求，等待完整响应 |
| **场景2：流式调用** | `BaseAIService.stream()` | LLM服务层 | 流式调用 | 发送请求，逐块接收响应 |
| **场景3：流式SSE处理** | `chat_stream_query()` | 聊天流层 | 流式SSE主循环 | 处理流式SSE事件 |
| **场景4：Agent执行** | `BaseAgent.run()` | Agent层 | ReAct循环 | Agent执行任务的主循环 |

### 3.2 4个重试场景的关系

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent层（BaseAgent）                      │
│                                                             │
│  BaseAgent.run()  ←── Agent执行任务的主循环                  │
│       │                                                     │
│       └── 调用LLM获取响应                                    │
│              │                                              │
│              ▼                                              │
├─────────────────────────────────────────────────────────────┤
│                    LLM服务层（BaseAIService）                │
│                                                             │
│  ├── chat()      ←── 非流式调用（返回完整响应）              │
│  │                                                              │
│  └── stream()    ←── 流式调用（返回流式响应）                │
│         │                                                    │
│         └── 使用_StreamRetryContext处理429重试               │
│                │                                            │
│                ▼                                            │
├─────────────────────────────────────────────────────────────┤
│                    聊天流层（chat_stream_query）              │
│                                                             │
│  chat_stream_query()  ←── 处理流式SSE事件                   │
│         │                                                    │
│         └── 使用RetryCounter处理重试                         │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 4个重试场景是4个分支还是4个阶段？

**答案：是4个不同的使用场景/分支，不是流程的4个阶段。**

| 场景 | 函数 | 说明 |
|------|------|------|
| **场景1：非流式调用** | `BaseAIService.chat()` | 用户发送消息，等待完整响应 |
| **场景2：流式调用** | `BaseAIService.stream()` | 用户发送消息，逐块接收响应 |
| **场景3：流式SSE处理** | `chat_stream_query()` | 处理流式SSE事件 |
| **场景4：Agent执行** | `BaseAgent.run()` | Agent执行任务的主循环 |

### 3.4 4个重试场景的调用关系图

```
用户发送消息
    │
    ├── 场景1：非流式调用
    │       └── BaseAIService.chat()
    │               └── _post_with_retry() → RetryEngine
    │
    ├── 场景2：流式调用
    │       └── BaseAIService.stream()
    │               └── _StreamRetryContext → RetryEngine.execute_async_context()
    │
    ├── 场景3：流式SSE处理
    │       └── chat_stream_query()
    │               └── RetryCounter（纯计数器）
    │
    └── 场景4：Agent执行
            └── BaseAgent.run()
                    └── RetryEngine（空响应重试）
```

### 3.5 4个重试场景的区别

| 维度 | chat() | stream() | chat_stream_query() | BaseAgent.run() |
|------|--------|----------|---------------------|-----------------|
| **调用方式** | 同步等待 | 异步流式 | 异步流式 | 异步循环 |
| **响应类型** | 完整响应 | 流式响应 | SSE事件 | Agent执行结果 |
| **重试机制** | RetryEngine | RetryEngine.execute_async_context() | RetryCounter | RetryEngine |
| **使用场景** | 简单问答 | 实时对话 | 流式展示 | 任务执行 |

| 维度 | 说明 |
|------|------|
| **重试机制数量** | 4套（无变化） |
