## 一、问题背景

### 1.1 为什么要区分错误类型

当前后端代码的错误处理存在以下问题：

1. **错误处理不统一**：分散在多处调用 `create_error_response()` + `create_error_step()`
2. **没有区分错误场景**：工具执行错误和会话级错误混为一谈
3. **杀死LLM继续的机会**：工具错误后直接yield error，相当于"杀死"了LLM继续思考的机会

**说明**：本设计方案基于第2、3、4章的设计要求，用于指导后续代码重构。第7、8、9章是具体的修改说明。

## 二、错误分类设计

### 2.1 两种错误场景

```
错误发生后的两种情况：

┌─────────────────────────────────────────────────────────────┐
│ 场景A：会话级错误（Session-Level Error）                     │
├─────────────────────────────────────────────────────────────┤
│ - API超时耗尽、认证失败、API完全不可用                       │
│ - 发生错误后就是最后一步了                                   │
│ - 需要：yield error → 结束会话 → 保存DB                     │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 场景B：工具级错误（Tool-Level Error）                       │
├─────────────────────────────────────────────────────────────┤
│ - 文件操作失败、网络请求失败、命令执行错误                     │
│ - 发生错误后可能不是最后一步，LLM会收到observation           │
│ - LLM判断：继续循环 or 进入final                           │
│ - 需要：复用action_tool字段 → 传给LLM观察                   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 错误分类对照表

| 错误类型 | 分类 | 说明 | 处理方式 |
|---------|------|------|---------|
| API超时耗尽 | 会话级 | 重试3次后仍然超时 | 结束会话 |
| 认证/权限失败 | 会话级 | API key无效、权限不足 | 结束会话 |
| 安全拦截 | 会话级 | 危险操作被拦截 | 结束会话 |
| API完全不可用 | 会话级 | 429、500等 | 结束会话 |
| 工具执行失败 | 工具级 | 文件操作、网络请求失败 | 传给LLM决定 |
| 命令执行错误 | 工具级 | shell命令返回非0 | 传给LLM决定 |
| 工具未实现 | 会话级 | 功能模块尚未实现 | 结束会话 |

### 3.1 现有字段结构

### 3.2 工具级错误必须复用的字段

**工具级错误完全复用 `action_tool` 类型，不需要新增字段：**

| 字段 | 正常值 | 错误时值 | 说明 |
|------|--------|---------|------|
| `type` | `'action_tool'` | `'action_tool'` | ✅ 保持不变 |
| `step` | 数字 | 数字 | ✅ 保持不变 |
| `tool_name` | 工具名 | 工具名 | ✅ 保持不变 |
| `tool_params` | 参数 | 参数 | ✅ 保持不变 |
| `execution_status` | `'success'` | `'error'` | ✅ **关键：标记为错误** |
| `summary` | 成功摘要 | 错误信息 | ✅ **放错误消息** |
| `raw_data` | 原始数据 | 详细错误 | ✅ **可选：详细错误信息** |
| `action_retry_count` | 0 | 重试次数 | ✅ **放重试次数** |

### 3.3 现有字段覆盖情况

| 需求 | 现有字段 | 是否满足 | 说明 |
|------|---------|---------|------|
| 标记错误 | `execution_status='error'` | ✅ 已满足 | 用 `'error'` 标记 |
| 错误消息 | `summary` | ✅ 已满足 | summary放错误信息 |
| 详细错误 | `raw_data` | ✅ 已满足 | 可放完整错误堆栈 |
| 重试次数 | `action_retry_count` | ✅ 已满足 | 已有字段 |
| 工具名称 | `tool_name` | ✅ 已满足 | 保持不变 |
| 工具参数 | `tool_params` | ✅ 已满足 | 保持不变 |
| 错误码 | - | ⚠️ 可扩展 | summary可包含错误码 |

### 3.4 可能需要新增的字段

**如果确实需要，可以考虑新增：**

| 字段名 | 类型 | 说明 | 优先级 |
|--------|------|------|--------|
| `error_code` | `str` | 错误码（如 `FILE_NOT_FOUND`） | 可选：`summary` 已包含 |
| `is_retryable` | `bool` | 是否可重试 | 可选：已有 `action_retry_count` 暗示 |
| `error_type` | `str` | 错误分类（如 `file_error`） | 可选：已有 `execution_status='error'` 暗示 |

**优先级原则**：
1. ✅ **优先使用现有字段**
2. ⚠️ **确实不够用时再新增**
3. ❌ **不要为了"完美"而新增字段**

## 四、核心设计

### 4.1 函数设计

```python
# ============================================================
# 会话级错误 - 统一的函数
# ============================================================

def create_session_error_result(
    error_type: str,           # 错误类型标识
    original_error: str,        # 原始错误消息
    step_num: int,
    model: str = None,
    provider: str = None,
    display_name: str = None,
    chat_timeout: int = None,
    max_retries: int = None,
    retryable: bool = True,
    retry_after: int = 3,
) -> tuple[str, dict]:
    """
    统一的会话级错误处理函数
    
    用于：
    - 聊天API超时耗尽
    - 认证/权限失败
    - 安全拦截
    - API完全不可用
    - 功能未实现
    
    特点：
    - 发生错误后就是最后一步
    - yield给前端 + 保存到DB + 结束会话
    """
    # 1. 解析错误码
    resolved_type = resolve_http_error_type(original_error) if original_error else None
    final_type = resolved_type if resolved_type else error_type
    
    # 2. 组装错误信息
    error_code, error_message = get_stream_error_info(final_type, original_error)
    
    # 3. 特殊情况处理
    if error_type == 'idle_timeout' and display_name and chat_timeout and max_retries:
        total_timeout = chat_timeout * max_retries
        error_message = f"请求超时：AI模型({display_name}) {chat_timeout}秒内未返回任何内容，已重试{max_retries}次，合计{total_timeout}秒"
    
    # 4. 创建 error_response（yield给前端）
    error_response = create_error_response(
        error_type=error_code,
        message=error_message,
        model=model,
        provider=provider,
        retryable=retryable,
        retry_after=retry_after,
        step=step_num
    )
    
    # 5. 创建 error_step（保存到DB）
    error_step = create_error_step(
        type='error',
        code=error_code,
        message=error_message,
        error_type=error_code,
        step_num=step_num,
        model=model,
        provider=provider,
        retryable=retryable,
        retry_after=retry_after
    )
    
    return error_response, error_step


# ============================================================
# 工具级错误 - 复用现有 format_action_tool_sse
# ============================================================

def create_tool_error_result(
    tool_name: str,            # 工具名称
    error_message: str,        # 错误消息
    step_num: int,
    tool_params: dict = None,  # 工具参数
    retry_count: int = 0,      # 当前重试次数
    max_retries: int = 3,     # 最大重试次数
    raw_data: Any = None,      # 详细错误信息（可选）
) -> dict:
    """
    统一的工具级错误处理函数
    
    【重要】必须复用现有 action_tool 字段，不另搞一套！
    
    用于：
    - 文件操作失败
    - 网络请求失败
    - 命令执行错误
    
    特点：
    - 发生错误 ≠ 最后一步
    - LLM会收到这个作为 observation
    - LLM决定：继续循环 or 进入final
    - 复用 format_action_tool_sse() 格式化
    """
    # 计算重试状态
    can_retry = retry_count < max_retries
    
    # 构建错误摘要
    if can_retry:
        summary = f"[错误] {tool_name} 执行失败: {error_message}，正在重试 ({retry_count + 1}/{max_retries})..."
    else:
        summary = f"[错误] {tool_name} 执行失败: {error_message}，已重试{max_retries}次"
    
    # 【关键】复用 format_action_tool_sse()，不需要新建函数
    # 返回 dict，由调用方用 format_action_tool_sse() 格式化
    return {
        'tool_name': tool_name,
        'tool_params': tool_params or {},
        'execution_status': 'error',     # 标记为错误
        'summary': summary,             # 错误摘要
        'raw_data': raw_data or error_message,  # 详细错误
        'action_retry_count': retry_count,
        'step': step_num,
        'timestamp': create_timestamp()
    }
```

async def run_stream(self, ...):
    for step in range(max_steps):

            # 调用工具
            result = await self.execute_tool(tool_name, tool_params)

## 七、工具级重试机制

### 7.1 核心问题

**不是所有错误都适合重试！**

盲目重试同样的tool不会成功：
- 文件不存在 → 重试100次也不会存在
- 权限不足 → 重试不会改变权限
- 工具内部bug → 重试不会解决

### 7.2 错误分类

| 错误类型 | 是否可重试 | 原因 | 示例 |
|---------|-----------|------|------|
| **可重试错误** | ✅ | 临时故障，可能自动恢复 | 网络超时、连接断开、服务暂时不可用 |
| **不可重试错误** | ❌ | 永久故障，重试不会改变结果 | 文件不存在、权限不足、工具内部错误 |

### 7.3 错误模式定义

```python
# tool_executor.py 中的错误分类

# 可重试错误模式（Transient Error）
TRANSIENT_ERROR_PATTERNS = [
    "timeout",           # 超时
    "timed out",        # 超时
    "connection",       # 连接问题
    "network",          # 网络问题
    "temporarily unavailable",  # 暂时不可用
    "service unavailable",     # 服务不可用
    "refused",          # 连接被拒绝
    "reset",            # 连接重置
]

# 不可重试错误模式（Permanent Error）
PERMANENT_ERROR_PATTERNS = [
    "not found",        # 文件/资源不存在
    "does not exist",   # 不存在
    "no such file",     # 文件不存在
    "permission denied", # 权限不足
    "access denied",    # 访问被拒绝
    "invalid",          # 无效参数
    "forbidden",        # 禁止访问
    "unauthorized",     # 未授权
]
```

### 7.4 重试策略

```python
async def execute(self, action, action_input):
    """
    执行工具调用（带智能重试）
    """
    tool = self.available_tools[action]
    
    try:
        result = await tool(**params)
        return self._format_result(result, action)
    
    except Exception as e:
        error_msg = str(e).lower()
        
        # ===== 判断错误类型 =====
        if self._is_transient_error(error_msg):
            # 可重试错误：自动重试
            return await self._execute_with_retry(action, params, error_msg)
        else:
            # 不可重试错误：立即返回
            return {
                "status": "error",
                "summary": f"[不可重试] {str(e)}",
                "data": None,
                "retryable": False,  # 标记为不可重试
                "retry_count": 0
            }
    
    async def _execute_with_retry(self, action, params, original_error):
        """带退避的重试机制"""
        max_retries = 3
        last_error = original_error
        
        for retry in range(max_retries):
            try:
                # 退避等待（指数退避）
                if retry > 0:
                    wait_time = 1 * (2 ** retry)  # 1s, 2s, 4s
                    await asyncio.sleep(wait_time)
                
                # 尝试执行
                result = await self.available_tools[action](**params)
                return self._format_result(result, action)
            
            except Exception as e:
                last_error = str(e)
                # 如果是不可重试错误，立即退出
                if not self._is_transient_error(last_error):
                    return {
                        "status": "error",
                        "summary": f"[不可重试] {last_error}",
                        "data": None,
                        "retryable": False,
                        "retry_count": retry + 1
                    }
                continue
        
        # 重试3次都失败
        return {
            "status": "error",
            "summary": f"[重试耗尽] 已重试{max_retries}次仍失败: {last_error}",
            "data": None,
            "retryable": False,
            "retry_count": max_retries
        }
    
    def _is_transient_error(self, error_msg: str) -> bool:
        """判断是否为可重试错误"""
        # 先检查不可重试模式
        for pattern in PERMANENT_ERROR_PATTERNS:
            if pattern in error_msg:
                return False  # 不可重试
        
        # 再检查可重试模式
        for pattern in TRANSIENT_ERROR_PATTERNS:
            if pattern in error_msg:
                return True  # 可重试
        
        # 默认不可重试
        return False
```

### 7.5 LLM收到的message

**无论重试成功还是失败，LLM都收到完整的上下文：**

```python
# base_react.py 中的处理

# 工具执行完成后
raw_data = execution_result.get('data')

if execution_result.get('status') == 'error':
    # 错误情况
    retryable = execution_result.get('retryable', False)
    retry_count = execution_result.get('retry_count', 0)
    
    if retryable and retry_count > 0:
        # 重试中...
        observation_text = f"Observation: 重试中 ({retry_count}/3) - {execution_result.get('summary', '')}"
    elif retry_count >= 3:
        # 重试耗尽
        observation_text = f"Observation: [失败] {execution_result.get('summary', '')}，该错误不可重试，请尝试其他方法"
    else:
        # 不可重试错误
        observation_text = f"Observation: [不可重试错误] {execution_result.get('summary', '')}，请尝试其他方法"
else:
    # 成功情况
    if raw_data:
        observation_text = f"Observation: 成功 - {execution_result.get('summary', '')}\n实际数据: {raw_data}"
    else:
        observation_text = f"Observation: 成功 - {execution_result.get('summary', '')}"

# 添加到对话历史，LLM据此决定下一步
self._add_observation_to_history(observation_text)
```

```python
def create_tool_error_result(
    tool_name: str,            # 工具名称
    error_message: str,        # 错误消息
    step_num: int,
    tool_params: dict = None,  # 工具参数
    retry_count: int = 0,      # 当前重试次数
    max_retries: int = 3,     # 最大重试次数
    raw_data: Any = None,      # 详细错误信息（可选）
) -> dict:
    """
    统一的工具级错误处理函数
    用于：文件操作失败、网络请求失败、命令执行错误
    特点：发生错误 ≠ 最后一步，LLM会收到这个作为observation
    """
    # 构建错误摘要
    can_retry = retry_count < max_retries
    if can_retry:
        summary = f"[错误] {tool_name} 执行失败: {error_message}，正在重试 ({retry_count + 1}/{max_retries})..."
    else:
        summary = f"[错误] {tool_name} 执行失败: {error_message}，已重试{max_retries}次"
    
    # 返回dict，由调用方用 format_action_tool_sse() 格式化
    return {
        'tool_name': tool_name,
        'tool_params': tool_params or {},
        'execution_status': 'error',
        'summary': summary,
        'raw_data': raw_data or error_message,
        'action_retry_count': retry_count,
        'step': step_num,
        'timestamp': create_timestamp()
    }
```

**新增代码示意**：

```python
# === 在 tool_executor.py 文件开头新增 ===

# 可重试错误模式（Transient Error）
TRANSIENT_ERROR_PATTERNS = [
    "timeout", "timed out", "connection", "network",
    "temporarily unavailable", "service unavailable",
    "refused", "reset", "502", "503", "504"
]

# 不可重试错误模式（Permanent Error）
PERMANENT_ERROR_PATTERNS = [
    "not found", "does not exist", "no such file",
    "permission denied", "access denied", "invalid",
    "forbidden", "unauthorized", "401", "403", "404"
]

# === 在 execute() 方法中修改 ===

async def execute(self, action, action_input):
    tool = self.available_tools[action]
    params = action_input or {}
    
    try:
        result = await tool(**params)
        return self._format_result(result, action)
    
    except Exception as e:
        error_msg = str(e).lower()
        
        # 判断错误类型，决定是否重试
        if self._is_transient_error(error_msg):
            # 可重试错误：调用重试方法
            return await self._execute_with_retry(action, params, str(e))
        else:
            # 不可重试错误：立即返回
            return {
                "status": "error",
                "summary": f"[不可重试] {str(e)}",
                "data": None,
                "retryable": False,      # 标记为不可重试
                "retry_count": 0
            }

# === 新增辅助方法 ===

async def _execute_with_retry(self, action, params, original_error):
    """带退避的重试机制"""
    max_retries = 3
    last_error = original_error
    
    for retry in range(max_retries):
        try:
            # 指数退避：1s, 2s, 4s
            if retry > 0:
                wait_time = 1 * (2 ** retry)
                await asyncio.sleep(wait_time)
            
            # 再次尝试执行
            tool = self.available_tools[action]
            result = await tool(**params)
            return self._format_result(result, action)
        
        except Exception as e:
            last_error = str(e)
            # 检查是否变为不可重试错误
            if not self._is_transient_error(last_error.lower()):
                return {
                    "status": "error",
                    "summary": f"[不可重试] {last_error}",
                    "data": None,
                    "retryable": False,
                    "retry_count": retry + 1
                }
            continue
    
    # 重试3次都失败
    return {
        "status": "error",
        "summary": f"[重试耗尽] 已重试{max_retries}次仍失败: {last_error}",
        "data": None,
        "retryable": False,
        "retry_count": max_retries
    }

def _is_transient_error(self, error_msg: str) -> bool:
    """判断是否为可重试错误"""
    # 先检查不可重试模式（优先）
    for pattern in PERMANENT_ERROR_PATTERNS:
        if pattern in error_msg:
            return False
    
    # 再检查可重试模式
    for pattern in TRANSIENT_ERROR_PATTERNS:
        if pattern in error_msg:
            return True
    
    # 默认不可重试
    return False
```

## 十、核心原则

### 10.1 必须遵守的原则

1. **错误分类**：区分会话级错误和工具级错误
2. **工具级错误复用 action_tool**：`execution_status = 'error'` + `summary = 错误信息`
