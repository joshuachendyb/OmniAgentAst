## 🟡 问题三：时间戳信息显示不完整

### 问题位置

`MessageItem.tsx:412-422`

### 视觉现状

```tsx
// 只有 start 类型显示时间戳
{step.timestamp && (
  <span style={{ 
    color: Colors.TEXT.TERTIARY,
    backgroundColor: Colors.BG.SECONDARY,
    padding: "2px 8px",
    borderRadius: 4,
    fontSize: FontSize.TERTIARY,
  }}>
    {formatTimestamp(step.timestamp)}
  </span>
)}
```

### 问题分析

1. **仅start步骤有时间戳**：其他步骤类型（thought、action_tool、observation等）不显示时间戳
2. **用户无法追踪耗时**：无法知道每个步骤的具体执行时间
3. **调试困难**：出错时无法定位是哪个步骤耗时过长

### ⚠️ 特殊说明（小资补充）

**chunk类型特殊处理**：
- chunk是流式内容片段，不是独立执行步骤
- **chunk不应显示step序号和时间戳**，保持现有显示方式即可
- 时间戳和序号只对thought、action_tool、observation等独立步骤有意义

### 可行改进方案

**方案A：除chunk外的所有步骤显示相对时间（推荐）**

```tsx
// 在每个步骤右上角显示相对时间（chunk类型除外）
const StepTimestamp: React.FC<{ 
  timestamp?: string;
  stepType: string;  // 新增：判断是否为chunk
  isFirst?: boolean;
}> = ({ timestamp, stepType, isFirst }) => {
  // ⚠️ chunk类型不显示时间戳（小资补充）
  if (!timestamp || stepType === "chunk") return null;
  
  // 计算相对时间（相对于第一个步骤）
  const getRelativeTime = (ts: string, baseTs?: string) => {
    const date = new Date(ts);
    const baseDate = baseTs ? new Date(baseTs) : new Date();
    const diffMs = date.getTime() - baseDate.getTime();
    
    if (diffMs < 1000) return `${diffMs}ms`;
    if (diffMs < 60000) return `${(diffMs / 1000).toFixed(1)}s`;
    return `${Math.floor(diffMs / 60000)}m${Math.floor((diffMs % 60000) / 1000)}s`;
  };
  
  return (
    <span style={{
      position: "absolute",
      right: 8,
      top: 8,
      fontSize: 10,
      color: "#8c8c8c",
      backgroundColor: "#f5f5f5",
      padding: "2px 6px",
      borderRadius: 4,
    }}>
      {isFirst ? formatTimestamp(timestamp) : `+${getRelativeTime(timestamp)}`}
    </span>
  );
};
```

**实现复杂度**: 低
**预期效果**: 用户可直观了解每个步骤的执行时间
