## 1 Omni 系统工具定义说明书 6.1 章节参数分析方法论

### 1.1 分析框架：基于 5.5.1 原则的"三问法"

对每一个 Tool 的每一个参数，必须连续问自己三个问题：

| 问题 | 目的 | 判断标准 |
|------|------|---------|
| **Q1：LLM 必须提供吗？** | 区分必填/可选 | 没有它 Agent 完全无法执行 → 必填；否则 → 可选 |
| **Q2：Agent 能智能补全吗？** | 挖掘缺少的可选参数 | Agent 能否根据环境、上下文、文件类型自动设置？能 → 补充为可选参数 |
| **Q3：参数描述准确吗？** | 确保符合 5.5.1 精神 | 是否混淆了 LLM 和 Agent 的职责？是否写了冗余描述？ |

### 1.2 执行流程：五步走

```
第 1 步：逐 Tool 通读现有参数
    ↓
第 2 步：按"场景维度"挖掘缺失参数
    ├─ 编码处理场景（encoding）
    ├─ 安全确认场景（overwrite/force）
    ├─ 搜索增强场景（ignore_case/type/include_hidden）
    ├─ 目录操作场景（create_parents/recursive）
    └─ 系统信息场景（follow_symlinks/max_depth）
    ↓
第 3 步：补充到参数分析表 + Function 定义
    ↓
第 4 步：交叉审查（小健角色）
    ├─ 命名是否统一？
    ├─ 默认值是否安全？
    ├─ 描述是否准确？
    └─ 是否违背 5.5.1？
    ↓
第 5 步：修正并同步两处（分析表 + Function 定义）
```

### 1.3 发现的 4 类典型问题及纠正

| 问题类型 | 具体表现 | 纠正方法 | 教训 |
|---------|---------|---------|------|
| **1. 参数缺失** | 只关注核心参数，忽略编码、安全、搜索增强等细节 | 按"场景维度"逐个排查，补充 15 个可选参数 | **不能囫囵吞枣，必须逐个 Tool 深入思量** |
| **2. 命名不统一** | `case_sensitive` vs `ignore_case`；`show_hidden` vs `include_hidden` | 全文统一命名，正负逻辑保持一致 | **命名统一是 Agent 开发的基础，否则需写两套转换逻辑** |
| **3. 默认值不安全** | `copy_file` 的 `overwrite` 默认 `true` | 改为 `false`，遵循"安全优先"原则 | **AI Agent 默认覆盖可能误删重要文件** |
| **4. 描述冗余** | "若 LLM 明确指定，则优先使用 LLM 的值" | 全部删除，明确 Agent 自主处理边界 | **LLM 不需要关心的参数，不要画蛇添足** |

### 1.4 核心原则（必须死守）

| 原则 | 说明 | 反例 |
|------|------|------|
| **可选参数是 Agent 智能补全的载体** | 每个可选参数都必须有明确的 Agent 补全策略 | 写死默认值，不让 Agent 动态判断 |
| **命名必须全局统一** | 同类功能用同一参数名，正负逻辑不混用 | `case_sensitive` 和 `ignore_case` 混用 |
| **默认值安全优先** | 涉及覆盖、删除、修改的默认值必须保守 | `overwrite: true` |
| **描述忠于 5.5.1** | 只说 Agent 怎么处理，不写 LLM 权限边界 | "若 LLM 明确指定..." |
| **分析表与 Function 定义必须同步** | 两处参数名、类型、描述必须完全一致 | 只改一处，另一处遗漏 |

### 1.5 经验沉淀：后续章节复用检查清单

后续分析 6.2-6.10 章节时，直接套用以下检查清单：

- [ ] **编码类**：所有读写文件的 Tool 是否都有 `encoding` 参数？
- [ ] **安全类**：覆盖/删除/修改类 Tool 是否有 `overwrite`、`force` 参数？默认值是否为 `false`？
- [ ] **搜索类**：搜索类 Tool 是否有 `ignore_case`、`include_hidden`、`type` 参数？命名是否统一？
- [ ] **目录类**：目录操作 Tool 是否有 `create_parents`、`recursive` 参数？
- [ ] **系统类**：信息获取 Tool 是否有 `follow_symlinks`、`max_depth` 参数？
- [ ] **描述类**：是否删除了所有"若 LLM 明确指定..."的冗余描述？
- [ ] **同步类**：参数分析表和 Function 定义是否完全一致？

### 1.6 总结一句话

**参数分析不是"数参数"，而是"想场景"。每一个可选参数都是 Agent 智能补全的抓手，必须逐个 Tool 深入思量，绝不能囫囵吞枣。**
