### 1.4 Key Takeaways（對第5章的設計啟發）

| OpenCode做法 | 我們設計要點（差異） |
|-------------|------------------|
| 純ReAct循環（while true: LLM→tool→loop） | 我們拆為兩步：執行計劃(Plan)→按計劃執行(Execute)，非純ReAct |
| Tool執行順序（for循環，無並行） | ArtTool是即興生成代碼+沙箱執行，非預註冊靜態工具 |
| ProviderEvent 8種事件 | 我們的LLMChunk更簡潔（text/tool_call/tool_result/error） |
| Session級併發控制 | 我們直接複用 |
| 上下文注入（AGENTS.md/project context） | 我們的經驗學習機制是持久化+檢索+注入，比靜態文件更動態 |
| Summarizer獨立provider無tool | 我們的經驗總結直接寫入DB，非LLM生成摘要 |
| SubAgent as tool（嵌套代理） | 類似我們的FlowTool概念，但我們的FlowTool是執行計劃的子任務 |
