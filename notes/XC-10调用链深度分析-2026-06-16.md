## 2 根因分析（真正原因）

### 关键发现：observation内容已正确传递给LLM

**数据库验证**：
- Observation 24 (step 24): search_web结果 - 10352字符，包含完整10条搜索结果
- Observation 26 (step 26): search_web结果 - 9964字符，包含完整10条搜索结果
- Observation 28 (step 28): search_web结果 - 9412字符，包含完整10条搜索结果

**结论**：observation内容**已正确插入**给LLM，数据传递没有问题。

### 真正根因：trim_history导致上下文丢失

**trim_history逻辑**（message_builder.py:142-156）：
```python
def trim_history(self) -> None:
    total = self._total_chars(self.conversation_history)
    if total < self.MAX_CONTEXT_CHARS * 0.8:  # 150000 * 0.8 = 120000
        return
    # 从最旧消息开始裁剪
    available_budget = max(10000, int(self.MAX_CONTEXT_CHARS * 0.7) - system_chars)
    trimmed = self._trim_to_budget(obs_list, assistant_msgs, available_budget)
```

**XC-10计算**：
- MAX_CONTEXT_CHARS = 150000
- trim_history触发条件：total > 120000
- 可用预算：≈ 100000字符
- XC-10有31个observation，每个约10000字符
- 总observation：31 * 10000 = 310000字符
- **超出预算210000字符，旧observation被裁剪！**

**后果**：
- 第一次search_web的observation（step 2）被裁剪掉
- LLM不记得已经搜索过
- 所以重复搜索相同的query

### 对比：XC-04为什么成功？

**XC-04**：
- 只有5个observation（tool_search, search_web, fetch_webpage ×2, send_notification）
- 总observation约50000字符
- **在budget内，不会触发trim_history**
- LLM记得完整的调用链

**XC-10**：
- 有31个observation
- 总observation约310000字符
- **超出budget，触发trim_history**
- LLM丢失早期上下文

### 根因总结

| 问题 | 原因 | 影响 |
|------|------|------|
| observation内容丢失 | trim_history裁剪旧消息 | LLM不记得已搜索 |
| LLM重复搜索 | 丢失早期搜索记录 | 陷入搜索循环 |
| 任务未完成 | 超时导致失败 | 最终超时 |

## 4 真正的解决方案

### 解决方案1：增加observation budget（推荐）

**位置**: `backend/app/constants.py`

**方案**：
```python
# 增加observation budget

```

**效果**：
- 每个observation可以保留更多内容
- 减少trim_history触发频率
- LLM可以保留更多上下文

### 解决方案2：优化trim_history策略

**位置**: `backend/app/services/agent/message_builder.py`

**方案**：
```python
def trim_history(self) -> None:
    """容量感知的对话历史裁剪 — 保留关键上下文"""
    total = self._total_chars(self.conversation_history)
    if total < self.MAX_CONTEXT_CHARS * 0.9:  # 从0.8增加到0.9
        return
    if len(self.conversation_history) <= 2:
        return

    system_msgs, obs_list, assistant_msgs = self._classify_messages()
    system_chars = self._total_chars(system_msgs)
    available_budget = max(20000, int(self.MAX_CONTEXT_CHARS * 0.8) - system_chars)  # 增加budget
    trimmed = self._trim_to_budget(obs_list, assistant_msgs, available_budget)
    rebuilt = self._rebuild_and_validate(system_msgs, trimmed)
    if rebuilt is not None:
        self.conversation_history = rebuilt
```
