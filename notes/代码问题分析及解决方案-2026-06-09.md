```python
def _get_openai_tools(self) -> list:

    current_time = time.time()
    if (self._cached_openai_tools and
        current_time - self._cache_timestamp < self._cache_ttl):
        return self._cached_openai_tools
    from app.services.tools.registry import tool_registry
    category = getattr(self, 'tool_category', None)
    self._cached_openai_tools = tool_registry.to_openai_tools(category=category)

```

### 4.4 时区-001

**文件**: `backend/app/utils/time_utils.py:20-27`
**验证时间**: 2026-06-09 20:47:47

**现状代码**:
```python
def create_timestamp() -> int:
    """生成统一的时间戳(毫秒)"""
    return int(datetime.now().timestamp() * 1000)

def get_timestamp_ms() -> int:
    """获取毫秒时间戳(UTC)"""
    return int(datetime.now(timezone.utc).timestamp() * 1000)
```

**问题分析**:
create_timestamp 用 datetime.now()（本地时间），get_timestamp_ms 用 datetime.now(timezone.utc)（UTC）。虽然 .timestamp() 最终都转为 POSIX 时间戳（本质 UTC），但在夏令时切换等边界情况下可能产生微妙差异。更关键的是语义混乱：同一事件的 Step 时间戳用本地时间，数据库记录时间用 UTC，差8小时。

**解决方法**:
统一使用 UTC：
```python
def create_timestamp() -> int:
    """生成统一的时间戳(毫秒, UTC)"""
    return int(datetime.now(timezone.utc).timestamp() * 1000)
```

**修复难度**: 低（改一行代码）
