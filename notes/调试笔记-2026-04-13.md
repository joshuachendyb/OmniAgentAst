用户在聊天时发现：AI 响应的每个 step（start/thought/action_tool/observation/final）没有逐个显示，而是等所有 step 收到完成后才一次性显示。

```
20:01:38 收到 observation
20:01:51 收到 final
20:01:51 AI响应完成END (所有step收到后才显示)
```

## 3 问题定位

### 版本历史确认

| 版本 | 时间 | 功能状态 |
|------|------|---------|
| v0.9.0 | 2026-04-12 18:14:25 | ✅ step逐个显示正常 |
| v0.9.4 | 2026-04-13 15:15:00 | ❌ step不逐个显示 |

### v0.9.4 改动了什么？

**v0.9.4 性能优化**（version.txt 第 132-177 行）：
- 实施 50ms throttle + ref累积 + leading edge 策略
- 减少 setMessages 调用次数（55次 → 5-10次）
- **关键改动**：`UPDATE_INTERVAL = 50`（后来改成0）

**当前代码状态**：
```typescript
// NewChatContainer.tsx 第155行
const UPDATE_INTERVAL = 0;  // 已是0ms，无throttle
```

### 真正的根因

**不是 throttle 的问题！**

throttle 已经设置为 0ms，每次 step 都应该更新。**真正的根因是：前端根本没有渲染每个 step 的详细 content！**

现在的渲染逻辑：

| 渲染内容 | 位置 | 说明 |
|----------|------|------|
| `message.content` | MessageItem.tsx:1249-1259 | 只渲染 final 的文本内容 |
| `DynamicStatusDisplay` | MessageItem.tsx:1265-1269 | 只显示状态文字（如"正在思考"），不显示 step 详细内容 |

### 问题代码位置

`MessageItem.tsx` 第 1206-1269 行：

```tsx
// 现有逻辑：
// 1. 先判断 hasAction（有没有 action_tool）
// 2. 然后判断 hasChunk、hasErrorStep 等
// 3. 最后只显示 message.content
// 4. DynamicStatusDisplay 只显示状态，不显示 step 内容
```

**结论**：代码已有 `StepRow` 组件渲染每个 step，在消息气泡内部显示。

### 确认问题

**StepRow 在消息气泡内渲染**，使用过滤逻辑：
- chunk 和 final 在有 chunk 时被过滤掉

**用户期望**可能是：
- A) 在消息气泡下方显示？
- B) 在 ExecutionPanel（单独的底部面板）中显示？

### 需要确认

请确认 v0.9.0 时 step 逐个显示的位置？

## 4 解决方案

### 4.1 方案A：在 MessageItem.tsx 添加 step 内容渲染（推荐）

**思路**：直接修改 MessageItem.tsx，在渲染内容时同时渲染每个 step 的详细内容。

**优点**：
- 代码改动集中在一处
- 不影响现有数据流
- 可控性强

**缺点**：
- 需要修改 MessageItem.tsx

---

### 4.2 方案B：修改 NewChatContainer.tsx 的 step 渲染时机

**思路**：在 onStep 回调时立即触发渲染，而不是等 50ms throttle。

**改动位置**：
- `NewChatContainer.tsx` 第 287-292 行的 throttle 逻辑
- 移除 throttle，每次收到 step 都触发 setMessages

**代码改动**：
```typescript
// 移除 throttle 逻辑，改为每次都更新
setMessages((prev) => {
  // ... 更新逻辑不变
});
```

**优点**：
- 真正实现"逐个显示"

**缺点**：
- 可能导致性能问题（每个 step 都触发重渲染）
- 需要配合节流/防抖

---

### 4.3 方案C：复用现有的 DynamicStatusDisplay

**思路**：扩展 DynamicStatusDisplay 组件，显示每个 step 的内容。

**改动位置**：
- `dynamicStatus.tsx` 组件
- 或者在 MessageItem.tsx 中添加新组件

**优点**：
- 复用现有代码

**缺点**：
- 需要较大改动

---

### 4.5 实施选择：方案A

**改动位置**: `MessageItem.tsx` 第 1259 行之后（`</div>` 前面）

**改动位置**: `MessageItem.tsx` 内容显示区域

**改动代码示例**:

```tsx
{/* 【新增】渲染每个 step 的内容 */}
{message.executionSteps && message.executionSteps.length > 0 && (
  <div className="execution-steps-container" style={{ marginTop: 8 }}>
    {message.executionSteps.map((step, index) => {
      // 只渲染有内容的 step
      if (!step.content && step.type !== 'start') return null;
      
      // step 类型的中文标签
      const stepLabel: Record<string, string> = {
        start: '🚀 开始',
        thought: '💭 思考',
        action_tool: '🔧 工具',
        observation: '👁️ 观察',
      };
      
      return (
        <div 
          key={index}
          style={{
            padding: '8px 12px',
            margin: '4px 0',
            background: step.type === 'action_tool' ? '#f0f7ff' : '#f9f9f9',
            borderRadius: 6,
            borderLeft: step.type === 'action_tool' ? '3px solid #2196F3' : '3px solid #ddd',
            fontSize: '0.9em',
          }}
        >
          <div style={{ fontWeight: 600, color: '#666', marginBottom: 4 }}>
            {stepLabel[step.type] || step.type} · {step.timestamp || ''}
          </div>
          <div style={{ whiteSpace: 'pre-wrap', wordBreak: 'break-word' }}>
            {step.content}
          </div>
        </div>
      );
    })}
  </div>
)}
```
