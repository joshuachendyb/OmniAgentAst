## 3 配置同步机制说明 【技术实现】2026-02-22 19:03:43

### 3.1 同步机制总览

| 场景 | 同步方式 | 无需重启 |
|------|---------|---------|
| 人为修改配置文件 | 后端reload() + 前端点击下拉框 | ✅ |
| 设置页面修改 | 前端点击下拉框自动刷新 | ✅ |

### 3.2 同步流程

```
配置文件变更 → 后端API请求时reload() → 读取最新配置 → 前端点击下拉框(onDropdownVisibleChange) → 获取最新模型列表
```

### 3.3 关键代码

**后端** (`config.py`):
```python
# 每次API请求时重新加载配置
config = get_config_instance()
config.reload()
```

**前端** (`Layout/index.tsx`):
```typescript
// 打开下拉框时刷新
onDropdownVisibleChange={async (open) => {
  if (open) {
    await refreshModelList();
  }
}}
```

### 5.1 问题发现

**问题1：使用的组件不对**

| 问题 | 说明 |

**根本原因**：前端页面加载的是 `Chat/index.tsx`，不是修改的 `ChatContainer.tsx`！

**问题2：api.ts 中 sendMessageStream 没有传 model**

文件：`frontend/src/services/api.ts` 第192-193行
```typescript
} else if (data.type === 'final') {
  callbacks.onComplete(data.content);  // 没有传model！
}
```

**问题3：Chat/index.tsx 没有设置 model**

文件：`frontend/src/components/Chat/index.tsx` 第212-240行
- 创建消息时没有 model 字段
- onComplete 回调没有更新 model

### 5.2 修复方案（优先解决问题1和问题2）

#### 修复1：api.ts - sendMessageStream 传递 model

```typescript
} else if (data.type === 'final') {
  callbacks.onComplete(data.content, data.model);  // 传递model
}
```

#### 修复2：Chat/index.tsx - 设置 model

1. 创建消息时设置 model：
```typescript
setMessages((prev) => [...prev, {
  id: assistantMessageId,
  role: 'assistant',
  content: '',
  timestamp: new Date(),
  model: 'minimax-m2.5-free',  // 从serviceStatus获取
}]);
```

2. onComplete 时更新 model：
```typescript
onComplete: (finalContent, model) => {
  setMessages((prev) => prev.map(msg => 
    msg.id === assistantMessageId 
      ? { ...msg, content: finalContent, model: model || msg.model }
      : msg
  ));
}
```
