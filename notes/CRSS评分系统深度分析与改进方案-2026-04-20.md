## 一、CRSS评分系统架构概述

### 1.1 系统设计

CRSS评分系统是一个基于多维度权重的命令风险评估系统，用于评估用户输入命令的风险等级。

**评分公式：**
```
最终分数 = (操作类型权重 + 操作对象权重) / 2 × 影响范围系数
```

**三个维度：**
1. **操作类型（Operation Type）：** READ、CREATE、UPDATE、DELETE、EXEC
2. **操作对象（Operation Target）：** TEMP、USER、PROJECT、SYSTEM
3. **影响范围（Impact Scope）：** SINGLE_FILE、DIRECTORY、CROSS_DIR、SYSTEM

### 1.2 核心代码结构

```python
def calculate_risk_score(command: str) -> int:
    """计算命令风险分数"""
    # 1. 黑名单命令直接10分
    if not checker.is_safe(command):
        return 10
    
    # 2. 解析三个维度
    op_type = parse_operation_type(command)
    op_target = parse_operation_target(command)
    scope = parse_impact_scope(command)
    
    # 3. 获取各维度权重
    type_score = OPERATION_WEIGHTS[op_type]['default']
    target_score = TARGET_WEIGHTS[op_target]['default']
    scope_multiplier = SCOPE_MULTIPLIERS[scope]
    
    # 4. 计算总分
    raw_score = (type_score + target_score) / 2 * scope_multiplier
    final_score = min(10, int(round(raw_score)))
    
    return final_score
```

## 二、深层次问题分析

### 2.1 问题1：评分公式设计缺陷

#### 问题描述

**当前公式：**
```python
raw_score = (type_score + target_score) / 2 * scope_multiplier
```

**问题分析：**

1. **简单平均不合理**
   - 操作类型和操作对象的重要性不同，简单平均会掩盖高风险因素
   - 例如：删除操作(8分) + 用户数据(4分) = 6分
   - 即使影响范围是SYSTEM(3.0)，最终也只有9分
   - 但实际上，删除用户数据应该比读取系统文件更危险

2. **乘法放大效应**
   - 影响范围系数直接相乘，会导致分数跳跃式增长
   - 例如：基础分数4分 × SYSTEM系数3.0 = 12分 → 10分
   - 这导致很多普通操作被误判为高风险

3. **缺乏权重平衡**
   - 三个维度的重要性没有体现
   - 操作类型应该是最重要的（删除 > 修改 > 创建 > 读取）
   - 操作对象次之（系统 > 项目 > 用户 > 临时）
   - 影响范围应该作为调节因子，而不是主要因子

#### 影响范围

- **严重程度：** 高
- **影响范围：** 所有评分计算
- **用户体验：** 导致误判，影响用户操作

### 2.2 问题2：模式匹配过于简单

#### 问题描述

**parse_operation_target函数：**
```python
for target_type, config in TARGET_WEIGHTS.items():
    for pattern in config['patterns']:
        if re.search(pattern, command_lower, re.IGNORECASE):
            return target_type
```

**问题分析：**

1. **优先级问题**
   - 使用字典遍历顺序，但Python 3.7+字典保持插入顺序
   - 这可能导致误判
   - 例如：如果TEMP在SYSTEM之前，"C:\Windows\temp"可能被识别为TEMP而不是SYSTEM

2. **缺乏上下文理解**
   - 简单的模式匹配无法理解命令的上下文
   - 例如："复制E:\微信到D:\备份"会被识别为USER
   - 但"删除C:\Windows"应该识别为SYSTEM

3. **模式冲突**
   - 多个模式可能匹配同一个命令，但只返回第一个匹配
   - 例如："删除C:\Windows\System32\temp"可能匹配多个模式

#### 影响范围

- **严重程度：** 中
- **影响范围：** 操作对象识别
- **用户体验：** 导致误判，影响评分准确性

### 2.3 问题3：关键词匹配过于宽泛

#### 问题描述

**parse_operation_type函数：**
```python
for keyword in OPERATION_WEIGHTS[op_type]['keywords']:
    if keyword.lower() in command_lower:
        return op_type
```

**问题分析：**

1. **子串匹配问题**
   - 使用`in`进行子串匹配，会误判
   - 例如："directory"包含"dir"，会被识别为READ操作
   - 例如："remove_old_files"包含"remove"，会被识别为DELETE操作

2. **缺乏词边界检查**
   - 没有检查关键词是否是独立的单词
   - 例如："update"会匹配"updated"、"updater"等

3. **优先级固定**
   - 优先级顺序是固定的，无法根据上下文调整
   - 例如：在"复制"命令中，"删除"关键词不应该触发DELETE操作

#### 影响范围

- **严重程度：** 中
- **影响范围：** 操作类型识别
- **用户体验：** 导致误判，影响评分准确性

### 2.4 问题4：缺乏命令解析的语义理解

#### 问题描述

**问题分析：**

1. **没有解析命令结构**
   - 没有区分源路径和目标路径
   - 例如："复制A到B"中，A是源，B是目标，风险主要来自目标

2. **没有考虑操作方向**
   - 没有区分读取和写入的方向
   - 例如："读取C:\Windows"和"写入C:\Windows"的风险完全不同

3. **没有考虑操作数量**
   - 没有区分单个文件和批量操作
   - 例如："删除一个文件"和"删除所有文件"的风险不同

#### 影响范围

- **严重程度：** 高
- **影响范围：** 命令理解
- **用户体验：** 导致误判，影响评分准确性

### 2.5 问题5：评分结果缺乏可解释性

#### 问题描述

**问题分析：**

1. **只有最终分数**
   - 没有提供评分过程的详细信息
   - 用户不知道为什么被评高分
   - 开发者难以调试和优化

2. **缺乏中间结果**
   - 没有保存各维度的评分细节
   - 无法追溯评分过程
   - 无法进行细粒度调整

3. **缺乏置信度**
   - 没有提供评分的置信度
   - 某些评分可能基于不确定的匹配
   - 应该提供置信度来辅助决策

#### 影响范围

- **严重程度：** 中
- **影响范围：** 调试和优化
- **用户体验：** 用户不理解评分结果

### 2.6 问题6：配置缺乏灵活性

   - 难以追踪配置变更
   - 难以回滚到之前的配置

#### 影响范围

- **严重程度：** 低
- **影响范围：** 系统维护
- **用户体验：** 影响系统灵活性

## 三、全面的改进方案

### 3.1 改进方案1：改进评分公式

#### 新的评分公式

```python
def calculate_risk_score_v2(command: str) -> dict:
    """
    改进的CRSS评分系统 v2.0
    
    新公式：
    1. 使用加权平均代替简单平均
    2. 使用对数函数平滑影响范围系数
    3. 添加风险叠加机制
    """
    # 1. 解析三个维度
    op_type = parse_operation_type(command)
    op_target = parse_operation_target(command)
    scope = parse_impact_scope(command)
    
    # 2. 获取各维度权重
    type_score = OPERATION_WEIGHTS[op_type]['default']
    target_score = TARGET_WEIGHTS[op_target]['default']
    scope_multiplier = SCOPE_MULTIPLIERS[scope]
    
    # 3. 新的评分公式
    # 3.1 加权平均（操作类型权重更高）
    type_weight = 0.5  # 操作类型权重
    target_weight = 0.3  # 操作对象权重
    base_score = type_score * type_weight + target_score * target_weight + (type_score + target_score) / 2 * (1 - type_weight - target_weight)
    
    # 3.2 对数平滑影响范围系数（避免跳跃式增长）
    import math
    smoothed_multiplier = 1 + math.log(scope_multiplier, 2)  # log2(系数) + 1
    
    # 3.3 风险叠加（高风险操作额外加分）
    risk_bonus = 0
    if op_type == 'DELETE' and op_target == 'SYSTEM':
        risk_bonus = 3  # 删除系统文件额外加分
    elif op_type == 'DELETE' and scope == 'SYSTEM':
        risk_bonus = 2  # 系统级删除额外加分
    elif op_type == 'EXEC' and op_target == 'SYSTEM':
        risk_bonus = 2  # 执行系统命令额外加分
    
    # 3.4 计算最终分数
    raw_score = base_score * smoothed_multiplier + risk_bonus
    final_score = min(10, max(0, int(round(raw_score))))
    
    # 4. 返回详细的评分结果
    return {
        'score': final_score,
        'details': {
            'operation_type': op_type,
            'operation_target': op_target,
            'impact_scope': scope,
            'type_score': type_score,
            'target_score': target_score,
            'scope_multiplier': scope_multiplier,
            'base_score': base_score,
            'smoothed_multiplier': smoothed_multiplier,
            'risk_bonus': risk_bonus,
            'confidence': calculate_confidence(command, op_type, op_target, scope)
        }
    }
```

#### 改进效果

**改进前：**
- 基础分数 = (4 + 4) / 2 = 4
- 最终分数 = 4 × 3.0 = 12 → 10（跳跃式增长）

**改进后：**
- 基础分数 = 4 × 0.5 + 4 × 0.3 + 4 × 0.2 = 4
- 平滑系数 = 1 + log2(3.0) = 1 + 1.58 = 2.58
- 最终分数 = 4 × 2.58 = 10.32 → 10（平滑增长）

### 3.2 改进方案2：改进模式匹配

#### 新的模式匹配函数

```python
def parse_operation_target_v2(command: str) -> tuple:
    """
    改进的操作对象解析 v2.0
    
    改进：
    1. 使用优先级队列而不是字典遍历
    2. 添加上下文理解
    3. 返回所有匹配结果和置信度
    """
    command_lower = command.lower().strip()
    matches = []
    
    # 1. 定义优先级顺序（从高到低）
    priority_order = ['SYSTEM', 'PROJECT', 'USER', 'TEMP']
    
    # 2. 按优先级检查所有模式
    for target_type in priority_order:
        config = TARGET_WEIGHTS[target_type]
        for pattern in config['patterns']:
            match = re.search(pattern, command_lower, re.IGNORECASE)
            if match:
                # 计算匹配置信度
                confidence = calculate_pattern_confidence(pattern, match, command_lower)
                matches.append({
                    'type': target_type,
                    'pattern': pattern,
                    'match': match.group(),
                    'confidence': confidence
                })
    
    # 3. 如果没有匹配，返回默认值
    if not matches:
        return 'USER', 1.0, []
    
    # 4. 选择置信度最高的匹配
    best_match = max(matches, key=lambda x: x['confidence'])
    
    return best_match['type'], best_match['confidence'], matches

def calculate_pattern_confidence(pattern: str, match, command: str) -> float:
    """
    计算模式匹配置信度
    
    考虑因素：
    1. 匹配长度占命令长度的比例
    2. 匹配位置（开头、中间、结尾）
    3. 模式的特异性（正则表达式的复杂度）
    """
    match_length = len(match.group())
    command_length = len(command)
    
    # 匹配长度比例
    length_ratio = match_length / command_length
    
    # 匹配位置权重（开头和结尾更重要）
    match_start = match.start()
    position_weight = 1.0
    if match_start == 0:  # 开头匹配
        position_weight = 1.2
    elif match.end() == command_length:  # 结尾匹配
        position_weight = 1.1
    
    # 模式特异性（正则表达式越复杂，置信度越高）
    specificity = 1.0
    if '\\' in pattern:  # 包含转义字符
        specificity = 1.1
    if '[' in pattern or '(' in pattern:  # 包含字符集或分组
        specificity = 1.2
    if pattern.count('\\') > 2:  # 多个转义字符
        specificity = 1.3
    
    # 综合置信度
    confidence = min(1.0, length_ratio * position_weight * specificity)
    
    return confidence
```

### 3.3 改进方案3：改进关键词匹配

#### 新的关键词匹配函数

```python
def parse_operation_type_v2(command: str) -> tuple:
    """
    改进的操作类型解析 v2.0
    
    改进：
    1. 使用词边界匹配
    2. 添加否定词检查
    3. 返回置信度
    """
    command_lower = command.lower().strip()
    matches = []
    
    # 定义优先级顺序
    priority_order = ['DELETE', 'EXEC', 'UPDATE', 'CREATE', 'READ']
    
    # 定义否定词（如果命令中包含这些词，不应该匹配）
    negation_words = {
        'DELETE': ['复制', '移动', '查看', '读取'],
        'EXEC': ['查看', '读取', '复制'],
        'UPDATE': ['查看', '读取', '复制', '删除'],
    }
    
    for op_type in priority_order:
        config = OPERATION_WEIGHTS[op_type]
        for keyword in config['keywords']:
            # 使用词边界匹配
            pattern = r'\b' + re.escape(keyword.lower()) + r'\b'
            match = re.search(pattern, command_lower)
            
            if match:
                # 检查否定词
                has_negation = False
                if op_type in negation_words:
                    for neg_word in negation_words[op_type]:
                        if neg_word in command_lower:
                            has_negation = True
                            break
                
                if not has_negation:
                    # 计算置信度
                    confidence = 1.0 if match.group() == keyword.lower() else 0.8
                    matches.append({
                        'type': op_type,
                        'keyword': keyword,
                        'confidence': confidence
                    })
    
    # 如果没有匹配，返回默认值
    if not matches:
        return 'READ', 0.5, []
    
    # 选择置信度最高的匹配
    best_match = max(matches, key=lambda x: x['confidence'])
    
    return best_match['type'], best_match['confidence'], matches
```

### 3.4 改进方案4：添加命令语义解析

#### 新的命令解析器

```python
class CommandParser:
    """
    命令语义解析器
    
    功能：
    1. 解析命令结构（源路径、目标路径）
    2. 识别操作方向（读取、写入）
    3. 识别操作数量（单个、批量）
    """
    
    def parse(self, command: str) -> dict:
        """解析命令"""
        result = {
            'operation': None,
            'sources': [],
            'targets': [],
            'direction': None,
            'quantity': None,
            'context': {}
        }
        
        # 1. 识别操作类型
        operation_patterns = {
            'copy': [r'复制', r'copy', r'cp'],
            'move': [r'移动', r'move', r'mv'],
            'delete': [r'删除', r'delete', r'rm', r'del'],
            'read': [r'读取', r'查看', r'read', r'cat', r'type'],
        }
        
        for op, patterns in operation_patterns.items():
            for pattern in patterns:
                if re.search(pattern, command, re.IGNORECASE):
                    result['operation'] = op
                    break
            if result['operation']:
                break
        
        # 2. 解析路径（源和目标）
        # 中文模式："将A复制到B"、"把A移动到B"
        cn_pattern = r'(?:将|把)\s*(.+?)\s*(?:复制|移动|删除|读取|查看)\s*(?:到|至)?\s*(.+)?'
        match = re.search(cn_pattern, command)
        if match:
            result['sources'] = [match.group(1).strip()]
            if match.group(2):
                result['targets'] = [match.group(2).strip()]
        
        # 3. 识别操作方向
        if result['operation'] in ['copy', 'move']:
            result['direction'] = 'write'  # 复制和移动涉及写入
        elif result['operation'] == 'delete':
            result['direction'] = 'delete'
        elif result['operation'] == 'read':
            result['direction'] = 'read'
        
        # 4. 识别操作数量
        quantity_patterns = {
            'single': [r'一个', r'单个', r'这个'],
            'batch': [r'所有', r'全部', r'批量', r'多个', r'几个'],
        }
        
        for qty, patterns in quantity_patterns.items():
            for pattern in patterns:
                if re.search(pattern, command):
                    result['quantity'] = qty
                    break
            if result['quantity']:
                break
        
        if not result['quantity']:
            result['quantity'] = 'single'  # 默认单个
        
        return result
```

```python
import yaml
from typing import Any

class ScoringConfig:
    """评分配置管理"""
    
    def __init__(self, config_file: str = 'config/scoring_config.yaml'):
        self.config_file = config_file
        self.version = '2.0'
        self.config = self._load_config()
    
    def _load_config(self) -> dict:
        """加载配置"""
        try:
            with open(self.config_file, 'r', encoding='utf-8') as f:
                config = yaml.safe_load(f)
            
            # 验证配置版本
            if config.get('version') != self.version:
                logger.warning(f"配置版本不匹配: {config.get('version')} != {self.version}")
            
            return config
        except Exception as e:
            logger.error(f"加载配置失败: {e}")
            return self._get_default_config()
    
    def _get_default_config(self) -> dict:
        """获取默认配置"""
        return {
            'version': '2.0',
            'formula': {
                'type_weight': 0.5,
                'target_weight': 0.3,
                'use_log_smoothing': True,
                'risk_bonus_enabled': True,
            },
            'thresholds': {
                'low': 3,
                'medium': 6,
                'high': 8,
                'critical': 10,
            },
            'confidence': {
                'min_threshold': 0.5,
                'high_threshold': 0.8,
            }
        }
    
    def update_config(self, key: str, value: Any):
        """更新配置"""
        keys = key.split('.')
        config = self.config
        for k in keys[:-1]:
            config = config.setdefault(k, {})
        config[keys[-1]] = value
        
        # 保存配置
        self._save_config()
    
    def _save_config(self):
        """保存配置"""
        with open(self.config_file, 'w', encoding='utf-8') as f:
            yaml.dump(self.config, f, allow_unicode=True, default_flow_style=False)
```

```python
def calculate_pattern_confidence(pattern: str, match, command: str) -> float:
    """
    计算模式匹配置信度
    
    考虑因素：
    1. 匹配长度占命令长度的比例
    2. 匹配位置（开头、中间、结尾）
    3. 模式的特异性（正则表达式的复杂度）
    """
    match_length = len(match.group())
    command_length = len(command)
    
    # 匹配长度比例
    length_ratio = match_length / command_length
    
    # 匹配位置权重（开头和结尾更重要）
    match_start = match.start()
    position_weight = 1.0
    if match_start == 0:  # 开头匹配
        position_weight = 1.2
    elif match.end() == command_length:  # 结尾匹配
        position_weight = 1.1
    
    # 模式特异性（正则表达式越复杂，置信度越高）
    specificity = 1.0
    if '\\' in pattern:  # 包含转义字符
        specificity = 1.1
    if '[' in pattern or '(' in pattern:  # 包含字符集或分组
        specificity = 1.2
    if pattern.count('\\') > 2:  # 多个转义字符
        specificity = 1.3
    
    # 综合置信度
    confidence = min(1.0, length_ratio * position_weight * specificity)
    
    return confidence
```

```python
def parse_operation_type_v2(command: str) -> tuple:
    """
    改进的操作类型解析 v2.0
    
    改进：
    1. 使用词边界匹配
    2. 添加否定词检查
    3. 返回置信度
    """
    command_lower = command.lower().strip()
    matches = []
    
    # 定义优先级顺序
    priority_order = ['DELETE', 'EXEC', 'UPDATE', 'CREATE', 'READ']
    
    # 定义否定词（如果命令中包含这些词，不应该匹配）
    negation_words = {
        'DELETE': ['复制', '移动', '查看', '读取'],
        'EXEC': ['查看', '读取', '复制'],
        'UPDATE': ['查看', '读取', '复制', '删除'],
    }
    
    for op_type in priority_order:
        config = OPERATION_WEIGHTS[op_type]
        for keyword in config['keywords']:
            # 使用词边界匹配
            pattern = r'\b' + re.escape(keyword.lower()) + r'\b'
            match = re.search(pattern, command_lower)
            
            if match:
                # 检查否定词
                has_negation = False
                if op_type in negation_words:
                    for neg_word in negation_words[op_type]:
                        if neg_word in command_lower:
                            has_negation = True
                            break
                
                if not has_negation:
                    # 计算置信度
                    confidence = 1.0 if match.group() == keyword.lower() else 0.8
                    matches.append({
                        'type': op_type,
                        'keyword': keyword,
                        'confidence': confidence
                    })
    
    # 如果没有匹配，返回默认值
    if not matches:
        return 'READ', 0.5, []
    
    # 选择置信度最高的匹配
    best_match = max(matches, key=lambda x: x['confidence'])
    
    return best_match['type'], best_match['confidence'], matches
```

```python
def calculate_risk_score_v2(command: str) -> dict:
    """
    改进的CRSS评分系统 v2.0
    
    新公式：
    1. 使用加权平均代替简单平均
    2. 使用对数函数平滑影响范围系数
    3. 添加风险叠加机制
    """
    # 1. 解析三个维度
    op_type = parse_operation_type(command)
    op_target = parse_operation_target(command)
    scope = parse_impact_scope(command)
    
    # 2. 获取各维度权重
    type_score = OPERATION_WEIGHTS[op_type]['default']
    target_score = TARGET_WEIGHTS[op_target]['default']
    scope_multiplier = SCOPE_MULTIPLIERS[scope]
    
    # 3. 新的评分公式
    # 3.1 加权平均（操作类型权重更高）
    type_weight = 0.5  # 操作类型权重
    target_weight = 0.3  # 操作对象权重
    base_score = type_score * type_weight + target_score * target_weight + (type_score + target_score) / 2 * (1 - type_weight - target_weight)
    
    # 3.2 对数平滑影响范围系数（避免跳跃式增长）
    import math
    smoothed_multiplier = 1 + math.log(scope_multiplier, 2)  # log2(系数) + 1
    
    # 3.3 风险叠加（高风险操作额外加分）
    risk_bonus = 0
    if op_type == 'DELETE' and op_target == 'SYSTEM':
        risk_bonus = 3  # 删除系统文件额外加分
    elif op_type == 'DELETE' and scope == 'SYSTEM':
        risk_bonus = 2  # 系统级删除额外加分
    elif op_type == 'EXEC' and op_target == 'SYSTEM':
        risk_bonus = 2  # 执行系统命令额外加分
    
    # 3.4 计算最终分数
    raw_score = base_score * smoothed_multiplier + risk_bonus
    final_score = min(10, max(0, int(round(raw_score))))
    
    # 4. 返回详细的评分结果
    return {
        'score': final_score,
        'details': {
            'operation_type': op_type,
            'operation_target': op_target,
            'impact_scope': scope,
            'type_score': type_score,
            'target_score': target_score,
            'scope_multiplier': scope_multiplier,
            'base_score': base_score,
            'smoothed_multiplier': smoothed_multiplier,
            'risk_bonus': risk_bonus,
            'confidence': calculate_confidence(command, op_type, op_target, scope)
        }
    }
```
