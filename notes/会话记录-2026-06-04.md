**关键厘清**:
- **agent**：系统中的"执行主体"模块，接收 task → 决策 → 调工具 → 产出结果。在代码里对应 `BaseAgent` 及子类。
- **react**：agent 内部采用的一种"工作方法"（Reasoning + Acting 循环算法），对应 `run_stream()` 核心循环。
- **关系**：agent 是"谁"，react 是"怎么干"。没有 agent，react 没有主体；没有 react，agent 没有"思考-行动"的执行模式。
- **明确否定**：我们这里说的 agent **不是**"agent 系统"这个独立产品，而是 OmniAgentAs-desk 平台里的一个功能模块。

**未来进化方向**（4 条）:
1. 多种执行模式并存（ReAct / Plan-Execute / 直调 LLM 可选）
2. 多 agent 协作（Master 协调多个 Sub-Agent）
3. 自适应模式选择（agent 根据任务特征自动选最佳模式）
4. 反思与学习（从历史 task 中沉淀经验，反哺未来执行）
