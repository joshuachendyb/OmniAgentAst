# 从聊天到编程：我的AI认知升级之路

2023年，我第一次用大模型软件。

我输入问题，它回答。很简单。

我那时候想，"哦，原来AI就是个聊天机器人啊。"

它能帮我解释代码，帮我找bug，但不能替我做什么完整的东西。我觉得，这大概就是AI的极限了。

但我错了，错得很离谱。

---

## 第一年：AI是聊天机器人

还记得第一次用ChatGPT吗？

我输入："帮我解释这段代码的意思"，它解释了。

我又输入："这段代码有个bug，你能帮我找找吗"，它找到了。

我觉得挺神奇，但也有个疑问：就这样吗？

它能回答问题，能解释代码，能提供建议。但如果你说"帮我做个系统"，它会说："我理解你的需求，但作为一个文本模型，我无法直接生成可运行的系统..."

换句话说，它懂理论，但不会实践。

你得懂编程，会写代码，知道怎么把AI的建议变成真正的代码。

如果你不会写代码，AI也帮不了你多少。

那时候，AI还是个"理论派"。

懂很多，能说会道，但不会动手干活。

---

## 第二年：AI变成代码助手

2023到2024年，一个叫GitHub Copilot的东西火了起来。

它的名字起得挺形象：Copilot，副驾驶。

你写代码，它陪你一起写。你写一行，它猜下一行你想写什么。

这不叫"问答"，这叫"协作"。

我也试过。你写个函数名，它帮你补全函数体。你写个循环，它帮你写循环里的内容。

感觉很爽——确实快了很多。

但是！

你得会写代码的根本，否则它补全的东西你可能看不懂。

你得知道你要什么，否则它补全的可能不是你想要的。

它是个"助手"，但不是"代劳者"。它在帮你写得更快，但前提是你能写。

如果你不会写代码，它还是帮不了你太多。

那时候，AI是"实操派"，但只是个"半吊子"的实操派。

能干活，但得有人带着。

---

## 第三年：AI成了系统创作者

2024年下半年，有个新词：**vibe coding**。

翻译成中文，大概是"氛围编程"的意思。

什么意思呢？

你描述你的"氛围"——你的想法、你的需求、你的预期——它帮你实现细节。

这就是2025我遇到trae的时候。

我说："做个集装箱管理系统，买集装箱要记录付款和编号，运到各地要记费用和时间，卖集装箱要记价格收款人，还要能查库存和资金状况。"

它没问："你想用什么数据库？""前端用什么框架？""API怎么设计？"

它直接给我一个完整的系统：

数据有了，页面有了，功能有了。打开浏览器，直接用。

这是真实的"系统创作者"，不是"代码助手"。

它不需要你会写代码，也不需要你懂框架。你只需要能用大白话把需求说清楚，它就能给你一个能用的系统。

我觉得这个变化太大了——

从"问答"，到"协作"，到"代劳"。

AI从一个"会说的理论家"，变成了一个"会干的实践家"。

---

## 三年，三个改变

简单总结一下，这三年的变化：

### 2022-2023：你问，它答

- 能理解问题
- 能解释代码
- 能找bug
- 但它只是"助手"

**结论**：你要会点编程，否则用处有限

### 2023-2024：你写，它补

- 能预测代码
- 能加速开发
- 能协作编写
- 但它还是"副驾驶"

**结论**：你得是主驾驶，否则它不知道去哪

### 2024-2025：你说，它做

- 能生成系统
- 能完成需求
- 能开箱即用
- 它是"全栈开发者"

**结论**：你只要能用自然语言描述需求就行

## 举个例子

我用同样的需求，在不同阶段问AI：

### 第一阶段（聊天机器人）

我："帮我做个集装箱管理系统。"

AI："我理解你的需求。集装箱管理系统应该包括集装箱信息的管理、库存追踪、资金记录等功能。你可以考虑使用Python的Flask框架..."

它给了我一堆建议，但没给我系统能行吗？不行。我得自己写代码，自己设计数据库，自己搭前端。

### 第二阶段（代码补全）

我开始写代码了，让我写个函数吧：

```
def add_container():
    
```

光标闪烁，等着我继续写。AI帮我补全：

```
def add_container(container_id, buy_price, status):
    # 添加集装箱逻辑
    pass
```

行吧，比我自己写快。但整个系统呢？还是得我一个一个写。

### 第三阶段（vibe coding）

我："做个集装箱管理系统，买集装箱要记录付款和编号，运到各地要记费用和时间，卖集装箱要记价格收款人，还要能查库存和资金状况。"

它："好的，系统已经生成了。包含数据库设计、前端页面、后端API。你可以打开浏览器，输入localhost:3000访问。"

我打开浏览器——真的是一个完整的系统，能用。

你觉得哪个阶段最爽？我选第三个。

## 这个变化意味着什么？

以前，如果你是个业务人，想做系统，你得：

1. 学编程，学几个月
2. 学数据库，学几周
3. 学框架，学几周
4. 然后才能开始做系统

现在，如果你是个业务人，想做系统，你只需要：

1. 能把你的需求说清楚
2. 打开AI工具
3. 输入需求
4. 系统生成好了

门槛从"几个月的学习时间"降到了"几分钟的自然语言描述"。

这不是说程序员要失业了，也不是说开发公司要关门了。

复杂系统、大型平台、专业领域，依然需要专业的开发者。

但对于"做个小系统"、"快速验证一个idea"、"把业务流程数字化"这些需求，AI确实给普通人提供了一个新选择。

---

## 如果你也是个"业务人"

我不知道你是个什么样的人。

可能是创业者，可能是产品人，可能是运营，也可能是和我一样，就是想做个工具提高效率的人。

如果你不是个程序员，但以前"想做系统不会写代码"的想法困扰过你，那现在，机会来了。

你不需要学会编程，也不需要报班学习。

你只需要做两件事：

1. **把你的需求想清楚**。你想做什么？你要解决什么问题？

2. **用大白话把需求说清楚**。别用术语，就按日常说话的方式。

然后，就用AI工具试试。

失败了没关系，花了十分钟，总比花了几个月学编程划算吧。

成功了，你就做出了一个能用的系统。

这不是让你放弃学习，而是让你用更高效的方式实现目标。

## 三个关键认知

这三年，我对AI的认知发生了三个关键改变：

### 认知1：AI从工具变成伙伴

以前，AI是个工具，我用它来"辅助"我。

现在，AI是个伙伴，我和它一起"协作"甚至"委托"它。

工具是你用来干活的，伙伴是和你一起干活的。这个区别很大。

### 认知2：编程门槛被大幅降低了

以前，"做系统"是个技术活，需要专业训练。

现在，"做系统"是个表达活，需要的是"说得清楚"。

这个变化，会让很多人受益。

### 认知3：机会多了，竞争也多了

门槛降低了，意味着更多人能做系统了。

如果你之前因为"不会写代码"被挡在门外，现在你有机会进来了。

但同时，更多人进来，竞争也变激烈了。

所以，光会用AI工具是不够的，你还得有好的想法、清晰的思路、独特的价值。

---

## 最后，说点实际的

这篇文章不是在说"AI万能"，也不是在说"程序员失业"。

AI有它的局限，它做不到的还有很多。

复杂系统的设计，性能的优化，安全的保障，这些依然需要人类的智慧和经验。

但我想说的是：

**对于"做个简单的系统"这个需求，AI已经足够了。**

如果你有这个需求，不妨试试。

也许，你的集装箱管理系统，你的客户关系管理工具，你的项目追踪系统，几分钟就做好了。

然后，你不会说"我不会写代码"，你会说"我用AI帮我做了一个系统"。

这不也是挺好的吗？
