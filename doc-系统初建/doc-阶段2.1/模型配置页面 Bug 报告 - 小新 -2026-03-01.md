## 三、发现的 Bug

### Bug 1：硬编码 Provider 显示名称

**位置**: 第 452-460 行

**问题代码**:
```typescript
const getProviderDisplayName = (name: string) => {
  const nameMap: Record<string, string> = {
    zhipuai: "智谱 GLM",
    opencode: "OpenCode",
    longcat: "LongCat",
  };
  return nameMap[name] || name;
};
```

**影响**:
- ❌ 添加新 Provider 时无法显示正确的中文名称
- ❌ 违反了禁止硬编码 provider 名称的设计原则
- ❌ 不支持动态添加的 Provider

**修复方案**:
```typescript
const getProviderDisplayName = (name: string) => {
  // 直接使用 Provider 名称，不硬编码映射
  // 或者从后端获取 Provider 的显示名称
  return name;
};
```

**优先级**: P2-中

---

### Bug 2：添加 Provider 时默认值硬编码

**位置**: 第 440-441 行

**问题代码**:
```typescript
await configApi.addProvider({
  // ...
  timeout: values.timeout || 60,
  max_retries: values.max_retries || 3,
});
```

**影响**:
- ❌ 前端硬编码默认值，与后端默认值可能不一致
- ❌ 应该由后端提供默认值

**修复方案**:
```typescript
await configApi.addProvider({
  // ...
  // 不设置默认值，让后端处理
  timeout: values.timeout,
  max_retries: values.max_retries,
});
```

**优先级**: P3-低

---

### Bug 3：缺少 Provider 切换功能

**位置**: 整个 ProviderSettings 组件

**问题描述**:
- ❌ 模型配置页面没有提供切换当前使用的 Provider 和 Model 的功能
- ❌ 用户需要回到对话页面才能切换模型

**影响**:
- ❌ 用户体验不佳
- ❌ 配置管理页面应该支持完整的配置管理功能

**修复方案**:
在 ProviderSettings 组件顶部添加当前使用的 Provider 和 Model 选择器：
```typescript
<Select
  value={currentProvider}
  onChange={(value) => {
    configApi.updateConfig({ ai_provider: value });
    setCurrentProvider(value);
  }}
>
  {providers.map(p => (
    <Option key={p.name} value={p.name}>{p.name}</Option>
  ))}
</Select>
```

**优先级**: P2-中

---

### Bug 4：缺少 Model 切换功能

**位置**: 整个 ProviderSettings 组件

**问题描述**:
- ❌ 选择了 Provider 后，无法直接切换该 Provider 下的 Model
- ❌ 需要额外的步骤

**影响**:
- ❌ 用户体验不佳

**修复方案**:
在 ProviderSettings 组件中添加 Model 选择器：
```typescript
<Select
  value={currentModel}
  onChange={(value) => {
    configApi.updateConfig({ ai_model: value });
    setCurrentModel(value);
  }}
>
  {selectedProvider?.models.map(m => (
    <Option key={m} value={m}>{m}</Option>
  ))}
</Select>
```

**优先级**: P2-中

### Bug 5：批量删除模型时缺少错误处理

**位置**: 第 356-416 行

**问题描述**:
- ⚠️ 批量删除时，如果某个模型删除失败，没有详细错误提示
- ⚠️ 只显示"X 成功，Y 失败"，没有说明失败原因

**影响**:
- ⚠️ 用户不知道失败的具体原因

**修复方案**:
```typescript
const failDetails = results
  .filter(r => !r.success && !r.cancelled)
  .map(r => `${r.model}: ${r.error?.response?.data?.detail || '未知错误'}`)
  .join('\n');

if (failCount > 0) {
  message.warning(
    `批量删除完成：${successCount} 成功，${failCount} 失败\n\n失败详情:\n${failDetails}`
  );
}
```

**优先级**: P3-低
