| LLM调用 | `llm_caller.py` | 获取工具定义，发送给LLM |

**文件**: `backend/app/services/tools/schema_utils.py`

```python
def _generate_input_schema(input_model, input_schema):
    if input_model is None:
        return input_schema or {}
    
    schema = input_model.model_json_schema()  # Pydantic V2 API
    schema = _fix_schema_types(schema)        # 修复Union/Optional类型
    return schema
```

**转换过程**:
```
ReadTextFileInput (Pydantic类)
    ↓ input_model.model_json_schema()

  "type": "object",
  "properties": {
    "file_path": {"type": "string", "description": "要读取的文件路径"},
    "head": {"type": "integer", "description": "读取前N行"}
  },
  "required": ["file_path"]
}
```

## 三、生成OpenAI工具定义

### 3.1 核心函数

**文件**: `backend/app/services/tools/tool_description.py`

```python
def to_openai_tools(registry, categories, strict=True):
    tools = []
    for name, meta in sorted(registry._tools.items(), key=lambda x: x[0]):
        if not meta.expose_to_llm:
            continue
        if categories is not None and meta.category not in categories:
            continue
        
        func_def = {
            "name": meta.name,              # ← 工具名
            "description": meta.description, # ← 工具描述
            "parameters": meta.input_schema, # ← JSON Schema
            "strict": strict,
        }
        if meta.examples:
            func_def["examples"] = meta.examples  # ← 示例
        
        tools.append({
            "type": "function",
            "function": func_def
        })
    
    return tools
```

### 3.2 字段映射

| OpenAI字段 | 来源 | 说明 |
|-----------|------|------|
| `name` | `meta.name` | 工具名称 |
| `description` | `meta.description` | 工具描述（来自FILE_TOOL_DESCRIPTIONS） |
| `parameters` | `meta.input_schema` | JSON Schema（来自Pydantic模型） |
| `strict` | `strict`参数 | 强制参数校验 |
| `examples` | `meta.examples` | 示例（来自FILE_TOOL_EXAMPLES） |

## 四、LLM调用时

### 4.1 调用链

```
llm_caller.py
    ↓
agent._get_openai_tools()
    ↓
tool_cache_manager.py: get_openai_tools(agent)
    ↓
tool_registry.to_openai_tools(categories=agent._loaded_categories)
    ↓
tool_description.py: to_openai_tools(registry, categories)
    ↓
返回 [{"type": "function", "function": {...}}, ...]
```

### 4.2 缓存机制

**文件**: `backend/app/services/agent/tool_cache_manager.py`

```python
def get_openai_tools(agent):
    # 1. 尝试从TTL缓存获取
    cached = agent._tool_cache.get()
    if cached is not None:
        return cached
    
    # 2. 缓存未命中，重新生成
    from app.services.tools.registry import tool_registry
    tools = tool_registry.to_openai_tools(categories=agent._loaded_categories)
    
    # 3. 存入缓存
    agent._tool_cache.set(tools)
    return tools
```

### 4.3 分类过滤

**关键参数**: `categories=agent._loaded_categories`

- 只返回已加载分类的工具
- 例如：`{ToolCategory.FILE, ToolCategory.NETWORK}`
- 未加载的工具不会出现在LLM的工具列表中

### 6.2 tool_categories.json 文件说明

**文件路径**: `backend/app/services/tools/meta/tool_categories.json`

**作用**: 为 `tool_search` 工具提供未加载分类的概要信息，动态更新工具描述。

**使用场景**: 
- Agent 初始化时，只加载 `fundamental` 和 `shell` 分类
- 其他分类（file/network/desktop等）未加载
- `tool_search` 需要告诉 LLM 有哪些未加载的工具可以搜索
- 从 `tool_categories.json` 读取分类概要和工具列表，动态拼接到 `tool_search` 的描述中

**文件结构**:
```json
{
  "file": {
    "summary": "文本文件读写编辑、目录内容浏览、按文件名/内容模式搜索...",
    "tools": {
      "read_text_file": "读取文本文件，支持分页和编码自动检测",
      "write_text_file": "写入文本文件，创建新文件或追加内容",
      "list_directory": "列出目录内容，支持扁平或树状结构",
      ...
    }
  },
  "network": {
    "summary": "HTTP请求发送、文件下载、网页内容获取...",
    "tools": {
      "http_request": "发送HTTP请求，支持GET/POST/PUT/DELETE",
      ...
    }
  },
  ...
}
```

**动态拼接逻辑**:

**文件**: `backend/app/services/agent/tool_cache_manager.py`

```python
def patch_search_desc(agent):
    """动态更新 tool_search 描述: 列出未加载分类的概要+工具名"""
    
    # 1. 读取 tool_categories.json
    categories_config = read_json_file(_CATEGORIES_CONFIG_PATH)
    
    # 2. 找出未加载的分类
    unloaded = [cat for cat in ToolCategory
                if cat not in agent._loaded_categories]
    
    # 3. 拼接描述
    lines = []
    for cat in sorted(unloaded):
        cfg = categories_config.get(cat.value, {})
        summary = cfg.get("summary", cat.name_cn)          # ← 分类概要
        tools = cfg.get("tools", {})                       # ← 工具列表
        name_str = ", ".join(f"{k}:{v}" for k, v in tools.items()[:5])
        lines.append(f"- {cat.name_cn}({cat.value}): {summary} [{name_str}]")
    
    # 4. 动态更新 tool_search 的描述
    ts_meta.description = base_desc + "\n\n当前未加载分类:\n" + "\n".join(lines)
```

**最终效果**:

`tool_search` 的描述会动态变为：
```
搜索并注入未加载的工具，按关键词在全部工具中做BM25全文检索...

当前未加载分类:
- 文件(file): 文本文件读写编辑、目录内容浏览... [read_text_file:读取文本文件, write_text_file:写入文本文件, ...]
- 网络(network): HTTP请求发送、文件下载... [http_request:发送HTTP请求, download_file:从URL下载文件, ...]
- 桌面(desktop): 窗口信息查询和控制... [window_info:查询窗口信息, window_control:控制窗口状态, ...]
...
```

**信息来源**:
- `summary` ← `tool_categories.json[category]["summary"]`
- `tools` ← `tool_categories.json[category]["tools"]`
- 拼接后追加到 `tool_search` 的 `description` 字段

## 七、关键设计要点

### 7.1 职责分离

| 文件 | 职责 | 不负责 |
|------|------|--------|
| `file_schema.py` | 参数验证（Pydantic模型） | 工具描述、示例 |
| `file_register.py` | 工具描述、示例、注册 | 参数验证 |
| `tool_description.py` | 生成OpenAI格式 | 注册、验证 |
| `tool_cache_manager.py` | 缓存、分类过滤 | 生成格式 |

### 7.2 禁止事项

**在`file_schema.py`中禁止写Pydantic类docstring**：

```python
# ❌ 错误示例
class ReadTextFileInput(BaseModel):
    """读取文本文件 - 这个会进入JSON Schema发给LLM"""
    file_path: str = Field(...)

# ✅ 正确示例
class ReadTextFileInput(BaseModel):
    file_path: str = Field(description="要读取的文件路径")
```

**原因**: Pydantic class docstring会自动进入JSON Schema的`parameters.description`并发给LLM，导致重复。

### 7.3 编制方法

#### 7.3.1 register 文件编制方法

**格式**: `功能描述。适用场景:xxx时使用。`

**示例**:
```python
FILE_TOOL_DESCRIPTIONS = {
    "read_text_file": """读取文本文件。支持分页读取。encoding默认utf-8。适用场景:需要读取源代码、日志文件、配置文件时使用。""",
}
```

**要点**: 第一句（核心功能）+ 中间（关键特性）+ 最后（适用场景）

#### 7.3.2 schema 文件编制方法

**要点**: 禁止写 class docstring，只写 Field description

**示例**:
```python
class ReadTextFileInput(BaseModel):
    # ❌ 禁止写 class docstring（会发给LLM重复）
    
    file_path: str = Field(description="要读取的文件路径(绝对路径)")
    head: Optional[int] = Field(default=None, ge=1, le=1000000, description="读取前N行")
```

#### 7.3.3 编制流程

1. schema 文件：定义 Pydantic 模型（无 class docstring）
2. tools 文件：实现工具函数
3. register 文件：添加描述（功能+适用场景）+ 示例 + 注册

### 7.4 缓存机制

- **TTL缓存**: 避免每次LLM调用都重新生成工具定义
- **分类过滤**: 只返回已加载分类的工具，减少LLM token消耗
- **缓存失效**: 工具注册/注销后调用`invalidate_tool_cache()`
