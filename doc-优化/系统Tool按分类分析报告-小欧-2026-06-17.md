## 三、架构说明

### 3.1 目录结构

```
tools/
├── __init__.py                   # 导出: tool_registry / register_tool / ensure_tools_registered
├── lazy_loader.py                # 按需注册懒加载引擎
├── registry.py                   # ToolRegistry核心类 + register_tool装饰器
├── schema_utils.py               # Pydantic JSON Schema修复/生成
├── tool_types.py                 # ToolCategory枚举 + ToolMetadata dataclass
├── tool_constants.py             # 运行时常量(超时/文件大小/安全模式)
├── tool_description.py           # OpenAI格式转换 + Parameter Reminder生成
├── tool_response.py              # 统一返回结构(build_success/error/warning)
│
├── file/                         # 15个工具 (FILE)
├── fundamental/                  # 5个工具 (FUNDAMENTAL)
├── shell/                        # 6个工具 (SHELL)
├── network/                      # 6个工具 (NETWORK)
├── desktop/                      # 17个工具 (DESKTOP)
├── document/                     # 9个工具 (DOCUMENT)
├── dataanalysis/                 # 6个工具 (DATAANALYSIS)
├── system/                       # 10个工具 (SYSTEM)
├── timer/                        # 3个工具 (TIMER)
├── win_registry/                 # 3个工具 (WIN_REGISTRY)
└── toolhelper/                   # 15个内部辅助文件(不注册为工具)
```

### 3.2 每个分类目录的标准三文件模式

```
{category}/
├── {category}_register.py    # 注册函数(定义哪些工具注册到哪个分类)
├── {category}_tools.py       # 工具函数实现
└── {category}_schema.py      # Pydantic输入模型
```

### 3.3 安全机制

- **确认工具**: 部分工具设 `needs_confirmation=True`(如delete_file, kill_process, registry_write等)，执行前需用户确认
- **安全检查**: `safety/tool_safety_checker.py` 提供统一安全检查
- **文件安全**: `safety/file_safety/` 提供文件操作安全保障(备份/回滚)

### 3.4 统一返回格式

所有工具返回: `Dict[str, Any]`
```python
{
    "code": int,           # 0=成功, 非0=失败
    "data": Any,           # 返回数据
    "message": str,        # 状态消息
    # 可选字段:
    "warning": str,        # 警告信息
    "llm_data": Any,       # LLM附加数据
```
