## 一、问题发现

### 1.1 主要问题

**🔴 关键问题**: Vite配置中使用了开发时不需要的插件，导致启动慢

### 1.2 问题位置

**文件**: `vite.config.ts` (第1-24行)

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import eslint from 'vite-plugin-eslint';        // 🔴 问题1
import prettier from 'vite-plugin-prettier';  // 🔴 问题2

export default defineConfig({
  plugins: [
    react(),
    eslint(),              // 🔴 开发时每次都运行ESLint
    prettier({             // 🔴 开发时每次都运行Prettier
      parser: 'typescript',
    }),
  ],
  // ...
});
```

---

## 二、根因分析

### 2.1 问题1: vite-plugin-eslint 在开发环境运行

**影响**:
- 每次启动开发服务器时，ESLint会检查所有代码
- 每次文件保存时，ESLint会重新检查
- 增加了启动时间和热更新时间

**为什么不应该在开发时运行**:
- ESLint应该在**提交前**或**构建时**运行
- 开发时可以使用IDE的实时检查
- 不应该阻塞开发服务器启动

### 2.2 问题2: vite-plugin-prettier 在开发环境运行

**影响**:
- 每次启动开发服务器时，Prettier会检查格式
- 每次文件保存时，Prettier会重新格式化
- 进一步增加了启动时间

**为什么不应该在开发时运行**:
- Prettier应该在**保存时**通过IDE自动运行
- 或者在**提交前**通过git hooks运行
- 不应该阻塞开发服务器启动

### 2.3 其他可能的问题

虽然依赖不多，但也可以检查：

| 依赖 | 大小 | 说明 |
|------|------|------|
| antd | ~50MB | UI组件库，正常 |
| react-router-dom | ~20MB | 路由库，正常 |
| axios | ~5MB | HTTP客户端，正常 |
| dayjs | ~2MB | 日期库，正常 |

## 三、解决方案

### 3.1 方案1: 只在构建时运行ESLint和Prettier（推荐）

**修改 `vite.config.ts`**:

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import eslint from 'vite-plugin-eslint';
import prettier from 'vite-plugin-prettier';

// https://vitejs.dev/config/
export default defineConfig(({ command }) => {
  const isBuild = command === 'build';
  
  return {
    plugins: [
      react(),
      // 只在构建时运行ESLint和Prettier
      isBuild && eslint(),
      isBuild && prettier({
        parser: 'typescript',
      }),
    ].filter(Boolean),
    server: {
      port: 3000,
      proxy: {
        '/api': {
          target: 'http://localhost:8000',
          changeOrigin: true,
        },
      },
    },
  };
});
```

### 3.2 方案2: 完全移除这两个插件（最简单）

**修改 `vite.config.ts`**:

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [
    react(),
    // 移除 eslint 和 prettier 插件
  ],
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
    },
  },
});
```

**然后使用npm scripts运行检查**:

```json
{
  "scripts": {
    "dev": "vite",
    "build": "tsc && vite build",
    "lint": "eslint src --ext .ts,.tsx",
    "lint:fix": "eslint src --ext .ts,.tsx --fix",
    "format": "prettier --write .",
    "format:check": "prettier --check .",
    "check": "npm run lint && npm run format:check"
  }
}
```

## 四、优化建议

### 4.1 短期优化（立即实施）

1. **修改vite.config.ts** - 移除或条件化ESLint和Prettier插件
2. **验证启动速度** - 修改后测试启动时间
3. **配置IDE** - 确保IDE在保存时自动运行ESLint和Prettier

### 4.2 中期优化（本周）

1. **添加git hooks** - 使用husky在提交前运行检查
2. **配置TypeScript** - 确保tsconfig.json配置合理
3. **检查依赖更新** - 确保所有依赖都是最新版本

### 4.3 长期优化（后续）

1. **考虑依赖按需加载** - 如果antd太大，可以考虑按需导入
2. **添加bundle分析** - 使用rollup-plugin-visualizer分析bundle大小
3. **配置预构建** - 优化Vite的依赖预构建
