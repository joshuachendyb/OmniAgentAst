# 数据库迁移记录 - 添加缺失字段

**创建时间**: 2026-03-03 16:30:00
**编写人**: 小沈
**问题发现**: 新建会话功能报错 500

---

## 一、问题描述

### 现象
前端显示"创建会话失败，正在重试"，后端报 500 错误。

### 错误信息
```
sqlite3.OperationalError: table chat_sessions has no column named title_locked
```

---

## 二、根因分析

### 缺失的字段
| 字段名 | 类型 | 用途 | 添加时间 |
|--------|------|------|---------|
| `title_locked` | BOOLEAN | 标题锁定标记（防止自动更新标题） | 2026-02-25 |
| `title_updated_at` | TIMESTAMP | 标题最后更新时间 | 2026-02-25 |
| `version` | INTEGER | 乐观锁版本号（防止并发冲突） | 2026-02-25 |

### 为什么会出现问题
1. **代码已更新**：2026-02-25 的代码提交添加了这三个字段的使用
2. **数据库未迁移**：数据库迁移脚本未执行，字段实际未添加到表中
3. **兼容逻辑被移除**：最近的修改移除了 `check_db_fields_exist` 的兼容逻辑，直接假设所有字段都存在
4. **INSERT 失败**：创建会话时 INSERT 语句包含不存在的字段名，导致报错

---

## 三、解决方案

### 执行数据库迁移
```sql
-- 添加 title_locked 字段
ALTER TABLE chat_sessions ADD COLUMN title_locked BOOLEAN DEFAULT 0;

-- 添加 title_updated_at 字段
ALTER TABLE chat_sessions ADD COLUMN title_updated_at TIMESTAMP;
UPDATE chat_sessions SET title_updated_at = created_at WHERE title_updated_at IS NULL;

-- 添加 version 字段
ALTER TABLE chat_sessions ADD COLUMN version INTEGER DEFAULT 1;
UPDATE chat_sessions SET version = 1 WHERE version IS NULL;
```

### 字段默认值说明
| 字段 | 默认值 | 合理性说明 |
|------|--------|-----------|
| `title_locked` | 0 (FALSE) | 老会话标题从未被手动锁定过 |
| `title_updated_at` | created_at | 标题创建后从未被更新过 |
| `version` | 1 | 初始版本号，老会话从未被并发修改过 |

## 五、经验教训

### 问题
1. 代码修改添加了新字段，但未同步执行数据库迁移
2. 依赖兼容性逻辑掩盖了问题，移除兼容逻辑后问题暴露
3. 缺少数据库迁移的自动化检查和提醒机制

### 改进措施
1. **代码与数据库同步**：添加新字段时，必须同时提供迁移脚本并执行
2. **迁移检查**：启动时检查必要字段是否存在，缺失时提示执行迁移
3. **文档记录**：记录所有数据库结构变更，便于追踪和排查
