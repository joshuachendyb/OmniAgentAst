### 2.2 三重工具查找机制

**文件**: `backend/app/services/agent/tool_retry_engine.py` (第 135-146 行)

**查找逻辑**:
```python
def _lookup_tool(self, action, tool=None):
    # 第 1 层：直接传入的工具
    if tool is not None:
        return tool
    
    # 第 2 层：本地缓存
    tool = self.available_tools.get(action)
    if tool is not None:
        return tool
    
    # 第 3 层：从 registry 动态获取
    from app.services.tools.registry import tool_registry
    impl = tool_registry.get_implementation(action)
    if impl is not None:
        self.available_tools[action] = impl  # 缓存到本地
    return impl
```

**"绕"的问题**:
- ✅ 3 次查找 + 1 次缓存
- ✅ 每次失败都要从 registry 重新查询
- ✅ 可能产生多次数据库/内存查找

**复杂度**: ⭐⭐⭐⭐☆ (高 - 多层兜底)

## 六、建议优化方案

### 6.1 立即实施 (高优先级)

#### 6.1.1 实现真正的 ReAct 循环

**当前问题**: 占位符只有循环框架，没有业务逻辑

**建议实现**:

```python
async def run_stream(self, task, context, max_steps, task_id):
    """真正的 ReAct 循环"""
    if max_steps is None:
        max_steps = get_config().get_max_steps()
    
    conversation = []
    
    for step_count in range(1, max_steps + 1):
        # Step 1: 思考
        self.status = AgentStatus.THINKING
        llm_response = await self.llm.call(conversation)
        yield {"type": "thought", "content": llm_response}
        
        # Step 2: 解析 Action
        action_name, action_input = parse_react_response(llm_response)
        yield {"type": "action", "name": action_name, "input": action_input}
        
        # Step 3: 执行工具
        impl = tool_registry.get_implementation(action_name)
        if not impl:
            yield {"type": "error", "message": f"Unknown tool: {action_name}"}
            break
        
        observation = await impl(**action_input)
        yield {"type": "observation", "content": observation}
        
        # Step 4: 添加到对话
        conversation.append({"role": "assistant", "content": llm_response})
        conversation.append({"role": "user", "content": observation})
        
        # Step 5: 检查终止条件
        if is_finished(observation):
            yield {"type": "finish"}
            break
    
    yield {"type": "done"}
```

**复杂度**: ⭐⭐⭐⭐⭐ (高 - 核心逻辑)

---

#### 6.1.2 统一参数验证

**当前问题**: `normalize_params` 和 `_validate_params` 重复

**建议合并**:

```python
def validate_and_normalize(self, action, action_input, tool):
    """统一参数验证"""
    # 1. 获取 schema
    metadata = tool_registry.get_tool(action)
    if not metadata or not metadata.input_schema:
        return None, "Invalid tool"
    
    # 2. 删除非法参数
    params = dict(action_input)
    valid_params = metadata.input_schema.get("properties", {}).keys()
    for key in list(params.keys()):
        if key not in valid_params:
            del params[key]
    
    # 3. 检查必需参数
    sig = inspect.signature(tool)
    required = [p for p, v in sig.parameters.items() 
                if v.default == inspect.Parameter.empty]
    missing = [p for p in required if p not in params]
    if missing:
        return None, f"Missing required params: {missing}"
    
    return params, None
```

**复杂度**: ⭐⭐⭐☆☆ (中 - 消除重复)
