# AI助手消息显示名称功能需求说明

**创建时间**: 2026-03-03 22:49:56
**版本**: v1.0
**编写人**: 小新

---

## 一、需求背景

### 1.1 需求描述

在 AI 助手的聊天界面中，AI 助手消息气泡上方需要显示当前使用的 AI 模型名称，格式为：`AI 助手【模型显示名称】`

例如：`AI 助手【OpenAI (GPT-4)】` 或 `AI 助手【Claude (claude-sonnet-4-20250514)】`

### 1.2 当前状态

- **流式响应**：后端在 SSE 的 `start` 事件中已经返回了 `display_name` 字段，前端可以正常接收
- **历史消息**：后端 `get_session_messages` 接口返回的消息不包含 `display_name` 字段，导致刷新页面后历史消息无法显示模型名称

---

## 二、功能需求

### 2.1 需要后端处理的场景

| 场景 | 是否需要后端处理 | 说明 |
|------|----------------|------|
| 新发送消息 | ✅ 是 | 流式响应时已返回 display_name，前端已处理 |
| 历史消息显示 | ✅ 是 | 需要从数据库读取并返回 display_name |

### 2.2 后端修改清单

#### 2.2.1 数据库 - chat_messages 表

需要给 `chat_messages` 表添加 `display_name` 字段：

```sql
ALTER TABLE chat_messages ADD COLUMN display_name TEXT;
```

**说明**：存储 AI 助手的显示名称（如 "OpenAI (GPT-4)"）

#### 2.2.2 保存消息 API - save_message

修改 `POST /api/v1/sessions/{session_id}/messages` 接口：

- **用户消息**：不需要处理
- **AI 回复消息**：需要接收并保存 `display_name` 字段

**请求体扩展**：
```python
class MessageCreate(BaseModel):
    role: str
    content: str
    display_name: Optional[str] = None  # ⭐ 新增：AI助手显示名称
```

#### 2.2.3 获取消息历史 API - get_session_messages

修改 `GET /api/v1/sessions/{session_id}/messages` 接口：

- 返回的 Message 响应中添加 `display_name` 字段

**响应体扩展**：
```python
class MessageResponse(BaseModel):
    id: int
    session_id: str
    role: str
    content: str
    timestamp: str
    execution_steps: Optional[List[dict]] = None
    display_name: Optional[str] = None  # ⭐ 新增：AI助手显示名称
```

#### 2.2.4 流式响应 API - chat/stream

**无需修改**：已在 SSE `start` 事件中返回 `display_name` 字段，格式正确。

---

## 三、字段格式规范

### 3.1 display_name 格式

后端在流式响应中生成的格式为：

```
display_name = f"{get_provider_display_name(provider)} ({model})"
```

**示例**：
- `OpenAI (gpt-4)` → `OpenAI (gpt-4)`
- `Anthropic (claude-sonnet-4-20250514)` → `Anthropic (claude-sonnet-4-20250514)`

### 3.2 前端显示格式

前端直接使用后端返回的 `display_name`，用【】包住显示：

```
AI 助手【{display_name}】
```

**示例**：
- `AI 助手【OpenAI (gpt-4)】`
- `AI 助手【Anthropic (claude-sonnet-4-20250514)】`

## 四、前端处理说明

### 4.1 已完成的前端修改

1. **sse.ts**：SSE 解析时将 `display_name` 传递到 `onComplete` 回调
2. **NewChatContainer.tsx**：接收并存储 `displayName` 到消息对象
3. **MessageItem.tsx**：直接使用 `message.displayName` 显示，不做 fallback 处理

### 4.2 待后端完成后

- 后端返回带 `display_name` 的历史消息后，刷新页面即可正常显示
