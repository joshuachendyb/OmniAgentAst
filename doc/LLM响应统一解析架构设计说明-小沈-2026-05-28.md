# LLM响应统一解析架构设计说明

**创建时间**: 2026-05-28 09:10:05  
**编写人**: 小沈  
**版本**: v1.0  
**适用范围**: backend `app/services/agent/llm_response_parser/` 包

---

## 一、概述

LLM响应解析器是 Agent ReAct 循环中**唯一**的 LLM 输出解析入口。所有 Agent 子类（`BaseAgent.run()` → `_process_llm_response()`）统一调用 `parse_react_response(output)`，将 LLM 返回的原始字符串解析为结构化结果 dict。

### 1.1 核心设计原则

| 原则 | 体现 |
|------|------|
| **统一入口** | 整个系统只有一个解析入口 `parse_react_response()`，所有 Agent 共享 |
| **分类输出** | 无论输入格式如何，输出统一为 8 字段的标准化 dict |
| **优先级链** | 10 个 handler 按确定性优先排列，第一个匹配成功即返回 |
| **层状架构** | 6 个模块从底层工具到顶层入口逐层依赖，底层不依赖高层 |

## 三、解析流程

### 3.1 主入口：`parse_react_response(output)`

```
parse_react_response(output)
    │
    ▼
遍历 _HANDLERS 链（10 个 handler，按优先级）
    │
    ├─ 第1个返回非 None 的 handler → 返回其结果
    └─ 全部返回 None → 返回 parse_error（"Parser chain exhausted"）
```

### 3.2 Handler 优先级链

| 优先级 | Handler | 识别条件 | 功能 | 典型输入 |
|--------|---------|----------|------|---------|
| 1 | `_handle_dict_input` | `isinstance(output, dict)` | 直接解析 dict | LLM 直接返回 dict |
| 2 | `_handle_list_input` | `isinstance(output, list)` | 解析 list（并行工具调用） | 多个工具调用的列表 |
| 3 | `_handle_json_array_string` | 字符串以 `[` 开头 | JSON 数组字符串 → list 处理 | `[{"tool_name":"..."}]` |
| 4 | `_handle_empty_input` | `not output` 或非 string | 空值/非字符串降级 | `None`, `""`, 数字 |
| 5 | `_handle_standard_json` | `json.loads()` 成功且返回 dict | 标准 JSON 解析 | `{"tool_name":"read_file"...}` |
| 6 | `_handle_non_standard_json` | 单引号/多余逗号等非标准 JSON | 6 级容错解析 | `{'tool_name':'finish'}` |
| 7 | `_handle_mixed_text_json` | 文本中包含 `{...}` JSON 块 | 混合文本 → JSON 提取 | `思考过程 {"tool_name":"read_file"}` |
| 8 | `_handle_regex_fallback` | 正则匹配到 `"tool_name"` | 正则兜底提取工具调用 | 不完整 JSON 片段 |
| 9 | `_handle_known_tool_match` | 内容包含已注册工具名 | 按已知工具名匹配 | 纯文本提及工具 |
| 10 | `_handle_keyword_match` | 最后兜底 | 关键词（Thought/Action/Answer）匹配 | ReAct 格式输出 |

### 3.3 7号 Handler 详解：`_handle_mixed_text_json`

这是最复杂的 handler，处理 LLM 在思考文本中嵌入 JSON 的情况：

```
输入: "我需查文档{"tool_name":"search_files","tool_params":{"path":"/data"}}"
                      │
                      ▼
    1. _extract_json_block(output) → 提取JSON块
                      │
              ┌───────┴────────┐
              ▼                 ▼
        提取成功            提取失败
              │                 │
        ┌─────┼──────┐         └─→ 检查是否为不完整JSON
        ▼     ▼      ▼                 │
    finish  tool  content/        正则兜底→chunk
    handler name  reasoning
    (answer)  (action)  (implicit)
```

### 3.4 结果统一结构

所有 handler 返回的 dict 统一为 8 字段格式：

```python
{
    "type": str,          # "action" | "answer" | "implicit" | "chunk" | "thought_only" | "parse_error"
    "thought": str,       # 思考过程/推理文本
    "content": str,       # 内容文本（通常与 thought 相同或为最终内容）
    "reasoning": str,     # 推理说明
    "tool_name": str|None,  # 工具名（answer/chunk 时为 None）
    "tool_params": dict|None,  # 工具参数
    "response": str|None, # 最终返回给用户的响应
    "error": str|None,    # 解析错误信息
}
```

### 3.5 输出类型说明

| type 值 | 含义 | 后续处理 |
|---------|------|---------|
| `action` | 需调用工具 | `BaseAgent._execute_tool()` |
| `answer` | LLM 返回最终答案 | 直接输出给用户 |
| `implicit` | 隐含回复（LLM 直接说话，无工具调用） | 当文本输出处理 |
| `chunk` | 流式块（长文本分段） | 拼接到已有内容 |
| `thought_only` | 仅有思考过程 | 显示为推理过程 |
| `parse_error` | 解析失败 | 返回错误信息给用户 |

## 四、各模块功能分解

### 4.1 `_utils.py` — 共享工具函数（8 函数）

| 函数 | 功能 | 被调用方 |
|------|------|---------|
| `_add_reasoning_warning()` | 校验 reasoning 字段长度，空或过短时添加警告 | `_result_builders`, `_handlers` |
| `_normalize_result_to_str()` | bool/int/float/list/dict/str 统一转字符串 | `_handlers`, `_result_builders` |
| `REACT_KEYWORDS` | Thought/Action/Action Input/Answer 的中英文正则 | `_keyword_parsers` |
| `_get_all_tool_names()` | 从 ToolRegistry 获取注册工具名列表 | `_handlers` (handler #9) |
| `_build_handler_result()` | 构造统一 8 字段结果 dict | `_handlers` |
| `_make_action_result_dict()` | 构造 action 结果 dict（附带 reasoning 校验） | `_result_builders` |
| `_extract_json_with_balanced_braces()` | 按括号匹配提取 JSON/数组片段 | `_tool_params`, `_json_strategies`, `_keyword_parsers`, `_handlers` |
| `_extract_key_value_pairs()` | 从文本中提取 key:value 键值对 | `_keyword_parsers` |

### 4.2 `_tool_params.py` — 工具参数处理（12 函数）

按处理流水线排列：

```
原始 tool_params
    │
    ▼
_fallback_tool_name()       ← 若 tool_name 为空，从 params 兜底提取
    │
    ▼
_normalize_tool_params()    ← 统一键名（params→tool_params 等）
    │
    ▼
_normalize_tool_params_content()  ← 非字符串值（bool/list/dict）→ str
    │
    ▼
_filter_tool_params()       ← 清除非参数字段 + camelCase→snake_case
    │
    ▼
_process_tool_params()      ← 总装函数：上面 3 步顺序执行
```

另外 5 个函数用于从各种格式中提取 params：

| 函数 | 提取来源 |
|------|---------|
| `_extract_tool_params_from_thought()` | 从 thought 文本中的 JSON 提取 |
| `_extract_tool_params_from_text()` | 从内容文本中定位 `tool_params` 键 |
| `_extract_content_value_from_json_str()` | 从 JSON 字符串中提取 content 值 |
| `_extract_params_by_regex_from_json_str()` | 从 JSON 字符串中按正则提取参数 |
| `_extract_params_by_regex()` | 从任意文本中按正则提取参数 |

### 4.3 `_json_strategies.py` — JSON 解析策略（14 函数）

**5 级策略链**（`STRATEGIES` 列表，按宽严程度排列）：

| 策略 | 功能 | 处理例 |
|------|------|-------|
| `_strategy_direct_parse` | 标准 `json.loads()` | `{"a":1}` |
| `_strategy_encoding_fix` | UTF-8 编码修复 | `{"a":"\xff"}` |
| `_strategy_chinese_quotes` | 中文引号 `""` → 半角 `""` | `{"a":"数据"}` |
| `_strategy_newline_fix` | 换行符替换为空格 | `{"a":"\n数据\n"}` |
| `_strategy_trailing_comma` | 去除末尾多余逗号 | `{"a":1,}` |

**字段提取**（当 JSON 完整解析失败时逐字段补救）：

| 函数 | 功能 |
|------|------|
| `_try_extract_single_field()` | 从 JSON 字符串中提取单个字段（含别名映射） |
| `_extract_fields_from_json_str()` | 批量提取多个字段（thought/content/tool_name/tool_params） |
| `_extract_json_block()` | **对外接口**：从文本中提取 JSON 块 → 策略链解析 → 字段提取兜底 |

**非标准 JSON 容错**（6 级降级）：

| 级别 | 处理 |
|------|------|
| 1 | 标准 `json.loads()` |
| 2 | 单引号 `'` → 双引号 `"` |
| 3 | 去末尾逗号 + 单引号替换 |
| 4 | 去掉 `//` 注释 |
| 5 | 括号平衡性检查 |

### 4.4 `_result_builders.py` — 结果构建（12 函数）

**3 种输入格式 → 统一结果**：

| 原始格式 | 构建函数 | 典型 JSON |
|---------|---------|-----------|
| **新格式** `{tool_name, tool_params}` | `_build_action_from_new_format()` | `{"tool_name":"read_file","tool_params":{...}}` |
| **旧格式** `{action, action_input}` | `_build_action_from_old_format()` | `{"action":"read_file","action_input":{...}}` |
| **FC 格式** `{name, arguments}` | `_build_action_from_fc_format()` | `{"name":"read_file","arguments":{...}}` |
| **answer 格式** | `_build_answer_result()` | `{"type":"answer","content":"..."}` |
| **chunk 格式** | `_build_chunk_result()` | `{"type":"chunk","content":"..."}` |
| **parse_error 格式** | `_build_parse_error_result()` | `{"type":"parse_error","error":"..."}` |

**2 个入口函数**：

| 函数 | 处理输入 | 说明 |
|------|---------|------|
| `_create_action_result_from_dict()` | dict | 检查 type→action→tool_name 分支 → `_resolve_return_type()` |
| `_create_action_result_from_list()` | list | 取最后一个有效元素，支持 FC 格式头 |

**特殊处理**：
- finish 工具：`tool_name="finish"` → 输出 type 转为 `answer`，`response` 设为 result 字符串
- 空 tool_name：降级为 `implicit` 或检查 finish 关键词

### 4.5 `_keyword_parsers.py` — 关键词/ReAct 解析（14 函数）

**2 条解析线**：

**线 A：代码块解析**（优先于关键词）

| 入口 | 内部调用 |
|------|---------|
| `_try_codeblock_parse()` | `_create_action_result()` (来自 `_result_builders`) |

**线 B：ReAct 关键词解析**

```
_try_keyword_parse(output)
    │
    ├─ Thought + Action 同时存在
    │   └─ _parse_action()
    │       └─ _parse_action_input() → 6 级子解析器链
    │              ├─ _try_markdown_parse
    │              ├─ _try_json_parse
    │              ├─ _try_balanced_braces
    │              ├─ _try_single_quotes
    │              ├─ _extract_fields_partial
    │              └─ _try_kv_parse
    │
    ├─ Answer 存在 → _parse_answer()
    │
    └─ 只有 Thought → _parse_thought_only()
```

**兜底逻辑 `_determine_parse_type()`**：

```python
def _determine_parse_type(output):
    ① _try_codeblock_parse(output)      # ```json 包裹？
    ② _try_keyword_parse(output)        # Thought/Action/Answer 关键词？
    ③ _make_fallback_result()           # 长度≥5 → implicit, 否则 parse_error
```

### 4.6 `_handlers.py` — 入口与分发（11 函数 + 1 常量）

**10 Handler 职责**：

| Handler | 输入特征 | 核心逻辑 | 返回格式 |
|---------|---------|---------|---------|
| 1. `_handle_dict_input` | dict 类型 | 直接调用 `_create_action_result_from_dict()` | dict |
| 2. `_handle_list_input` | list 类型 | 调用 `_create_action_result_from_list()` | dict |
| 3. `_handle_json_array_string` | 字符串以 `[` 开头 | `json.loads()` → list → `_create_action_result_from_list()` | dict |
| 4. `_handle_empty_input` | None/非字符串 | 直接返回 parse_error 结果 | dict |
| 5. `_handle_standard_json` | `json.loads()` 成功 | `_process_json_result()` 分发到 3 种格式构建 | dict |
| 6. `_handle_non_standard_json` | `_try_parse_non_standard_json()` | 6 级非标准 JSON 容错 → `_process_json_result()` | dict |
| 7. `_handle_mixed_text_json` | 文本含 `{...}` | `_extract_json_block()` + 3 路分支 | dict |
| 8. `_handle_regex_fallback` | 含 `"tool_name"` 文本 | `_try_regex_tool_call_fallback()` 正则提取 | dict |
| 9. `_handle_known_tool_match` | 含注册工具名 | 全局工具名匹配 + 简单参数提取 | dict |
| 10. `_handle_keyword_match` | 最终兜底 | `_determine_parse_type()` 关键词解析 | dict |

**正则兜底 `_try_regex_tool_call_fallback()`**：

特殊处理 finish 和普通工具两种情况：
- finish：提取 prefix_text + result → answer 格式
- 普通工具：校验 tool_registry 存在性 → 提取 tool_params → action 格式

## 五、数据流完整路径

```
┌─────────────────────────────────────────────────────────────────┐
│  LLM 原始响应                                                    │
│  (str / dict / list / None)                                      │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  parse_react_response(output)                                    │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │  _HANDLERS 链（10 handler 按优先级遍历）                     │ │
│  │                                                             │ │
│  │  第1-3号: 类型预检（dict/list/array）                        │ │
│  │  第4号:   空值降级                                          │ │
│  │  第5-6号: JSON 解析（标准→非标准 6 级容错）                  │ │
│  │  第7号:   混合文本 JSON 提取                                 │ │
│  │  第8号:   正则兜底                                          │ │
│  │  第9号:   工具名匹配                                        │ │
│  │  第10号:  关键词/ReAct 匹配                                  │ │
│  └─────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  标准化输出 dict                                                 │
│  {type, thought, content, reasoning,                            │
│   tool_name, tool_params, response, error}                       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  BaseAgent._process_llm_response()                               │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │  type == "action"    → _execute_tool(tool_name, params)     │ │
│  │  type == "answer"    → 直接返回给用户                        │ │
│  │  type == "implicit"  → 当文本输出处理                        │ │
│  │  type == "chunk"     → 拼接到流式内容                        │ │
│  │  type == "parse_error" → 重试或返回错误                      │ │
│  └─────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

```python
__all__ = [

    "_parse_action",             # 关键词解析 - action

    "_parse_thought_only",       # 关键词解析 - thought only
