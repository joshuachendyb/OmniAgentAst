## 一、工具参数详解

### 1.1 list_directory - 列出目录

```
【参数】
  - dir_path:      目录的完整路径（必填，绝对路径）
  - recursive:     是否递归子目录（默认 False）
  - max_depth:     最大递归深度（默认10，1-50）
  - page_token:    分页令牌（默认 None）
  - page_size:     每页数量（默认 None，返回全部）

【返回】
  - entries[]:     条目数组
  - total:         总数
  - has_more:      是否还有更多
  - directory:     查询的目录路径

【entries 结构】
  {
    "name": "src",                  // 名称
    "path": "D:/项目/src",          // 路径（递归时可能是相对路径）
    "type": "directory",            // "directory" 或 "file"
    "size": null                    // 目录无 size，文件有 size（字节）
  }
```

---

### 1.2 read_file - 读取文件

```
【参数】
  - file_path:     文件的完整路径（必填）
  - offset:        起始行号，从1开始（默认1）
  - limit:         最大读取行数（默认2000，最大10000）
  - encoding:      文件编码（默认utf-8）

【返回】
  - content:       文件内容（带行号格式: "1: 第一行\n2: 第二行\n...")
  - total_lines:   总行数
  - start_line:    本次返回的起始行
  - end_line:      本次返回的结束行
  - has_more:      是否还有更多
  - file_size:     文件大小（字节）
  - encoding:      使用的编码

【支持的格式】
  任何文本文件：.txt, .md, .py, .js, .ts, .json, .html, .css, .xml, .yaml, .yml, .csv, .sql, .sh, .bat, .c, .cpp, .h, .go, .rs, .java 等

【不支持】
  二进制文件：.exe, .dll, .jpg, .png, .pdf, .docx, .xlsx 等
```

---

### 1.3 write_file - 写入文件

```
【参数】
  - file_path:     文件的完整路径（必填）
  - content:       要写入的内容（字符串）
  - encoding:      文件编码（默认utf-8）

【返回】
  - file_path:     写入的文件路径
  - bytes_written: 写入的字节数
  - operation_id:  操作ID（可用于回滚）

【关于格式】
  ✅ 可以指定格式！content 是什么格式，写入就是什么格式：
  
  例子：
    - 写 .json → content = '{"name": "test", "value": 123}'
    - 写 .py   → content = 'def hello():\n    print("Hello")'
    - 写 .md   → content = '# 标题\n\n这是内容'
    - 写 .csv  → content = 'name,age\n张三,25\n李四,30'
    - 写 .xml  → content = '<?xml version="1.0"?>\n<root>...</root>'

【注意】
  - 会覆盖已有文件
  - 自动创建父目录
  - 备份到回收站（通过 safety 模块）
```

---

### 1.4 delete_file - 删除文件

```
【参数】
  - file_path:     要删除的文件或目录路径（必填）
  - recursive:     是否递归删除（默认 False）
                    // 当目录非空时需要设为 True

【返回】
  - deleted_path:  删除的路径
  - message:       提示信息
  - operation_id:  操作ID（可用于回滚）

【注意】
  - 不是永久删除，而是移动到回收站
  - 可以回滚恢复
  - recursive=False 时，目录必须为空
```

---

### 1.5 move_file - 移动/重命名

```
【参数】
  - source_path:      源文件/目录的完整路径（必填）
  - destination_path:  目标路径（必填）
                       // 可以是新位置 + 新名称

【返回】
  - source:      源路径
  - destination: 目标路径
  - message:     提示信息
  - operation_id: 操作ID（可用于回滚）

【使用场景】
  1. 移动文件到另一个目录
     source: "C:/文档/a.txt"
     dest:   "D:/备份/a.txt"
  
  2. 重命名文件（同一目录）
     source: "C:/文档/old.txt"
     dest:   "C:/文档/new.txt"
  
  3. 移动并重命名
     source: "C:/文档/file.txt"
     dest:   "D:/备份/report.txt"
  
  4. 移动整个目录
     source: "C:/文档/项目"
     dest:   "D:/备份/项目"
```

---

### 1.6 search_files - 搜索文件内容

```
【参数】
  - pattern:       搜索关键字或正则表达式（必填）
  - path:          搜索的起始目录（默认 "."）
  - file_pattern: 文件名匹配模式（默认 "*"，支持通配符）
                   // 例如: "*.py", "*.js", "*.txt"
  - use_regex:    是否使用正则表达式（默认 False）
  - max_results:  最大搜索结果数（默认1000，最大10000）

【返回】
  - pattern:          搜索的关键词
  - path:             搜索的目录
  - files_searched:   搜索的文件总数
  - files_matched:    匹配的文件数
  - total_matches:    总匹配次数
  - matches[]:        匹配详情数组

【matches 结构】
  [
    {
      "file": "src/main.py",           // 相对路径
      "match_count": 3,                // 该文件匹配次数
      "matches": [
        {
          "start": 100,               // 匹配开始位置（字符偏移）
          "end": 104,                 // 匹配结束位置
          "matched": "TODO",          // 匹配的文本
          "context": "... TODO: 修复bug ..."  // 前后各50字符
        }
      ]
    }
  ]

【排序】
  按 match_count 降序排列（匹配最多的文件排在前面）

【示例】
  pattern: "TODO"
  file_pattern: "*.py"
  → 搜索所有 .py 文件中包含 "TODO" 的内容
```

---

### 1.7 generate_report - 生成报告

```
【参数】
  - output_dir:   报告输出目录（默认 None，使用系统默认目录）

【返回】
  - session_id:   会话ID
  - reports{}:    报告字典 {报告类型: 文件路径}

【reports 结构】
  {
    "tree": "D:/报告/tree_20260323.html",           // 目录树报告
    "operation": "D:/报告/operation_20260323.html", // 操作记录报告
    "summary": "D:/报告/summary_20260323.html"     // 摘要报告
  }
```

## 二、工具返回数据对比表

| 工具 | 核心数据 | 数据结构 | 数量 |
|------|---------|---------|------|
| list_directory | entries[] | entries[{name, path, type, size}] | 数组 |
| read_file | content | 字符串（带行号） | 单值 |
| write_file | file_path, bytes_written | {字符串, 数字} | 多值 |
| delete_file | deleted_path | 字符串 | 单值 |
| move_file | source, destination | 两个字符串 | 多值 |
| search_files | matches[] | matches[{file, matches[]}] | 嵌套数组 |
| generate_report | reports{} | 字典 | 字典 |

## 三、前端解析方案

### 3.1 核心结论

**不同工具的 `raw_data.data` 结构完全不同，前端必须根据 `tool_name` 区分处理。**

```
action_tool 步骤数据结构：
{
  "type": "action_tool",
  "tool_name": "xxx",           // ← 用这个判断工具类型
  "tool_params": {...},
  "raw_data": {
    "status": "...",
    "data": {...}               // ← 这个结构因工具而异
  }
}
```

### 3.2 各工具的前端显示策略

```tsx
// 根据 tool_name 选择显示组件
switch (step.tool_name) {
  
  case 'list_directory':
    // raw_data.data.entries[] - 文件列表
    // raw_data.data.total - 总数
    return (
      <div>
        <span>📁 文件列表 ({raw_data.data.total}个)</span>
        <div className="file-list">
          {raw_data.data.entries.map(entry => (
            <div key={entry.path}>
              {entry.type === 'directory' ? '📁' : '📄'} {entry.name}
              {entry.size && <span> ({entry.size} bytes)</span>}
            </div>
          ))}
        </div>
      </div>
    );
  
  case 'read_file':
    // raw_data.data.content - 带行号内容
    // raw_data.data.total_lines - 总行数
    return (
      <div>
        <span>📄 共 {raw_data.data.total_lines} 行</span>
        <pre className="file-content">
          {raw_data.data.content}
        </pre>
      </div>
    );
  
  case 'write_file':
    // raw_data.data.file_path - 文件路径
    // raw_data.data.bytes_written - 字节数
    return (
      <div>
        <span>✅ 已写入 {raw_data.data.file_path}</span>
        <span>（{raw_data.data.bytes_written} 字节）</span>
      </div>
    );
  
  case 'delete_file':
    // raw_data.data.deleted_path - 删除路径
    // raw_data.data.message - 提示
    return (
      <div>
        <span>🗑️ 已删除 {raw_data.data.deleted_path}</span>
        <span className="hint">{raw_data.data.message}</span>
      </div>
    );
  
  case 'move_file':
    // raw_data.data.source - 源路径
    // raw_data.data.destination - 目标路径
    return (
      <div>
        <span>📦 {raw_data.data.source}</span>
        <span> → </span>
        <span>{raw_data.data.destination}</span>
      </div>
    );
  
  case 'search_files':
    // raw_data.data.matches[] - 匹配数组
    // raw_data.data.files_matched - 匹配文件数
    // raw_data.data.total_matches - 总匹配数
    return (
      <div>
        <span>🔍 找到 {raw_data.data.files_matched} 个文件，{raw_data.data.total_matches} 处匹配</span>
        {raw_data.data.matches.map(match => (
          <div key={match.file}>
            <div>📄 {match.file} ({match.match_count}处)</div>
            {match.matches.slice(0, 3).map((m, i) => (
              <pre key={i} className="context">...{m.context}...</pre>
            ))}
          </div>
        ))}
      </div>
    );
  
  case 'generate_report':
    // raw_data.data.reports{} - 报告字典
    return (
      <div>
        <span>📊 已生成 {Object.keys(raw_data.data.reports).length} 个报告</span>
        {Object.entries(raw_data.data.reports).map(([type, path]) => (
          <div key={type}>
            {type}: {path}
          </div>
        ))}
      </div>
    );
  
  default:
    // 未知工具，显示原始数据
    return <pre>{JSON.stringify(raw_data.data, null, 2)}</pre>;
}
```

## 四、observation 步骤的数据位置

### 4.1 action_tool vs observation

```
action_tool 步骤：
  - 工具执行前的"正在执行"状态
  - raw_data: 包含工具返回的原始数据

observation 步骤：
  - 工具执行后的"观察结果"状态
  - obs_raw_data: 包含原始返回数据（与 action_tool.raw_data 相同）
```

### 4.2 observation 的数据结构

```python
{
    "type": "observation",
    "tool_name": "list_directory",
    "obs_reasoning": "我已获取目录内容，现在分析...",  # 可选
    "obs_raw_data": {
        "status": "success",
        "data": {
            "entries": [...],
            "total": 100,
            ...
        }
    }
}
```

### 4.3 建议：统一使用 raw_data.data

前端解析时，应该统一访问 `step.raw_data?.data`：

```tsx
// action_tool 步骤
const data = step.raw_data?.data;

// observation 步骤  
const data = step.obs_raw_data?.data;

// 统一处理
if (data) {
    // 根据 tool_name 显示
}
```

## 五、修复优先级建议

| 优先级 | 工具 | 复杂度 | 原因 |
|-------|------|--------|------|
| P0 | list_directory | 中 | 用户最常用，已发现显示问题 |
| P1 | search_files | 高 | matches 嵌套结构复杂 |
| P2 | read_file | 低 | 直接显示 content |
| P3 | write_file | 低 | 显示写入结果 |
| P4 | delete_file | 低 | 显示删除确认 |
| P5 | move_file | 低 | 显示移动路径 |
| P6 | generate_report | 低 | 显示报告列表 |
