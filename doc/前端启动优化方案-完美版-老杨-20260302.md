# 前端启动优化方案-完美版

**创建时间**: 2026-03-02 19:50:00  
**作者**: 老杨  
**需求理解**: 
- 用户运行系统时：启动快，不检查
- 开发测试调试时：需要检查，但不影响日常开发
- E2E测试时：保证代码质量

---

## 方案设计原则

| 场景 | 是否需要ESLint/Prettier | 说明 |
|------|----------------------|------|
| **用户运行 (npm run dev)** | ❌ 不需要 | 启动要快，用户体验优先 |
| **开发调试** | ⚠️ 可选 | 通过IDE或手动命令运行，不阻塞启动 |
| **构建 (npm run build)** | ✅ 需要 | 保证发布代码质量 |
| **E2E测试** | ✅ 需要 | 通过npm scripts单独运行，不在vite.config中 |

---

## 实施方案

### 1. vite.config.ts - 只在构建时检查

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import eslint from 'vite-plugin-eslint';
import prettier from 'vite-plugin-prettier';

export default defineConfig(({ command }) => {
  const isBuild = command === 'build';
  
  return {
    plugins: [
      react(),
      // ⭐ 老杨完美方案：只在构建时运行检查
      // - 用户运行(npm run dev): 不检查，启动快
      // - 构建(npm run build): 检查，保证质量
      isBuild && eslint(),
      isBuild && prettier({
        parser: 'typescript',
      }),
    ].filter(Boolean),
    server: {
      port: 3000,
      proxy: {
        '/api': {
          target: 'http://localhost:8000',
          changeOrigin: true,
        },
      },
    },
  };
});
```

### 2. package.json - 完善scripts

```json
{
  "scripts": {
    "dev": "vite",                    // 用户用：快，不检查
    "build": "tsc && vite build",     // 构建：自动检查
    "preview": "vite preview",
    
    // ⭐ 开发测试用：单独的检查命令
    "lint": "eslint src --ext .ts,.tsx",
    "lint:fix": "eslint src --ext .ts,.tsx --fix",
    "format": "prettier --write .",
    "format:check": "prettier --check .",
    
    // ⭐ 完整检查：提交前或测试前运行
    "check": "npm run lint && npm run format:check",
    
    // ⭐ 测试相关
    "test": "vitest run",
    "test:watch": "vitest",
    "test:coverage": "vitest run --coverage",
    
    // ⭐ E2E测试前先检查
    "test:e2e": "npm run check && playwright test",
    "test:e2e:ui": "npm run check && playwright test --ui",
    "test:e2e:debug": "npm run check && playwright test --debug"
  }
}
```

## 使用指南

### 用户使用（最简单）

```bash
npm run dev
# ✅ 启动快！没有ESLint/Prettier检查
```

### 开发人员使用

**方式1: IDE自动检查**（推荐）
- 配置VS Code/WebStorm在保存时自动运行ESLint和Prettier
- 不影响启动速度

**方式2: 手动检查**
```bash
# 开发过程中随时检查
npm run lint
npm run lint:fix  # 自动修复

npm run format
npm run format:check
```

**方式3: 提交前完整检查**
```bash
npm run check
# 同时运行ESLint和Prettier检查
```

### E2E测试时

```bash
npm run test:e2e
# ✅ 自动先运行 npm run check，再运行E2E测试
# 保证测试时代码质量
```

### 构建时

```bash
npm run build
# ✅ vite.config.ts中配置了，自动运行检查
# 保证发布代码质量
```
