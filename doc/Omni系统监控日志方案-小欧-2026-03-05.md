## 一、日志方案概述

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| **统一目录** | 所有日志集中在 `backend/logs/` 目录 |
| **统一命名** | 主体部分统一为 `app`，格式为 `app_YYYY-MM-DD.log` |
| **统一配置** | 所有模块使用 `app.utils.logger` 模块管理日志 |
| **分级记录** | 支持 DEBUG/INFO/WARNING/ERROR 四个级别 |
| **自动轮转** | 单文件最大 10MB，保留 5 个备份 |

### 1.2 目录结构

```
OmniAgentAs-desk/
├── backend/
│   ├── logs/                      # 日志根目录
│   │   ├── app_2026-02-16.log    # 按日期轮转
│   │   ├── app_2026-02-17.log
│   │   ├── ...
│   │   └── app_2026-03-05.log    # 当天日志
│   ├── app/
│   │   ├── utils/
│   │   │   └── logger.py         # 日志配置模块
│   │   └── main.py               # 启动入口
│   └── start.bat                 # 启动脚本
└── config/
    └── config.yaml               # 日志配置
```

## 二、配置文件说明

### 2.1 config.yaml 日志配置

**位置**: `config/config.yaml`

```yaml
logging:
  level: INFO              # 日志级别: DEBUG/INFO/WARNING/ERROR
  max_file_size: 10485760 # 单文件最大 10MB
  backup_count: 5         # 保留 5 个备份
```

**配置项说明**:

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| level | string | INFO | 日志级别 |
| max_file_size | int | 10485760 | 单文件最大字节数（10MB） |
| backup_count | int | 5 | 保留的备份文件数量 |

### 2.2 日志级别说明

| 级别 | 值 | 使用场景 |
|------|-----|---------|
| DEBUG | 10 | 调试信息，仅开发环境使用 |
| INFO | 20 | 正常业务日志 |
| WARNING | 30 | 警告信息，不影响功能 |
| ERROR | 40 | 错误信息，功能异常 |

### 3.2 使用方法

#### 3.2.1 基本使用

```python
# 1. 导入日志模块
from app.utils.logger import logger

# 2. 记录日志
logger.debug("调试信息")
logger.info("普通信息")
logger.warning("警告信息")
logger.error("错误信息")
```

#### 3.2.2 格式化输出

```python
# 使用 f-string 格式化
logger.info(f"用户 {user_id} 登录成功")
logger.error(f"请求失败: {error_message}")

# 或使用 % 格式化（旧式）
logger.info("用户 %s 登录成功", user_id)
```

#### 3.2.3 异常追踪

```python
try:
    # 可能抛出异常的代码
    result = risky_operation()
except Exception as e:
    # 记录异常堆栈
    logger.error(f"操作失败: {e}", exc_info=True)
```

## 四、日志文件管理

### 4.1 文件轮转机制

```
┌─────────────────────────────────────────────────────────────┐
│                   日志轮转流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   app_2026-03-05.log (当前活动文件)                        │
│        ↓ 达到 10MB                                         │
│   重命名为 app_2026-03-05.log.1                            │
│        ↓ 保留 5 个备份                                      │
│   删除 app_2026-03-05.log.5 (最旧)                        │
│                                                             │
│   创建新的 app_2026-03-05.log                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 备份文件命名

| 文件 | 说明 |
|------|------|
| `app_2026-03-05.log` | 当天活动日志 |
| `app_2026-03-05.log.1` | 1号备份（昨天） |
| `app_2026-03-05.log.2` | 2号备份 |
| `app_2026-03-05.log.3` | 3号备份 |
| `app_2026-03-05.log.4` | 4号备份 |
| `app_2026-03-05.log.5` | 5号备份（最旧） |

### 4.3 手动清理脚本

```python
# cleanup_logs.py
from pathlib import Path
from datetime import datetime, timedelta

LOG_DIR = Path("backend/logs")
DAYS_TO_KEEP = 7  # 保留7天

def cleanup_old_logs():
    cutoff_date = datetime.now() - timedelta(days=DAYS_TO_KEEP)
    
    for log_file in LOG_DIR.glob("app_*.log*"):
        if log_file.suffix == ".log":
            # 跳过当天日志
            continue
        
        # 解析日期
        try:
            date_str = log_file.stem.replace("app_", "").split(".")[0]
            file_date = datetime.strptime(date_str, "%Y-%m-%d")
            
            if file_date < cutoff_date:
                log_file.unlink()
                print(f"已删除: {log_file.name}")
        except:
            continue

if __name__ == "__main__":
    cleanup_old_logs()
```

## 六、最佳实践

### 6.1 日志记录规范

| 规范 | 说明 | 示例 |
|------|------|------|
| **使用合适级别** | INFO 记录正常流程，ERROR 记录异常 | 登录成功用 INFO，登录失败用 ERROR |
| **包含上下文** | 记录足够的上下文信息 | `logger.info(f"用户 {user_id} 登录成功")` |
| **避免敏感信息** | 不记录密码、密钥等敏感数据 | 记录 `user_id` 而非 `password` |
| **异常堆栈** | 记录异常时包含堆栈信息 | `logger.error(msg, exc_info=True)` |

### 6.2 禁止事项

| 禁止 | 说明 |
|------|------|
| ❌ 禁止使用 print | 必须使用 logger |
| ❌ 禁止硬编码路径 | 必须使用 LOG_DIR 常量 |
| ❌ 禁止多个主体 | 只能有一个主体 `app` |
| ❌ 禁止随意创建 logger | 必须使用 `app.utils.logger` |

### 6.3 新增模块日志

如果新增模块需要使用日志，按以下步骤：

```python
# 1. 导入统一的日志模块
from app.utils.logger import logger

# 2. 直接使用，无需额外配置
logger.info("模块初始化完成")

# 3. 禁止创建新的 logger
# ❌ 错误示例:
# logger = logging.getLogger("my_module")  # 禁止

# ✅ 正确示例:
# from app.utils.logger import logger
# logger.info("我的模块日志")
```

## 七、监控集成

### 7.1 日志监控指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| ERROR 数量 | > 10/min | 触发告警 |
| 响应时间 | > 5s | 性能警告 |
| 日志文件大小 | > 100MB | 存储告警 |

### 7.2 日志告警示例

```python
# alert_monitor.py
from pathlib import Path
import time

def monitor_errors():
    LOG_FILE = Path("backend/logs/app_2026-03-05.log")
    ERROR_THRESHOLD = 10  # 每分钟最多10个错误
    
    while True:
        # 统计最近1分钟的错误数
        error_count = 0
        current_time = time.time()
        
        with open(LOG_FILE, 'r', encoding='utf-8') as f:
            for line in f:
                if "ERROR" in line:
                    # 简单解析时间（实际应更精确）
                    error_count += 1
        
        if error_count > ERROR_THRESHOLD:
            send_alert(f"错误数量过多: {error_count}/min")
        
        time.sleep(60)
```

## 八、故障排查

### 8.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 日志不写入文件 | LOG_DIR 不存在 | 检查 `backend/logs/` 目录 |

| 控制台无输出 | 非 DEBUG 模式 | 修改 config.yaml level |
| 文件过大 | 未启用轮转 | 检查 max_file_size 配置 |
