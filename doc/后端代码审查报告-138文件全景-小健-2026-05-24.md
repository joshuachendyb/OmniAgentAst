#### 13.1.4 重构方案

**目标**: 将 `run_stream` 从 498 行压缩到 ~80 行（高层循环骨架），其余逻辑下沉到 8 个私有方法。

```

├── _should_interrupt_and_yield(step_count, running_tasks, task_id)  # 15 行
├── _handle_llm_response(response, step_count, running_tasks, task_id)  # 30 行

│   │   ├── _handle_completion_type(parsed, step_count, ...)  # 40 行
```

**关键设计决策**:

1. **提取 `ChunkBuffer` 类**（独立文件 `agent/chunk_buffer.py`）

   - 消除 3 处重复的 flush 逻辑

2. **提取 `_dispatch_parsed_type()` 分发器**
   - 将 6-way if/elif 链（chunk/answer/thought_only/parse_error/action/隐含兜底）提取为映射表
   - 使用 `HANDLER_MAP = {"chunk": _handle_chunk_type, ...}` 替代长 if/elif

3. **合并 4 处中断检查**
   - 当前 4 处中断检查代码几乎相同（仅日志前缀不同）
   - 提取为 `_should_interrupt_and_yield()`，接受 `check_point: str` 参数用于日志区分

4. **`_handle_action_type` 委托给 `_execute_action_flow()`**
   - 包含: ThoughtStep yield → 中断检查 → _execute_tool_step → 中断检查 → Observation → pending calls
   - 这是当前唯一还比较大的私有方法（~80行），但职责单一："执行完整 Action→Observation 周期"
