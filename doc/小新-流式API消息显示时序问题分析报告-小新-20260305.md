## 一、分析背景与问题说明

### 1.1 项目背景

OmniAgentAs-desk 是一款基于AI的智能桌面助手应用，提供聊天对话、任务执行、数据分析等多种功能。其中，流式API消息显示功能是核心交互体验的重要组成部分，直接影响用户对AI响应的理解和使用体验。

### 1.2 问题描述

#### 1.2.1 核心问题

在当前版本中，流式API消息显示存在严重的时序颠倒和布局问题：

**问题1：消息显示时序颠倒**
- **现象**：AI思考过程（thinking字段）显示在下方，而最终结果显示在上方
- **影响**：不符合用户阅读和理解的逻辑顺序，导致混淆
- **严重程度**：P1-高

**问题2：字段获取错误**
- **现象**：前端错误地获取了 `rawData.observation` 字段而非 `rawData.result` 字段
- **影响**：导致显示大片JSON数据而非用户友好的文本内容
- **严重程度**：P0-紧急

**问题3：数据库未保存执行步骤**
- **现象**：前端未保存 `execution_steps` 到数据库
- **影响**：刷新页面后执行步骤信息消失，用户无法回顾完整会话历史
- **严重程度**：P1-高

**问题4：布局层次不合理**
- **现象**：消息内容和执行步骤的视觉层次关系不正确
- **影响**：降低了信息的可读性和用户体验
- **严重程度**：P2-中

#### 1.2.2 使用场景分类

根据消息类型和内容特点，将流式API消息分为以下3种主要场景：

| 场景分类 | 特点 | 需要显示的内容 |
|---------|------|--------------|
| **简单问答** | 无思考过程，直接输出 | 只显示最终答案 |
| **复杂问答** | 思考过程（连续文本）+ 答案 | 思考过程（可折叠）+ 最终答案 |
| **动作操作类** | 思考过程和操作步骤交替 | 思考过程 + 执行动作（带参数/结果）+ 最终答案 |

### 1.3 边界情况处理

#### 1.3.1 网络异常情况
- **网络中断**：显示加载失败提示，支持重试功能
- **超时**：设置合理的超时时间，显示超时提示
- **连接错误**：显示连接失败信息，引导用户检查网络

#### 1.3.2 空值和异常值处理
- **null/undefined字段**：显示默认文本或隐藏对应部分
- **异常JSON结构**：显示解析错误提示，记录详细错误信息
- **空消息内容**：显示友好提示，避免空界面

#### 1.3.3 特殊内容处理
- **过长文本**：支持文本折叠和展开功能
- **代码块**：优化代码显示格式，支持语法高亮
- **表格数据**：优化表格布局，支持响应式显示

async function handleStreamResponse(response) {
  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    
    const chunk = decoder.decode(value);
    const parsed = parseSSEChunk(chunk);
    
    if (parsed.event === 'thinking') {
      updateThinkingProcess(parsed.data);
    } else if (parsed.event === 'execution_step') {
      addExecutionStep(parsed.data);
    } else if (parsed.event === 'result') {
      showFinalResult(parsed.data);
    }
  }
}

def format_execution_steps(steps):
    formatted_steps = []
    for step in steps:
        formatted_steps.append({
            'id': step.get('id', str(uuid.uuid4())),
            'action': step.get('action', 'unknown'),
            'parameters': step.get('parameters', {}),
            'observation': step.get('observation', ''),
            'result': step.get('result', ''),
            'status': step.get('status', 'completed'),
            'timestamp': step.get('timestamp', datetime.now().isoformat())

- **可能性**：高
- **影响**：前端代码需要相应调整
- **缓解措施**：与后端团队密切沟通，制定版本兼容性策略

**风险2：浏览器兼容性风险**
- **可能性**：中
- **影响**：部分浏览器可能不支持某些功能
- **缓解措施**：使用现代化浏览器支持的API，添加必要的polyfill

#### 9.2.2 业务风险

**风险1：用户接受度风险**
- **可能性**：中
- **影响**：新界面可能需要用户适应期
- **缓解措施**：提供清晰的引导和帮助文档

**风险2：功能回归风险**
- **可能性**：低
- **影响**：修改可能引入新的问题
- **缓解措施**：完善的测试覆盖和回归测试

### 9.3 最终建议

1. **立即修复紧急问题**：优先处理字段获取错误和消息显示时序颠倒问题
