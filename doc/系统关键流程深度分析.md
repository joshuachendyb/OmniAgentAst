## 一、概述

OmniAgentAs-desk 是全栈（React+FastAPI）AI Agent 桌面应用。核心模式是 **用户输入 → 意图检测 → Agent工厂 → ReAct循环 → SSE流式返回**。

本文深度梳理系统中 10 个关键流程，从入口到出口追踪完整数据流。

## 二、10 大关键流程

### 流程1：用户请求 → SSE 流式响应（核心主流程）

**文件链**: `frontend/src/utils/sse.ts:673` → `chat_stream_v2.py:28` → `run_sse_stream.py:16` → `react_cycle.py:137`

```
Frontend fetch POST /api/v1/chat/stream
  │
  ▼
chat_stream_v2 (chat_stream_v2.py:28)
  ├── 1. detect_intent(user_input)         → 意图检测(CRSS)
  ├── 2. register_task(task_id)            → 注册任务
  ├── 3. task_interrupt_check              → 中断检查
  ├── 4. task_pause_check                  → 暂停检查
  ├── 5. step_start()                      → 发出start SSE事件
  ├── 6. run_sse_stream()                  → Agent执行
  │     ├── AgentFactory.create(intent)    → 创建Agent
  │     ├── agent.run_react_cycle(task)    → ReAct循环
  │     │     ├── _call_llm()              → 调用LLM
  │     │     ├── parse_llm_response()     → 解析LLM响应
  │     │     ├── handle type → yield step → 按类型分发
  │     │     └── _execute_tool()          → 执行工具
  │     └── yield SSE                      → 流式返回
  │
  ├── 7. task_cancel_check_and_yield       → 取消检查
  └── 8. task_cleanup                      → 清理
```

**关键设计**:
- task 生命周期注册在 `chat_stream_v2`，异常在 `finally` 保证 cleanup
- SSE 事件在 `run_sse_stream` 层直接 `to_dict()`，不经过 SSE 字符串往返（**P1 修复**）
- execution_steps 累积到 final 时**批量保存一次**（**P1 修复**）

### 流程2：意图检测流程（CRSS 双维度评分）

**文件链**: `detect_intent.py:16` → `crss_scorer.py` → `crss_definitions.py`

```
detect_intent(user_input)
  │
  ▼
crss_scorer.detect_intent_v2(user_input)
  ├── 第一层：类型评分
  │     └── 匹配 ToolCategory (FILE/SYSTEM/NETWORK/DOCUMENT/DESKTOP)
  │         关键词来自 INTENT_MAPPING + CRSS_TYPE_KEYWORDS
  │
  ├── 第二层：动作评分（兼容矩阵）
  │     └── 匹配 ACTION_DEFINITIONS
  │         命中后通过兼容系数调制类型分
  │
  ├── 无类型匹配 → 动作推断类型（兜底）
  ├── 无任何匹配 → primary=None → 返回 "system"（LLM兜底）
  │
  └── 返回: (primary, candidates, confidence)
```

**关键设计**:
- CRSS 评分器是纯规则引擎，无 LLM 调用，速度极快
- 无匹配时走 `system` 兜底，不会阻断用户请求
- 仅支持 5 类意图（file/system/network/document/desktop），由 `_CRSS_REGISTRY` 驱动

### 流程4：ReAct 循环流程（核心推理引擎）

**文件链**: `react_cycle.py:137` → `parse_llm_response()` → type handlers

```
run_react_cycle(task, context)
  │
  ├── _initialize_run_state()       → 初始化steps/message_builder/chunk_buffer
  │
  ├── while step < max_steps:
  │     │
  │     ├── step_counter += 1
  │     ├── _call_llm()             → 调用LLM获取响应
  │     ├── parse_llm_response()    → 解析为 {type, tool_name, thought, ...}
  │     │
  │     ├── type = parsed["type"]
  │     │
  │     ├── type == "action" ───────────────────── _handle_action()
  │     │     ├── ThoughtStep                     → 思考步骤
  │     │     ├── ActionToolStep                  → 工具调用步骤
  │     │     └── _execute_tool() + ObservationStep → 执行+观察
  │     │
  │     ├── type == "answer" / "implicit" ──────── _handle_answer()
  │     │     ├── ThoughtStep                     → 思考（可选）
  │     │     └── FinalStep + status=COMPLETED    → 最终回答
  │     │
  │     ├── type == "chunk" ────────────────────── _handle_chunk()
  │     │     └── ChunkStep + ChunkBuffer         → 流式片段
  │     │
  │     └── type == "parse_error" ──────────────── _handle_parse_error()
  │           └── ErrorStep + status=FAILED
  │
  ├── COMPLETED/FAILED → break
  └── finally:
        ├── _on_after_loop()
        └── _complete_tracked_task()
```

**关键设计**:
- 类型分发使用 `OrderedDict` 注册式分派（**P2-5 修复**），非 if/elif 硬编码
- handler 系统开放可扩展（`_TYPE_HANDLERS: OrderedDict[str, callable]`）
- 每个 handler yield 一个或多个 Step 对象，保持生成器协程语义
- `_execute_tool` 调用 `ToolRetryEngine`（带重试逻辑）

### 流程6：SSE 事件流（后端 → 前端）

**文件链**: `react_cycle.py` → `chat_stream.py:33(format_agent_sse)` → `sse.ts:956(processSSEData)`

```
后端 (Python)                              前端 (TypeScript)
────────────                             ────────────────
Step对象 (.to_dict())                    fetch POST /chat/stream
    │                                          │
    ▼                                          ▼
format_agent_sse(step)                    ReadableStream reader
    │  → SSE字符串                           │
    ▼  "data: {json}\n\n"                     ▼
yield sse_data                            processSSEData(line)
                                              │
                                              ├── type == "start" → startStep
                                              ├── type == "thought" → update executionSteps
                                              ├── type == "chunk" → onChunk + content累加
                                              ├── type == "action_tool" → tool step
                                              ├── type == "observation" → result step
                                              ├── type == "final" → onComplete + disconnect
                                              ├── type == "error" → onError + disconnect
                                              └── type == "incident" → incident handling
```

**8种 SSE type**:

| Type | 含义 | 前端处理 |
|------|------|---------|
| `start` | 开始，含模型/安全信息 | 显示"正在思考" |
| `thought` | LLM思考过程 | 步骤列表展示 |
| `chunk` | AI流式回复片段 | 内容区累加显示 |
| `action_tool` | 工具调用 | 显示工具执行状态 |
| `observation` | 工具执行结果 | 显示结果摘要 |
| `final` | 最终回答 | 完成回调 |
| `error` | 错误信息 | 错误提示 |
| `incident` | 中断/暂停/恢复/重试 | 状态切换 |

**前端关键设计**:
- sessionStorage 备份 SSES 数据（页面隐藏后恢复）
- 指数退避 + Full Jitter 重连策略（最多3次）
- 空闲超时检测（60s 无数据判定断开）
- token 性能指标（TTFT、TPS等）

### 流程7：任务控制流程（取消/暂停/恢复/确认）

**文件链**: `chat_router.py:28` → `services/task/`

```
用户操作                API端点                        后端处理
────────              ────────                      ────────
点击取消    →    POST /chat/stream/cancel/{id}  →   task_cancel.py
                      设置 task.cancelled=true
                      检查 agent._cancelled
                      react_cycle 检测 yield 中断chunk

点击暂停    →    POST /chat/stream/pause/{id}   →   task_pause.py
                      设置 task.paused=true
                      yield paused 状态

点击恢复    →    POST /chat/stream/resume/{id}  →   task_resume.py

确认操作    →    POST /chat/stream/confirm      →   confirm_operation.py
```

**关键设计**:
- task 状态在 `task_registry` 中管理，跨协程可访问
- `_cancelled` 标志位在 `react_cycle` 每轮检测
- 暂停/恢复通过 `asyncio.Event` + `wait()` 挂起协程

### 流程8：会话与消息持久化流程

**文件链**: `api/v1/sessions/` + `api/v1/messages.py` → `db/`

```
创建会话    POST /sessions         → 生成 session_id, 存入 DB
获取会话    GET /sessions          → 分页列表
获取消息    GET /sessions/{id}/messages → 含 execution_steps
保存消息    POST /sessions/{id}/messages → 保存 AI 回复
保存步骤    POST /sessions/{id}/execution_steps → 保存步骤 + content

保存时机：
  ├── start 步骤 → 立即保存（create_execution_steps API）
  ├── final 步骤 → 批量保存（run_sse_stream 的 final 时刻）
  └── 异常/中断 → 立即保存（error_step 时）
```

**关键设计**:
- 乐观锁版本号（version 字段）管理会话标题并发更新
- `save_execution_steps_to_db` 是唯一保存入口（**合并后**）
- 无效 session_id 加入 `_INVALID_SESSION_IDS` 集合并跳过（避免高频重试）

## 四、关键架构决策

| 决策 | 说明 | 文件位置 |
|------|------|---------|
| 配置驱动 Agent | 所有 intents 共用 UniversalAgent | `agent_config.py:47` |
| CRSS 纯规则引擎 | 不调用 LLM 做意图分类 | `crss_scorer.py` |
| SSE 直接 to_dict | 避免 Step→SSE→dict 往返解析 | `run_sse_stream.py:53` |
| 批量保存 | 不在每个 chunk 保存，final 时走一次 | `run_sse_stream.py:74` |
| 注册式 Handler | OrderedDict 分派，可扩展 | `react_cycle.py:127` |
| 任务生命周期 | 在 chat_stream_v2 统一管理 | `chat_stream_v2.py:84` |

## 五、流程中的关键风险点

| 风险点 | 描述 | 影响 |
|--------|------|------|
| react_cycle 缺少 max_steps 兜底 | 死循环可能导致无限调用 | 资源耗尽 |
| 工具执行无超时 | 工具可能被阻塞（如 shell） | 协程泄漏 |
| LLM 策略状态管理 | tools/text/response_format 切换复杂 | 推理错误 |
| SSE 重连与状态同步 | 重连时 steps/ref/state 不一致 | 前端显示异常 |
| 批量保存丢失 | final 前如果崩溃，steps 未保存 | 数据丢失 |
| 空 session_id 保护 | `_INVALID_SESSION_IDS` 集合在内存中 | 重启后不持久 |

## 六、总结

系统共 10 大关键流程，核心是 **CRSS → AgentFactory → ReAct循环 → SSE流式** 的完整链路。

| 流程编号 | 流程名称 | 核心文件 | 数据流向 |
|---------|---------|---------|---------|
| 1 | 主请求流 | `chat_stream_v2.py` → `run_sse_stream.py` | REST → SSE |
| 2 | 意图检测 | `detect_intent.py` → `crss_scorer.py` | 规则引擎 |
| 3 | Agent创建 | `agent_factory.py` → `agent_config.py` | 声明式工厂 |
| 4 | ReAct循环 | `react_cycle.py` | 推理引擎核心 |
| 5 | 工具系统 | `tool_manager.py` → `registry.py` | 注册式加载 |
| 6 | SSE事件流 | `chat_stream.py` ↔ `sse.ts` | 流式双向 |
| 7 | 任务控制 | `services/task/` | REST控制 |
| 8 | 持久化 | `api/v1/conversation.py` → `db/` | 异步写入 |
| 9 | LLM通讯 | `llm_core.py` → `client_sdk.py` | HTTP流式 |
| 10 | 配置管理 | `api/v1/ai_config/` → `config.py` | CRUD |
