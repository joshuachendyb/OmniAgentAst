## 一、问题背景

### 1.1 现状问题

现有 Word（write_docx）和 PDF（write_pdf）工具的参数存在**严重重叠**：

| 参数 | Word | PDF |
|------|------|-----|
| file_path | ✅ | ✅ |
| title | ✅ | ✅ |
| content | ✅ | ✅ |
| paragraphs | ✅ | ✅ |
| table_data | ✅ | ✅ |
| data | ✅ | ❌ |

共 `6个`/`5个` 参数，但实际**只有 2 种**数据处理逻辑：

1. **标题** — `title` 一个就够了
2. **正文** — `content` 的 list 模式（支持 heading/paragraph/table）能覆盖全部，但 `paragraphs`、`table_data`、`data` 重复实现

### 1.2 核心矛盾

| 参数 | 能力 | 状态 |
|------|------|------|
| content(list) | 支持 heading/h1~h5/paragraph/table 4种格式 | ✅ 完整 |
| paragraphs | 只支持纯文本 str(list) | ❌ 和 content 重叠 |
| table_data | 只能传表格 | ❌ content(list) 的 table type 已包含 |
| data | content+title 的包装壳 | ❌ 多余 |

---

## 二、设计目标

### 2.1 统一原则

1. **参数最小化** — 每个工具不超过 3 个参数
2. **格式统一化** — 同一种格式在 Word/PDF/PPT 中语义一致
3. **向后不兼容** — 旧参数直接废弃，不兼容旧调用方式
4. **功能不丢失** — 现有 4 种内容类型（heading/h1~h5/paragraph/table）全部保留

### 2.2 覆盖范围

| 工具 | 改前参数数 | 改后参数数 | 状态 |
|------|-----------|-----------|------|
| write_docx | 6 | 3 | 本次 |
| write_pdf | 5 | 3 | 本次 |
| write_pptx | 3 | 3 | 本次 |
| write_xlsx | 3 | 不动 | 不动 |

## 三、参数设计

### 3.1 最终参数表

```
file_path:  str               → 输出文件路径（必需）
title:      Optional[str]     → 文档标题（可选）
paragraphs: Union[str, List, Dict]  → 正文内容（可选）
```

### 3.2 paragraphs 的三种格式

#### 格式1：str — 纯文本段落

```json
"这是正文内容"
```

处理：`doc.add_paragraph("这是正文内容")`

适用：简单单段文档。

#### 格式2：list — 混合内容列表

```json
[
  {"type": "heading", "text": "第一章", "level": 1},
  {"type": "h2", "text": "二级标题"},
  "纯文本段落",
  {"type": "paragraph", "text": "格式段落"},
  {"type": "table", "rows": [["a","b"],["1","2"]]}
]
```

每个元素处理：

| 元素类型 | 格式 | Word 处理 | PDF 处理 |
|---------|------|----------|---------|
| str | "文本" | add_paragraph | Paragraph(style) |
| dict type=heading/h1 | {"text":"标题","level":1} | add_heading(level=1) | Paragraph(font size=18, bold) |
| dict type=h2~h5 | {"text":"副标题","level":2} | add_heading(level=N) | Paragraph(font size=16-N, bold) |
| dict type=paragraph | {"text":"正文"} | add_paragraph | Paragraph(style) |
| dict type=table | {"rows":[[...]]} | add_table | Table |

#### 格式3：dict — 结构化文档

```json
{
  "title": "文档标题",
  "content": [
    {"type": "heading", "text": "第一章", "level": 1},
    "正文内容"
  ]
}
```

处理：
1. 取出 `title`（如果外部 `title` 参数为空则用这个）
2. `content` 按 list 格式逐项处理

适用：需要标题+内容的完整文档。

### 3.3 废弃参数

| 参数 | 替代方式 |
|------|---------|
| content | paragraphs[str] / paragraphs[list][str] |
| table_data | paragraphs[list][dict type=table] |
| data | paragraphs[dict] |

## 四、改前 vs 改后对比

### Word 改前

```python
write_docx(
    file_path, content=None, paragraphs=None,
    title=None, table_data=None, data=None
)
```

内部处理流程：
```
data → 解包到 title + content
title → add_heading(level=0)
content(list) → 逐项判断 heading/paragraph/table
content(str)  → add_paragraph
paragraphs(list) → str() 逐项 add_paragraph
table_data → Table
```

### Word 改后

```python
write_docx(
    file_path, title=None, paragraphs=None
)
```

内部处理流程：
```
paragraphs(dict) → 拆出 title + content → 转 list/str
paragraphs(list) → 逐项判断 type:
  dict: heading/h1/h2~h5/paragraph/table
  str:  add_paragraph
paragraphs(str) → add_paragraph
title → add_heading(level=0)
```

### PDF 改前

```python
write_pdf(
    file_path, title=None, content=None,
    paragraphs=None, table_data=None
)
```

### PDF 改后

```python
write_pdf(
    file_path, title=None, paragraphs=None
)
```

处理逻辑同 Word，PDF 的 heading 用字体大小+粗体模拟标题级别。

## 五、PPT (write_pptx) 参数统一设计

### 5.1 PPT 与 Word/PDF 的本质差异

PPT 是**多页幻灯片集合**，不是线性文档，不能复用 `title`+`paragraphs` 模式：

| 维度 | Word/PDF | PPT |
|------|---------|-----|
| 页面结构 | 线性单文档 | 多页幻灯片 |
| 封面 | title 参数 | 主标题 + 副标题 |
| 标题 | 全文一个 | 每页独立 |
| 内容类型 | heading/paragraph/table | paragraph/bullets |
| table 位置 | 内嵌文本流 | **独立 shape**，不能放 text_frame |

**核心差异**: PPT 的 table 是独立 shape（`slide.shapes.add_table`），不能和文本混在一个 text_frame 里。所以设计上 table 必须从 content 中分离出来。

### 5.2 参数设计

去掉 `title` 参数（封面是 `slides[0]`），只有 2 参数：

```
file_path:  str                 → 输出PPT路径（必需）
slides:     Optional[List]      → 幻灯片列表（可选，不填则空演示）
```

### 5.3 每页幻灯片结构

```python
{
    "type":     int | str,          # 类型（可选，默认1）
    "title":    str,                # 标题（可选）
    "subtitle": str,                # 副标题（仅type=cover有效）
    "content":  str | List,         # 正文（可选）
    "tables":   List[List[List]],   # 表格（可选，独立shape）
}
```

**type 可选值**：

| 值 | 含义 | 说明 |
|----|------|------|
| 0 或 "cover" | 封面 | 主标题 + 副标题 |
| 1 或 "content" | 标题+正文 | 默认 |
| 2 或 "two" | 两栏内容 | title + 两栏 |

超出范围回退到 type=1。

### 5.4 content 格式

复用 paragraphs 的 3 格式原则，但：
- **去掉 heading** — 标题由 slide.title 承担
- **去掉 table** — table 是独立 shape，放到 `tables` 字段
- **新增 bullets** — PPT 特有列表

#### str — 纯文本

```json
"正文内容"
```

→ 放到 content placeholder 作为一段。

#### list — 混合内容

```python
[
    "段落文本",                              # str: 段落
    {"type": "paragraph", "text": "正文"},    # 段落
    {"type": "bullets", "items": ["项1","项2"]},  # 列表
]
```

| 元素类型 | 格式 | PPT 处理 |
|---------|------|---------|
| str | "文本" | text_frame.add_paragraph(文本) |
| dict type=paragraph | {"text":"正文"} | text_frame.add_paragraph(文本) |
| dict type=bullets | {"items":["项1","项2"]} | for i in items: p=add_paragraph(); p.text=i; p.level=1 |

### 5.5 tables 格式

table 在 slide 级独立传入，不和 content 混在一起：

```python
"tables": [
    [["列1", "列2"], ["a", "b"]],     # 第一个表格
    [["h1", "h2"], ["v1", "v2"]],     # 第二个表格
]
```

每个表格用 `slide.shapes.add_table(rows, cols, left, top, width, height)` 添加。

### 5.6 废弃参数

| 参数 | 替代方式 |
|------|---------|
| write_pptx 的 `title` 参数 | `slides[0].type="cover"` + `slides[0].title` + `slides[0].subtitle` |
| slides[i].content(type=table) | slides[i].tables |

### 5.7 改前 vs 改后

#### 改前

```python
write_pptx(file_path, title="封面", slides=[
    {"title": "页1", "content": "纯文本"},
])
```

#### 改后

```python
write_pptx(file_path, slides=[
    {"type": "cover", "title": "报告标题", "subtitle": "副标题"},
    {"title": "页1", "content": ["段落1", {"type": "bullets", "items": ["a","b"]}], "tables": [[["列1","列2"],["v1","v2"]]]},
])
```

### 5.8 代码结构

```python
def _write_pptx(file_path, slides=None):
    prs = _build_pptx_presentation(slides or [])
    prs.save(file_path)

def _build_pptx_presentation(slides):
    """构建全部幻灯片 — 小欧 2026-06-19"""
    prs = Presentation()
    for slide_data in slides:
        _add_pptx_slide(prs, slide_data)
    return prs

def _add_pptx_slide(prs, slide_data):
    """添加一页幻灯片（SRP: 只负责1页）— 小欧 2026-06-19"""
    slide_type = slide_data.get("type", 1)
    title = slide_data.get("title", "")
    subtitle = slide_data.get("subtitle", "")
    content = slide_data.get("content")
    tables = slide_data.get("tables")

    layout = _select_layout(prs, slide_type)
    slide = prs.slides.add_slide(layout)

    # 标题
    if title and slide.shapes.title:
        slide.shapes.title.text = title

    # 封面副标题
    if subtitle and slide_type in (0, "cover"):
        for shape in slide.placeholders:
            if shape.placeholder_format.idx == 1:
                shape.text = subtitle; break

    # 正文
    if content:
        _add_pptx_content(slide, content)

    # 表格（独立shape）
    if tables:
        for td in tables:
            _add_pptx_table(slide, td)

def _select_layout(prs, slide_type):
    """选布局 — 小欧 2026-06-19"""
    m = {0: 0, "cover": 0, 1: 1, "content": 1, 2: 2, "two": 2}
    return prs.slide_layouts[m.get(slide_type, 1)]

def _add_pptx_content(slide, content):
    """处理正文到 content placeholder — 小欧 2026-06-19"""
    body = None
    for sh in slide.placeholders:
        idx = sh.placeholder_format.idx
        if idx == 1:  # subtitle 已处理, 跳过
            continue
        if idx >= 2:  # content/body placeholder
            body = sh.text_frame; break
    if body is None:
        return

    if isinstance(content, str):
        body.paragraphs[0].text = content
    elif isinstance(content, list):
        for i, item in enumerate(content):
            if isinstance(item, str):
                p = body.paragraphs[0] if i == 0 else body.add_paragraph()
                p.text = item
            elif isinstance(item, dict):
                t = item.get("type", "paragraph")
                txt = item.get("text", "")
                if t == "paragraph":
                    p = body.paragraphs[0] if i == 0 else body.add_paragraph()
                    p.text = txt
                elif t == "bullets":
                    for j, b in enumerate(item.get("items", [])):
                        p = body.paragraphs[0] if (i == 0 and j == 0) else body.add_paragraph()
                        p.text = str(b); p.level = 1

def _add_pptx_table(slide, table_data):
    """添加表格到幻灯片 — 小欧 2026-06-19"""
    if not table_data or not table_data[0]:
        return
    rows, cols = len(table_data), len(table_data[0])
    from pptx.util import Inches
    left, top, width, height = Inches(1), Inches(2), Inches(6), Inches(0.4 * rows)
    tbl = slide.shapes.add_table(rows, cols, left, top, width, height).table
    for ri, row in enumerate(table_data):
        for ci, val in enumerate(row):
            tbl.cell(ri, ci).text = str(val)
```

## 六、代码设计优化

### 6.1 SLAP — 同层抽象

三个 `isinstance` 分支（str/list/dict）是**高层路由**，各 type 的处理是**底层细节**，不能混在一起。

正确做法：抽一个 `_add_content_item(doc, item)` 处理**单个元素**。

```python
def _add_docx_content_item(doc, item):
    """处理单个内容元素(Word)"""
    if isinstance(item, str):
        if item.strip():
            doc.add_paragraph(item)
    elif isinstance(item, dict):
        item_type = item.get("type", "paragraph")
        item_text = item.get("text", "")
        if item_type in ("h1", "heading"):
            doc.add_heading(item_text, item.get("level", 1))
        elif item_type in ("h2", "h3", "h4", "h5"):
            doc.add_heading(item_text, level=int(item_type[1]))
        elif item_type == "paragraph":
            doc.add_paragraph(item_text)
        elif item_type == "table":
            rows = item.get("rows", [])
            if rows:
                t = doc.add_table(rows=len(rows), cols=len(rows[0]))
                for ri, rd in enumerate(rows):
                    for ci, cv in enumerate(rd):
                        t.rows[ri].cells[ci].text = str(cv)
    else:
        text = str(item)
        if text.strip():
            doc.add_paragraph(text)
```

```python
# _add_pdf_content_item 必须是 _write_pdf 内部的嵌套函数
# 原因: reportlab 的 Paragraph/Spacer/Table/TableStyle/ParagraphStyle/mm
# 在 _write_pdf 的 try 块内导入，如果 import 失败整个函数返回错误
# 嵌套函数在 try 作用域内，能正确访问这些导入

def _write_pdf(file_path, title=None, paragraphs=None):
    if not _check_module("reportlab"):
        return build_error(...)
    try:
        from reportlab.lib.pagesizes import A4
        from reportlab.lib.styles import getSampleStyleSheet, ParagraphStyle
        from reportlab.lib.units import mm
        from reportlab.platypus import SimpleDocTemplate, Paragraph, Table, TableStyle, Spacer
        from reportlab.lib import colors
        from reportlab.pdfbase import pdfmetrics
        from reportlab.pdfbase.ttfonts import TTFont
        
        def _add_content_item(elements, item, chinese_style, title_style):
            """处理单个PDF内容元素(嵌套函数,可访问reportlab导入)"""
            if isinstance(item, str):
                if item.strip():
                    elements.append(Paragraph(item, chinese_style))
                    elements.append(Spacer(1, 3*mm))
            elif isinstance(item, dict):
                item_type = item.get("type", "paragraph")
                item_text = item.get("text", "")
                if item_type in ("h1", "heading"):
                    heading_style = ParagraphStyle('h1', parent=chinese_style,
                        fontSize=18, spaceBefore=12, spaceAfter=6)
                    elements.append(Paragraph(item_text, heading_style))
                    elements.append(Spacer(1, 3*mm))
                elif item_type in ("h2", "h3", "h4", "h5"):
                    fs = max(18 - int(item_type[1]) * 2, 12)
                    h_style = ParagraphStyle(f'h{item_type[1]}', parent=chinese_style,
                        fontSize=fs, spaceBefore=8, spaceAfter=4)
                    elements.append(Paragraph(item_text, h_style))
                    elements.append(Spacer(1, 2*mm))
                elif item_type == "paragraph":
                    elements.append(Paragraph(item_text, chinese_style))
                    elements.append(Spacer(1, 3*mm))
                elif item_type == "table":
                    rows = item.get("rows", [])
                    if rows and len(rows) > 0:
                        try:
                            t = Table(rows)
                            t.setStyle(TableStyle(...))
                            elements.append(t)
                            elements.append(Spacer(1, 5*mm))
                        except Exception:
                            pass
            else:
                text = str(item)
                if text.strip():
                    elements.append(Paragraph(text, chinese_style))
                    elements.append(Spacer(1, 3*mm))
```

### 6.2 复用优先 — 公用函数

Word 和 PDF 的 `paragraphs` 路由逻辑（str/list/dict 三路分发）**完全相同**。应该抽成一个公用函数，避免重复。

```python
def _resolve_paragraphs(paragraphs):
    """解析paragraphs参数: 统一输出为list格式
    
    输入: str / list / dict
    输出: (title_from_dict, content_list)
    """
    title = None
    items = []
    
    if isinstance(paragraphs, str):
        items = [paragraphs]
    elif isinstance(paragraphs, list):
        items = paragraphs
    elif isinstance(paragraphs, dict):
        title = paragraphs.get("title")
        content = paragraphs.get("content", [])
        items = content if isinstance(content, list) else [content]
    
    return title, items
```

这样 Word 和 PDF 的路由函数变成：

```python
# _write_docx
dict_title, items = _resolve_paragraphs(paragraphs)
if dict_title and not title:
    title = dict_title
if title:
    doc.add_heading(title, 0)
for item in items:
    _add_docx_content_item(doc, item)
```

```python
# _write_pdf
dict_title, items = _resolve_paragraphs(paragraphs)
if dict_title and not title:
    title = dict_title
if title:
    elements.append(Paragraph(title, title_style))
    elements.append(Spacer(1, 10*mm))
for item in items:
    _add_pdf_content_item(elements, item, chinese_style, title_style)
```

### 6.3 复查发现的3个问题（已修正）

| # | 问题 | 发现轮次 | 修正 |
|---|------|---------|------|
| 1 | list 元素含非str/非dict类型（如数字）会被静默跳过 | 第2遍 | `_add_docx_content_item` 和 `_add_pdf_content_item` 加 `else: str(item)` 兜底 |
| 2 | `_add_pdf_content_item` 依赖 reportlab 导入，但 import 在 `_write_pdf` try 块内，抽成模块级函数不可用 | 第3遍 | 改为 `_write_pdf` 内部嵌套函数，正确访问 try 作用域内的导入 |
| 3 | `title` 参数和 `paragraphs` dict 里都有 title 时谁优先 | 第1遍 | 已有标注：`title` 参数优先 |
