### 方案2：DRY治理 — 重复代码消除（高优先级）

#### 2.1 DB连接管理统一化

```python
# 新建 db/db_context.py
from contextlib import contextmanager

@contextmanager
def get_db_connection(db_path: str):
    """统一DB连接上下文管理器，替换所有手动conn/cursor/try/finally/close"""
    conn = sqlite3.connect(db_path)
    try:
        yield conn
    finally:
        conn.close()

# 所有调用方统一改用:
with get_db_connection(db_path) as conn:
    cursor = conn.cursor()
    ...
```

### 方案4：SLAP治理 — 抽象层分离（中优先级）

**核心方法**: "骨架-细节"模式 — 每个函数只做编排，底层操作提取为私有方法

```python
# Before (SLAP违反):
async def _handle_action_type(self, ...):
    yield step              # 高层：编排
    self.chunk_buffer.flush()  # 底层：chunk细节
    thought += chunk_content   # 底层：拼接细节

# After (SLAP修复):
async def _handle_action_type(self, ...):
    yield step
    await self._flush_and_update_thought(...)  # 底层细节封装

async def _flush_and_update_thought(self, ...):
    self.chunk_buffer.flush()
    thought += chunk_content
```

**影响范围**: `base_react.py`、`llm_strategies.py`、`llm_core.py`、`chat_stream_query.py`、`operation_history.py` 共6个文件18处

| 问题 | 解决方案 |
|------|---------|
| O(n²)累加 (`operation_history.py`) | 改为O(n)前缀和累加 |
| 200硬编码命令 (`command_security.py`) | 移到YAML配置文件，支持动态加载 |
