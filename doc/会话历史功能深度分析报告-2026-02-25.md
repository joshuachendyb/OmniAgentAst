## 四、修复方案建议

### 🔧 修复方案1：明确会话标题的数据源优先级

**原则**：
1. **最高优先级**：用户明确修改过的标题（存储在后端）
2. **次优先级**：后端API返回的 `session.title` 字段
3. **最低优先级**：消息内容（只在没有其他标题时使用）

**修复代码示意**:
```typescript
// 加载会话时
const loadSessionFromApi = async (sessionId: string) => {
  const sessionData = await sessionApi.getSessionMessages(sessionId);
  
  // 优先使用后端返回的title
  let title = sessionData.title;  // ⭐ 新增：从sessionData获取title
  
  // 如果后端没有title，才用消息内容（降级方案）
  if (!title && sessionData.messages && sessionData.messages.length > 0) {
    title = sessionData.messages[0]?.content?.substring(0, 30) || '会话';
  }
  
  setSessionTitle(title || '会话');
};
```

### 🔧 修复方案2：解决sessionStorage和URL参数冲突

**原则**：
1. **URL参数优先**：如果URL有 `session_id`，**忽略sessionStorage**
2. **清除sessionStorage**：切换历史会话时，清除旧的sessionStorage
3. **只在无URL参数时**：才使用sessionStorage

**修复代码示意**:
```typescript
const loadSession = async () => {
  const urlSessionId = searchParams.get('session_id');

        setMessages(sessionData.messages.map(...));

      console.warn('加载URL会话失败:', error);
    }

      console.log('从缓存恢复会话状态');
      return;
    }
  }

  // 2. 次优先级: 用户已经修改过的标题（从状态中获取）
  if (sessionTitle && sessionTitle !== '新会话' && sessionTitle !== '会话') {
    return sessionTitle;
  }
  
  // 3. 最低优先级: 消息内容（只在没有其他标题时使用）
  if (sessionData?.messages?.[0]?.content) {
    return sessionData.messages[0].content.substring(0, 30) + '...';
  }
  
  // 4. 默认标题
  return '新会话';
};

// API接口更新
export interface SessionMessagesResponse {
  messages: ChatMessage[];
  title?: string;  // ✅ 新增: 可选的title字段
}
```

**修复效果**: 会话标题来源统一，不再被消息内容意外覆盖
