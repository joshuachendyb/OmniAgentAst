## 二、分阶段实施计划

### 阶段1：类型定义（第1天）

| 任务 | 文件 | 说明 |
|------|------|------|
| 1.1 | `frontend/src/types/chat.ts` | 新建文件，定义8种消息类型 |

**产出**：根据设计文档10.2节，创建完整的类型定义

```typescript
// 需要定义的类型
export type StreamMessage = 
  | StartMessage      // type: 'start'
  | ThoughtMessage   // type: 'thought'
  | ActionToolMessage // type: 'action_tool'
  | ObservationMessage // type: 'observation'
  | ChunkMessage     // type: 'chunk'
  | FinalMessage     // type: 'final'
  | ErrorMessage     // type: 'error'
  | StatusMessage    // type: 'incident'
```

### 阶段2：SSE处理层（第2天）

| 任务 | 文件 | 说明 |
|------|------|------|
| 2.1 | `frontend/src/utils/sse.ts` | 修改字段映射 |

**修改要点**：

| 原字段 | 新字段 | 说明 |
|--------|--------|------|
| `type: "action"` | `type: "action_tool"` | type名称变更 |

| `rawData.action_input` | `rawData.tool_params` | 工具参数 |
| `rawData.result` | 适配新格式 | 简化后的结果 |
| - | 新增 `tool_name` | 工具名称 |
| - | 新增 `execution_status` | 执行状态 |

### 阶段3：API服务层（第3天）

| 任务 | 文件 | 说明 |
|------|------|------|
| 3.1 | `frontend/src/services/api.ts` | 添加任务控制API |

**新增方法**（根据设计文档10.10-10.12节）：

```typescript
// 任务控制
cancelTask(taskId: string): Promise<{ success: boolean; message: string }>
pauseTask(taskId: string): Promise<{ success: boolean; message: string }>
resumeTask(taskId: string): Promise<{ success: boolean; message: string }>

// 用户确认
confirmTask(taskId: string, confirmed: boolean, modifiedCommand?: string)

// 分页请求
requestNextPage(taskId: string, toolName: string, nextPageToken: string)
```

### 阶段5：用户确认流程（第6天）

| 任务 | 文件 | 说明 |
|------|------|------|
| 5.1 | NewChatContainer.tsx | 实现确认对话框 |

**处理逻辑**（根据设计文档10.6节）：

```
收到 status:paused → 显示确认对话框
    ↓
用户点击"确认" → 调用 confirmTask(taskId, true)
用户点击"拒绝" → 调用 confirmTask(taskId, false)
用户修改命令 → 调用 confirmTask(taskId, true, modifiedCommand)
```
