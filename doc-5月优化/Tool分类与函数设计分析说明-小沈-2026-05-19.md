## 一、总体架构

### 1.1 分类体系

系统采用 `ToolCategory` 枚举定义7个工具分类，共59个LLM可见工具：

```python
class ToolCategory(Enum):
    FILE = "file"          # 文件操作 — 11个工具
    SHELL = "shell"        # 命令执行 — 5个工具
    NETWORK = "network"    # 网络通信 — 5个工具
    SYSTEM = "system"      # 系统管理 — 10个工具
    DOCUMENT = "document"  # 文档处理 — 9个工具
    DESKTOP = "desktop"    # 桌面GUI — 10个工具
    META = "meta"          # 元工具+时间 — 9个工具
```

### 1.2 设计原则（依据精简方案§2）

| 编号 | 原则 | 轨道 | 在本系统中的体现 |
|------|------|------|-----------------|
| P11 | 统一入口+内部分发 | A | read_document按后缀路由、window_control按action路由、get_time按action路由 |
| P12 | 工具间互相调用 | A | read_document内部自动调用_check_pdf_readable等前置检查 |
| P13 | 分层架构Tool/Helper | A | env_check验证工具降级为document内部helper、gui_helpers降级为内部helper |
| P14 | 优雅降级 | A | read_excel缺pandas降级openpyxl、desktop缺pywin32返回ERR |
| P15 | 返回值自解释 | A | 工具返回含next_actions建议下一步操作 |
| P16 | 幂等性 | A | kill_process对已退出PID返回"无需终止" |
| P17 | 参数正交 | A | rename_file用mode区分single/batch，替代互斥参数组 |
| P1 | 重复合并 | B | read_pdf+read_docx+read_xlsx→read_document |
| P2 | 相似整合 | B | set_window_state+focus_window+resize_window→window_control |
| P3 | 功能互斥 | B | 任意两个工具功能无交集，LLM看到场景只匹配一个工具 |
| P5 | 后缀自动路由 | B | read_document("a.pdf")自动调pdfplumber |
| P9 | 直接删除 | B | 旧函数直接删除不保留委托 |

### 1.3 每个工具的4部分代码结构

| 部分 | 文件类型 | 职责 |
|------|---------|------|
| **Tool函数** | `*_tools.py` | 功能实现，接受参数→执行逻辑→返回{code,data,message} |
| **Schema** | `*_schema.py` | Pydantic参数模型，定义字段名/类型/必填/默认值/描述 |
| **Register** | `*_register.py` | 注册到tool_registry，关联函数+Schema+description+examples |
| **Prompt** | `prompts/*/*.py` | Agent的系统提示，列出可用工具名+参数+用例 |

## 二、FILE分类（11个工具）

### 2.1 分类概述

文件操作分类，覆盖文件的读取、写入、编辑、搜索、重命名、压缩、移动/复制/删除、结构化格式读写等全部文件管理场景。

### 2.2 涉及代码文件

| 文件 | 路径 | 职责 |
|------|------|------|
| file_tools.py | `backend/app/services/tools/file/file_tools.py` | 11个工具函数实现 |
| file_schema.py | `backend/app/services/tools/file/file_schema.py` | 11个Pydantic参数模型 |
| file_register.py | `backend/app/services/tools/file/file_register.py` | 注册点：description+examples+注册 |
| file_prompts.py | `backend/app/services/prompts/file/file_prompts.py` | Prompt模板(F1-F11工具说明+用例) |

### 2.3 工具清单

#### F1. read_file — 统一文件读取

- **设计原则**: P11统一入口(合并read_text_file+read_batch_file)
- **Schema**: `ReadFileInput` — file_paths（必填List[str]，1个元素=单文件模式，多个=批量模式）, head, tail, offset, limit, encoding
- **参数**:
  - **必填**: `file_paths`(文件路径列表，传1个=单文件模式支持分页，传多个=批量模式)
  - **可填**: `head`(取前N行) / `tail`(取后N行) / `offset`(从第N行开始读，1-indexed) / `limit`(限制返回行数，配合offset分页)，仅单文件模式生效；`encoding`(不填则自动检测utf-8/gbk/gb2312/utf-8-sig)
- **实现逻辑**: 单文件→逐行读取支持分页；批量→并行读取多个文件返回列表
- **适用场景**: 读取代码文件、配置文件、日志文件；批量读取多文件；分页读取大文件

#### F2. write_text_file — 写入文本文件

- **设计原则**: P7悲观默认值(encoding默认utf-8, append默认False)
- **Schema**: `WriteTextFileInput` — file_path, text, encoding, append, create_parents, unescape
- **参数**:
  - **必填**: `text`(写入内容，禁止传思考/计划)；`file_path`(文件绝对路径)
  - **可填**: `encoding`(不填则追加时自动检测已有文件编码，新建时utf-8)；`append`(默认False)；`create_parents`(默认True)；`unescape`(默认True，反转义\\n等)
- **实现逻辑**: append=False→覆盖写入；append=True→追加写入；自动创建父目录；自动反转义\n等
- **适用场景**: 创建新文件、覆盖写入、追加日志、保存代码

#### F3. read_media_file — 读取媒体文件

- **设计原则**: P4单一职责(只做媒体读取)
- **Schema**: `ReadMediaFileInput` — file_path
- **参数**:
  - **必填**: `file_path`(媒体文件路径，支持JPG/PNG/GIF/BMP/WebP/MP3/WAV/OGG/M4A)
  - **可填**: 无
- **实现逻辑**: 读取文件→Base64编码→返回{mime_type, data, size}
- **适用场景**: 读取图片用于OCR、读取音频、展示图片内容

#### F4. edit_file — 统一文件编辑

- **设计原则**: P11统一入口(合并precise_replace_in_file+edit_text_file)，P17互斥(old_string与edits二选一)
- **Schema**: `EditFileInput` — file_path, old_string, new_string, edits, replace_all, dry_run, encoding
- **参数**:
  - **必填**: `file_path`(目标文件绝对路径)；`old_string`+`new_string`(单处替换) 或 `edits`(多处编辑数组)，二选一互斥
  - **可填**: `replace_all`(默认False，仅old_string模式)；`dry_run`(默认False，预览不修改)；`encoding`(默认utf-8)
- **实现逻辑**: 单处→精确查找替换；多处→按edits数组顺序依次替换；dry_run→只返回预览diff
- **适用场景**: 修改代码函数名、批量替换配置项、预览修改效果
- **【小沈 2026-05-19 精简8→7】**: 砍`ignore_case`(极少使用，精确替换不应忽略大小写)

#### F5. list_directory — 统一目录列表

- **设计原则**: P11统一入口(合并list_directory+get_directory_tree+file_statistics)，P15返回值自解释(始终含statistics)
- **Schema**: `ListDirectoryInput` — dir_path, format, recursive, max_depth, sortBy, include_hidden, page_token
- **参数**:
  - **必填**: `dir_path`(目录绝对路径)
  - **可填**: `format`("list"扁平/"tree"JSON树，默认list)；`recursive`(默认False)；`max_depth`(默认10)；`sortBy`(name/size/mtime，默认name)；`include_hidden`(默认False)；`page_token`(分页令牌)
- **实现逻辑**: format=list→扁平文件列表+统计；format=tree→JSON树结构+统计；分页支持
- **适用场景**: 查看目录内容、查看项目树结构、统计文件数量/大小
- **【小沈 2026-05-19 精简8→7】**: 砍`exclude_patterns`(普通用户不需要，高级用search_files)

#### F6. search_files — 搜索文件名

- **设计原则**: P8自描述性(名字即功能)
- **Schema**: `SearchFilesInput` — pattern, search_dir, recursive, max_depth, ignore_case, type, page_token
- **参数**:
  - **必填**: `pattern`(glob通配符，如"*.py"/"**/*.ts"/中文文件名)；`search_dir`(搜索起始目录，必填绝对路径)
  - **可填**: `recursive`(默认True)；`max_depth`(默认50)；`ignore_case`(默认True)；`type`(file/directory过滤)；`page_token`(分页)
- **实现逻辑**: 用glob递归匹配文件名；支持中文文件名
- **适用场景**: 查找所有Python文件、查找配置文件、定位特定文件
- **【小沈 2026-05-19 精简9→7】**: 砍`excludePatterns`(驼峰命名不一致)；砍`sortBy`(搜索排序意义不大)

#### F7. grep_file_content — 搜索文件内容

- **设计原则**: P8自描述性
- **Schema**: `GrepFileContentInput` — pattern, search_dir, output_mode, glob, context, ignore_case, head_limit, multiline, page_token
- **参数**:
  - **必填**: `pattern`(正则表达式，支持中文搜索)
  - **可填**: `search_dir`(默认当前目录)；`output_mode`(content/files_with_matches/count)；`glob`(文件过滤如"*.py")；`context`(上下文行数对象，如{"around":3}上下各3行，{"after":2,"before":1})；`ignore_case`(默认True)；`head_limit`(最大返回数)；`multiline`(默认False)；`page_token`(分页)
- **实现逻辑**: Python re模块递归搜索文件内容；支持正则+中文；context对象解构为after_lines/before_lines/context_lines
- **适用场景**: 搜索TODO标记、查找函数定义、搜索错误关键字
- **【小沈 2026-05-19 精简13→9】**: `after_lines`+`before_lines`+`context_lines`→收编`context:Dict`(LLM只需传{"around":3})；砍`show_line_no`(永远显示行号)；砍`type`(glob更明确)

#### F8. rename_file — 统一重命名

- **设计原则**: P11统一入口(合并rename_file+batch_rename)，mode参数显式分发(single/batch)
- **Schema**: `RenameFileInput` — mode, path, new_name, directory, pattern, replacement
- **参数**:
  - **必填**: `mode`("single"单文件 / "batch"批量)；mode=single时需`path`+`new_name`；mode=batch时需`directory`+`pattern`+`replacement`
  - **可填**: 无
- **实现逻辑**: 单文件→直接重命名；批量→遍历目录+正则匹配+替换
- **适用场景**: 重命名单个文件、批量添加前缀、正则替换文件名
- **【小沈 2026-05-19 精简9→6】**: 砍`preview`(批量预览可先dry_run)；砍`recursive`(默认不递归)；砍`conflict_strategy`(默认skip，冲突覆盖用file_operation)

#### F9. archive_tool — 统一压缩/解压

- **设计原则**: P11统一入口(合并compress_files+extract_archive)，action路由
- **Schema**: `ArchiveToolInput` — action, source, destination, format, compression_level, password, overwrite, exclude_patterns
- **参数**:
  - **必填**: `action`("compress"/"extract")；compress时需`source`+`destination`；extract时需`source`
  - **可填**: `destination`(extract时可选，默认自动创建同名目录)；`format`(zip/tar/tar.gz/tar.bz2，默认zip)；`compression_level`(0-9，默认6)；`password`(仅ZIP)；`overwrite`(默认False)；`exclude_patterns`(排除模式)
- **实现逻辑**: compress→zipfile/tarfile打包；extract→解压到目标目录；支持密码加密ZIP
- **适用场景**: 压缩项目备份、解压下载的压缩包、加密压缩
- **【小沈 2026-05-19 精简11→8】**: `source_path`/`output_path`/`archive_path`/`output_dir`→统一`source`+`destination`(与F10对齐)；砍`preserve_permissions`(默认True)

#### F10. file_operation — 统一文件操作

- **设计原则**: P11统一入口(合并move_file+copy_file+delete_file)，action路由，P16幂等性
- **Schema**: `FileOperationInput` — action, source, destination, recursive, overwrite, force, preserve_metadata
- **参数**:
  - **必填**: `action`("move"/"copy"/"delete")；`source`(源路径)；move/copy时需`destination`
  - **可填**: `recursive`(默认False，copy目录/delete非空目录时需True)；`overwrite`(默认False)；`force`(delete跳过回收站永久删除，move/copy跳过确认)；`preserve_metadata`(copy保留元数据，默认True)
- **实现逻辑**: move→shutil.move；copy→shutil.copy2(保留元数据)；delete→send2trash(回收站)或os.remove(force)
- **适用场景**: 移动文件到新目录、复制文件、删除文件(安全到回收站)

#### F11. data_file_format — 结构化数据文件读写（配置文件工具）

- **设计原则**: P11统一入口(合并json/yaml/toml/ini/xml读写)，P5后缀自动路由。当LLM需要读取/修改JSON/YAML/TOML等配置文件时使用
- **Schema**: `DataFileFormatInput` — action, file_path, format, data, encoding, indent
- **参数**:
  - **必填**: `file_path`(绝对路径)；write时需`data`(写入数据)
  - **可填**: `action`("read"/"write"，默认read)
  - **可填**: `format`(json/yaml/toml/ini/xml/properties，不填按扩展名自动检测)；`encoding`(默认utf-8)；`indent`(JSON/YAML缩进空格数)
- **实现逻辑**: read→按格式解析(json.loads/yaml.safe_load/toml.load等)；write→按格式序列化
- **适用场景**: 读取JSON配置、修改YAML文件、解析TOML/INI/XML

- **适用场景**: 明天是周末吗、国庆放不放假、下个工作日是几号

#### M8. timezone_convert — 时区转换

- **设计原则**: P11统一入口(三方向：utc_to_local+local_to_utc+any)
- **Schema**: `TimezoneConvertInput` — time_value, direction, tz, source_tz, target_tz
- **参数**:
  - **必填**: `time_value`(时间值，Unix时间戳或日期字符串)
  - **可填**: `direction`(utc_to_local/local_to_utc/any，默认utc_to_local)；`tz`(时区，默认本地时区)；`source_tz`(direction=any时必填)；`target_tz`(direction=any时必填)
- **实现逻辑**: pytz.timezone→astimezone转换
- **适用场景**: UTC转北京时间、北京时间转纽约时间、跨国时间转换

#### M9. timer — 定时器管理

- **设计原则**: P11统一入口(三合一：timer_set+timer_clear+timer_list)
- **Schema**: `TimerInput` — action, delay, callback, callback_data, timer_id, limit
- **参数**:
  - **必填**: `action`(set/clear/list)；set时需`delay`+`callback`；clear时需`timer_id`
  - **可填**: `delay`(1~86400秒，最长24小时)；`callback`(提醒内容)；`callback_data`(附加数据)；`timer_id`(定时器ID)；`limit`(返回数量限制，默认10)
- **实现逻辑**: set→asyncio loop.call_later(delay, callback)创建TimerHandle；clear→timer.cancel()；list→返回所有活跃定时器
- **适用场景**: 3分钟后提醒、设置定时任务、取消提醒
