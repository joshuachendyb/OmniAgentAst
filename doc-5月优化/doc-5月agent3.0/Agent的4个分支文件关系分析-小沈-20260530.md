### 2.1 UniversalReactAgent（通用Agent）

```
UniversalReactAgent
  继承: BaseAgent（地基）
  混入3个mixin:
    ├── ToolStepMixin      ← 编排一次工具执行
    ├── ReactAgentMixin    ← 初始化逻辑
    └── RollbackMixin      ← 回滚能力
  实现4个抽象方法:
    ├── _get_llm_response()   → self._call_llm()
    ├── _execute_tool()       → execute_tool_with_unified_retry()
    ├── _get_system_prompt()  → self.prompts.get_system_prompt()
    └── _get_task_prompt()    → self.prompts.get_task_prompt()
  特点:
    - 配置驱动（AgentConfig）
    - 支持回滚（继承了RollbackMixin）
    - 取代原来9个重复的Agent子类
```

### 2.2 DesktopReactAgent（桌面Agent）

```
DesktopReactAgent
  继承: BaseAgent（地基）
  混入2个mixin（无Rollback）:
    ├── ToolStepMixin      ← 编排一次工具执行
    └── ReactAgentMixin    ← 初始化逻辑
  实现4个抽象方法:
    ├── _get_llm_response()   → self._call_llm()
    ├── _execute_tool()       → execute_tool_with_unified_retry()
    ├── _get_system_prompt()  → self.prompts.get_system_prompt()
    └── _get_task_prompt()    → self.prompts.get_task_prompt()
  特点:
    - 桌面操作专用
    - 不支持回滚（桌面操作不需要）
    - 硬编码 DesktopPrompts
```

### 2.3 RollbackMixin（回滚能力）

```
RollbackMixin
  提供方法:
    └── rollback(step_number) → execute_tool_with_unified_retry()
  被谁继承:
    ├── UniversalReactAgent  ✅ 继承（文件操作需要回滚）
    └── DesktopReactAgent    ❌ 不继承（桌面操作不需要）
  依赖:
    └── tool_executor.py::execute_tool_with_unified_retry()
```

**关键区别**：
- UniversalReactAgent**继承了**RollbackMixin → 支持回滚
- DesktopReactAgent**没有继承**RollbackMixin → 不支持回滚
- RollbackMixin不是独立分支，是UniversalReactAgent的父类之一

## 三、每个文件的职责

| 角色 | 文件 | 类 | 职责 | 关键代码 |
|------|------|-----|------|---------|
| **地基** | `base_react.py` | `BaseAgent` | ReAct循环骨架，定义抽象方法 | `run_stream()` + 4个抽象方法 |
| **Agent子类1** | `universal_react.py` | `UniversalReactAgent` | 通用Agent，继承BaseAgent+3个mixin | `_execute_tool()` → `execute_tool_with_unified_retry()` |
| **Agent子类2** | `desktop_react.py` | `DesktopReactAgent` | 桌面Agent，继承BaseAgent+2个mixin（无Rollback） | `_execute_tool()` → `execute_tool_with_unified_retry()` |
| **Agent子类3** | `generic_react.py` | `GenericReactAgent` | 兜底Agent，继承BaseAgent，不调工具 | `_execute_tool()` → `return {}` |
| **mixin能力** | `rollback_mixin.py` | `RollbackMixin` | 回滚能力，仅被UniversalReactAgent继承 | `rollback()` → `execute_tool_with_unified_retry()` |

## 四、调用流程

```
Agent.run(task)
    │
    ▼
BaseAgent.run_stream()          ← ReAct循环（定义在base_react.py）
    │
    │ 每轮循环:
    │   1. _get_llm_response()   ← 子类实现（调LLM）
    │   2. parse → action
    │   3. _execute_tool(action)  ← 子类实现（调工具）
    │       │
    │       └──→ tool_executor.py::execute_tool_with_unified_retry()
    │                 │
    │                 └──→ tool_retry_engine.py::execute_tool_with_retry()
    │
    │ 观察结果 → 继续循环/退出
    │
    ▼
AgentResult
```

## 六、Agent层 vs Tool层

```
┌─────────────────────────────────────────────────────────────┐
│  UniversalReactAgent   DesktopReactAgent                    │
│       │                    │                               │
│       │ 继承               │ 继承                           │
│       └──────────┬─────────┘                               │
│                  ▼                                         │
│         ┌───────────────┐                                  │
│         │ base_react.py │  ← 地基                           │
│         └───────┬───────┘                                  │
│                 │ 委托                                      │
│                 ▼                                          │
│           tool_executor.py                                  │
│                 │                                          │
│                 ▼                                          │
│           tool_retry_engine.py → 各category_tools.py       │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  RollbackMixin（回滚能力，仅被UniversalReactAgent继承）       │
│       │                                                     │
│       └── rollback() → tool_executor.py::execute_tool_with_ │
│                         unified_retry()                     │
└─────────────────────────────────────────────────────────────┘
```

- **继承链**：UniversalReactAgent/DesktopReactAgent → BaseAgent（地基）
- **混入链**：ToolStepMixin + ReactAgentMixin → 被2个Agent子类继承
- **继承链**：RollbackMixin → 仅被UniversalReactAgent继承
- **委托链**：base_react.py → tool_executor.py（Agent层向下委托）
- **调用链**：RollbackMixin.rollback() → tool_executor.py::execute_tool_with_unified_retry()

**Agent层**定义"什么时候调工具、调哪个工具"，**Tool层**负责"怎么把工具跑起来"。Agent层不碰工具实现，只通过`_execute_tool()`这个抽象方法向下层委托。RollbackMixin是Agent层的一个能力组件，不是独立分支。

子类	_execute_tool() 实现	是否调工具

最终方案（一次性解决）：

base_react.py：_execute_tool() 从抽象方法改为默认实现，直接调 execute_tool_with_unified_retry()
universal_react.py：删掉 _execute_tool() 方法，继承base的默认实现
desktop_react.py：同上
generic_react.py：保留自己的 _execute_tool()（它不调工具，返回空）
tool_init_mixin.py：删掉 _execute_tool_func lambda（死代码，没人用）
改完后：

所有需要调工具的Agent → 继承base的默认实现，不用写任何代码
不调工具的Agent（GenericReactAgent）→ 自己override
新增Agent子类 → 自动获得工具调用能力，不用再重复写
符合DRY：executor逻辑只写一次，在base_react.py。
