### 1.3 设计原则

| 原则 | 说明 | 违反后果 |
|------|------|---------|
| **复制优先** | 拆分时复制原代码逻辑，只改导入路径，不改业务逻辑 | 重写引入新bug |
| **向后透明** | 拆分后外部import路径不变（通过`__init__.py`重导出） | 87个调用者全要改 |
| **逐个拆分** | 每个问题独立完成、独立测试，不一次性全改 | 8批并行出问题难定位 |
| **先测后改** | 修改前运行现有测试，修改后再次运行确认无回归 | 不知道自己改坏了 |
| **转发层保留** | 原文件改为转发层（`from new_pkg import X`），兼容期1个月 | 调用者逐个迁移风险大 |

### 1.4 通用迁移模式

```
拆分前:  origin.py (含ClassA, ClassB, func_c)
                          ↓
拆分后:  origin.py (转发层: from .new_pkg import ClassA, ClassB, func_c)
         new_pkg/
         ├── __init__.py (重导出: from .class_a import ClassA; from .class_b import ClassB)
         ├── class_a.py  (复制原ClassA代码)
         ├── class_b.py  (复制原ClassB代码)
         └── func_c.py   (复制原func_c代码)
```

### 1.5 风险定义

| 风险等级 | 定义 | 处理要求 |
|---------|------|---------|
| **P0-紧急** | 导致系统崩溃、数据丢失 | 立即回滚 |
| **P1-高** | 功能异常，性能严重下降 | 修复后再提交 |
| **P2-中** | 边界问题、轻微性能影响 | 应该修复 |
| **P3-低** | 代码质量问题 | 建议修复 |

```python
@dataclass
class RenameContext:
    """批量重命名上下文"""
    # 业务参数
    directory: str
    pattern: str
    replacement: str
    recursive: bool = False
    preview: bool = False
    conflict_strategy: str = "skip"
    
    # 依赖注入回调
    validate_path_func: Optional[Callable] = None
    safety_service: Optional[Any] = None
    task_id: Optional[str] = None
    record_operation_func: Optional[Callable] = None
    execute_with_safety_func: Optional[Callable] = None
    get_next_sequence_func: Optional[Callable] = None
    
    def __post_init__(self):
        """延迟计算 regex"""
        if self.pattern:
            try:
                self._regex = re.compile(self.pattern)
                self._use_regex = True
            except re.error:
                self._regex = None
                self._use_regex = False
        else:
            self._regex = None
            self._use_regex = False
```

```python
class MonitoringMiddleware:
    async def __call__(self, scope, receive, send):
        if scope["type"] != "http":
            return await self.app(scope, receive, send)
        
        path = scope.get("path", "")
        method = scope.get("method", "GET")
        start_time = time.time()
        request_size = 0
        
        wrapped_send = self._make_send_wrapper(send, start_time, method, path, request_size)
        try:
            await self.app(scope, receive, wrapped_send)
        except Exception as e:
            self._record_metric_error(method, path, e)
            raise
    
    def _make_send_wrapper(self, send, start_time, method, path, request_size):
        """创建包装send — 闭包提取为工厂方法"""
        async def wrapper(message):
            if message["type"] == "http.response.start":
                self._record_response_metrics(message, start_time, method, path, request_size)
            await send(message)
        return wrapper
    
    def _record_response_metrics(self, message, start_time, method, path, request_size):
        """记录响应指标 — 提取的独立方法"""
        status_code = message["status"]
        self.collector.record_metric(
            name="http_requests_total", value=1,
            labels={"method": method, "path": path, "status": str(status_code)}
        )
        duration = time.time() - start_time
        self.collector.record_metric(
            name="http_request_duration_seconds", value=duration,
            labels={"method": method, "path": path}
        )
        self.collector.record_metric(
            name="http_request_size_bytes", value=request_size,
            labels={"method": method, "path": path}
        )
        if status_code >= 400:
            self.collector.record_metric(
                name="errors_total", value=1,
                labels={"method": method, "path": path, "status": str(status_code)}
            )
    
    def _record_metric_error(self, method, path, exception):
        """记录异常指标"""
        self.collector.record_metric(
            name="errors_total", value=1,
            labels={"method": method, "path": path, "error": type(exception).__name__}
        )
```

```python
    def _on_after_loop(self) -> None: ...
    async def _execute_tool_step(
        self, tool_name: str, tool_params: Dict[str, Any],
        step_count: int, is_primary: bool = True
    ) -> Any: ...
```

### 13.4 Protocol 使用

Protocol 是**结构子类型**（structural subtyping），不需要显式声明实现：

```python
# react_handler_mixin.py
from typing import Protocol
from .protocols import HandlerHostProtocol

class ReActHandlerMixin:
    """ReAct循环响应处理逻辑 — 依赖 HandlerHostProtocol 的宿主类"""
    
    async def _handle_empty_response(self, step_count: int) -> AsyncGenerator[Dict[str, Any], None]:
        # 通过 self 访问协议属性（假设宿主满足 HandlerHostProtocol）
        _retry_cnt = self._empty_response_retry_engine.attempt_count  # type: ignore[attr-defined]
        ...
```

`# type: ignore[attr-defined]` 可加在依赖宿主特有属性上。
