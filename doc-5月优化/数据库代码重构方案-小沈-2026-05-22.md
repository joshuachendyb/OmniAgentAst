## 四、风险分析

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| `sessions.py` 模块级 `_init_database()` 调用被删除后，迁移代码 `init_database()` 在 import 时未执行 | 数据库表未创建，所有API失败 | 低 | `db/chat_db.py` 末尾保留模块级 `init_database()` 调用，与旧行为一致 |
| `execution.py` 单独加载时缺少 `init_database()` 调用 | 表未创建，SSE读取失败 | 低 | `execution.py` 现在就没有 `init_database()`，依赖 sessions.py 先加载；迁移后从 `db/chat_db.py` import，`db/chat_db.py` 有模块级初始化 |
| **`FileSafetyConfig.DB_PATH` 被外部引用（初版遗漏）** | `file_visualization.py` 和 `agent/session.py` 使用 `FileSafetyConfig().DB_PATH`，移除后崩溃 | **高** | §三Step 6 已补充：这两个文件改为直接 `from app.db.config import OPERATIONS_DB_PATH` |
| **`models/file_operations/` 被多处引用（初版遗漏）** | `file_tools.py`、`file_helpers.py`、`operation_history.py`、`agent/session.py` 导入路径断裂 | **高** | §三Step 6 已补充：所有引用改为 `from app.db.models.operation_*` |
| **`init_history_db.py` 的 monkey-patch 失效（初版遗漏）** | 调试脚本无法创建历史数据库 | 中 | §三Step 6 已补充：改为导入 `CHAT_DB_PATH` 覆盖和 `db/chat_db.py` 的 `init_database` |
| 并行执行 `pytest` 时多个测试同时 init_database | 表已存在，IF NOT EXISTS 安全 | 低 | 当前 SQL 已用 `CREATE TABLE IF NOT EXISTS`，不影响 |
| 旧 `models/file_operations/` 删除后，远程分支仍存在 | git 冲突 | 低 | commit 时明确说明文件删除 |
