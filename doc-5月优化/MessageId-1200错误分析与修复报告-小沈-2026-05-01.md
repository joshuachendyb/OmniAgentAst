### 系统BUG-1: precise_replace_in_file 解包崩溃

| 项目 | 内容 |
|------|------|
| **发生位置** | Step 57 |
| **工具** | precise_replace_in_file |
| **错误信息** | `cannot unpack non-iterable bool object` |

**什么情况下触发：** 任何调用precise_replace_in_file都会崩溃。

**根因：** `_replace_sync`函数返回`tuple`，但外层`execute_with_safety`只返回`bool`，原始tuple被丢弃。解包`count, used_enc, name = True`报错。

**BUG代码：**
```python
def _replace_sync() -> tuple[int, str, str]:
    return count, used_enc, path.name

count, used_enc, name = await asyncio.to_thread(
    self.safety.execute_with_safety,   # 返回bool！
    operation_id=operation_id,
    operation_func=_replace_sync
)
```

**修复代码：**
```python
replace_result = {}

def _replace_sync() -> bool:
    ...
    replace_result['count'] = count
    replace_result['used_enc'] = used_enc
    replace_result['name'] = path.name
    return True

success = await asyncio.to_thread(
    self.safety.execute_with_safety,
    operation_id=operation_id,
    operation_func=_replace_sync
)
if success:
    return {..., "replaced_count": replace_result['count'], ...}
```

**修复commit:** `27c2dc0d` ✅ 已修复

### 系统BUG-3: execute_command 成功执行却标记为失败

| 项目 | 内容 |
|------|------|
| **发生位置** | Step 27 |
| **工具** | execute_command |
| **错误现象** | returncode=0、stdout正常输出50个文件，但execution_status="error" |

**根因：** 工具可能只检查了stderr非空就标记error，忽略了returncode=0。

**修复方案：** `returncode==0` → success；`returncode!=0` → error。stderr有内容只是warning。

**修复文件：** backend/app/services/tools/shell/ 相关代码

**修复commit:** `0e2a3006` ✅ 已修复

**深层问题：** `_format_result`之前只检查`success`字段，但shell工具返回`code/data/message`格式（无`success`字段），导致所有shell工具结果都被误判为error。修复后新增`code`字段判定链：`code=SUCCESS`→检查`returncode`→0=success/非0=error；`code=ERR_*`→error。

同时增强：execute_command默认timeout 30→60秒；超时时也返回已捕获输出；message区分returncode和stderr。

### 7.1 序号1: precise_replace_in_file + edit_file 解包崩溃

**修复commit:** `27c2dc0d` ✅ 已完成

两个工具的问题相同：`execute_with_safety`返回`bool`，但原代码期望解包函数原始返回值。

修复方式：用闭包字典存结果，`_sync`函数只返回`True`。

```python
# precise_replace_in_file 修复
replace_result = {}

def _replace_sync() -> bool:
    # ... 原有逻辑 ...
    replace_result['count'] = count
    replace_result['used_enc'] = used_enc
    replace_result['name'] = path.name
    return True

success = await asyncio.to_thread(
    self.safety.execute_with_safety,
    operation_id=operation_id,
    operation_func=_replace_sync
)
if success:
    return _to_unified_format({
        "success": True,
        "replaced_count": replace_result['count'],
        "encoding": replace_result['used_enc'],
        ...
    }, "precise_replace_in_file")
else:
    return _to_unified_format({"success": False, ...}, "precise_replace_in_file")

# edit_file 同理，edit_result = {} 存结果
```
