## 一、背景与设计目标

### 1.1 现有架构痛点

| 痛点 | 现状 | 影响 |
|------|------|------|
| **意图路由割裂** | CRSS正则 + LLM兜底，两套关键词维护 | 准确率低、维护成本高 |
| **安全防线脆弱** | `command_security.py` 黑名单模式 | 易被绕过，无法应对变量拼接 |
| **Agent同质化** | 9个子类中7个代码完全相同 | 修改需同步9处 |
| **无审计能力** | 工具调用无全量记录 | 无法回溯、无法检测异常 |
| **预处理空壳** | `PreprocessingPipeline` 仅做strip | 增加调用层级无价值 |

### 1.2 设计目标

| 编号 | 目标 | 优先级 | 验收标准 |
|------|------|--------|---------|
| G1 | 废除CRSS正则路由，改为LLM语义路由 | P0 | 意图识别准确率 ≥ 90% |
| G2 | 废除黑名单安全模式，改为工具声明式安全分级 | P0 | 135个工具全部标注安全等级 |
| G3 | 7个同质Agent合并为GenericReactAgent | P0 | 删除7个文件，功能不变 |
| G4 | 引入分级HITL机制 | P0 | DANGEROUS工具100%拦截确认 |
| G5 | 引入ToolObserver全量审计 | P1 | 所有工具调用可查询、可回溯 |
| G6 | 工具加载量控制在30个以内 | P1 | 单次请求加载工具数 ≤ 30 |
| G7 | 清理死代码 | P2 | 删除PreprocessingPipeline、CRSS等 |

### 1.3 核心设计原则

```
┌─────────────────────────────────────────────────┐
│  奥卡姆剃刀：如无必要，勿增实体                    │
│  分级安全：SAFE免确认，DANGEROUS必确认            │
│  语义优先：LLM理解替代正则匹配                     │
│  防御纵深：LLM自评 → 工具自检 → Observer监控      │
│  渐进迁移：灰度切换，随时可回退                   │
└─────────────────────────────────────────────────┘
```

### 2.2 调用链路（与现有对比）

| 阶段 | 现有架构 | 新架构 | 变化 |
|------|---------|--------|------|
| 预处理 | PreprocessingPipeline | `user_input.strip()` | **删除空壳管线** |
| 意图识别 | CRSS正则 + LLM兜底 | SemanticRouter（Function Calling） | **替换为语义路由** |
| Agent分发 | AgentFactory.create(intent) | GenericReactAgent(profile) | **统一Agent** |
| 工具加载 | 按intent加载单分类 | 按Router推荐加载多分类子集 | **动态子集加载** |
| 安全检测 | command_security黑名单 | ToolSafetyLayer分级检查 | **替换为声明式安全** |
| 工具执行 | 直接执行 | 执行前安全检查 → HITL → 执行 → Observer记录 | **增加三道防线** |
| 审计 | 无 | ToolObserver全量记录 | **新增审计能力** |

### 2.3 防御纵深模型

```
第一道防线：LLM安全自评
  ↓ 失败时
第二道防线：ToolSafetyLayer（工具声明式自检）
  ↓ 失败时（DANGEROUS工具）
第三道防线：HITL人工确认
  ↓ 执行后
第四道防线：ToolObserver异常检测 + 审计回溯

信任层级：
  LLM自评  →  信任但验证（可能被注入）
  工具自检 →  信任代码（声明式，注入无效）
  HITL     →  信任人类（绝对安全）
  Observer →  信任模式（事后检测）
```

## 三、核心模块详细设计

### 3.1 Semantic Router（语义路由器）

#### 3.1.1 设计定位

**不是意图分类器**，而是**工具子集推荐器**。输入用户自然语言，输出应该加载哪些工具分类。

#### 3.1.2 核心流程

```
用户输入: "帮我看看D盘有什么文件，然后搜索一下里面的PDF"
  ↓
LLM分析: 涉及"文件查看" + "PDF搜索"
  ↓
推荐: [ToolCategory.FILE, ToolCategory.DOCUMENT]
  ↓
Agent加载: FILE分类工具 + DOCUMENT分类工具 + META分类工具（始终加载）
  ↓
工具总数: ~25个（而非135个全量）
```

#### 3.1.3 实现代码

```python
# backend/app/services/agent/semantic_router.py

from typing import List, Optional
from app.services.tools.registry import ToolCategory
from app.services.llm_core import get_llm_client
from app.utils.logger import logger


# 工具分类描述 — 供LLM理解每个分类的能力
CATEGORY_DESCRIPTIONS = {
    ToolCategory.FILE: "文件和目录的读写、搜索、编辑、归档、压缩等操作",
    ToolCategory.SHELL: "执行Shell命令、Python代码、JavaScript代码",
    ToolCategory.NETWORK: "HTTP请求、下载、网页抓取、网络搜索、网络诊断",
    ToolCategory.DESKTOP: "窗口管理、鼠标键盘控制、截屏、OCR、剪贴板、通知",
    ToolCategory.SYSTEM: "系统信息查询、进程管理、服务控制、环境变量",
    ToolCategory.DOCUMENT: "文档读写、格式转换、SQL查询、图表生成、数据分析",
    ToolCategory.META: "工具帮助、工具搜索、时间日期、定时器",
}

# 始终加载的分类（提供基础能力）
ALWAYS_LOAD_CATEGORIES = [ToolCategory.META]

# 路由兜底：如果路由失败，加载这些分类
FALLBACK_CATEGORIES = [
    ToolCategory.FILE,
    ToolCategory.SHELL,
    ToolCategory.NETWORK,
    ToolCategory.SYSTEM,
    ToolCategory.DOCUMENT,
]


class SemanticRouter:
    """
    语义路由器 — 基于LLM Function Calling的工具子集推荐器
    
    职责：
    1. 分析用户输入的自然语言
    2. 推荐需要加载的工具分类（通常2-4个）
    3. 控制单次请求的工具加载量（≤ 30个）
    """
    
    def __init__(self, llm_client=None):
        self._llm_client = llm_client
    
    async def recommend_categories(self, user_input: str) -> List[ToolCategory]:
        """
        推荐工具分类子集
        
        Args:
            user_input: 用户原始输入
            
        Returns:
            推荐的ToolCategory列表（包含ALWAYS_LOAD_CATEGORIES）
        """
        try:
            categories = await self._call_llm_route(user_input)
            # 合并始终加载的分类
            result = list(set(categories + ALWAYS_LOAD_CATEGORIES))
            logger.info(f"[SemanticRouter] 推荐分类: {[c.value for c in result]}")
            return result
        except Exception as e:
            logger.warning(f"[SemanticRouter] 路由失败，使用兜底分类: {e}")
            return FALLBACK_CATEGORIES + ALWAYS_LOAD_CATEGORIES
    
    async def _call_llm_route(self, user_input: str) -> List[ToolCategory]:
        """调用LLM进行Function Calling路由"""
        if self._llm_client is None:
            self._llm_client = get_llm_client()
        
        # 构造Function Calling Schema
        category_names = [c.value for c in ToolCategory]
        tools = [{
            "type": "function",
            "function": {
                "name": "select_tool_categories",
                "description": "根据用户请求选择需要使用的工具分类",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "categories": {
                            "type": "array",
                            "items": {
                                "type": "string",
                                "enum": category_names,
                                "description": f"工具分类列表。分类能力：{CATEGORY_DESCRIPTIONS}"
                            },
                            "description": "推荐的工具分类，通常2-4个"
                        }
                    },
                    "required": ["categories"]
                }
            }
        }]
        
        response = await self._llm_client.chat(
            messages=[{"role": "user", "content": user_input}],
            tools=tools,
            tool_choice={"type": "function", "function": {"name": "select_tool_categories"}},
            max_tokens=100,  # 限制输出长度，加快响应
        )
        
        # 解析Function Calling结果
        tool_call = response.tool_calls[0]
        import json
        arguments = json.loads(tool_call.function.arguments)
        selected = arguments.get("categories", [])
        
        # 转换为ToolCategory枚举
        result = []
        for cat_name in selected:
            try:
                result.append(ToolCategory(cat_name))
            except ValueError:
                logger.warning(f"[SemanticRouter] 未知分类: {cat_name}")
        
        return result if result else FALLBACK_CATEGORIES


# 全局单例
semantic_router = SemanticRouter()
```

#### 3.1.4 性能优化策略

| 策略 | 说明 | 预期效果 |
|------|------|---------|
| **轻量模型** | 路由使用TTFT < 0.5s的专属模型 | 路由延迟 < 500ms |
| **max_tokens=100** | 限制LLM输出长度 | 减少token消耗 |
| **内存缓存** | 相同输入在5秒内缓存结果 | 重复请求零延迟 |
| **兜底机制** | 路由失败时使用FALLBACK_CATEGORIES | 保证系统可用性 |

```python
from enum import Enum


class ToolSafetyLevel(Enum):
    """
    工具安全等级 — 每个工具在注册时必须声明
    
    分级原则：
    - READ_ONLY: 纯读取，无任何副作用
    - SAFE: 有副作用但可逆或无害（如创建临时文件）
    - DESTRUCTIVE: 破坏性操作，不可逆（如删除文件、覆盖内容）
    - DANGEROUS: 危险操作，可能影响系统稳定性（如Shell命令、系统配置修改）
    """
    READ_ONLY = "read_only"
    SAFE = "safe"
    DESTRUCTIVE = "destructive"
    DANGEROUS = "dangerous"


# 各等级的处理策略
SAFETY_POLICY = {
    ToolSafetyLevel.READ_ONLY: {
        "needs_confirmation": False,    # 不需要确认
        "needs_safety_check": False,    # 不需要参数检查
        "log_level": "DEBUG",           # 低级别日志
    },
    ToolSafetyLevel.SAFE: {
        "needs_confirmation": False,
        "needs_safety_check": False,
        "log_level": "INFO",
    },
    ToolSafetyLevel.DESTRUCTIVE: {
        "needs_confirmation": True,     # 需要用户确认
        "needs_safety_check": True,     # 需要参数安全检查
        "log_level": "WARNING",
    },
    ToolSafetyLevel.DANGEROUS: {
        "needs_confirmation": True,     # 必须HITL确认
        "needs_safety_check": True,     # 必须参数安全检查
        "log_level": "ERROR",
    },
}
```

### 3.3 ToolSafetyLayer（工具安全自检层）

#### 3.3.1 核心职责

在工具执行前进行安全检查，根据安全等级决定是否放行、是否需要确认、是否需要参数检查。

#### 3.3.2 实现代码

```python
# backend/app/services/agent/tool_safety_layer.py

from typing import Dict, Any, Optional, Tuple
from app.services.tools.safety import ToolSafetyLevel, SAFETY_POLICY
from app.services.tools.registry import get_tool_metadata
from app.utils.logger import logger


class SafetyCheckResult:
    """安全检查结果"""
    
    def __init__(
        self,
        passed: bool,
        message: str = "",
        need_confirmation: bool = False,
        confirm_info: Optional[Dict] = None,
    ):
        self.passed = passed
        self.message = message
        self.need_confirmation = need_confirmation
        self.confirm_info = confirm_info or {}
    
    @classmethod
    def allow(cls) -> "SafetyCheckResult":
        return cls(passed=True)
    
    @classmethod
    def block(cls, reason: str) -> "SafetyCheckResult":
        return cls(passed=False, message=reason)
    
    @classmethod
    def need_confirm(cls, info: Dict) -> "SafetyCheckResult":
        return cls(passed=False, need_confirmation=True, confirm_info=info)


class ToolSafetyLayer:
    """
    工具安全自检层
    
    执行前检查：
    1. 工具安全等级 → 决定是否放行
    2. 参数安全检查 → 检查危险参数（如Shell命令）
    3. HITL确认 → DANGEROUS/DESTRUCTIVE工具需要用户确认
    
    设计原则：
    - 安全责任附在工具上，不在用户输入层
    - 每个工具自己声明安全属性，不依赖集中式规则
    - Prompt注入无效（检查的是参数本身，不是LLM输出）
    """
    
    def __init__(self):
        self._observer = None  # 延迟注入，避免循环依赖
    
    def set_observer(self, observer):
        self._observer = observer
    
    async def check(
        self,
        tool_name: str,
        tool_params: Dict[str, Any],
    ) -> SafetyCheckResult:
        """
        执行前安全检查
        
        Args:
            tool_name: 工具名称
            tool_params: 工具参数
            
        Returns:
            SafetyCheckResult
        """
        meta = get_tool_metadata(tool_name)
        if meta is None:
            logger.error(f"[ToolSafetyLayer] 未知工具: {tool_name}")
            return SafetyCheckResult.block(f"未知工具: {tool_name}")
        
        policy = SAFETY_POLICY.get(meta.safety_level, SAFETY_POLICY[ToolSafetyLevel.SAFE])
        
        # 1. 参数安全检查（如Shell命令检查）
        if meta.needs_safety_check and meta.safety_check_params:
            for param_name in meta.safety_check_params:
                param_value = tool_params.get(param_name, "")
                if param_value:
                    block_result = self._check_param_safety(
                        tool_name, param_name, str(param_value)
                    )
                    if block_result:
                        if self._observer:
                            self._observer.record(
                                tool_name, "blocked",
                                f"参数安全检查拦截: {block_result}"
                            )
                        return SafetyCheckResult.block(block_result)
        
        # 2. 需要用户确认
        if policy["needs_confirmation"]:
            confirm_info = self._build_confirm_info(meta, tool_params, policy)
            if self._observer:
                self._observer.record(
                    tool_name, "pending_confirm", tool_params
                )
            return SafetyCheckResult.need_confirm(confirm_info)
        
        # 3. 放行
        if self._observer:
            self._observer.record(
                tool_name, "safety_passed", f"安全等级: {meta.safety_level.value}"
            )
        return SafetyCheckResult.allow()
    
    def _check_param_safety(
        self, tool_name: str, param_name: str, param_value: str
    ) -> Optional[str]:
        """
        参数安全检查
        
        当前复用现有的command_security检查逻辑，
        远期应替换为更精细的参数级检查。
        """
        if param_name in ("command", "cmd", "code", "script"):
            from app.services.command_security import check_command_safety
            result = check_command_safety(param_value)
            if result.get("blocked", False):
                return result.get("risk", "未知风险")
        return None
    
    def _build_confirm_info(
        self, meta, tool_params: Dict, policy: Dict
    ) -> Dict:
        """构建确认信息（发送给前端展示）"""
        risk_descriptions = {
            ToolSafetyLevel.DESTRUCTIVE: "此操作不可撤销，可能丢失数据",
            ToolSafetyLevel.DANGEROUS: "此操作有安全风险，可能影响系统稳定性",
        }
        
        return {
            "tool": meta.name,
            "tool_description": meta.description,
            "safety_level": meta.safety_level.value,
            "risk_description": risk_descriptions.get(meta.safety_level, ""),
            "params": self._mask_sensitive_params(tool_params),
            "log_level": policy["log_level"],
        }
    
    def _mask_sensitive_params(self, params: Dict) -> Dict:
        """脱敏敏感参数（如密码、密钥）"""
        SENSITIVE_KEYS = {"password", "secret", "key", "token", "credential"}
        masked = {}
        for k, v in params.items():
            if k.lower() in SENSITIVE_KEYS:
                masked[k] = "***MASKED***"
            else:
                masked[k] = str(v)[:200]  # 截断过长参数
        return masked
```

### 3.4 HITL（人机协同确认）

#### 3.4.1 交互流程

```
ToolExecutor检测到需要确认的工具
  ↓
挂起当前ReAct循环
  ↓
通过SSE发送 AUTHORIZATION_REQUIRED 事件
  ↓

  - 工具名称 + 描述
  - 安全等级 + 风险说明

  ↓
用户选择：
  ├─ 【允许执行】 → 恢复ReAct循环，执行工具
  ├─ 【拒绝】 → 恢复ReAct循环，返回"用户拒绝执行"给LLM

```

```python
# backend/app/services/agent/authorization_manager.py

import asyncio
import time
from typing import Dict, Optional
from app.utils.logger import logger


class AuthorizationRequest:
    """单次授权请求"""
    
    def __init__(self, task_id: str, tool: str, params: Dict, timeout: int = 60):
        self.task_id = task_id
        self.tool = tool
        self.params = params
        self.timeout = timeout
        self.created_at = time.time()
        self._event = asyncio.Event()
        self._approved: Optional[bool] = None
        self._reason: str = ""
    
    async def wait(self) -> bool:
        """等待用户授权，超时返回False"""
        try:
            await asyncio.wait_for(self._event.wait(), timeout=self.timeout)
            return self._approved or False
        except asyncio.TimeoutError:
            logger.warning(f"[Authorization] 超时自动拒绝: {self.tool}")
            return False
    
    def respond(self, approved: bool, reason: str = ""):
        """响应用户授权"""
        self._approved = approved
        self._reason = reason
        self._event.set()


class AuthorizationManager:
    """
    授权管理器 — 管理所有待确认的授权请求
    
    职责：
    1. 创建授权请求
    2. 等待用户响应
    3. 处理超时自动拒绝
    """
    
    def __init__(self):
        self._pending: Dict[str, AuthorizationRequest] = {}
    
    async def request_authorization(
        self, task_id: str, tool: str, params: Dict, timeout: int = 60
    ) -> bool:
        """
        请求用户授权
        
        Returns:
            True = 允许执行, False = 拒绝/超时
        """
        req = AuthorizationRequest(task_id, tool, params, timeout)
        self._pending[task_id] = req
        
        logger.info(f"[Authorization] 请求授权: {tool} (task={task_id})")
        
        try:
            approved = await req.wait()
            logger.info(f"[Authorization] 授权结果: {tool} → {'允许' if approved else '拒绝'}")
            return approved
        finally:
            self._pending.pop(task_id, None)
    
    def respond(self, task_id: str, approved: bool, reason: str = ""):
        """处理前端回传的授权响应"""
        req = self._pending.get(task_id)
        if req:
            req.respond(approved, reason)
        else:
            logger.warning(f"[Authorization] 未知task_id: {task_id}")


# 全局单例
authorization_manager = AuthorizationManager()
```

#### 3.4.4 Session Trust（会话信任）机制

**问题**：用户在同一会话中多次确认相同操作，造成骚扰。

**方案**：提供"本会话对该命令信任"选项，一定时间内对相同模式的高危命令免审批。

```python
class SessionTrustManager:
    """
    会话信任管理器
    
    用户可选择"本次会话信任此操作"，在有效期内自动放行相同模式的工具调用。
    """
    
    def __init__(self):
        # task_id → {tool: expire_time}
        self._trust_map: Dict[str, Dict[str, float]] = {}
        self._default_ttl = 300  # 5分钟
    
    def grant_trust(self, task_id: str, tool: str, ttl: int = 300):
        """授予会话信任"""
        if task_id not in self._trust_map:
            self._trust_map[task_id] = {}
        self._trust_map[task_id][tool] = time.time() + ttl
    
    def is_trusted(self, task_id: str, tool: str) -> bool:
        """检查是否已信任"""
        trust = self._trust_map.get(task_id, {}).get(tool, 0)
        if time.time() > trust:
            return False
        return True
    
    def cleanup(self, task_id: str):
        """清理过期信任"""
        self._trust_map.pop(task_id, None)
```

### 3.5 ToolObserver（反应式观察者）

#### 3.5.1 核心职责

**不拦截，只记录和检测**：
1. 全量审计日志 — 记录所有工具调用
2. 异常模式检测 — 短时间大量同类操作
3. 极端模式自动暂停 — 触发安全阈值时暂停Agent

#### 3.5.2 实现代码

```python
# backend/app/services/agent/tool_observer.py

import time
from collections import defaultdict
from typing import Any, Dict, List, Optional
from dataclasses import dataclass, field
from app.utils.logger import logger


@dataclass
class AnomalyThreshold:
    """异常检测阈值"""
    max_per_minute: int       # 每分钟最大调用次数
    action: str               # 触发后的动作: "warn" / "pause" / "block"
    description: str = ""     # 说明


class ToolObserver:
    """
    反应式工具观察者
    
    职责：
    1. 实时记录所有工具调用（全量审计日志）
    2. 检测异常模式（如短时间内大量删除操作）
    3. 极端模式下自动触发暂停
    4. 提供查询接口供用户/管理者审查
    
    与ToolSafetyLayer的区别：
    - SafetyLayer: 执行前检查"这个调用是否安全"
    - Observer: 执行后记录"发生了什么"，检测"是否异常"
    """
    
    def __init__(self, task_id: str = ""):
        self.task_id = task_id
        self._records: List[Dict] = []
        self._pattern_window: Dict[str, List[float]] = defaultdict(list)
        self._paused = False
        
        # 异常检测阈值
        self._anomaly_thresholds: Dict[str, AnomalyThreshold] = {
            "delete_file": AnomalyThreshold(
                max_per_minute=10, action="pause",
                description="短时间内大量删除文件"
            ),
            "delete_directory": AnomalyThreshold(
                max_per_minute=5, action="pause",
                description="短时间内大量删除目录"
            ),
            "execute_shell_command": AnomalyThreshold(
                max_per_minute=5, action="pause",
                description="短时间内大量执行Shell命令"
            ),
            "execute_python_code": AnomalyThreshold(
                max_per_minute=3, action="pause",
                description="短时间内大量执行Python代码"
            ),
            "overwrite_file": AnomalyThreshold(
                max_per_minute=5, action="warn",
                description="短时间内大量覆盖文件"
            ),
        }
    
    def record(
        self,
        tool_name: str,
        status: str,
        detail: Any = "",
        result: str = "",
    ):
        """
        记录一次工具调用
        
        Args:
            tool_name: 工具名称
            status: 状态 (executed/blocked/pending_confirm/safety_passed)
            detail: 详细信息（参数）
            result: 执行结果（截断）
        """
        now = time.time()
        record = {
            "timestamp": now,
            "task_id": self.task_id,
            "tool": tool_name,
            "status": status,
            "detail": str(detail)[:200],
            "result": str(result)[:500] if result else "",
        }
        self._records.append(record)
        
        # 更新滑动窗口
        if tool_name in self._anomaly_thresholds:
            self._pattern_window[tool_name].append(now)
            self._check_anomaly(tool_name)
        
        # 日志
        policy_log_level = {
            "executed": "INFO",
            "blocked": "WARNING",
            "pending_confirm": "INFO",
            "safety_passed": "DEBUG",
        }
        log_method = getattr(logger, policy_log_level.get(status, "INFO").lower(), logger.info)
        log_method(f"[Observer] {status}: {tool_name} - {str(detail)[:60]}")
    
    def _check_anomaly(self, tool_name: str):
        """检测异常模式"""
        threshold = self._anomaly_thresholds.get(tool_name)
        if not threshold:
            return
        
        # 滑动窗口：最近1分钟内
        window_start = time.time() - 60
        recent = [
            t for t in self._pattern_window[tool_name]
            if t >= window_start
        ]
        
        if len(recent) > threshold.max_per_minute:
            msg = (
                f"[Observer] ⚠ 异常检测: {tool_name} "
                f"在1分钟内调用了{len(recent)}次 "
                f"(阈值={threshold.max_per_minute})"
            )
            
            if threshold.action == "pause":
                self._paused = True
                logger.warning(f"{msg} → 自动暂停")
            elif threshold.action == "block":
                logger.error(f"{msg} → 自动拦截")
            else:
                logger.warning(msg)
    
    @property
    def is_paused(self) -> bool:
        return self._paused
    
    def resume(self):
        """恢复执行"""
        self._paused = False
    
    def get_recent_calls(self, limit: int = 20) -> List[Dict]:
        """获取最近的工具调用记录"""
        return self._records[-limit:]
    
    def get_stats(self) -> Dict:
        """获取统计信息"""
        stats = defaultdict(int)
        for r in self._records:
            stats[f"{r['tool']}:{r['status']}"] += 1
        return dict(stats)
    
    def export_audit_log(self) -> List[Dict]:
        """导出完整审计日志"""
        return self._records.copy()
```

```python
# backend/app/services/agent/generic_react.py

from typing import Dict, Optional, List
from app.services.agent.base_react import BaseAgent
from app.services.agent.mixins.react_agent_mixin import ReactAgentMixin
from app.services.agent.profile import AgentProfile
from app.services.agent.tool_safety_layer import ToolSafetyLayer
from app.services.agent.tool_observer import ToolObserver
from app.services.agent.authorization_manager import authorization_manager
from app.services.tools.registry import ToolCategory
from app.utils.logger import logger


class GenericReactAgent(ReactAgentMixin, BaseAgent):
    """
    通用ReAct Agent — 通过AgentProfile配置化驱动
    
    替代7个同质Agent（Shell/Network/Desktop/System/Document/Database/CodeExecution），
    差异化完全由Profile描述。
    
    新增能力：
    - 集成ToolSafetyLayer（工具安全自检）
    - 集成ToolObserver（全量审计）
    - 集成HITL（人机协同确认）
    """
    
    def __init__(
        self,
        profile: AgentProfile,
        llm_client,
        task_id: str,
        tool_categories: Optional[List[ToolCategory]] = None,
        max_steps: Optional[int] = None,
        candidates: Optional[List[str]] = None,
        **kwargs,
    ):
        self.profile = profile
        self.task_id = task_id
        
        # 工具分类：优先使用Router推荐的，否则使用Profile的
        self._tool_categories = tool_categories or [profile.tool_category]
        effective_max_steps = max_steps or profile.max_steps
        
        # 初始化安全层和观察者
        self._safety_layer = ToolSafetyLayer()
        self._observer = ToolObserver(task_id=task_id)
        self._safety_layer.set_observer(self._observer)
        
        super().__init__(
            llm_client=llm_client,
            task_id=task_id,
            tool_category=self._tool_categories[0],  # 主分类（兼容基类）
            max_steps=effective_max_steps,
            **kwargs,
        )
        
        # task_id校验
        if profile.require_task_id and not task_id:
            raise ValueError(f"{profile.name} Agent requires a valid task_id")
        
        # 工具+策略+追踪初始化
        self._init_tools_and_executor(self._tool_categories)
        self._init_llm_strategies()
        self._init_task_tracking(enable=profile.enable_session)
        self._init_candidates(candidates or profile.candidates_default)
        
        # Prompt实例
        self.prompts = profile._prompt_instance
    
    def _init_tools_and_executor(self, categories: List[ToolCategory]):
        """
        加载指定分类的工具
        
        与现有逻辑的区别：
        - 现有：只加载单个分类
        - 新：可加载多个分类（Router推荐）
        """
        from app.services.tools.registry import get_all_tools_dict, get_tools_by_category
        
        tools_dict = {}
        for cat in categories:
            cat_tools = get_tools_by_category(cat)
            tools_dict.update(cat_tools)
            logger.info(f"[GenericReactAgent] 加载分类 {cat.value}: {len(cat_tools)} 个工具")
        
        self._tools_dict = tools_dict
        
        # 初始化executor（注入安全层和观察者）
        from app.services.agent.tool_executor import ToolExecutor
        self.executor = ToolExecutor(
            tools_dict=tools_dict,
            safety_layer=self._safety_layer,
            observer=self._observer,
        )
    
    async def _execute_tool(self, action: str, params: dict) -> str:
        """
        工具执行 — 集成安全检查 + HITL
        
        流程：
        1. 安全检查（ToolSafetyLayer）
        2. 如果需要确认 → HITL等待
        3. 执行工具
        4. 记录结果（ToolObserver）
        """
        # 1. 安全检查
        check_result = await self._safety_layer.check(action, params)
        
        if check_result.need_confirmation:
            # 2. HITL确认
            approved = await authorization_manager.request_authorization(
                task_id=self.task_id,
                tool=action,
                params=check_result.confirm_info.get("params", {}),
            )
            if not approved:
                return f"Error: 危险操作被用户拒绝执行。"
        
        if not check_result.passed:
            return f"Error: 安全拦截 — {check_result.message}"
        
        # 3. 参数标准化（根据Profile策略）
        if self.profile.param_normalize.value == "none":
            result = await self.executor.execute_raw(action, params)
        elif self.profile.param_normalize.value == "alias":
            resolved_action = self._resolve_tool_alias(action)
            normalized = self.executor._normalize_params(resolved_action, params)
            result = await self.executor.execute_raw(resolved_action, normalized)
        else:  # standard
            normalized = self.executor._normalize_params(action, params)
            result = await self.executor.execute_raw(action, normalized)
        
        # 4. 记录执行结果
        self._observer.record(action, "executed", params, str(result)[:200])
        
        return result
    
    def _resolve_tool_alias(self, action: str) -> str:
        """工具别名解析（仅File类Agent需要）"""
        if self.profile.alias_resolver:
            return self.profile.alias_resolver(action)
        return action
    
    async def rollback(self, step_number=None) -> bool:
        """回滚 — 根据Profile的rollback_strategy"""
        if self.profile.rollback_strategy.value == "noop":
            return True
        elif self.profile.rollback_strategy.value == "session":
            return await self._session_rollback(step_number)
        return False
    
    def _get_system_prompt(self) -> str:
        return self._build_system_prompt(self.profile.name)
    
    async def _get_llm_response(self) -> str:
        return await self._call_llm_with_summary()
```

```python
# backend/app/services/agent/tool_executor.py — 改造要点

class ToolExecutor:
    """
    工具执行器 — 集成安全层和观察者
    
    改造：
    1. 新增 safety_layer 和 observer 参数
    2. execute() 方法改为 execute_raw()（安全检查在Agent层做）
    3. 保留原有 execute() 作为兼容接口
    """
    
    def __init__(
        self,
        tools_dict: Dict[str, callable],
        safety_layer: ToolSafetyLayer = None,
        observer: ToolObserver = None,
    ):

        self._safety_layer = safety_layer
        self._observer = observer
    
    async def execute_raw(self, tool_name: str, params: dict) -> str:
        """
        执行工具（不做安全检查，安全检查在Agent层已完成）

        tool = self._tools_dict.get(tool_name)
        if not tool:
            return f"Error: 未知工具 '{tool_name}'"
        
        try:
            result = await tool(**params)
            return result
        except Exception as e:
            error_msg = f"Error: 工具执行失败 — {str(e)}"
            if self._observer:
                self._observer.record(tool_name, "error", params, error_msg)
            return error_msg
    
    # 保留原有接口兼容
    async def execute(self, tool_name: str, params: dict) -> str:
        """兼容旧接口 — 直接执行（无安全检查）"""
        return await self.execute_raw(tool_name, params)
```

```python
# backend/app/services/chat_router.py — 重构后

import uuid
from app.services.agent.semantic_router import semantic_router
from app.services.agent.generic_react import GenericReactAgent
from app.services.agent.agent_registry import agent_registry
from app.utils.logger import logger


class ChatRouter:
    """
    聊天路由器 — 重构版
    
    架构：
    1. strip() 清洗
    2. SemanticRouter 推荐工具子集
    3. 创建 GenericReactAgent
    4. agent.run_stream() → SSE
    """
    
    async def route(self, user_input, provider, model, session_id, **kwargs):
        # ===== 1. 清洗输入 =====
        user_input = user_input.strip()
        if not user_input:
            yield {"type": "error", "data": {"message": "输入不能为空"}}
            return
        
        # ===== 2. 语义路由 — 推荐工具子集 =====
        recommended_categories = await semantic_router.recommend_categories(user_input)
        logger.info(f"[ChatRouter] 推荐工具分类: {[c.value for c in recommended_categories]}")
        
        # ===== 3. 创建Agent =====
        task_id = str(uuid.uuid4())
        
        # 从注册表获取Profile（用于Prompt和策略配置）
        profile = agent_registry.get_profile_for_categories(recommended_categories)
        if profile is None:
            profile = agent_registry.get_default_profile()
        
        agent = GenericReactAgent(
            profile=profile,
            llm_client=self._get_llm_client(provider, model),
            task_id=task_id,
            tool_categories=recommended_categories,
            max_steps=profile.max_steps,
        )
        
        # ===== 4. 执行并流式输出 =====
        async for event in agent.run_stream(user_input):
            yield event
    
    def _get_llm_client(self, provider, model):
        # 现有逻辑不变
```

## 九、风险分析与缓解

## 十一、总结

### 11.1 方案C核心优势

| 维度 | 方案A | 方案B | 方案C |
|------|-------|-------|-------|
| 工具加载量 | 子集✅ | 全量❌ | **子集✅** |
| 安全机制 | HITL全量❌ | 工具自检✅ | **分级HITL✅** |
| 延迟 | 路由+HITL❌ | 无路由✅ | **仅路由✅** |
| 维护成本 | 意图定义❌ | 安全声明✅ | **安全声明✅** |
| 回滚能力 | HITL✅ | 无❌ | **分级HITL✅** |
| 审计能力 | 无❌ | 全量✅ | **全量✅** |
| 架构复杂度 | 中 | 低 | **中** |

### 11.2 关键设计决策

1. **语义路由替代意图分类** — 不分类意图，只推荐工具子集，更灵活、更准确
2. **分级安全替代全量HITL** — SAFE/READ_ONLY免确认，仅DANGEROUS需要HITL，用户体验提升90%+
3. **工具声明式安全** — 安全责任附在工具上，Prompt注入无效
4. **反应式Observer** — 不拦截，只记录和检测，性能影响最小
5. **渐进式迁移** — Feature Flag灰度，随时可回退
