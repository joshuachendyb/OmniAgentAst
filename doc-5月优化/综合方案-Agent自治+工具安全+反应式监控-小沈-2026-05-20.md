## 一、从哪里拿什么

| 方向 | 拿来什么 | 为什么好 |
|------|---------|---------|
| **Agent自治** 🧠 | 统一Agent、不意图路由、LLM自己理解文本和选工具 | 零规则维护，LLM语义理解天生强大 |
| **工具安全** 🛡️ | 安全下放到每个工具自己声明，不搞集中输入检测 | 安全责任到"执行者"，更精确，无漏检 |
| **全LLM语义** 📝 | 语义理解替代CRSS规则和黑名单 | 灵活，无死角，错别字/变体不影响 |
| **反应式监控** 👁️ | 被动监控记录工具调用+检测异常模式，不前置拦截 | 审计能力强，性能影响小，可回溯 |

**不要什么**：
- ❌ 不要前置管线（不搞预处理→意图→安全的顺序步骤）
- ❌ 不要CRSS规则匹配
- ❌ 不要在用户输入层做安全检查
- ❌ 不要意图分发到不同Agent

## 三、具体设计

### 3.1 Agent层：统一Agent（来自 Agent自治 + 全LLM语义）

**不做什么**：
- 不跑预处理（不矫正错别字——LLM自己能理解）
- 不跑CRSS规则（不维护关键词——LLM语义理解）
- 不跑安全检测（不维护黑名单——安全下放到工具层）
- 不分发意图（不区分file/shell/network Agent——一个Agent全部处理）

**做什么**：
- 加载全部工具（统一Agent，不分路由）
- 用一份system prompt指导行为

### 3.2 工具安全自检层（来自 工具自治安全）

**核心思想**：安全责任附在工具上，不在用户输入层。每个工具注册时声明自己的安全特性，执行前自检。

from enum import Enum

class ToolSafetyLevel(Enum):
    READ_ONLY = "read_only"       # 只读，无副作用（读文件、查时间等）
    SAFE = "safe"                # 安全，有副作用但无害（创建临时文件等）

    DANGEROUS = "dangerous"      # 危险（shell命令、修改系统配置等）

@dataclass
class ToolMetadata:
    """工具元数据（扩展安全信息）"""
    name: str
    description: str
    category: ToolCategory
    # === 新增安全字段 ===
    safety_level: ToolSafetyLevel = ToolSafetyLevel.SAFE
    needs_confirmation: bool = False       # 执行前需要用户确认
    needs_safety_check: bool = False       # 执行前需要参数安全检查
    safety_check_param: str = ""           # 需要检查的参数名（如 "command"）

#### 安全自检执行器

```python
# backend/app/services/agent/tool_safety_layer.py

class ToolSafetyLayer:
    """
    工具安全自检层
    
    在工具执行前检查：
    1. 工具的安全等级
    2. 是否需要用户确认
    3. 是否需要参数安全检查（复用check_command_safety）
    
    这是去中心化的安全——每个工具自己声明安全属性，
    不需要集中式的用户输入安全检测。
    """
    
    def __init__(self, observer: "ToolObserver"):
        self._observer = observer
    
    async def check(
        self,
        tool_name: str,
        tool_params: Dict[str, Any],
        tool_meta: ToolMetadata,
    ) -> Tuple[bool, str, Optional[Dict]]:
        """
        执行前安全检查
        
        Returns:
            (pass, message, confirm_info)
            pass=True → 放行
            pass=False, message="拒绝原因" → 拦截
            confirm_info={} → 需要用户确认
        """
        # 1. 参数安全检查（如Shell命令）
        if tool_meta.needs_safety_check and tool_meta.safety_check_param:
            param_value = tool_params.get(tool_meta.safety_check_param, "")
            if param_value:
                from app.services.command_security import check_command_safety
                result = check_command_safety(param_value)
                if result.get("blocked", False):
                    self._observer.record(
                        tool_name, "blocked", 
                        f"参数安全检查拦截: {result.get('risk')}"
                    )
                    return False, f"安全拦截: {result.get('risk')}", None
        
        # 2. 需要用户确认
        if tool_meta.needs_confirmation:
            risk_info = self._build_risk_info(tool_meta, tool_params)
            return False, "需要确认", risk_info
        
        return True, "", None
    
    def _build_risk_info(self, meta: ToolMetadata, params: dict) -> Dict:
        """构建确认信息"""
        risk_descriptions = {
            ToolSafetyLevel.DESTRUCTIVE: "此操作不可撤销",
            ToolSafetyLevel.DANGEROUS: "此操作有安全风险",
        }
        return {
            "tool": meta.name,
            "safety_level": meta.safety_level.value,
            "description": risk_descriptions.get(meta.safety_level, ""),
            "params": params,
        }
```

#### 执行流程

```python
# ToolExecutor.execute() 改造

class ToolExecutor:
    def __init__(self, tools_dict):
        self._tools = tools_dict
        self._safety = ToolSafetyLayer(self._observer)
        self._observer = ToolObserver()
    
    async def execute(self, tool_name: str, params: dict) -> str:
        meta = registry.get_tool_metadata(tool_name)
        
        # 1. 安全自检
        passed, message, confirm_info = await self._safety.check(
            tool_name, params, meta
        )
        
        if not passed and message == "需要确认":
            # 需要确认 → 记录观察者，走确认流程
            self._observer.record(tool_name, "pending_confirm", params)
            return json.dumps({
                "status": "need_confirm",
                "tool": tool_name,
                "params": params,
                "risk_info": confirm_info,
            })
        
        if not passed:
            # 拦截
            self._observer.record(tool_name, "blocked", message)
            return json.dumps({
                "status": "blocked",
                "message": message,
            })
        
        # 2. 正常执行
        result = await self._tools[tool_name](**params)
        self._observer.record(tool_name, "executed", params)
        return result
```

### 3.3 反应式观察者（来自 反应式监控）

**核心思想**：不做前置拦截（那是工具自检的事），做被动监控+异常检测+审计回溯。

```python
# backend/app/services/agent/tool_observer.py

import time
from collections import defaultdict
from typing import Any, Dict, List, Optional
from app.utils.logger import logger


class ToolObserver:
    """
    反应式工具观察者
    
    职责（不拦截，只记录和检测）：
    1. 实时记录所有工具调用（全量审计日志）
    2. 检测异常模式（如短时间内大量删除操作）
    3. 极端模式下自动触发暂停
    4. 提供查询接口供用户/管理者审查
    
    与安全自检层的区别：
    - 安全自检：执行前检查"这个调用是否安全"
    - 观察者：执行后记录"发生了什么"，检测"是否异常"
    """
    
    def __init__(self):
        self._records: List[Dict] = []
        self._pattern_window: Dict[str, List[float]] = defaultdict(list)
        self._paused = False
        
        # 异常检测阈值
        self._anomaly_thresholds = {
            "delete_file": {"max_per_minute": 10, "action": "pause"},
            "execute_shell_command": {"max_per_minute": 5, "action": "pause"},
            "format_disk": {"max_per_minute": 1, "action": "block"},
        }
    
    def record(self, tool_name: str, status: str, detail: Any):
        """记录一次工具调用"""
        now = time.time()
        record = {
            "timestamp": now,
            "tool": tool_name,
            "status": status,  # "executed" / "blocked" / "pending_confirm"
            "detail": str(detail)[:200],
        }
        self._records.append(record)
        
        # 更新窗口计数
        if tool_name in self._anomaly_thresholds:
            self._pattern_window[tool_name].append(now)
            self._check_anomaly(tool_name)
        
        # 日志
        logger.info(f"[Observer] {status}: {tool_name} - {str(detail)[:60]}")
    
    def _check_anomaly(self, tool_name: str):
        """检测异常模式"""
        threshold = self._anomaly_thresholds.get(tool_name)
        if not threshold:
            return
        
        # 滑动窗口：最近1分钟内
        window_start = time.time() - 60
        recent = [
            t for t in self._pattern_window[tool_name] 
            if t >= window_start
        ]
        
        if len(recent) > threshold["max_per_minute"]:
            logger.warning(
                f"[Observer] ⚠ 异常检测: {tool_name} "
                f"在1分钟内调用了{len(recent)}次 "
                f"(阈值{threshold['max_per_minute']})"
            )
            
            if threshold["action"] == "pause":
                self._paused = True
                logger.warning(
                    f"[Observer] 🚫 自动暂停: {tool_name} 调用过于频繁"
                )
    
    def get_recent_calls(self, limit: int = 20) -> List[Dict]:
        """获取最近的工具调用记录"""
        return self._records[-limit:]
    
    def get_stats(self) -> Dict:
        """获取统计信息"""
        stats = defaultdict(int)
        for r in self._records:
            stats[f"{r['tool']}:{r['status']}"] += 1
        return dict(stats)
    
    @property
    def is_paused(self) -> bool:
        return self._paused
    
    def resume(self):
        self._paused = False
```

## 四、三种场景演示

### 场景1：正常请求"帮我看看D盘有什么文件"

```
用户: "帮我看看D盘有什么文件"
  ↓ strip → Agent接收
  ↓ LLM: Thought→"用户想看D盘文件，用list_directory"
  ↓ Action: list_directory(path="D:/")
  ↓   → ToolSafetyLayer: READ_ONLY → 放行 ✓
  ↓   → ToolExecutor执行 → 返回结果
  ↓   → Observer记录: list_directory → executed
  ↓ 输出结果
```

**无管线、无意图路由、无安全检测。纯LLM决策+工具自检。**

### 场景2：危险请求"帮我删除所有文件"

```
用户: "帮我删除所有文件"
  ↓ strip → Agent接收
  ↓ LLM: Thought→"用户想删除文件，但这是危险操作，我应该谨慎"
  ↓ LLM: 先问用户确认 (Agent内部对话)
  用户: "确认"
  ↓ LLM: Action → delete_file(path="*")
  ↓   → ToolSafetyLayer: DESTRUCTIVE + needs_confirmation
  ↓     → 返回 need_confirm
  ↓   → LLM看到确认信息，再次问用户
  ↓   → Observer记录: delete_file → pending_confirm
```

**LLM自评 + 工具自检双重保护。即使LLM安全判断出问题，工具层也会要求确认。**

### 场景3：错别字"帮我删处所有文件"

```
用户: "帮我删处所有文件"
  ↓ strip → Agent接收
  ↓ LLM: Thought→"用户说'删处'应该是'删除'的意思，他要删除所有文件..."
  ↓ (LLM自然理解错别字，不需要矫正器)
  ↓ 后续同场景2
```

**LLM自然理解错别字，不需要字典映射。这就是语义理解比规则强的根本原因。**

## 五、和之前方案的根本区别

| 维度 | 管线方案（不要了） | 纯Agent（上次方案） | **综合方案（本次）** |
|------|----------------|-------------------|-------------------|
| **架构** | 预处理→意图→安全→Agent | Agent独自搞定 | Agent + 工具自检 + 观察者 |
| **安全方式** | 集中式前置检测 | LLM自评（或无安全） | **工具声明式自检 + 被动监控** |
| **安全维护** | 维护黑名单/规则 | 不维护 | **工具注册时声明安全等级** |
| **安全盲区** | 规则遗漏/管线断裂 | LLM被注入 | **工具层免疫注入（检查参数本身）** |
| **审计** | 弱 | 无 | **全量审计日志 + 异常检测** |
| **代码量** | +500行 | -850行 | **-600行**（删管线，加自检+观察者） |
| **规则维护** | 需要 | 不需要 | **不需要**（安全等级是一次性声明） |

**这个方案解决了纯Agent方案最弱的点——安全**：
- 纯Agent高度依赖LLM的安全对齐（可能被prompt注入绕过）
- 综合方案中每个工具的安全等级是代码声明的（prompt注入影响不了）
- 即使LLM被注入说"执行rm -rf /"，工具层看到DANGEROUS等级+command安全检查，照样拦截

## 六、实施步骤
