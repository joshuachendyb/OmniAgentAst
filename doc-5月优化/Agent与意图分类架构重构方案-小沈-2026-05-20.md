## 一、现状分析

### 1.1 当前架构总览

```
用户输入
  └─ ChatRouter.route()
       ├─ 预处理(PreprocessingPipeline, 仅strip)
       ├─ 意图检测(route_with_fallback, CRSS+LLM两阶段)
       ├─ 安全检测(check_command_safety)
       └─ Agent分发(AgentFactory.create → XxxReactAgent.run_stream)
```

当前Agent体系包含9个Agent子类，全部采用 `ReactAgentMixin + BaseAgent` 多继承模式：

```
BaseAgent(ABC)                 ← ReAct循环核心(1100行)
ReactAgentMixin(ToolLoaderMixin) ← 公用逻辑混入(487行)
        ↓ MRO
XxxReactAgent × 9              ← 各意图Agent(30~386行)
```

### 1.2 九大Agent子类差异分析

| Agent | 代码行数 | 差异项 | 实质差异 |
|-------|---------|--------|---------|
| FileReactAgent | 386 | rollback+session+alias+Hook | **有实质差异** |
| TimeReactAgent | 60 | rollback=True+无normalize+无task_id校验 | **有实质差异** |
| ShellReactAgent | 45 | 仅Prompt+Category | 同质 |
| NetworkReactAgent | 45 | 仅Prompt+Category | 同质 |
| DesktopReactAgent | 45 | 仅Prompt+Category | 同质 |
| SystemReactAgent | 45 | 仅Prompt+Category | 同质 |
| DocumentReactAgent | 45 | 仅Prompt+Category | 同质 |
| DatabaseReactAgent | 50 | 仅Prompt+Category(=DOCUMENT) | 同质 |
| CodeExecutionReactAgent | 50 | 仅Prompt+Category(=SHELL) | 同质 |

**关键发现：7个Agent代码结构完全同质，差异仅来自不同的Prompts类和ToolCategory。**

### 1.3 意图分类现状

| 组件 | 状态 | 问题 |
|------|------|------|
| CRSS评分器 | 活跃使用 | 关键词硬编码、缺META/SUPPORT_TOOL条目、否定检测简单 |

## 二、问题清单（按严重程度排序）

### 2.1 [P0-严重] Agent同质化导致代码膨胀和维护困难

**问题**：9个Agent子类中7个代码结构完全相同，每个都重复`__init__/_get_system_prompt/_get_llm_response/_execute_tool/rollback`五方法。修改任何一个公用逻辑需确认9个文件。

**影响**：新增分类需复制整个Agent文件、修改逻辑需同步9处、代码审查困难。

### 2.2 [P0-严重] AgentFactory注册顺序导致覆盖风险

**问题**：`database`和`document`都注册到`DOCUMENT`分类，但映射不同Agent类（DatabaseReactAgent vs DocumentReactAgent）。注册顺序导致`document`键被覆盖。

**影响**：`intent_type="document"` 和 `intent_type="database"` 走不同Agent，但ToolCategory相同（DOCUMENT），行为可能不一致。

### 2.3 [P1-中等] 意图分类与Agent层紧耦合

**问题**：意图标签直接决定Agent类型（intent_type → AgentFactory.create），意图分类的任何变化都需要同步修改AgentFactory注册表、CRSS关键词、LLM定义、Prompt类等多处。

**影响**：扩展新意图分类需要改动6+处代码，容易遗漏。

### 2.4 [P1-中等] CRSS关键词与definitions目录不同步

**问题**：CRSS的TYPE_KEYWORDS硬编码在crss_scorer.py中，与definitions/目录（已废弃）完全脱节。两套关键词定义无法保持一致。

**影响**：关键词维护困难，新增/修改关键词需改代码重启。

### 2.5 [P1-中等] PreprocessingPipeline已成空壳

**问题**：`PreprocessingPipeline.process()` 仅做 `user_input.strip()`，意图检测逻辑已全部移到 `route_with_fallback()`，但ChatRouter仍调用它。

**影响**：多一次无意义的async调用，代码误导（以为有预处理逻辑）。

### 2.6 [P1-中等] IntentClassifier类成死代码

**问题**：`IntentClassifier` 类（preprocessing/intent_classifier.py）和 `classify_intent` 函数共存，route_with_fallback直接调用后者绕过了类。

**影响**：代码冗余，新开发者可能误用类接口。

### 2.7 [P2-低] candidates机制不完整

**问题**：candidates存储在`self._candidates`中，但不自动触发跨分类工具加载。仅通过Prompt提示LLM"可以访问候选分类工具"，依赖LLM主动发起跨分类调用。

**影响**：跨分类工具访问依赖LLM"自觉"，不可靠。

### 2.8 [P2-低] 未匹配意图硬编码fallback到network

**问题**：`intent_type = intent_type_value.value if intent_type_value else "network"`，纯对话类输入（"你好"/"讲个笑话"）走network Agent会尝试搜索网络。

**影响**：用户体验不合理，对简单对话发起网络搜索。

### 2.9 [P2-低] CRSS否定检测过于简单

**问题**：`_is_negated()` 只检查关键词前2个字符是否含否定词，无法处理"我没有想要删除文件"。

**影响**：否定意图误判，可能触发相反操作。

### 2.10 [P2-低] gui_register.py残留大量死代码

**问题**：gui_register.py保留约400行旧GUI工具描述，但注册函数已改为空操作。

**影响**：代码膨胀，维护干扰。

## 四、重构方案

### 4.1 核心设计：Agent配置化 + 意图注册表

#### 4.1.1 设计理念

当前9个Agent的差异化仅来自三个维度：
1. **ToolCategory** — 决定加载哪些工具
2. **Prompt类** — 决定system prompt内容
3. **行为特性** — rollback策略/参数标准化/工具别名/session管理

其中只有 **FileReactAgent** 和 **TimeReactAgent** 有实质行为差异，其余7个完全同质。

**核心思路**：将"同质Agent"合并为1个通用Agent（`GenericReactAgent`），通过**AgentProfile配置对象**描述差异；保留2个有实质差异的Agent为独立子类。

#### 4.1.2 AgentProfile 配置对象

```python
# backend/app/services/agent/profile.py

from dataclasses import dataclass, field
from typing import Optional, Dict, Callable, List
from enum import Enum

from app.services.tools.registry import ToolCategory


class RollbackStrategy(Enum):
    NONE = "none"           # 不支持回滚，return False
    NOOP = "noop"           # 无需回滚，return True（如时间操作）
    SESSION = "session"     # 基于session的文件回滚


class ParamNormalize(Enum):
    STANDARD = "standard"   # 标准参数标准化（_normalize_params）
    ALIAS = "alias"         # 别名映射+标准化（FileReactAgent）
    NONE = "none"           # 不做标准化（TimeReactAgent）


@dataclass
class AgentProfile:
    """
    Agent配置档案 — 描述一个Agent的全部差异化属性
    
    设计原则：
    - 同质Agent只需创建Profile实例，无需新建Agent子类
    - 有实质差异的Agent（File/Time）可覆盖Profile中的行为策略
    """
    name: str                                # Agent显示名（如"文件操作"）
    intent_type: str                         # 意图标签（如"file"）
    tool_category: ToolCategory              # 工具分类
    prompt_class_name: str                   # Prompt类全限定名
    rollback_strategy: RollbackStrategy = RollbackStrategy.NONE
    param_normalize: ParamNormalize = ParamNormalize.STANDARD
    max_steps: int = 100                     # 最大步数
    require_task_id: bool = True             # 是否要求task_id
    enable_session: bool = False             # 是否启用session生命周期管理
    alias_resolver: Optional[Callable] = None  # 工具别名解析器
    candidates_default: List[str] = field(default_factory=list)  # 默认候选意图
    
    # 运行时引用（由AgentRegistry填充，不序列化）
    _prompt_instance: Optional[object] = field(default=None, repr=False)
    _agent_class: Optional[type] = field(default=None, repr=False)
```

#### 4.1.3 AgentRegistry 意图注册表（替代AgentFactory）

```python
# backend/app/services/agent/agent_registry.py

from typing import Dict, Optional, Type
from app.services.agent.profile import AgentProfile, RollbackStrategy, ParamNormalize
from app.services.tools.registry import ToolCategory


class AgentRegistry:
    """
    Agent意图注册表 — 统一管理意图→AgentProfile映射
    
    替代AgentFactory，解决：
    1. 注册覆盖风险（字典key冲突检测）
    2. 意图与Agent紧耦合（Profile解耦）
    3. 新增分类需改多处代码（注册表单一数据源）
    """
    
    def __init__(self):
        self._profiles: Dict[str, AgentProfile] = {}  # intent_type → Profile
        self._category_to_intents: Dict[ToolCategory, List[str]] = {}  # ToolCategory → [intent_types]
    
    def register(self, profile: AgentProfile, agent_class: Optional[Type] = None) -> None:
        """
        注册一个意图的AgentProfile
        
        Args:
            profile: Agent配置档案
            agent_class: 自定义Agent类（仅File/Time等有实质差异的Agent需要）
                         None则使用GenericReactAgent
        """
        intent = profile.intent_type
        
        # 冲突检测：同一intent_type不允许重复注册
        if intent in self._profiles:
            existing = self._profiles[intent]
            raise ValueError(
                f"意图 '{intent}' 已注册为 {existing.name}，"
                f"不允许覆盖。如需修改请先unregister。"
            )
        
        profile._agent_class = agent_class
        self._profiles[intent] = profile
        
        # 构建反向索引：ToolCategory → intent_types
        cat = profile.tool_category
        if cat not in self._category_to_intents:
            self._category_to_intents[cat] = []
        self._category_to_intents[cat].append(intent)
    
    def get_profile(self, intent_type: str) -> Optional[AgentProfile]:
        """获取意图的AgentProfile，支持新旧意图名兼容"""
        if intent_type in self._profiles:
            return self._profiles[intent_type]
        # 兼容旧意图名
        compat = self._resolve_compat_intent(intent_type)
        if compat and compat in self._profiles:
            return self._profiles[compat]
        return None
    
    def create_agent(self, intent_type: str, **kwargs):
        """根据intent_type创建Agent实例"""
        profile = self.get_profile(intent_type)
        if profile is None:
            raise ValueError(f"未注册的意图类型: {intent_type}")
        
        agent_class = profile._agent_class or GenericReactAgent
        return agent_class(profile=profile, **kwargs)
    
    def _resolve_compat_intent(self, intent_type: str) -> Optional[str]:
        """旧意图名→新意图名兼容映射"""
        COMPAT_MAP = {
            "time": "meta",
            "environment": "system", "env": "system",
            "database": "document",
            "code_execution": "shell",
            "data_format": "file",
            "data_analysis": "document",
            "support_tool": "meta",
        }
        return COMPAT_MAP.get(intent_type)
    
    def all_intents(self) -> List[str]:
        """返回所有已注册的意图标签"""
        return list(self._profiles.keys())


# 全局单例
agent_registry = AgentRegistry()
```

#### 4.1.4 GenericReactAgent 通用Agent（替代7个同质Agent）

```python
# backend/app/services/agent/generic_react.py

from app.services.agent.base_react import BaseAgent
from app.services.agent.mixins.react_agent_mixin import ReactAgentMixin
from app.services.agent.profile import AgentProfile, RollbackStrategy, ParamNormalize


class GenericReactAgent(ReactAgentMixin, BaseAgent):
    """
    通用ReAct Agent — 通过AgentProfile配置化驱动
    
    替代7个同质Agent（Shell/Network/Desktop/System/Document/Database/CodeExecution），
    差异化完全由Profile描述。
    """
    
    def __init__(self, profile: AgentProfile, llm_client, task_id, 
                 max_steps=None, candidates=None, **kwargs):
        self.profile = profile
        
        # 从Profile推导初始化参数
        effective_category = profile.tool_category
        effective_max_steps = max_steps or profile.max_steps
        
        super().__init__(
            llm_client=llm_client,
            task_id=task_id,
            tool_category=effective_category,
            max_steps=effective_max_steps,
            **kwargs
        )
        
        # task_id校验
        if profile.require_task_id and not task_id:
            raise ValueError(f"{profile.name} Agent requires a valid task_id")
        
        # 工具+策略+追踪初始化
        self._init_tools_and_executor(effective_category)
        self._init_llm_strategies()
        self._init_task_tracking(enable=profile.enable_session)
        self._init_candidates(candidates or profile.candidates_default)
        
        # Prompt实例
        self.prompts = profile._prompt_instance
    
    def _get_system_prompt(self) -> str:
        return self._build_system_prompt(self.profile.name)
    
    async def _get_llm_response(self) -> str:
        return await self._call_llm_with_summary()
    
    async def _execute_tool(self, action: str, params: dict) -> str:
        """工具执行 — 根据Profile的param_normalize策略选择执行方式"""
        if self.profile.param_normalize == ParamNormalize.NONE:
            return await self.executor.execute(action, params)
        elif self.profile.param_normalize == ParamNormalize.ALIAS:
            resolved_action = self._resolve_tool_alias(action)
            return await self.executor.execute(
                resolved_action, 
                self.executor._normalize_params(resolved_action, params)
            )
        else:  # STANDARD
            return await self.executor.execute(
                action, 
                self.executor._normalize_params(action, params)
            )
    
    async def rollback(self, step_number=None) -> bool:
        """回滚 — 根据Profile的rollback_strategy"""
        if self.profile.rollback_strategy == RollbackStrategy.NOOP:
            return True
        elif self.profile.rollback_strategy == RollbackStrategy.SESSION:
            return await self._session_rollback(step_number)
        return False
    
    def _resolve_tool_alias(self, action: str) -> str:
        """工具别名解析（仅File类Agent需要）"""
        if self.profile.alias_resolver:
            return self.profile.alias_resolver(action)
        return action
```

### 4.2 意图定义中心化

#### 4.2.1 设计理念

当前意图定义分散在4处：
1. `crss_scorer.py` — CRSS关键词
2. `intent_classifier.py` — LLM意图定义
3. `agent_factory.py` — Agent注册
4. `registry.py` — INTENT_TO_CATEGORY映射

**核心思路**：建立统一的 `IntentDefinition` 数据模型，所有组件从此单一数据源读取。

#### 4.2.2 IntentDefinition 数据模型

```python
# backend/app/services/intents/definition.py

from dataclasses import dataclass, field
from typing import Dict, List, Optional
from app.services.tools.registry import ToolCategory


@dataclass
class TypeKeywords:
    """类型层关键词定义"""
    zh: List[str] = field(default_factory=list)  # 中文关键词
    en: List[str] = field(default_factory=list)  # 英文关键词（正则模式）


@dataclass
class ActionDef:
    """动作层定义"""
    zh: List[str]                      # 中文关键词
    en: List[str]                      # 英文关键词
    compatibility: Dict[str, float]    # ToolCategory → 兼容系数


@dataclass
class IntentDefinition:
    """
    意图定义 — 单一数据源
    
    所有意图相关组件（CRSS/LLM/Agent/Prompt）统一从此定义读取配置。
    """
    intent_type: str                        # 意图标签（如"file"）
    category: ToolCategory                  # 对应的ToolCategory
    display_name: str                       # 显示名（中文，如"文件操作"）
    description: str                        # 意图描述（给LLM看）
    
    # CRSS配置
    type_keywords: Optional[TypeKeywords] = None   # 类型层关键词
    actions: Dict[str, ActionDef] = field(default_factory=dict)  # 动作定义
    
    # Agent配置
    agent_class_name: Optional[str] = None        # 自定义Agent类名（None=GenericReactAgent）
    prompt_class_name: Optional[str] = None       # Prompt类名
    rollback_strategy: str = "none"                # none/noop/session
    param_normalize: str = "standard"              # standard/alias/none
    enable_session: bool = False                   # 启用session管理
    require_task_id: bool = True                   # 要求task_id
    max_steps: int = 100                           # 最大步数
    
    # 兼容旧意图名
    compat_names: List[str] = field(default_factory=list)  # 如["time", "timer"]
    
    # 是否已废弃（仅保留兼容映射，不独立注册Agent）
    deprecated: bool = False
```

#### 4.2.3 意图定义注册表

```python
# backend/app/services/intents/registry.py

from typing import Dict, List, Optional
from app.services.intents.definition import IntentDefinition
from app.services.tools.registry import ToolCategory


class IntentRegistry:
    """
    意图定义注册表 — 单一数据源
    
    所有组件通过此注册表获取意图定义，消除4处分散定义。
    """
    
    def __init__(self):
        self._definitions: Dict[str, IntentDefinition] = {}
        self._compat_map: Dict[str, str] = {}  # 旧名 → 新名
    
    def register(self, definition: IntentDefinition) -> None:
        self._definitions[definition.intent_type] = definition
        for compat in definition.compat_names:
            self._compat_map[compat] = definition.intent_type
    
    def get(self, intent_type: str) -> Optional[IntentDefinition]:
        if intent_type in self._definitions:
            return self._definitions[intent_type]
        new_name = self._compat_map.get(intent_type)
        if new_name:
            return self._definitions.get(new_name)
        return None
    
    def active_intents(self) -> List[IntentDefinition]:
        """返回所有非废弃意图"""
        return [d for d in self._definitions.values() if not d.deprecated]
    
    def intent_labels(self) -> List[str]:
        """返回LLM分类器的候选标签"""
        return [d.intent_type for d in self.active_intents()]
    
    def crss_type_keywords(self) -> Dict[str, TypeKeywords]:
        """返回CRSS评分器需要的类型关键词"""
        result = {}
        for d in self.active_intents():
            if d.type_keywords:
                result[d.intent_type.upper()] = d.type_keywords
        return result
    
    def crss_actions(self) -> Dict[str, ActionDef]:
        """返回CRSS评分器需要的动作定义"""
        result = {}
        for d in self.active_intents():
            result.update(d.actions)
        return result


# 全局单例
intent_registry = IntentRegistry()
```

### 4.4 Candidates机制增强

#### 4.4.1 自动跨分类工具预加载

当前candidates仅在Prompt中提示LLM"可以访问候选分类工具"，但不自动加载。增强为：

```python
# backend/app/services/agent/mixins/react_agent_mixin.py 增强要点

def _init_candidates(self, candidates: Optional[List[str]] = None):
    """初始化候选意图 + 自动预加载候选分类工具"""
    self._candidates = candidates if candidates else []
    
    # 增强：自动预加载候选分类的工具
    if self._candidates:
        for candidate_intent in self._candidates:
            if candidate_intent not in self._loaded_categories:
                try:
                    self.load_tools_by_intent(candidate_intent)
                    logger.info(f"预加载候选分类工具: {candidate_intent}")
                except Exception as e:
                    logger.warning(f"预加载候选分类工具失败: {candidate_intent}, {e}")
```

**注意**：预加载会增加初始工具数量，可能影响LLM上下文长度。通过`_get_tools_detail`和`_get_tools_summary`的分级注入已有控制，但需增加工具数量上限检查。

#### 4.4.2 置信度驱动的选择性预加载

更精细的策略：只预加载置信度超过阈值的候选分类。

```python
def _init_candidates_with_confidence(self, candidates_with_confidence: List[Dict[str, float]]):
    """
    基于置信度的选择性预加载
    
    Args:
        candidates_with_confidence: [{"intent": "file", "confidence": 0.85}, ...]
    """
    PRELOAD_THRESHOLD = 0.15  # 置信度>15%才预加载
    
    for item in candidates_with_confidence:
        intent = item["intent"]
        confidence = item["confidence"]
        self._candidates.append(intent)
        
        if confidence >= PRELOAD_THRESHOLD and intent not in self._loaded_categories:
            try:
                self.load_tools_by_intent(intent)
                logger.info(f"预加载候选分类: {intent}(置信度={confidence:.2f})")
            except Exception as e:
                logger.warning(f"预加载失败: {intent}, {e}")
```

## 七、风险分析与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| GenericReactAgent无法覆盖未预见的Agent差异 | 某些Agent行为异常 | 保留独立子类扩展点（profile._agent_class），File/Time已保留 |
| 意图定义中心化后单点故障 | intent_registry初始化失败导致全系统不可用 | 兜底：若registry为空，回退到硬编码默认定义 |
| candidates预加载增加工具数量 | LLM上下文过长 | 置信度阈值过滤+工具数量上限检查 |
| CRSS关键词从registry动态读取性能 | 每次请求多一次dict查找 | registry单例内存缓存，性能影响可忽略 |
| Phase 4路由层改造风险最高 | 路由错误导致用户请求失败 | 先灰度：新旧Factory并存，通过feature flag切换 |

## 九、总结

本次重构方案的核心思想是 **"配置化替代代码化，单一数据源替代分散定义"**：

1. **Agent配置化**：7个同质Agent合并为1个GenericReactAgent + AgentProfile，消除代码膨胀
2. **意图定义中心化**：IntentRegistry统一管理意图定义，CRSS/LLM/Agent/Prompt从此单一数据源读取
3. **AgentRegistry替代AgentFactory**：消除注册覆盖风险，新增分类零代码改动
4. **Candidates增强**：置信度驱动的自动跨分类工具预加载
5. **Chat Fallback**：未匹配意图走chat而非network，改善用户体验
6. **死代码清理**：删除8+个冗余文件，减少维护负担

重构后，新增一个意图分类只需3步：定义IntentDefinition → 注册工具 → 编写Prompt，无需修改任何Agent/Factory/CRSS/LLM代码。
