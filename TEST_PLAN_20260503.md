## 专项测试

### 1. 别名机制测试（高优先级）
**目标**：验证4个别名正确映射
- `write_file` → `write_text_file`
- `glob_files` → `search_files`
- `read_file` → `read_text_file`
- `search_file_content` → `grep_file_content`

**测试项**：
- [ ] 调用别名是否自动转换
- [ ] 转换后功能是否一致
- [ ] 参数传递是否正确
- [ ] 错误处理是否一致

### 2. 二进制文件保护测试（高优先级）
**目标**：验证5个工具的保护机制
- `read_text_file`
- `write_text_file`
- `read_batch_file`
- `precise_replace_in_file`
- `edit_file`

**测试项**：
- [ ] 禁止的后缀是否正确拒绝
- [ ] 允许的后缀是否正确放行
- [ ] 无后缀文件如何处理
- [ ] 错误消息是否清晰

### 3. 交叉调用测试（中优先级）
**目标**：验证工具组合使用
- 文件读写+分析
- 系统信息+网络
- 时间+数据处理
