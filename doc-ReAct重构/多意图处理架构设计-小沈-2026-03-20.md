## 一、架构概述

### 1.1 设计理念

OmniAgent 不是"文件操作助手"，而是一个**通用的多意图处理 Agent**。它通过意图分类来决定处理路径，当前只实现了"文件操作"这一个意图，但框架层面支持所有意图类型的扩展。

**核心原则**：
- 框架通用，实现专用
- 意图分类驱动
- 预留扩展接口
- 当前只实现 file 意图，其他类型预留

### 1.2 架构分层

```
┌─────────────────────────────────────────────────────────┐
│                    用户输入                              │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│              意图识别层（通用）                           │
│  判断用户输入属于哪种意图类型                             │
│  file? network? system? code? ...                       │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│              安全检查层（通用）                           │
│  根据意图类型，走对应的安全检查路径                       │
│  file → 文件安全检查                                    │
│  network → 网络安全检查（以后）                          │
│  system → 系统安全检查（以后）                           │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│              工具执行层（通用）                           │
│  根据意图类型，调用对应的工具集                           │
│  file → file_operations 工具集（7个）                   │
│  network → network_tools 工具集（以后）                  │
│  system → system_tools 工具集（以后）                    │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│              结果处理层（通用）                           │
│  统一的结果格式化、日志记录、状态更新                     │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│                    返回用户                              │
└─────────────────────────────────────────────────────────┘
```

### 3.3 工具执行层（通用）

**职责**：根据意图类型，调用对应的工具集执行操作。

**当前实现**：只有 file_operations 工具集

**未来实现**：每种意图类型注册对应的工具集。

```python
# 接口定义
class ToolRegistry:
    """工具注册表（通用接口）"""
    
    def __init__(self):
        self._tools: Dict[IntentType, Dict[str, BaseTool]] = {}
    
    def register(self, intent_type: IntentType, tool_name: str, tool: BaseTool):
        """注册工具到指定意图类型"""
        if intent_type not in self._tools:
            self._tools[intent_type] = {}
        self._tools[intent_type][tool_name] = tool
    
    def get_tools(self, intent_type: IntentType) -> Dict[str, BaseTool]:
        """获取指定意图类型的所有工具"""
        return self._tools.get(intent_type, {})

# 注册示例
registry = ToolRegistry()
registry.register(IntentType.FILE, "read_file", read_file_tool)
registry.register(IntentType.FILE, "write_file", write_file_tool)
# 未来扩展：

# registry.register(IntentType.SYSTEM, "system_info", system_info_tool)
```

## 五、扩展接口设计

### 5.1 新增意图类型的步骤

```
1. 定义意图类型
   └── 在 IntentType 枚举中添加新类型

2. 定义工具集
   └── 创建 Pydantic 模型（参数）
   └── 写工具函数（实现）

3. 注册工具
   └── registry.register(IntentType.NETWORK, "http_request", tool)

4. 定义安全检查
   └── 实现对应的 _check_xxx_safety() 方法

5. 添加意图分类规则
   └── 在 IntentClassifier 中添加判断逻辑
```

## 六、ReAct 循环中的意图处理

### 6.1 意图识别的位置

意图识别发生在 ReAct 循环**之前**：

```
用户输入
    │
    ▼
意图识别层（判断 file/network/system/...）
    │
    ▼
加载对应意图的工具集
    │
    ▼
进入 ReAct 循环（thought → action_tool → observation → ...）
    │
    ▼
使用该意图类型的工具执行
```

### 6.2 与 ReAct 字段的关系

| ReAct 字段 | 与意图的关系 |
|-----------|-----------|
| `thought` | 意图判断后的推理过程 |
| `action_tool` | 使用对应意图的工具 |
| `observation` | 工具执行结果 |
| `final` | 最终回答 |
