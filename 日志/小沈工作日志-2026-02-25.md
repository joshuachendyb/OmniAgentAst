## 3 修复validate_config逻辑缺陷 【小健审查+小沈修复】2026-02-25 01:25:00

### 3.1 小健深度审查发现的问题

| 问题编号 | 问题描述 | 严重程度 |
|---------|---------|---------|
| **FRC-P0-001** | validate_config() 中 model 变量被覆盖，导致验证结果丢失 | 🔴 P0-紧急 |
| **FRC-P0-002** | validate_config() 没有使用最终确定的 provider | 🔴 P0-紧急 |
| **FRC-P1-003** | validate_config() 中的 provider 变量被多次重用，逻辑混乱 | 🟠 P1-高 |
| **FRC-P2-004** | 重复的 Fallback 逻辑代码（validate_config 和 get_service 都有） | 🟡 P2-中 |

### 3.2 小沈修复方案

#### 3.2.1 修复 FRC-P0-001、FRC-P0-002、FRC-P0-003

**问题根源**：变量命名混乱，多次重用同一个变量名导致逻辑错误

**修复方案**：
- 重命名变量，使用清晰的命名：
  - `selected_provider` - 用户在 `ai.provider` 中选择的
  - `selected_model` - 用户在 `ai.model` 中选择的
  - `selected_provider_config` - 对应 provider 的配置块
  - `final_provider` - 最终确定使用的 provider
  - `final_model` - 最终确定使用的 model

#### 3.2.2 修复 FRC-P0-004（重复代码）

**修复方案**：
- 提取公共方法 `_get_fallback_provider_and_model()`
- 在 `validate_config()` 和 `get_service()` 中都调用这个方法
- 消除了重复代码

### 3.3 新增公共方法

```python
@classmethod
def _get_fallback_provider_and_model(cls, ai_config: dict) -> tuple[str, str]:
    """获取 fallback 的 provider 和 model（统一 Fallback 逻辑）"""
    fallback_provider = ''
    fallback_model = ''
    for provider_name in ai_config.keys():
        if provider_name == 'provider' or provider_name == 'model':
            continue
        provider_data = ai_config.get(provider_name, {})
        if isinstance(provider_data, dict) and 'models' in provider_data and provider_data['models']:
            fallback_provider = provider_name
            fallback_model = provider_data['models'][0]
            break
    return fallback_provider, fallback_model
```
