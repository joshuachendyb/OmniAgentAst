## 3 SSE流式输出问题分析与修复 【深度研究】2026-02-23 06:30:00

### 3.1 问题现象

用户反馈：验收演示时完全看不到流式输出效果，所有数据一次性返回。

### 3.2 问题研究：FastAPI SSE流式输出要点

#### 3.2.1 SSE流式输出的核心实现

```python
from fastapi.responses import StreamingResponse

@router.post("/stream")
async def stream_api():
    async def generate():
        for i in range(10):
            yield f"data: {i}\n\n"  # SSE格式：data: + 内容 + \n\n
            await asyncio.sleep(0.5)
    
    return StreamingResponse(
        generate(),
        media_type="text/event-stream"
    )
```

#### 3.2.2 SSE流式输出失败的常见原因

| 原因 | 说明 | 解决方案 |
|------|------|---------|
| **缺少响应头** | Nginx/代理缓冲整个响应 | 添加 Cache-Control 等响应头 |
| **SSE格式错误** | 缺少 `\n\n` 结尾 | 确保 `data: xxx\n\n` 格式 |
| **中间件缓冲** | 全局异常处理缓冲 | 检查中间件配置 |
| **生成器问题** | yield后直接return | 检查生成器逻辑 |
| **客户端处理** | 前端未正确解析 | 检查前端SSE解析 |

#### 3.2.3 关键响应头（最重要！）

| Header | 作用 |
|--------|------|
| `Cache-Control: no-cache` | 禁止浏览器缓存 |
| `Connection: keep-alive` | 保持长连接 |
| `X-Accel-Buffering: no` | **禁用Nginx缓冲（关键）** |

### 3.3 根因分析

**问题代码位置**: `backend/app/api/v1/chat.py` 第428-431行

```python
# 问题代码（缺少headers）
return StreamingResponse(
    generate(),
    media_type="text/event-stream"
)
```

**影响**：
- Nginx默认缓冲整个响应，直到后端完成所有yield才发送
- 用户看不到流式效果，所有数据一次性返回

### 3.4 修复方案

```python
# 修复后代码
return StreamingResponse(
    generate(),
    media_type="text/event-stream",
    headers={
        "Cache-Control": "no-cache",
        "Connection": "keep-alive",
        "X-Accel-Buffering": "no"  # 禁用Nginx缓冲，确保流式输出
    }
)
```
