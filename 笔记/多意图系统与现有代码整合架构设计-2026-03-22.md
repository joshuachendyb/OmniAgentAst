## 1 背景与问题

### 1.1 问题描述

多意图预处理系统（PreprocessingPipeline + GLiClass）重构完成后，花费了约6天时间，但存在以下问题：

| 问题 | 说明 |
|------|------|
| 重构成果未被使用 | PreprocessingPipeline 完成后，chat_stream.py 没有调用它 |
| 旧逻辑仍在使用 | chat_stream.py 仍使用 detect_file_operation_intent() 旧函数 |
| 架构职责不清 | chat_stream.py (1537行) 同时处理：意图检测、安全检查、流式输出、中断处理 |
| 代码冗余 | 多处存在重复的意图检测逻辑 |

### 1.2 整合目标

| 目标 | 说明 |
|------|------|
| 统一入口 | 所有对话请求经过统一的意图检测 |
| 职责分离 | 每个文件只做一件事 |
| 复用重构成果 | PreprocessingPipeline 作为核心意图检测模块 |
| 清晰架构 | 各文件各司其职 |

### 1.3 重构原则

**核心原则**：用新架构替代旧架构，不是修修补补！

| 原则 | 说明 |
|------|------|
| 接口 | 重新设计，不用旧的 |
| 函数名 | 重新命名，不用旧的 |
| 流程 | 重新设计，不用旧的 |
| 渐进式 | 先新建文件，逐步迁移，最后删除废弃代码 |

**必须删除的旧代码**：
- `detect_file_operation_intent()` → 删除
- `intent_classifier.py` → 删除
- `chat_stream.py` 中的 `start1/thought1` → 移到对应 flow
- 前端需要适配新的 API

**测试要求**：每个新建/修改的文件都需要测试验证。

```python
def create_final_response(
    content: str,
    model: str,
    provider: str,
    display_name: Optional[str] = None,
    step: Optional[int] = None
) -> str:
    """
    创建统一的 final 响应格式
    返回 SSE 格式的 final 响应字符串
    """
    response = {
        'type': 'final',
        'content': content,
        'display_name': display_name if display_name else f"{provider} ({model})" or None,
        'timestamp': create_timestamp(),
    }
    if step is not None:
        response['step'] = step
    return f"data: {json.dumps(response)}\n\n"
```

FileOperationAgent.run_stream() 实现了标准的 ReAct 循环：

```
while step_count < max_steps:

    │  1. Thought (思考)      │
    │  LLM 分析任务，决定工具   │

    └─────────────────────────┘

    ┌─────────────────────────┐
    │  判断 is_finished       │
    │  是 → 发送 final 结束   │
```

### 7.7 总结

chat2.py 中的 ReAct 流程通过**两层结构**实现：

1. **外层**（chat2.py）：负责文件操作的初始化和安全检测，产生 action1 → observation1 → action2 序列
2. **内层**（FileOperationAgent.run_stream()）：执行真正的 ReAct 循环，产生 Thought → Action → Observation → ... → Final 序列

这种设计将**流程控制**（外层）和**业务逻辑**（内层）分离，便于维护和扩展。

```python
# app/chat_stream/sse_formatter.py

def format_sse_event(event_type: str, step: int, data: dict) -> str:
    """统一格式化 SSE 事件"""
    base = {
        'type': event_type,
        'step': step,
        'timestamp': datetime.now().strftime('%Y-%m-%d %H:%M:%S')
    }
    base.update(data)
    return f"data: {json.dumps(base)}\n\n"

def format_thought_sse(step: int, content: str, reasoning: str = '', 
                       action_tool: str = '', params: dict = None) -> str:
    """格式化 thought 事件"""
    return format_sse_event('thought', step, {
        'content': content,
        'reasoning': reasoning,
        'action_tool': action_tool,
        'params': params or {}
    })

def format_action_tool_sse(step: int, tool_name: str, tool_params: dict,
                           execution_status: str = 'success', summary: str = '',
                           raw_data=None, action_retry_count: int = 0) -> str:
    """格式化 action_tool 事件"""
    return format_sse_event('action_tool', step, {
        'tool_name': tool_name,
        'tool_params': tool_params,
        'execution_status': execution_status,
        'summary': summary,
        'raw_data': raw_data,
        'action_retry_count': action_retry_count
    })

def format_observation_sse(step: int, execution_status: str, summary: str,
                           content: str = '', reasoning: str = '',
                           action_tool: str = '', params: dict = None,
                           is_finished: bool = False, raw_data=None) -> str:
    """格式化 observation 事件"""
    return format_sse_event('observation', step, {
        'obs_execution_status': execution_status,
        'obs_summary': summary,
        'obs_raw_data': raw_data,
        'content': content,
        'obs_reasoning': reasoning,
        'obs_action_tool': action_tool,
        'obs_params': params or {},
        'is_finished': is_finished
    })
```
