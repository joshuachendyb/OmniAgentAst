## 一、核心概念

本文档分析**普通用户的日常场景**与**Function Call（工具调用）**之间的对应关系。

用户说的是**日常用语（人话）**，系统执行的是**工具调用序列（机器指令）**。中间通过**意图翻译（Intent Translation）**机制进行转换。

---

## 二、从日常场景到 Function Call 的转换逻辑

```
【用户日常场景】：日常用语（非结构化）
      ↓
【意图翻译层】（中间处理机制）：
  1. 意图识别（用户想干嘛？）
  2. 步骤拆解（需要分几步做？）
  3. 参数提取（从话里抠出关键信息）
      ↓
【Function Call 层】：生成执行计划（Plan）
  Step 1: Tool A(参数)
  Step 2: Tool B(参数)
  ...
```

## 三、典型日常场景与 Function Call 对应关系

> **说明**：本文档选取 15 个最典型的日常场景进行分析，覆盖文件操作、办公自动化、系统维护、信息获取等常见类别。实际应用中，用户场景可能更加多样化。

以下梳理日常最高频的 15 类场景，展示从"日常场景"到"Function Call 调用序列"的转换过程：

### 场景 1：找文件（跨类别组合）

*   **用户日常场景**："帮我找下上周下载的那个合同"
*   **意图翻译**：
    1.  **意图**：搜索文件，并通知用户结果。
    2.  **拆解**：算时间范围 -> 搜文件 -> 筛时间 -> 确认内容 -> 通知用户。
    3.  **参数**：目录=`Downloads`，关键词=`合同`，时间=`上周`。
*   **对应 Function Call 序列（跨类别）**：
    1.  `get_current_time()` -> **【5 类：时间/日期】**获取当前时间，计算"上周"范围
    2.  `search_files(search_dir="Downloads", pattern="*合同*")` -> **【1 类：文件操作】**搜索文件
    3.  `get_file_info(file_path="找到的文件 1")` -> **【1 类：文件操作】**检查修改时间是否在上周范围内
    4.  `read_text_file(file_path="确认的文件")` -> **【1 类：文件操作】**确认内容是合同
    5.  `send_notification(title="找到文件", message="已找到上周下载的合同：xxx.pdf")` -> **【28 类：通知】**通知用户结果

### 场景 2：改内容（跨类别组合）

*   **用户日常场景**："把这个文档里的旧地址改成新地址"
*   **意图翻译**：
    1.  **意图**：修改文件内容，并确保安全（备份）。
    2.  **拆解**：检查文件 -> 备份 -> 读文件 -> 替换内容 -> 通知用户。
    3.  **参数**：文件=`当前文档`，旧内容=`旧地址`，新内容=`新地址`。
*   **对应 Function Call 序列（跨类别）**：
    1.  `check_path_exists(path="当前文档")` -> **【附录：配套保障】**检查文件是否存在
    2.  `backup_file(file_path="当前文档")` -> **【附录：配套保障】**修改前先备份
    3.  `read_text_file(file_path="当前文档")` -> **【1 类：文件操作】**读取文件内容
    4.  `precise_replace_in_file(file_path="当前文档", old_string="旧地址", new_string="新地址")` -> **【1 类：文件操作】**替换内容
    5.  `send_notification(title="修改完成", message="已将文档中的旧地址改为新地址")` -> **【28 类：通知】**通知用户结果

### 场景 3：整理文件（跨类别组合）

*   **用户日常场景**："帮我把桌面上的照片整理到一个文件夹里"
*   **意图翻译**：
    1.  **意图**：分类移动文件。
    2.  **拆解**：看桌面有什么 -> 获取当前日期命名文件夹 -> 建文件夹 -> 移动照片 -> 通知用户。
    3.  **参数**：源=`Desktop`，目标=`照片整理_日期`，类型=`jpg/png`。
*   **对应 Function Call 序列（跨类别）**：
    1.  `list_directory(dir_path="Desktop")` -> **【1 类：文件操作】**查看桌面内容
    2.  `get_current_time(format="YYYYMMDD")` -> **【5 类：时间/日期】**获取当前日期用于命名
    3.  `create_directory(dir_path="Desktop/照片整理_20260405")` -> **【1 类：文件操作】**创建分类文件夹
    4.  `move_file(source="Desktop/照片.jpg", destination="Desktop/照片整理_20260405/照片.jpg")` -> **【1 类：文件操作】**移动照片（循环执行）
    5.  `send_notification(title="整理完成", message="已将桌面照片整理到'照片整理_20260405'文件夹")` -> **【28 类：通知】**通知用户结果

### 场景 4：看文档（跨类别组合）

*   **用户日常场景**："帮我看看这个 PDF 合同写了什么，重点看金额"
*   **意图翻译**：

*   **对应 Function Call 序列（跨类别）**：

    4.  `write_append_file(file_path="合同摘要.txt", text="合同金额：XXX 元")` -> **【1 类：文件操作】**保存摘要
    5.  `send_notification(title="文档分析完成", message="合同金额为 XXX 元，摘要已保存")` -> **【28 类：通知】**通知用户结果

### 场景 5：查信息（跨类别组合）

    2.  **拆解**：查当前时间 -> 算偏移日期 -> 通知用户。

## 四、意图翻译的三大核心能力

| 能力 | 说明 | 例子 |
|------|------|------|
| **1. 意图识别** | 听懂用户想干嘛 | "找下"=搜索，"改成"=编辑，"打包"=压缩 |
| **2. 步骤拆解** | 把一件事拆成几步 | 找文件 = 搜索 + 过滤 + 确认 |
| **3. 参数提取** | 从话里抠出关键信息 | "上周"=时间过滤，"合同"=关键词 |

## 五、总结

**日常场景与 Function Call 的对应关系**：

*   **输入**：日常场景（一句话，人话）
*   **中间处理**：意图翻译（Intent Translation）
*   **输出**：Function Call 调用序列（机器指令列表）
*   **本质**：AI 的**规划能力（Planning）**，将非结构化需求转化为结构化工具调用。
