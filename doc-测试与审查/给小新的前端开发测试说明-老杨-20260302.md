## 一、背景

老杨优化了前端启动速度，修改了配置，需要知会开发测试人员如何使用。

---

## 二、修改内容

### 2.1 vite.config.ts 修改

**之前**: 每次 `npm run dev` 都运行 eslint 和 prettier，启动慢

**现在**: 
- `npm run dev` → 不检查，启动快 ⚡
- `npm run build` → 自动检查，保证质量 ✅

### 2.2 package.json 修改

新增了 `check` 命令：
```json
"check": "npm run lint && npm run format:check"
```

E2E测试命令自动先检查：
```json
"test:e2e": "npm run check && playwright test"
```

---

## 三、小新（前端开发）使用指南

### 3.1 日常开发

```bash
npm run dev
# ✅ 启动快！没有eslint/prettier检查
# 🔥 启动速度提升 50-70%
```

### 3.2 代码检查（可选）

**方式1: IDE自动检查**（推荐）
- 配置 VS Code / WebStorm 在保存时自动运行 ESLint 和 Prettier
- 不影响启动速度

**方式2: 手动检查**
```bash
# 检查代码规范
npm run lint

# 自动修复问题
npm run lint:fix

# 格式化代码
npm run format

# 检查格式
npm run format:check

# 完整检查（ESLint + Prettier）
npm run check
```

### 3.3 提交代码前

```bash
npm run check
# 确保代码符合规范后再提交
```

## 四、小健（测试）使用指南

### 4.1 E2E测试

```bash
npm run test:e2e
# ✅ 自动先运行 npm run check，再运行E2E测试
# 保证测试时代码质量
```

其他E2E命令同样自动检查：
```bash
npm run test:e2e:ui
npm run test:e2e:debug
```

### 4.2 单元测试

```bash
npm run test
npm run test:watch
npm run test:coverage
# 单元测试不受影响，照常使用
```
