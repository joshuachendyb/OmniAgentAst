## 三、解决方案

### 3.1 方案1：ProviderSettings改为按需加载（推荐）

**修改ProviderSettings组件**:
```typescript
const ProviderSettings: React.FC = () => {
  // ... 现有state ...
  
  // ⭐ 新增：是否已加载标志
  const [hasLoaded, setHasLoaded] = useState(false);

  // ⭐ 修改：只在第一次访问时加载
  useEffect(() => {
    if (!hasLoaded) {
      handleLoadWithValidation();
      setHasLoaded(true);
    }
  }, [hasLoaded]);

  // ... 其余代码不变 ...
};
```

**优点**：
- 用户打开Settings页面时，默认Tab是"模型配置"，所以还是会加载
- 如果用户修改默认Tab，就不会加载
- 简单，改动小

---

### 3.2 方案2：完全按需加载，切换Tab时才加载（最佳）

**修改Settings主组件 + ProviderSettings组件**:

**Settings主组件**:
```typescript
const Settings: React.FC = () => {
  const [activeKey, setActiveKey] = useState("model");
  // ⭐ 新增：跟踪哪个Tab已加载
  const [loadedTabs, setLoadedTabs] = useState<Set<string>>(new Set());

  const handleTabChange = (key: string) => {
    if (isDirty) {
      setPendingKey(key);
      setConfirmModalVisible(true);
    } else {
      setActiveKey(key);
      // ⭐ 按需加载：切换到对应Tab时标记已加载
      setLoadedTabs(prev => new Set(prev).add(key));
    }
  };

  return (
    <Tabs activeKey={activeKey} onChange={handleTabChange}>
      <TabPane tab="模型配置" key="model">
        {/* ⭐ 传递loadedTabs给ProviderSettings */}
        <ProviderSettings shouldLoad={loadedTabs.has("model")} />
      </TabPane>
      {/* ... 其他Tab ... */}
    </Tabs>
  );
};
```

**ProviderSettings组件**:
```typescript
const ProviderSettings: React.FC<{ shouldLoad: boolean }> = ({ shouldLoad }) => {
  // ... 现有state ...

  // ⭐ 修改：只在shouldLoad为true时加载
  useEffect(() => {
    if (shouldLoad) {
      handleLoadWithValidation();
    }
  }, [shouldLoad]);

  // ... 其余代码不变 ...
};
```

**优点**：
- 完全按需加载，只有切换到"模型配置"Tab时才发起API请求
- 不影响其他Tab的加载
- 最佳性能

---

## 四、推荐方案

**推荐使用方案2（完全按需加载）**，因为：
1. 性能最佳
2. 逻辑清晰
3. 符合之前3f23a55 commit的优化思路
