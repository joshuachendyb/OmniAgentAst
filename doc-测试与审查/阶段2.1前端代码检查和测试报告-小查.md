## 三、问题详细分析

### 3.1 问题1：StreamStep类型缺少interrupted和error

**问题描述**：
```typescript
// 当前定义（缺少interrupted和error）
export interface StreamStep {
  type: 'thought' | 'action' | 'observation' | 'final';
  // 缺少 'interrupted' 和 'error'
}
```

**小沈第8章要求**：
| 事件类型 | 说明 |
|---------|------|
| `thought` | AI思考过程 |
| `action` | 执行的操作步骤 |
| `observation` | 操作观察结果 |
| `error` | 错误信息 |
| `interrupted` | 任务被中断 |
| `final` | 最终完成 |

**整改建议**：
```typescript
// 修改为：
export interface StreamStep {
  type: 'thought' | 'action' | 'observation' | 'error' | 'interrupted' | 'final';
  step?: number;
  thought?: string;
  action?: string;
  action_input?: any;
  observation?: any;
  content?: string;
}
```

### 3.3 问题3：任务中断用AbortController而不是取消接口

**问题描述**：
```typescript
// 当前实现（AbortController）
const [abortController, setAbortController] = useState<AbortController | null>(null);

const handleSend = async () => {
  const controller = new AbortController();
  setAbortController(controller);
  
  const response = await fetch('/api/v1/chat/stream', {
    method: 'POST',
    signal: controller.signal  // ← 使用signal
  });
};

const handleInterrupt = () => {
  abortController?.abort();  // ← 直接中断
};
```

**小沈第8章要求（推荐方案一）**：
```typescript
const [currentTaskId, setCurrentTaskId] = useState<string | null>(null);

const handleSend = async () => {
  const response = await fetch('/api/v1/chat/stream', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ messages })
  });
  
  // 从响应中获取task_id（或自己生成UUID）
  const taskId = crypto.randomUUID();
  setCurrentTaskId(taskId);
  
  // 处理SSE流...
};

const handleInterrupt = async () => {
  if (currentTaskId) {
    await fetch(`/api/v1/chat/stream/cancel/${currentTaskId}`, {
      method: 'POST'
    });
  }
};
```

**小沈推荐方案的优点**：
1. 后端可以更精确控制中断时机
2. 支持在服务端判断中断状态
3. 不依赖前端手动中断
4. 后端能更准确地返回 `interrupted` 事件

**整改建议**：
按小沈推荐的方案一修改为使用取消接口。
