### ✅ 问题1：乐观锁并发安全问题（P0-紧急）

**修复位置**: `sessions.py` 第696-760行，`update_session()`函数

**修复内容**:
```python

cursor.execute(
    '''SELECT id, title, version, ... FROM chat_sessions WHERE id = ? AND version = ?''',
    (session_id, update_data.version)
)
session = cursor.fetchone()
# cursor.execute(f'UPDATE chat_sessions SET ... WHERE id = ?', ...)

# 修复后：原子性UPDATE
cursor.execute(

       WHERE id = ? AND is_deleted = FALSE AND version = ?''',
    (update_data.title, utc_time, 1, utc_time, session_id, update_data.version)
```

### ✅ 问题2：事务边界不完整（P0-紧急）

**修复位置**: `sessions.py` 第666-853行，`update_session()`和`save_message()`函数

**修复内容**:
```python
# 修复前：没有显式事务
try:
    cursor.execute(f'UPDATE chat_sessions SET ...')
    cursor.execute(f'INSERT INTO chat_session_title_history ...')
    conn.commit()
except Exception as e:
    conn.close()
    raise

# 修复后：显式事务+错误回滚

    conn = _get_db_connection()
    cursor = conn.cursor()
    cursor.execute("BEGIN")
    
    # ... 业务逻辑 ...
    
    cursor.execute("COMMIT")
    conn.close()
except HTTPException as he:
    raise
except Exception as e:
    logger.error(f"更新会话失败: {e}")
    try:
        cursor.execute("ROLLBACK")
    except:
        pass
    conn.close()
    raise HTTPException(status_code=500, detail="更新会话失败，请重试")
```

**修复效果**:
- ✅ 显式开启和关闭事务
- ✅ 异常时自动回滚
- ✅ 确保数据一致性
- ✅ 防止部分更新风险

### ✅ 问题4：版本号无条件递增（P1-高）

**修复位置**: `sessions.py` 第805-810行

**修复内容**:
```python
# 修复前：无条件递增版本号

if fields_exist['version']:
    update_fields.append('version = ?')
    new_version = current_version + 1
    update_values.append(new_version)

# 修复后：只在标题变化时递增
if fields_exist['version']:
    # ⭐ 修复：检查标题是否真的改变了
    if old_title != update_data.title:
        update_fields.append('version = ?')
        new_version = current_version + 1
        update_values.append(new_version)
    else:
        # 标题未变化，不更新version
        new_version = current_version
```

**修复效果**:
- ✅ 只在标题实际变化时递增版本号
- ✅ 避免不必要的版本号递增
- ✅ 减少409冲突错误
- ✅ 提高并发性能
