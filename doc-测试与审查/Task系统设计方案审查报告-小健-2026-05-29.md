**审查时间**: 2026-05-29 07:15:00

## 一、设计方案概述

### 1.1 设计目标

统一任务追踪系统，服务于Agent系统中全部9个意图（file/shell/document/code_execution/network/system/desktop/time/meta），提供统一的任务生命期管理和操作明细记录。

### 1.2 核心设计

| 组件 | 说明 |
|------|------|
| 数据模型 | tasks表（任务）+ operations表（操作明细） |
| 核心类 | TaskTracker（统一任务追踪器） |
| 存储 | SQLite（通过系统统一DB基础设施） |
| 文件结构 | app/services/task/目录 |

## 三、数据模型分析

### 3.1 tasks表设计

| 字段 | 类型 | 必要性 | 分析 |
|------|------|--------|------|
| task_id | TEXT PK | ✅ 必要 | 唯一标识，格式合理 |
| intent | TEXT | ✅ 必要 | 支持按意图筛选 |
| agent_id | TEXT | ✅ 必要 | 标识执行者 |
| task_description | TEXT | ✅ 必要 | 任务描述 |
| status | TEXT | ✅ 必要 | 任务状态 |
| total_operations | INTEGER | ✅ 必要 | 操作计数 |
| success_count | INTEGER | ✅ 必要 | 成功计数 |
| failed_count | INTEGER | ✅ 必要 | 失败计数 |
| rolled_back_count | INTEGER | ✅ 必要 | 回滚计数 |
| report_generated | INTEGER | ✅ 必要 | 报告状态 |

| completed_at | TIMESTAMP | ⚠️ 可选 | 完成时间 |

- `idx_tasks_intent`：✅ 必要，支持按意图筛选
