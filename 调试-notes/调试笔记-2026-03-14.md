## 2 切换应用后消息消失问题分析 【北京老陈问题6】2026-03-15 21:23:21

### 2.1 问题描述

切换应用（切换到其他App再切回）后，聊天消息消失或丢失。

### 2.2 问题1：sessionStorage 保存逻辑使用闭包中的旧 messages

**代码位置**:
- `saveState`: NewChatContainer.tsx 第775-791行
- `visibilitychange useEffect`: NewChatContainer.tsx 第868-951行

**问题代码**:
```typescript
// 第951行 - useEffect 依赖数组
}, [sessionId, isReceiving]);  // ❌ 没有 messages！

// 第775行 - saveState 函数
const saveState = () => {
  const state = {
    messages,  // ❌ 访问的是闭包中的旧 messages
    ...
  };
};
```

**根因**:
1. `visibilitychange` useEffect 依赖 `[sessionId, isReceiving]`，**不包含 messages**
2. 当 messages 更新但 sessionId/isReceiving 不变时，useEffect **不会重新运行**
3. `handleVisibilityChange` 闭包中的 `saveState` 访问的是**旧的** messages

**影响**:
- 页面隐藏时，保存的是旧消息，不是最新消息
- 页面恢复时要么恢复旧消息，要么因缓存检查失败而不恢复

### 2.3 问题2：组件卸载导致内存状态丢失

**场景**:
- 切换应用到后台（尤其移动设备）
- 浏览器内存压力导致页面被销毁
- 组件被卸载，内存中的 messages 状态全部丢失

**当前恢复逻辑**（第880-943行）:
1. 页面恢复时，尝试从 sessionStorage 恢复
2. 如果缓存有效（5分钟内），恢复消息
3. 如果缓存无效，从 API 加载

**问题**:
- 组件卸载后，内存状态丢失
- 恢复依赖 sessionStorage，但 sessionStorage 保存的是旧消息（问题1）
- 从 API 加载会**覆盖**当前正在进行的对话

### 2.4 两个问题的关联

```
切换应用 → 页面隐藏 → saveState() [但保存的是旧消息]
              ↓
         组件可能卸载 → 内存状态丢失
              ↓
         页面恢复 → 恢复旧消息 OR 从API加载 → 覆盖当前对话
```

### 2.5 待修复方案

**修复方案（已验证）**：

#### 针对问题1（sessionStorage 保存旧消息）：

**推荐方案**：在 visibilitychange useEffect 内部直接使用 messagesRef.current

## 3 executionStepsRef 未同步更新问题 【小沈分析】2026-03-15 21:26:58

### 3.1 问题背景

|------|------|------|
| 第617/632/696/717行 | `setExecutionSteps(...)` | ✅ 更新React state |

### 3.4 修复方案（已验证）

**文件**：sse.ts，第717-724行

```typescript
// 修改前：
setExecutionSteps((prev) => [...prev, step]);
onComplete?.(..., [...handlers.getCurrentExecutionSteps(), step]);
```

**原理**：
- 在 setExecutionSteps 的回调函数中同步更新 executionStepsRef.current
- 确保 onComplete 调用时 ref 已经是最新值
- 不再依赖异步的 useEffect

## 3 executionStepsRef 未同步更新问题 【小沈分析】2026-03-15 21:26:58
