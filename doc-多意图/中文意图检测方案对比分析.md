## 一、研究背景

### 1.1 项目现状

项目当前使用**基于规则的关键词匹配**方法进行文本分类，核心函数为 `detect_file_operation_intent`，位于 `backend/app/api/v1/chat.py` 第215-325行。

**现有方法的问题**：
- 准确率较低（70-80%）
- 关键词覆盖率有限
- 误报和漏报率较高
- 难以处理复杂语义和上下文

### 1.2 研究目标

找到一种更智能的文本分类方法，能够：
- 准确区分"简单问答"和"事务性请求"
- 支持中文语义理解
- 快速集成到现有架构
- 不需要大量训练数据

## 七、风险评估

### 7.1 低风险方案：混合模式（推荐）

```python
def detect_file_operation_intent(message: str) -> tuple[bool, str, float]:
    """混合方案：规则匹配 + JioNLP增强"""
    
    # 先使用现有规则匹配（保证兼容性）
    is_file_op, op_type, confidence = existing_rule_based_detection(message)
    
    if confidence < 0.5:
        # 规则匹配置信度低时，使用JioNLP增强判断
        jio_result = jionlp_classification(message)
        if jio_result["is_file_operation"]:
            is_file_op = True
            op_type = jio_result["operation_type"]
            confidence = 0.7 + jio_result["confidence"] * 0.3
            
    return is_file_op, op_type, min(confidence, 1.0)
```

### 7.2 风险控制措施

1. **保留现有逻辑**：混合模式保证向后兼容
2. **逐步替换**：先在测试环境验证，再上线
3. **监控日志**：记录分类结果，便于优化
4. **回滚方案**：如果效果不好，可以快速回滚
