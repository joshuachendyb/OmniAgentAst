## 三、目录结构与说明

**说明**：

| 目录 | 命名依据 | 职责 |
|------|---------|------|
| `preprocessing/` | 【独立】用户输入预处理 | 语句纠偏（pycorrector）、意图检测（GLiClass） |
| `intent/` | 【独立】意图管理层 | 意图注册、确认、拆分（通用接口） |

| → `tools/network/` | network 意图 | 网络操作工具（代码以后） |
| → `tools/system/` | system 意图 | 系统操作工具（代码以后） |
| → `tools/database/` | database 意图 | 数据库操作工具（代码以后） |
| → `tools/desktop/` | desktop 意图 | Windows桌面操作工具（file完成后首个升级） |

| → `safety/file/` | file 意图的安全检查 | 白名单、路径验证 |
| → `safety/network/` | network 意图的安全检查 | 允许域名、协议限制 |

| → `safety/database/` | database 意图的安全检查 | SQL 注入防护 |

### 4.2 基于目录结构的函数说明
```
backend/app/services/
│
├── agent/                          ← 通用多意图Agent引擎
│   │
│   ├── agent.py                    → IntentAgent类
│   │     核心函数:
│   │       __init__(llm_client, session_id, intent_type="file", ...)
```

## 六、预处理模块详解

### 6.1 模块概述

preprocessing/ 是用户输入预处理模块，在 Agent 之前执行，负责对用户输入进行初步处理：

| 模块 | 职责 | 使用库 |
|------|------|--------|
| corrector.py | 语句校对纠错 | pycorrector |
| intent_classifier.py | 意图检测分类 | GLiClass |
| pipeline.py | 流水线整合 | 以上两个模块 |

### 6.2 corrector.py - 语句校对

**功能**：对用户输入进行拼写检查和纠错

**输入输出**：
```
输入: "帮我读起个文件"
输出: ("帮我读取个文件", [{'position': 4, 'error': '起个', 'correction': '读取个'}, ...])
```

**处理逻辑**：
1. 检查输入是否为 None 或空
2. 调用 pycorrector 进行纠错
3. 返回修正后文本和修正记录列表

### 6.3 intent_classifier.py - 意图检测

**功能**：对修正后的文本进行意图分类

**输入输出**：
```python
输入: classify("帮我读取个文件", labels=["file", "network", "system"])
输出: {
    "intent": "file",           # 最佳意图
    "confidence": 0.95,         # 置信度
    "all_intents": {            # 所有意图及置信度
        "file": 0.95,
        "network": 0.03,
        "system": 0.02
    }
}
```

**处理逻辑**：
1. 检查输入文本和标签是否有效
2. 调用 GLiClass 进行零样本分类
3. 返回最佳意图和置信度

### 6.4 pipeline.py - 流水线整合

**功能**：整合 corrector 和 classifier，提供统一入口

**输入输出**：
```python
输入: process("帮我读起个文件", intent_labels=["file", "network"])
输出: {
    "original": "帮我读起个文件",      # 原始输入
    "corrected": "帮我读取个文件",      # 修正后
    "errors": [...],                   # 修正记录
    "intent": "file",                  # 检测到的意图
    "confidence": 0.95,               # 置信度
    "all_intents": {...}              # 所有意图
}
```

**处理流程**：
1. 调用 corrector.correct() 进行语句校对
2. 调用 classifier.classify() 进行意图检测
3. 整合结果返回

### 6.5 第三方依赖

| 库 | 版本 | 用途 | 状态 |
|------|------|------|------|
| pycorrector | 2.3.8 | 中文文本纠错 | 测试用mock |
| gliclass | 0.0.8 | 零样本意图分类 | 测试用mock |

**依赖位置**：`backend/requirements.txt`

### 6.6 日志记录

预处理模块在运行过程中会记录以下日志：

| 场景 | 日志级别 | 内容 |
|------|---------|------|
| 修正成功 | INFO | "[Preprocessing] Corrector - input: '...' -> output: '...', errors: [...]" |
| 修正失败 | WARNING | "[Preprocessing] Corrector failed: {error}" |
| 分类成功 | INFO | "[Preprocessing] Classifier - input: '...', labels: [...] -> intent: '...', confidence: 0.xxxx, all_intents: {...}" |
| 分类失败 | WARNING | "[Preprocessing] Classifier failed: {error}" |

**日志文件位置**：`backend/logs/app_YYYY-MM-DD.log`

**示例日志**：
```

2026-03-22 14:00:00 - INFO - pipeline.py - [Preprocessing] Classifier - input: '帮我读取个文件', labels: ['file', 'network'] -> intent: 'file', confidence: 0.9500, all_intents: {'file': 0.95, 'network': 0.05}
```
