## 一、架构概述

### 1.1 重构 file 检查的设计意图

**当前代码的问题**：意图检测和安全检查都硬编码在文件操作里，和文件操作绑死了，无法扩展。

| 问题 | 说明 |
|------|------|
| **意图检测硬编码** | 只能识别文件操作意图，无法识别其他意图 |
| **安全检查硬编码** | 白名单、路径验证写在文件操作模块里 |
| **框架不够通用** | 如果只支持 file，框架就和文件助手绑死了 |

OmniAgent 不是"文件操作助手"，而是一个**通用的多意图处理 Agent**。它通过意图分类来决定处理路径，当前只实现了"文件操作"这一个意图，但框架层面支持所有意图类型的扩展。

**核心原则**：
- 框架通用，实现专用
- 意图分类驱动
- 预留扩展接口
- 当前只实现 file 意图，其他类型预留

## 二、设计理念与三个"为什么"

### 2.1 为什么要引入输入信息的校对和修正？

**用户输入不标准**，是实际场景中的高频问题：

| 用户输入 | 问题类型 |
|---------|---------|
| "帮我读起这个文件" | 错别字（读起→读取） |
| "帮我读下这个文件" | 口语化（缺字） |
| "文件读取一下" | 语序不完整 |
| "读文件" | 极简口语 |

这些问题会导致：
- 意图检测失败或错误（LLM 无法理解"读起"是什么意图）
- 工具参数提取错误（"读下"对应什么操作？）

**因此**：在意图检测前，先用 pycorrector 做语句校对修正，确保输入文本规范。

### 2.2 为什么要引入意图检测的外部方法？

**为什么不自己训练一个意图分类模型？**

| 问题 | 说明 |
|------|------|
| **需要标注数据** | 训练一个可用的分类模型，需要大量标注数据 |
| **需要持续维护** | 模型上线后需要监控效果、持续迭代 |
| **冷启动成本高** | 从零开始训练，时间和算力成本都高 |

**为什么不只用关键词匹配？**

| 问题 | 说明 |
|------|------|
| **准确率低** | 关键词匹配无法处理同义词、多义词、口语化表达 |
| **覆盖有限** | 用户说"帮我看看 D:/test.txt"，关键词"看"不在 file 意图列表中 |
| **无法置信度排序** | 关键词匹配只有"是/否"，没有置信度，无法做多意图排序 |

**为什么选择 GLiClass 作为外部方法？**

| 特性 | GLiClass |
|------|---------|
| **零样本分类** | 不需要训练，直接用标签列表分类 |
| **准确率高** | 基于大模型，语义理解能力强 |
| **轻量** | 库本身只有 ~5MB，总依赖 ~500MB，断网可用 |
| **延迟低** | ~20ms，单次推理可接受 |
| **开源** | MIT 许可证，可商用 |

**结论**：GLiClass 是"开箱即用"的最佳选择，平衡了准确率、延迟、部署成本三者。

### 2.3 为什么要从 file 支持扩展到多意图支持？

**当前只支持 file 意图的局限性：**

| 问题 | 说明 |
|------|------|
| **用户请求是复合的** | 用户经常说"帮我读取 D:/test.txt，并搜索一下天气" |
| **框架不够通用** | 如果只支持 file，框架就和文件助手绑死了 |
| **无法扩展** | 以后想支持 network、system，需要大改框架代码 |

**多意图支持的目标**：

```
用户输入: "帮我读取 D:/test.txt，并搜索一下天气"
    │
    ▼
意图拆分：
├── Intent 1: file（"读取 D:/test.txt"）
└── Intent 2: network（"搜索天气"）
    │
    ▼
按顺序处理：
├── 先执行 file 意图 → 读取文件
└── 再执行 network 意图 → 搜索天气
    │
    ▼
合并结果 → 返回用户
```

**核心原则**：框架通用，实现专用。当前只实现 file 意图，但框架层面支持所有意图类型的扩展。

### 2.4 意图检测的双层策略

意图检测采用**双层策略**，互为补充：

```
第一层：GLiClass（第五章预处理流水线）
  └─ 优先使用 GLiClass 分类
  └─ 优点：语义理解强、准确率高

第二层：关键词匹配（第六章意图识别层 fallback）
  └─ GLiClass 不可用时的兜底
  └─ 优点：零延迟、无依赖

第三层：回退 ReAct 循环
  └─ 两层都失败时，使用全量工具
```

**为什么这样设计？**

| 场景 | 使用哪层 | 原因 |
|------|---------|------|
| GLiClass 可用 | GLiClass | 准确率最高 |
| GLiClass 加载失败 | 关键词匹配 | 零延迟兜底 |
| 两者都失败 | 回退 ReAct | 保证系统可用 |

## 三、目录改造思路

### 3.1 当前问题

当前 `file_operations/` 目录存在以下问题：

| 问题 | 说明 |
|------|------|
| **目录名误导** | 叫"文件操作"，实际包含 Agent 主循环、LLM 适配等 |

| **安全检查硬编码** | 白名单、路径验证写在文件操作模块里 |
| **扩展困难** | 新增意图类型需要大改框架代码 |

### 3.2 改造目标

**核心原则**：框架通用，实现专用

| 改造目标 | 说明 |
|---------|------|
| **目录名准确** | `agent/` 反映通用多意图 Agent 引擎定位 |

**三大核心模块**：


## 四、意图类型定义（Registry 模式）

### 4.1 意图注册表

**为什么用 Registry 而不是 Enum？**

新增意图类型时不需要修改框架代码，只需调用 `register()` 注册即可。符合开闭原则。

```python
from typing import Dict, List, Optional, Callable
from pydantic import BaseModel

class Intent(BaseModel):
    """意图定义"""
    name: str                    # 意图名称（如 "file", "network"）
    description: str             # 意图描述
    keywords: List[str]          # 关键词列表（用于分类）
    tools: List[str]             # 关联的工具名称列表
    safety_checker: Optional[str] = None  # 安全检查器名称

class IntentRegistry:
    """意图注册表（通用接口）"""
    
    def __init__(self):
        self._intents: Dict[str, Intent] = {}
    
    def register(self, intent: Intent):
        """注册新意图类型"""
        self._intents[intent.name] = intent
    
    def get(self, name: str) -> Optional[Intent]:
        """获取指定意图"""
        return self._intents.get(name)
    
    def list_all(self) -> List[Intent]:
        """列出所有已注册的意图"""
        return list(self._intents.values())
    
    def get_all_names(self) -> List[str]:
        """获取所有意图名称"""
        return list(self._intents.keys())
```

### 4.2 当前已注册的意图

```python
# 创建注册表

# 注册 FILE 意图（已实现）
registry.register(Intent(

# registry.register(Intent(name="network", ...))
# registry.register(Intent(name="system", ...))
```

## 五、用户输入预处理流水线

### 5.1 流水线设计

```
用户输入: "帮我读起个文件"
    │
    ▼
┌─ 第一步：语句校对修正 ─────────────────────────┐
│  工具: pycorrector                              │
│  输入: "帮我读起个文件"（原始用户输入）          │
│  输出: "帮我读取个文件"（修正后的文本）          │
│  修正: 读起 → 读取                               │
└────────────────────────────────────────────────┘
    │
    ▼
┌─ 第二步：意图检测 ─────────────────────────────┐
│  工具: GLiClass                                 │

    │
    ▼

```

### 5.2 pycorrector（语句校对修正）

| **大小** | ~50MB（Kenlm 模型） |
| **延迟** | <10ms |

| **功能** | 中文文本纠错（音似、形似、语法错误） |

**输入输出**：

```python
from pycorrector import Corrector

corrector = Corrector()

# 输入：用户原始输入（可能有错别字）
result = corrector.correct("帮我读起这个文件")

# 输出：修正后的文本 + 修正记录
# result = ('帮我读取这个文件', ['读起→读取'])
```

**输入**: `str` - 用户原始输入文本
**输出**: `Tuple[str, List[str]]` - (修正后文本, 修正记录列表)

| 属性 | 说明 |
|------|------|
| **名称** | GLiClass |
| **安装** | `pip install gliclass` |
| **大小** | ~5MB（库） + ~500MB（模型，首次自动下载） |

**输入输出**：

```python

# 输出：每个标签的置信度
# result = {'labels': ['file', 'network', 'system'], 'scores': [0.85, 0.10, 0.05]}
```

**输出**: `Dict` - `{labels: [按置信度排序的标签], scores: [对应的置信度]}`

### 5.4 完整实现代码

```python
from pycorrector import Corrector
from gliclass import GLiClassPipeline

class UserInputProcessor:
    """用户输入预处理器"""
    
    def __init__(self):
        self.corrector = Corrector()
        self.classifier = GLiClassPipeline(
            model="knowledgator/gliclass-instruct-base-v1.0"
        )
    
    def process(self, user_input: str, intent_labels: list) -> dict:
        """
        处理用户输入
        
        输入：
            user_input: 用户原始输入文本（可能有错别字）
            intent_labels: 候选意图标签列表
        
        输出：
            {
                "original": 原始输入,
                "corrected": 修正后文本,
                "errors": 修正记录,
                "intent": 最佳意图,
                "confidence": 置信度,
            }
        """
        # 第一步：语句校对修正
        corrected, errors = self.corrector.correct(user_input)
        
        # 第二步：意图检测
        result = self.classifier(corrected, labels=intent_labels)
        best_label = result['labels'][0]
        best_score = result['scores'][0]
        
        return {
            "original": user_input,
            "corrected": corrected,
            "errors": errors,
            "intent": best_label,
            "confidence": best_score,
        }

# 使用示例
processor = UserInputProcessor()
result = processor.process(
    "帮我读起个文件",
    intent_labels=["file", "network", "system"]
)
# {
#   "original": "帮我读起个文件",
#   "corrected": "帮我读取个文件",
#   "errors": ["读起→读取"],
#   "intent": "file",
#   "confidence": 0.85
# }
```

## 六、意图识别层

### 6.1 接口定义

意图识别层接收预处理流水线输出的结果，进行意图确认和拆分：

```python
class IntentClassifier:
    """意图分类器（通用接口）"""
    
    def classify(self, preprocessed: dict, context: dict) -> List[Intent]:
        """
        基于预处理结果，确认意图类型
        
        输入：preprocessed（来自第五章预处理流水线）
            - preprocessed["corrected"]: 修正后文本
            - preprocessed["intent"]: GLiClass 初步检测的意图
            - preprocessed["confidence"]: GLiClass 置信度
        
        返回值：List[Intent]（支持多意图拆分）
        
        处理逻辑：
        1. 如果 GLiClass 置信度足够高，直接使用
        2. 如果置信度不足，尝试关键词匹配兜底
        3. 失败时 → 返回空列表（触发回退机制）
        """
        matched_intents = []
        confidence = preprocessed.get("confidence", 0)
        
        # 1. GLiClass 高置信度，直接使用
        if confidence >= 0.7:
            intent = registry.get(preprocessed["intent"])
            if intent:
                matched_intents.append(intent)
        
        # 2. 置信度不足，尝试关键词匹配兜底
        if not matched_intents:
            matched_intents = self._keyword_match(preprocessed["corrected"])
        
        # 3. 如果仍然失败，返回空列表（触发回退）
        return matched_intents
    
    def _keyword_match(self, text: str) -> List[Intent]:
        """关键词匹配兜底"""
        matched = []
        for intent in registry.list_all():
            if any(kw in text.lower() for kw in intent.keywords):
                matched.append(intent)
        return matched
```

### 6.2 回退机制

**核心原则**：Always have a fallback route。

```python
class MultiIntentProcessor:
    """多意图处理器（主流程）"""
    
    def process(self, user_input: str, context: dict) -> dict:
        """处理用户输入"""
        
        # 步骤1：获取候选意图标签（来自注册表）
        intent_labels = self.intent_registry.get_all_names()
        
        # 步骤2：预处理（第五章）
        preprocessed = self.preprocessor.process(user_input, intent_labels)
        
        # 步骤2：意图识别（本章）
        intents = self.classifier.classify(preprocessed, context)
        
        if not intents:
            # 回退：意图识别失败，走默认 ReAct 循环
            return self._fallback_react(user_input, context)
        
        # 步骤3：按顺序处理每个意图
        results = []
        for intent in intents:
            result = self._process_single_intent(intent, user_input, context)
            results.append(result)
        
        # 步骤4：合并结果
        return self._merge_results(results)
    
    def _process_single_intent(self, intent: Intent, user_input: str, context: dict) -> dict:
        """
        处理单个意图
        
        【重要】当前版本只支持 file 意图，其他意图返回"暂不支持"提示
        """
        # 【当前版本】只支持 file 意图
        if intent.name != "file":
            return {
                "intent": intent.name,
                "status": "unsupported",
                "summary": f"检测到 {intent.name} 意图，本版本暂不支持。{intent.description}功能将在后续版本升级支持，感谢您的耐心等待。",
                "data": None,
                "is_placeholder": True  # 标识为占位意图
            }
        
        # 【file 意图】正常执行 ReAct 循环
        tools = self.get_tools_for_intent(intent)
        return self._run_react_loop(user_input, tools, context, intent)
    
    def _fallback_react(self, user_input: str, context: dict) -> dict:
        """
        回退机制：意图识别失败时，走默认 ReAct 循环
        
        使用所有已注册意图的工具集，让 LLM 自己决定用哪个工具
        """
        all_tools = []
        for intent in registry.list_all():
            all_tools.extend(intent.tools)
        
        # 进入 ReAct 循环，使用全量工具
        return self._run_react_loop(user_input, all_tools, context)
```

## 七、Context/State 层

### 7.1 上下文传递

**问题**：意图信息如何传递给 ReAct 循环？

**答案**：通过 Context 对象。

```python
class ProcessingContext:
    """处理上下文（在各层之间传递）"""
    
    def __init__(self):
        self.user_input: str = ""              # 用户原始输入
        self.corrected_input: str = ""         # 修正后文本（第五章输出）
        self.detected_intents: List[Intent] = []  # 检测到的意图列表
        self.current_intent: Optional[Intent] = None  # 当前正在处理的意图
        self.intent_results: Dict[str, dict] = {}     # 各意图的处理结果
        self.safety_context: dict = {}        # 安全检查上下文
        self.execution_steps: list = []       # ReAct 执行步骤（现有）
        self.session_id: str = ""             # 会话ID
        self.message_id: str = ""             # 消息ID
```

### 7.2 Context 在各层的流转

```
用户输入
    │
    ▼
预处理流水线（第五章）：
│   context.user_input = 用户原始输入
│   context.corrected_input = 修正后文本
│
    ▼
意图识别层（第六章）：
│   context.detected_intents = [file, network]
│
    ▼
安全检查层：
│   context.safety_context = {file: {白名单: [...]}, network: {允许域名: [...]}}
│
    ▼
工具执行层：
│   context.current_intent = file
│   → ReAct 循环（使用 context.execution_steps）
│   context.intent_results["file"] = 执行结果
│
│   context.current_intent = network
│   → ReAct 循环
│   context.intent_results["network"] = 执行结果
│
    ▼
结果处理层：
│   context.intent_results → 合并所有意图的结果
│
    ▼
返回用户
```

## 八、安全检查层

### 8.1 两级安全检查

```python
class SafetyChecker:
    """安全检查器（通用接口）"""
    
    def check_intent_level(self, intent: Intent, context: ProcessingContext) -> bool:
        """
        意图级安全检查
        
        判断是否允许执行该类型的操作
        """
        if intent.name == "file":
            return self._check_file_intent_allowed(context)
        elif intent.name == "network":
            return self._check_network_intent_allowed(context)
        elif intent.name == "system":
            return self._check_system_intent_allowed(context)
        else:
            return False
    
    def check_param_level(self, intent: Intent, tool_name: str, params: dict) -> bool:
        """
        参数级安全检查
        
        在工具执行前检查具体参数
        """
        if intent.name == "file":
            return self._check_file_params(tool_name, params)
        elif intent.name == "network":
            return self._check_network_params(tool_name, params)
        else:
            return True
```

## 十、ReAct 循环中的意图处理

### 10.1 意图识别的位置

意图识别发生在 ReAct 循环**之前**：

```
用户输入
    │
    ▼
预处理流水线（第五章）
    │
    ▼
意图识别层（第六章）（判断 file/network/system/...）
    │
    ├── 匹配成功 → 加载对应意图的工具集
    ├── 匹配多个 → 按顺序处理每个意图
    └── 匹配失败 → 回退，使用全量工具进入 ReAct
    │
    ▼
进入 ReAct 循环（thought → action_tool → observation → ...）
    │
    ▼
使用该意图类型的工具执行
```

### 10.2 与 ReAct 字段的关系

| ReAct 字段 | 与意图的关系 |
|-----------|-----------|
| `thought` | 意图判断后的推理过程 |
| `action_tool` | 使用对应意图的工具 |
| `observation` | 工具执行结果 |
| `final` | 最终回答 |

```python
# backend/app/services/agent/preprocessing/intent_classifier.py
from gliclass import GLiClassPipeline

class IntentClassifier:
    """意图分类器"""
    
    def __init__(self):
        self.classifier = GLiClassPipeline(
            model="knowledgator/gliclass-instruct-base-v1.0"
        )
    
    def classify(self, text: str, labels: list) -> dict:
        """
        意图分类
        
        输入: 修正后的文本 + 候选意图标签
        输出: {intent: 最佳意图, confidence: 置信度}
        """
        result = self.classifier(text, labels=labels)
        return {
            "intent": result['labels'][0],
            "confidence": result['scores'][0],
            "all_intents": dict(zip(result['labels'], result['scores'])),
        }
```

```python
# backend/app/services/agent/intent/registry.py
from typing import Dict, List, Optional
from pydantic import BaseModel

class Intent(BaseModel):
    """意图定义"""
    name: str
    description: str
    keywords: List[str]
    tools: List[str]
    safety_checker: Optional[str] = None

class IntentRegistry:
    """意图注册表"""
    
    def __init__(self):
        self._intents: Dict[str, Intent] = {}
    
    def register(self, intent: Intent):
        self._intents[intent.name] = intent
    
    def get(self, name: str) -> Optional[Intent]:
        return self._intents.get(name)
    
    def list_all(self) -> List[Intent]:
        return list(self._intents.values())
    
    def get_all_names(self) -> List[str]:
        return list(self._intents.keys())
```

```python
# backend/app/services/agent/preprocessing/pipeline.py
from .corrector import TextCorrector
from .intent_classifier import IntentClassifier

class PreprocessingPipeline:
    """用户输入预处理流水线"""
    
    def __init__(self):
        self.corrector = TextCorrector()
        self.classifier = IntentClassifier()
    
    def process(self, user_input: str, intent_labels: list) -> dict:
        """
        处理用户输入
        
        输入: 用户原始输入 + 候选意图标签
        输出: {original, corrected, errors, intent, confidence}
        """
        # 第一步：语句校对修正
        corrected, errors = self.corrector.correct(user_input)
        
        # 第二步：意图检测
        intent_result = self.classifier.classify(corrected, intent_labels)
        
        return {
            "original": user_input,
            "corrected": corrected,
            "errors": errors,
            "intent": intent_result["intent"],
            "confidence": intent_result["confidence"],
            "all_intents": intent_result["all_intents"],
        }
```

```python
# backend/app/services/agent/agent.py（改造后）
from app.services.agent.preprocessing.pipeline import PreprocessingPipeline
from app.services.agent.intent.registry import IntentRegistry

class Agent:
    def __init__(self):
        self.preprocessor = PreprocessingPipeline()
        self.intent_registry = IntentRegistry()
    
    async def process_message(self, user_input: str) -> dict:
        """处理用户消息"""
        
        # 第一步：预处理（校对 + 意图检测）
        intent_labels = self.intent_registry.get_all_names()
        preprocessed = self.preprocessor.process(user_input, intent_labels)
        
        # 第二步：安全检查
        intent = self.intent_registry.get(preprocessed["intent"])
        if not self.safety_check(intent, preprocessed):
            return {"error": "安全检查未通过"}
        
        # 第三步：加载对应意图的工具集
        tools = self.get_tools_for_intent(intent)
        
        # 第四步：进入 ReAct 循环
        result = await self.run_react_loop(
            preprocessed["corrected"],  # 使用修正后的文本
            tools,
            preprocessed["intent"],      # 传递意图信息
        )
        
        return result
```

### 12.4 Agent.py 拆分建议

当前 `agent.py` 共 **1327 行**，承担了过多职责。分析如下：

| 当前问题 | 说明 |
|---------|------|
| **代码量过大** | 1327 行，难以维护和调试 |
| **职责混杂** | 数据结构、解析、执行、LLM 调用全在一起 |
| **难以测试** | 无法单独测试某个模块 |
| **难以扩展** | 新增 LLM 调用模式要改主文件 |

#### 12.4.1 拆分方案

**完整目录结构见 12.2 节**，此处仅详细说明 `agent/` 内部的拆分。

**当前问题分析**：

| 问题 | 说明 |
|------|------|
| **名称错误** | `FileOperationAgent` 名称误导——它不是"文件操作"，而是"运行 ReAct 循环" |
| **职责单一** | 它只硬编码了 file 意图的 ReAct 循环，无法处理其他意图 |
| **扩展困难** | 每个新意图都需要复制一个类似的 Agent（如 `DesktopOperationAgent`、`NetworkOperationAgent`） |

**解决思路**：重命名为 `IntentAgent`，让它成为**通用意图 Agent**，各意图共用同一套 ReAct 循环逻辑。

```
agent/
├── types/                           ← 【新增】数据结构目录
│   ├── __init__.py
│   ├── step_types.py               → Step/ThoughtStep/ActionToolStep/ObservationStep
│   └── result_types.py             → AgentResult
│
├── tool_parser.py                  → 【拆分】ToolParser 类（~128行）
│   ├── parse_response()            → 解析 LLM 响应
│   └── _extract_from_text()        → 从文本提取信息
│
├── tool_executor.py                → 【拆分】ToolExecutor 类（~146行）
│   ├── execute()                   → 执行工具调用
│   └── 参数映射逻辑                 → 参数名容错处理（path→file_path 等）
│
├── llm_strategies.py               → 【新增】LLM 调用策略（~172行）
│   ├── TextStrategy                → 文本模式：直接返回响应文本
│   ├── ToolsStrategy               → Function Calling 模式：通过 tools Schema 约束
│   └── ResponseFormatStrategy      → response_format 模式：通过 JSON Schema 约束
│
├── agent.py                        → 【改造】通用 IntentAgent（~922行 → ~300行）
│   ├── IntentAgent                 → 【改名】通用意图 Agent（非 file 专用）
│   │   构造参数：intent_type       → 决定加载哪套 tools/prompts/safety
│   │   ├── run()                  → 同步运行入口
│   │   ├── run_stream()           → 流式运行入口
│   │   ├── rollback()             → 回滚操作
│   │   └── get_execution_log()    → 获取执行日志
│   │
│   │   # 内部根据 intent_type 加载对应模块
│   │   if intent_type == "file":
│   │       self.tools = FileTools()
│   │       self.safety = FileSafety()
│   │       self.prompts = FilePrompts()
│   │   elif intent_type == "desktop":
│   │       self.tools = DesktopTools()
│   │       self.safety = DesktopSafety()
│   │       self.prompts = DesktopPrompts()
│   │   # network, system, database 同理
│   │
│   └── run_intent(intent_type, user_input)
│       → 入口方法：根据意图类型运行对应 ReAct 循环
│
└── prompts.py                      → 【保留】Prompt 模板（已独立）
```

**使用方式**（各意图共用同一个 Agent 类）：

```python
# file 意图
file_agent = IntentAgent(intent_type="file")
result = await file_agent.run(user_input)

# desktop 意图（file 完成后）
desktop_agent = IntentAgent(intent_type="desktop")
result = await desktop_agent.run(user_input)

# network 意图
network_agent = IntentAgent(intent_type="network")
result = await network_agent.run(user_input)
```

**关键设计**：一个 `IntentAgent` 类，通过 `intent_type` 参数决定加载哪套工具/安全检查/Prompt，各意图共用 ReAct 循环逻辑，无需为每个意图复制一个 Agent 类。
#### 12.4.2 各模块职责

| 模块文件 | 职责 | 依赖 |
|---------|------|------|
| `step_types.py` | 定义 ReAct 循环中的数据结构 | 无 |
| `result_types.py` | 定义 Agent 执行结果结构 | 无 |
| `tool_parser.py` | 解析 LLM 响应为结构化数据 | json, re |

| `agent.py` | **IntentAgent**：通用意图 Agent，根据 intent_type 加载对应工具/安全/Prompt | 上面所有模块 |
