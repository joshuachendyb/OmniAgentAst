## 一、改造目标

### 1.1 核心目标

| 目标 | 说明 |
|------|------|
| **目录重命名** | `file_operations/` → `agent/` |
| **新建预处理模块** | `preprocessing/` - 用户输入校对+意图检测 |
| **新建意图模块** | `intent/` - 意图注册+分类+拆分 |
| **拆分 agent.py** | 从1327行拆分为独立模块 |
| **按意图组织代码** | `tools/{intent}/`、`safety/{intent}/` |
| **迁移 file 专用代码** | tools.py、safety.py、prompts.py 等迁移到新结构 |

### 1.2 最终目录结构


## 四、执行原则

### 4.1 铁律

| 原则 | 说明 |
|------|------|
| **不丢功能** | 每步迁移后验证功能不变 |
| **不破坏逻辑** | 保持原有业务逻辑不变 |
| **边做边测** | 每步完成后运行测试验证 |
| **保持可用** | 迁移过程中 agent.py 始终可用 |

### 4.2 执行顺序

```
1. 第一步~第八步：新建框架（preprocessing + intent + agent.py 拆分）
2. 第九步~第十六步：迁移 file 专用代码
3. 第十七步：清理旧文件，收尾
```

### 4.3 验证标准

| 阶段 | 验证项 |
|------|--------|
| 每步完成 | 语法检查通过，测试通过 |
| 迁移完成 | 原有功能不受影响 |
| 全部完成 | 整体测试通过，E2E测试通过 |
