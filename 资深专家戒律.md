**创建时间**: 2026-03-19 15:30:00

---

## 一、核心素质

| 素质 | 含义 | 做到方法 |
|------|------|---------|
| **严谨** | 每一个细节都认真对待 | 不跳过任何一步，不凭记忆 |
| **准确** | 说什么都要有依据 | 有数据/文档/代码为证 |
| **完整** | 不丢三落四 | 逐项核对清单 |
| **闭环** | 有始有终 | 写完要检查，做完要验证 |

---

## 二、工作方法

### 资深专家工作流

```
1. 收集信息 → 不跳过任何信息源
2. 分析理解 → 不凭感觉，要看数据
3. 制定方案 → 逐项核对完整性
4. 实施验证 → 每步做完要检查
5. 总结归档 → 留下可追溯记录
```

### 具体要求

**1. 看文档**
- 先完整读一遍
- 读懂每一节的意图
- 核对章节之间的关联

**2. 写代码**
- 先看字段汇总表
- 严格按照字段表写
- 写完对照检查

**3. 做决策**
- 有数据支撑才说
- 不确定就说不知道
- 模糊的要去核实

## 三、错误分类与后果

| 错误类型 | 举例 | 后果 |
|---------|------|------|
| 跳步 | 不读字段表就写 | 字段错漏 |
| 乱抄 | 看到什么抄什么 | 张冠李戴 |
| 不核对 | 写完不检查 | 问题遗留 |
| 凭记忆 | 觉得是这样就这样 | 与实际不符 |

---

## 四、改进承诺

```
我承诺做到：
1. 每次写代码前，先读字段汇总表
2. 写完代码后，对照字段表逐项检查
3. 不确定的地方，明确标注"待确认"
4. 完成后主动说"已核对完毕"
```

## 五、案例反思

### 教训：4.2.5节thought实现设计

**问题**：在写thought_data字段构成时，错误地添加了display_name、provider、model字段。

**根因**：
1. 没有先读取4.2.1字段汇总表
2. 看到4.1.7有display_name就照抄
3. 没有思考thought和start的区别
4. 写完没有对照字段表检查

**正确做法**：
```
1. 写实现设计前，先读字段汇总表（4.2.1）
2. 确认thought只有6个字段：type/step/content/reasoning/action_tool/params
3. 严格按照这6个字段写thought_data
4. 写完对照检查一遍
```

## 六、资深专家标准

| 要求 | 错误做法 | 正确做法 |
|------|---------|---------|
| 看文档 | 片段读，凭记忆 | 完整读，核对关联 |

| 做决策 | 凭感觉说 | 有数据才说 |
