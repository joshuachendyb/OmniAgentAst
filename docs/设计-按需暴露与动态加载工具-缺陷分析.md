## 一、Mixin初始化缺陷

### 1.1 问题描述
`agent/mixins/react_agent_mixin.py` 406-549行设计中，`_call_llm_with_summary()`方法（499行）第一句：
```python
self.llm_call_count += 1  # ❌ 但_init_llm_strategies()中没初始化llm_call_count
```

**根因**：Mixin的`_init_llm_strategies()`（446-453行）没有初始化`self.llm_call_count = 0`，直接调用会报`AttributeError`。

**修复方案**：
```python
# react_agent_mixin.py:446-453 修正
def _init_llm_strategies(self):
    """初始化LLM调用策略"""
    self.llm_call_count = 0  # ✅ 补充初始化
    self.text_strategy = TextStrategy()
    self.adapter = None
    self.use_function_calling = False
    self.openai_tools = []
    self.tools_strategy = None
    self.response_format_strategy = None
```

## 二、动态加载置信度阈值无降级策略

### 2.1 问题描述
`react_schema.py` 1915-1917行：
```python
for intent, score in sorted_intents[:top_n]:
    if score < 0.3:  # ❌ 如果用户输入很模糊，所有意图都低于0.3，就什么都不加载了
        continue
```

**根因**：没有考虑“所有意图都低于阈值”的降级场景，导致初始无工具可用。

**修复方案**：
```python
# react_schema.py:1914-1917 修正
sorted_intents = sorted(intent_dist.items(), key=lambda x: x[1], reverse=True)

# 检查top_n个是否都低于阈值
top_scores = [score for _, score in sorted_intents[:top_n]]
if all(score < 0.3 for score in top_scores):
    # 降级：只加载最高置信度的1个意图
    top_n = 1
    if sorted_intents[0][1] < 0.1:  # 连最高的都低于0.1，给空清单
        return [], ""

for intent, score in sorted_intents[:top_n]:
    if score < 0.3:
        continue
```

## 三、简要Schema设计不一致

### 3.1 问题描述
**3.2.2节（1850-1865行）**说要生成简要Schema（只有name+一句话description）**但实际代码（1892-1940行）**只是把低置信度意图收集到`brief_intents`列表，然后添加到system prompt字符串中，**并没有生成简要Schema**！

**根因**：设计文档前后矛盾，前面的简要Schema设计被废弃了，但文档没说明。

**修复方案**：
1. **方案A（诚实）**：修改3.2.2节，说明实际采用“简要清单放入system prompt”方案
2. **方案B（实现）**：真正生成简要Schema：
```python
# react_schema.py 新增简要Schema生成
def _build_brief_schema(tool: Dict, score: float) -> Dict:
    """生成简要Schema（只有name+一句话）"""
    return {
        "type": "function",
        "function": {
            "name": tool["name"],
            "description": f"【简要】{tool.get('description', '')[:50]}... 置信度: {score:.2f}",
            "parameters": {"type": "object", "properties": {}, "required": []}
        }
    }
```

## 四、ShellReactAgent的rollback返回值误导

### 4.1 问题描述
`agent/shell_react.py` 819-821行：
```python
async def rollback(self, step_number=None) -> bool:
    return True  # ❌ Shell命令无法回滚，返回True会误导上层以为回滚成功了
```

**根因**：Shell命令执行后无法回滚，但返回`True`会让调用方认为回滚成功，导致状态不一致。

**修复方案**：
```python
# shell_react.py:819-821 修正
async def rollback(self, step_number=None) -> bool:
    logger.warning("[ShellReactAgent] Shell命令无法回滚，请手动检查系统状态")
    return False  # ✅ 更准确的返回值
```

## 五、FileReactAgent改进后两套工具并存

### 5.1 问题描述
`agent/file_react.py` 改进后（1536-1563行）：
```python
# Mixin加载
self._tools_dict = self.load_tools_by_category(effective_category)
# 保留原有
self.file_tools = FileTools(task_id=task_id)  # ❌ 两套工具并存
```

**根因**：Mixin已经通过`self._tools_dict`和`self.executor`管理工具，但还保留了原来的`self.file_tools`，导致两套工具并存，且`_execute_tool()`中用`self.executor.execute()`，但原来有些地方可能直接用`self.file_tools.xxx()`。

**修复方案**：
```python
# file_react.py 改进后
class FileReactAgent(ReactAgentMixin, BaseAgent):
    def __init__(self, ...):
        # ... Mixin初始化 ...
        # ✅ 删除 self.file_tools = FileTools(...)
        # 如果确实需要task_id追踪，在executor中处理
        # 或者：只保留用于需要task_id的写操作追踪
        self._task_id = task_id  # 简化：只存task_id
```

## 六、AgentFactory注册无fallback机制

### 6.1 问题描述
`agent/agent_factory.py` 1480-1531行注册代码：
```python
try:
    from app.services.agent.shell_react import ShellReactAgent
    AgentFactory.register('shell', ShellReactAgent, ToolCategory.SHELL)
except ImportError as e:
    print(f"[AgentFactory] ShellReactAgent: {e}")  # ❌ 只是print，没有fallback
```

**根因**：如果`prompts/shell/shell_prompts.py`或`agent/shell_react.py`不存在，import失败后只是print，没有fallback到基础Agent的机制，导致后续创建Agent失败。

**修复方案**：
```python
# agent_factory.py:1484-1488 修正
try:
    from app.services.agent.shell_react import ShellReactAgent
    AgentFactory.register('shell', ShellReactAgent, ToolCategory.SHELL)
except ImportError as e:
    print(f"[AgentFactory] ShellReactAgent: {e}")
    # ✅ fallback：使用基础BaseAgent
    from app.services.agent.base_react import BaseAgent
    AgentFactory.register('shell', BaseAgent, ToolCategory.SHELL)
```

## 七、动态加载关键词检测可能误触发

### 7.1 问题描述
`base_react.py` 2017-2034行：
```python
# 关键词检测
if any(kw in observation for kw in keywords):  # ❌ 可能误触发
    if intent not in self._loaded_categories:
        self.load_tools_by_intent(intent, reason=f"Observation关键词检测: {keywords}")
```

**根因**：用户说“不要执行shell命令”，结果因为包含“shell”就加载了。没有排除否定语境。

**修复方案**：
```python
# base_react.py:2017-2034 修正
trigger_map = {
    "shell": ["执行命令", "npm ", "pip ", "git ", "docker "],
    # ... 其他 ...
}

# 否定词检测
negative_words = ["不要", "别", "不需要", "禁止"]

for intent, keywords in trigger_map.items():
    if any(kw in observation for kw in keywords):
        # ✅ 检查是否是否定语境
        if any(nw in observation for nw in negative_words):
            continue  # 否定语境，不触发
        if intent not in self._loaded_categories:
            self.load_tools_by_intent(intent, reason=f"Observation关键词检测: {observation[:50]}...")
            return
```

## 八、Mixin的session管理与原有逻辑重复

### 8.1 问题描述
- Mixin提供了`_on_session_init()`和`_on_after_loop()`（534-548行）
- 但FileReactAgent原来已经有session管理逻辑（file_react.py:87-91, 351-377）

**根因**：如果FileReactAgent使用Mixin后又自己实现一遍，就会重复。文档1536-1563行说“改进后使用Mixin”，但没说明和原有session逻辑的取舍。

**修复方案**：
```python
# react_agent_mixin.py:455-458 修正
def _init_session(self, enable=True):  # ✅ 可关闭
    """初始化session管理"""
    if not enable:
        return
    self.session_service = get_session_service()
    self._session_created_by_agent = False
```

```python
# file_react.py 改进后
class FileReactAgent(ReactAgentMixin, BaseAgent):
    def __init__(self, ...):
        # ... 其他初始化 ...
        self._init_session(enable=True)  # ✅ 明确启用
```
