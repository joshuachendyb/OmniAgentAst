## 二、方案一：Jieba（分词+关键词）

### 2.1 基本信息

| 属性 | 说明 |
|------|------|
| **名称** | Jieba（结巴） |
| **安装** | `pip install jieba` |
| **大小** | ~15MB |
| **延迟** | 10-50ms |
| **Stars** | 33K+ (GitHub) |
| **特点** | 最流行的中文分词工具 |

### 2.2 核心功能与返回结果

#### 2.2.1 分词（4种模式）

| 模式 | 函数 | 返回类型 | 说明 |
|------|------|---------|------|
| **精确模式** | `jieba.cut(text)` | `Generator[str]` | 最精确的切分，适合文本分析 |
| **全模式** | `jieba.cut(text, cut_all=True)` | `Generator[str]` | 扫描所有可能的词，速度快但有歧义 |
| **搜索引擎模式** | `jieba.cut_for_search(text)` | `Generator[str]` | 在精确模式基础上再切分长词 |
| **paddle 模式** | `jieba.cut(text, use_paddle=True)` | `Generator[str]` | 深度学习模型，需安装 paddlepaddle-tiny |

**精确模式返回示例**：
```python
import jieba
words = jieba.cut("帮我读取 D:/test.txt 文件")
# Generator → ["帮", "我", "读取", " ", "D", ":", "/", "test", ".", "txt", " ", "文件"]

# 转为 list
words_list = jieba.lcut("帮我读取 D:/test.txt 文件")
# List[str] → ["帮", "我", "读取", " ", "D", ":", "/", "test", ".", "txt", " ", "文件"]
```

#### 2.2.2 关键词提取

| 函数 | 返回类型 | 说明 |
|------|---------|------|
| `jieba.analyse.extract_tags(text, topK=20)` | `List[str]` | TF-IDF 关键词列表 |
| `jieba.analyse.extract_tags(text, topK=20, withWeight=True)` | `List[Tuple[str, float]]` | 带权重的关键词列表 |
| `jieba.analyse.textrank(text, topK=20)` | `List[str]` | TextRank 关键词列表 |
| `jieba.analyse.textrank(text, topK=20, withWeight=True)` | `List[Tuple[str, float]]` | 带权重的关键词列表 |

**返回示例**：
```python
import jieba.analyse

# 不带权重
keywords = jieba.analyse.extract_tags("帮我读取 D:/test.txt 文件", topK=3)
# List[str] → ["test.txt", "读取", "文件"]

# 带权重
keywords_with_weight = jieba.analyse.extract_tags("帮我读取 D:/test.txt 文件", topK=3, withWeight=True)
# List[Tuple[str, float]] → [("test.txt", 0.85), ("读取", 0.72), ("文件", 0.65)]
```

**返回示例**：
```python

words = pseg.cut("我爱北京天安门")
```

### 2.3 意图检测实现

```python
import jieba
import jieba.analyse

class JiebaIntentDetector:
    """基于 Jieba 的意图检测器"""
    
    def __init__(self):
        # 意图关键词映射
        self.intent_keywords = {
            "file": ["文件", "读取", "写入", "删除", "移动", "目录", "搜索文件", 
                     "复制", "粘贴", "创建", "重命名", "备份"],
            "network": ["搜索", "下载", "http", "网址", "网络", "请求", 
                        "API", "访问", "打开网页"],
            "system": ["进程", "CPU", "内存", "剪贴板", "系统", "关机", 
                       "重启", "音量", "亮度"],
            "code": ["代码", "执行", "python", "运行", "脚本", "计算"],
        }
        
        # 加载自定义词典（可选）
        # jieba.load_userdict("user_dict.txt")
    
    def detect(self, text: str) -> dict:
        """意图检测"""
        # Step 1: 分词
        words = list(jieba.cut(text))
        
        # Step 2: 关键词提取（TF-IDF）
        keywords = jieba.analyse.extract_tags(text, topK=5)
        
        # Step 3: 计算各意图得分
        scores = {}
        for intent, intent_kws in self.intent_keywords.items():
            score = 0
            # 精确匹配
            for kw in intent_kws:
                if kw in text:
                    score += 2  # 精确匹配权重高
            # 分词匹配
            for word in words:
                if word in intent_kws:
                    score += 1
            # 关键词匹配
            for kw in keywords:
                if kw in intent_kws:
                    score += 1.5
            scores[intent] = score
        
        # Step 4: 选择最佳意图
        if not scores or max(scores.values()) == 0:
            return {"intent": "unknown", "confidence": 0, "words": words, "keywords": keywords}
        
        best_intent = max(scores, key=scores.get)
        confidence = scores[best_intent] / (sum(scores.values()) + 0.01)
        
        return {
            "intent": best_intent,
            "confidence": round(confidence, 2),
            "words": words,
            "keywords": keywords,
        }

# 使用示例
detector = JiebaIntentDetector()
result = detector.detect("帮我读取 D:/test.txt 文件")
print(result)
# {'intent': 'file', 'confidence': 0.85, 'words': ['帮', '我', '读取', ' ', 'D', ':', '/', 'test', '.', 'txt', ' ', '文件'], 'keywords': ['test.txt', '读取', '文件']}
```

### 2.4 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 最轻量（~15MB） | ❌ 只做分词，没有内置意图分类 |
| ✅ 最快（10-50ms） | ❌ 需要自己写关键词映射逻辑 |
| ✅ 支持自定义词典 | ❌ 语义理解能力弱 |
| ✅ 关键词提取（TF-IDF/TextRank） | ❌ 处理复杂表达时准确率低 |
| ✅ 词性标注 | ❌ 不支持模糊匹配 |
| ✅ 社区最大（33K Stars） | - |

### 3.3 意图检测实现

```python
from snownlp import SnowNLP

class SnowNLPIntentDetector:
    """基于 SnowNLP 的意图检测器"""
    
    def __init__(self):
        # 关键词映射（与 Jieba 类似）
        self.intent_keywords = {
            "file": ["文件", "读取", "写入", "删除", "移动", "目录", "搜索文件"],
            "network": ["搜索", "下载", "http", "网址", "网络", "请求"],
            "system": ["进程", "CPU", "内存", "剪贴板", "系统"],
            "code": ["代码", "执行", "python", "运行"],
        }
    
    def detect(self, text: str) -> dict:
        """意图检测"""
        s = SnowNLP(text)
        
        # Step 1: 分词
        words = s.words
        
        # Step 2: 关键词提取（TextRank）
        keywords = s.keywords(5)
        
        # Step 3: 计算各意图得分
        scores = {}
        for intent, intent_kws in self.intent_keywords.items():
            score = 0
            for kw in intent_kws:
                if kw in text:
                    score += 2
            for word in words:
                if word in intent_kws:
                    score += 1
            for kw in keywords:
                if kw in intent_kws:
                    score += 1.5
            scores[intent] = score
        
        if not scores or max(scores.values()) == 0:
            return {"intent": "unknown", "confidence": 0, "sentiment": s.sentiments}
        
        best_intent = max(scores, key=scores.get)
        confidence = scores[best_intent] / (sum(scores.values()) + 0.01)
        
        return {
            "intent": best_intent,
            "confidence": round(confidence, 2),
            "sentiment": s.sentiments,  # 额外：情感分析
            "keywords": keywords,
        }

# 使用示例
detector = SnowNLPIntentDetector()
result = detector.detect("帮我读取 D:/test.txt 文件")
print(result)
```

### 3.4 SnowNLP 文本分类功能（关键）

SnowNLP 内置了**朴素贝叶斯文本分类器**，可以自定义训练：

```python
train_data = {

        "创建一个新文件夹",
        "搜索所有txt文件",
    ],
    "network": [
        "帮我搜索天气",
        "下载这个文件",
        "访问这个网址",
        "发送HTTP请求",
```

### 4.4 混合方案（关键词 + LLM）

```python
from sage_intent import IntentClassifier, KeywordIntentRecognizer, LLMIntentRecognizer

# 创建混合分类器
classifier = IntentClassifier(
    recognizers=[
        KeywordIntentRecognizer(catalog),     # 快速路径
        LLMIntentRecognizer(catalog, llm_client),  # 准确路径
    ],
    strategy="cascade"  # 级联：先关键词，失败再LLM
)

# 分类
intent = classifier.classify("帮我把这个文件的内容复制到剪贴板")

```

### 4.5 三种策略

| `confidence` | 选择置信度最高的 | 需要快速决策 |
| `cascade` | 先快后慢级联 | **推荐**，兼顾速度和准确率 |

## 五、三方案对比

### 5.0 核心事实

**第一：三个库的核心能力是分词，不能直接返回意图检测结果。**

```
三个库的实际能力：
├── Jieba:       分词 → List[str]      （需要自己匹配关键词、自己算置信度）
├── SnowNLP:     分词 → List[str]      （需要自己匹配关键词、自己算置信度）
└── sage-intent: 分词+关键词匹配封装   （内置匹配逻辑，但仍需自己定义关键词）

意图检测的完整流程：
分词 → 关键词匹配 → 置信度计算 → 返回意图
  ↑         ↑            ↑            ↑
三个库都能做  都需自己写   都需自己写   都需自己写
                           (sage-intent内置)
```

**第二：不管用哪个方案，关键词都是自己定义的。**

三个方案在"关键词匹配"这件事上没有本质区别，区别在于工具的封装程度和附加功能。

```
用户输入: "帮我读取 D:/test.txt 文件"
    │
    ▼
分词: ["帮", "我", "读取", "D", ":", "/", "test", ".", "txt", "文件"]
    │
    ▼
关键词匹配: 在分词结果中查找预定义的关键词
    ├── 找到"读取" → 命中 file
    └── 找到"文件" → 命中 file
    │
    ▼
返回意图: file

三个方案都是这个流程，区别只在于：
- Jieba：分词最好，关键词匹配需要自己写
- SnowNLP：分词一般，可训练文本分类（需要训练数据）
- sage-intent：封装了关键词管理和匹配逻辑，开箱即用
```

### 5.1 基础指标对比

| 指标 | Jieba | SnowNLP | sage-intent |
|------|-------|---------|-------------|
| **安装大小** | ~15MB | ~5MB | ~5MB |
| **延迟** | 10-50ms | 50-100ms | <100ms（关键词） |
| **Stars** | 33K+ | 6.6K+ | 新项目（126 downloads/月） |
| **功能定位** | 分词 | 中文NLP | 意图识别 |
| **关键词定义** | 自己写 dict | 自己写 dict | catalog.add_intent() |
| **关键词匹配逻辑** | 自己实现 | 自己实现 | 内置 |
| **LLM 集成** | ❌ 无 | ❌ 无 | ✅ 有（需额外 API） |
| **训练能力** | ❌ 不支持 | ✅ 朴素贝叶斯 | ❌ 不支持 |

### 5.2 意图检测能力对比（修正版）

**注意：三个库的返回结果都是分词结果（List[str]），不是意图检测结果。意图检测需要在分词基础上自己实现。**

| 能力 | Jieba | SnowNLP | sage-intent |
|------|-------|---------|-------------|
| **分词** | ✅ List[str] | ✅ List[str] | ❌ 无（依赖其他库） |
| **关键词提取** | ✅ List[str]（TF-IDF/TextRank） | ✅ List[str]（TextRank） | ❌ 无 |
| **词性标注** | ✅ List[Tuple[str,str]] | ✅ List[Tuple[str,str]] | ❌ 无 |
| **关键词定义** | ❌ 自己写 dict | ❌ 自己写 dict | ✅ IntentCatalog 封装 |
| **关键词匹配** | ❌ 自己实现 | ❌ 自己实现 | ✅ 内置 |
| **置信度计算** | ❌ 自己算 | ❌ 自己算 | ✅ 内置 |
| **LLM 语义理解** | ❌ 需自己集成 | ❌ 需自己集成 | ✅ 内置（需 API） |
| **文本分类训练** | ❌ 不支持 | ✅ 朴素贝叶斯 | ❌ 不支持 |
| **级联回退** | ❌ 需自己实现 | ❌ 需自己实现 | ✅ 内置 |

**关键区别总结**：
- **Jieba**：分词最好（4种模式）→ 但意图检测全要自己写
- **SnowNLP**：最小最简单 → 可训练文本分类（需要标注数据）
- **sage-intent**：封装最好 → 专门做意图（但仍需自己定义关键词）

### 5.3 集成难度对比

| 维度 | Jieba | SnowNLP | sage-intent |
|------|-------|---------|-------------|
| **代码量** | ~100行 | ~100行 | ~30行 |
| **依赖复杂度** | 0（纯Python） | 0（纯Python） | 0（基础）/ 需LLM（高级） |
| **与现有架构冲突** | 无 | 无 | 无 |

## 六、推荐方案

### 6.1 推荐矩阵

| 场景 | 推荐 | 原因 |
|------|------|------|
| **最轻量、最快** | Jieba | 10-50ms，15MB，分词+关键词提取 |
| **最简单集成** | SnowNLP | 5MB，开箱即用，功能丰富 |
| **专门做意图** | sage-intent | 内置意图识别，混合方案 |
| **50-70% 准确率** | Jieba 或 SnowNLP | 关键词匹配即可达到 |
| **70-85% 准确率** | sage-intent（混合模式） | 关键词+LLM |

### 6.2 最终推荐

**对于 OmniAgent 的当前需求（50-70% 准确率，轻量级）**：

| 优先级 | 推荐方案 | 理由 |
|--------|---------|------|
| **首选** | **Jieba** | 最轻量、最快、社区最大、功能够用 |
| 备选 | SnowNLP | 如果需要文本分类训练能力 |
| 未来 | sage-intent | 如果需要 LLM 混合方案 |

### 6.3 实施建议

```
├── 实现 JiebaIntentDetector 类
├── 配置意图关键词映射
└── 达到 50-70% 准确率

第二步（以后）：需要更高准确率时
├── 升级到 SnowNLP（自定义训练）
└── 或升级到 sage-intent（LLM混合）
```
