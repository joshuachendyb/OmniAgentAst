## 一、现状分析

### 1.1 现有错误提示分布

| 文件 | 错误类型 | 问题 |
|------|---------|------|
| `chatMessages.ts` | 成功/错误/警告提示 | 函数分散，未统一 |
| `sse.ts` | SSE连接错误 | 独立处理，风格各异 |
| `api.ts` | API响应错误 | 拦截器处理，分散 |
| `NewChatContainer.tsx` | 31处catch块 | 直接调用message，风格不统一 |

### 1.2 现有问题

1. **风格不统一** - 有的用duration:3，有的用5，有的没有
2. **位置分散** - catch块里直接调用message.error()
3. **重试逻辑分散** - 每个地方自己实现重试
4. **提示文案不统一** - 类似的错误，提示文字不同
5. **遗漏特殊场景** - 危险操作、安全降级等未覆盖

## 二、需要统一处理的前端错误类型

### 2.1 核心错误类型（已确认）

| 序号 | 错误类型 | 代码标识 | 处理方式 |
|------|---------|---------|---------|
| 1 | 连接被拒绝 | CONNECTION_REFUSED | 提示+重连3次+指数退避 |
| 2 | 请求超时 | REQUEST_TIMEOUT | 提示+重连3次 |
| 3 | 服务器500 | SERVER_500 | 提示+停止+建议刷新 |
| 4 | 网络断开 | NETWORK_ERROR | 提示+重连3次 |
| 5 | 存储满了 | QUOTA_EXCEEDED | 提示+清理建议 |
| 6 | 会话冲突 | SESSION_CONFLICT | 提示+刷新页面 |
| 7 | 认证失败 | AUTH_401/403 | 提示+跳转配置 |
| 8 | 请求取消 | REQUEST_ABORT | **静默处理** |
| 9 | 组件卸载 | COMPONENT_UNMOUNTED | **静默处理** |

### 2.2 业务场景错误（v1.2新增）

| 序号 | 错误类型 | 代码标识 | 处理方式 |
|------|---------|---------|---------|
| 10 | 存储满了 | STORAGE_FULL | 提示+清理建议 |
| 11 | 危险操作拦截 | DANGEROUS_OPERATION | 提示+删除消息+记录日志 |
| 12 | 内容过长 | CONTENT_TOO_LONG | 提示精简内容 |
| 13 | 会话不存在 | SESSION_NOT_FOUND | 提示+返回 |
| 14 | 危险确认失败 | DANGER_CONFIRM_FAIL | 提示确认失败 |
| 15 | 安全服务异常 | SECURITY_SERVICE_DOWN | warning+降级普通模式 |
| 16 | 用户消息保存失败 | USER_MESSAGE_SAVE_FAILED | error提示+继续执行 |
| 17 | 重复点击 | DUPLICATE_CLICK | 提示"处理中"+忽略 |

### 2.3 不需要处理的错误

| 错误类型 | 原因 |
|---------|------|
| 后端发送的 `type=error` | 原样显示即可 |
| 用户主动取消(AbortError) | 静默处理 |
| 组件卸载后的异步错误 | 静默处理 |

## 三、UI一致性规范

### 3.1 提示样式标准

```typescript
const UI_CONFIG = {
  error: { duration: 5, maxCount: 1 },      // 错误提示5秒
  warning: { duration: 3, maxCount: 3 },    // 警告提示3秒
  success: { duration: 2, maxCount: 1 },    // 成功提示2秒
  info: { duration: 3, maxCount: 3 },        // 信息提示3秒
  loading: { duration: 0, maxCount: 1 },    // 不消失loading
};
```

### 3.2 提示文案规范

| 场景 | 格式 |
|------|------|
| 错误提示 | "操作失败：具体原因，请检查/重试/刷新" |
| 警告提示 | "正在重试 (X/3)..." 或 "请注意：具体事项" |
| 成功提示 | "操作成功" 或 "已完成" |
| 信息提示 | "提示：具体信息" |
| loading提示 | duration:0，手动关闭 |

### 3.3 错误去重规范

```typescript
const ERROR_DEDUP_WINDOW = 30000; // 30秒内不重复提示
const recentErrors = new Map<string, number>();
```

### 3.4 API与SSE处理差异

| 维度 | API错误处理 | SSE错误处理 |
|------|------------|------------|
| 模式 | 请求-响应 | 长连接 |
| 重试策略 | 失败立即提示，可手动重试 | 自动重连+指数退避 |
| 提示时机 | 立即显示 | 连接断开后延迟显示 |
| 恢复检测 | 手动刷新 | 自动检测恢复 |

## 六、统一错误处理中心设计

### 6.1 文件结构

```
src/utils/errorHandler.ts  ← 统一错误处理中心
```

### 6.2 核心模块

```
errorHandler.ts
    │
    ├── ErrorType          → 错误类型枚举（28种）
    ├── ErrorConfig        → 错误配置接口
    ├── ERROR_CONFIG_MAP  → 错误配置映射表
    │
    ├── classifyError()    → 分类错误
    ├── handleError()        → 统一处理入口
    ├── handleApiError()   → API错误处理
    ├── handleSSEError()   → SSE错误处理
    ├── handleReconnect() → 重连处理
    ├── showMessage()      → 统一显示
    │
    └── 去重与静默处理
          ├── shouldShowError()  → 判断是否显示
          ├── isSilentError()      → 静默错误判断
          └── dedupErrors()       → 错误去重
```

### 6.3 统一入口函数

```typescript
/**
 * 统一错误处理入口
 * @param error - 原始错误对象
 * @param context - 错误上下文(source/onRetry/continue)
 */
handleError(error: any, context: ErrorContext): ActionResult

interface ActionResult {
  handled: boolean;
  shouldContinue?: boolean;  // 保存失败但继续执行
  deleteMessage?: boolean;  // 危险操作需删除
  fallbackMode?: string;   // 降级模式
}
```

### 6.4 特殊处理示例

```typescript
// 危险操作拦截
if (errorType === 'DANGEROUS_OPERATION') {
  return { handled: true, deleteMessage: true };
}

// 安全服务降级
if (errorType === 'SECURITY_SERVICE_DOWN') {
  return { handled: true, fallbackMode: 'normal' };
}

// 用户消息保存失败但继续
if (errorType === 'USER_MESSAGE_SAVE_FAILED') {
  return { handled: true, shouldContinue: true };
}

// 重复点击
if (errorType === 'DUPLICATE_CLICK') {
  return { handled: true, shouldContinue: false };  // 忽略
}
```
