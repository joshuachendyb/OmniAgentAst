## 一、现状分析

### 1.1 现有错误提示分布

| 文件 | 错误类型 | 问题 |
|------|---------|------|
| `chatMessages.ts` | 成功/错误/警告提示 | 函数分散，未统一 |
| `sse.ts` | SSE连接错误 | 独立处理，风格各异 |
| `api.ts` | API响应错误 | 拦截器处理，分散 |
| `NewChatContainer.tsx` | 20+处catch块 | 直接调用message，风格不统一 |

### 1.2 现有问题

1. **风格不统一** - 有的用duration:3，有的用5，有的没有
2. **位置分散** - catch块里直接调用message.error()
3. **重试逻辑分散** - 每个地方自己实现重试
4. **提示文案不统一** - 类似的错误，提示文字不同
5. **缺少入口函数** - 没有统一的错误处理入口

### 3.2 提示文案规范

| 场景 | 格式 |
|------|------|
| 错误提示 | "操作失败：具体原因，请检查/重试/刷新" |
| 警告提示 | "正在重试 (X/3)..." 或 "请注意：具体事项" |
| 成功提示 | "操作成功" 或 "已完成" |
| 信息提示 | "提示：具体信息" |

### 3.3 错误去重规范（v1.1新增）

```typescript
// 相同错误在30秒内不重复提示
const ERROR_DEDUP_WINDOW = 30000; // 30秒

// 记录最近错误：{ "connection_refused": 1700000000000 }
```

### 3.4 API与SSE处理差异（v1.1新增）

| 维度 | API错误处理 | SSE错误处理 |
|------|------------|------------|
| 模式 | 请求-响应 | 长连接 |
| 重试策略 | 失败立即提示，可手动重试 | 自动重连+指数退避 |
| 提示时机 | 立即显示 | 连接断开后延迟显示 |
| 恢复检测 | 手动刷新 | 自动检测恢复 |
| 最大重试 | 3次后停止 | 3次后进入"连接失败"状态 |

### 3.5 用户体验规范

1. **错误带操作建议** - 不仅说错了，还说怎么办
2. **重试显示进度** - "正在重试 (1/3)..."
3. **关键错误不重复** - 相同错误30秒内不重复提示
4. **自动恢复提示** - 成功后自动消失
5. **静默处理场景** - 用户取消、组件卸载后错误不弹窗

### 4.4 错误配置

```typescript
interface ErrorConfig {
  // 是否可重试
  retryable: boolean;
  // 最大重试次数
  maxRetries: number;
  // 重试延迟(毫秒)
  retryDelay: number;
  // 指数退避基数

  // 错误提示消息

  // 错误描述(用户操作建议)
  description?: string;
  // 严重级别
  severity: "critical" | "warning" | "info";
  // 是否静默处理（不弹窗）[v1.1新增]
  silent?: boolean;
  // 去重时间窗口(毫秒)[v1.1新增]
  dedupWindow?: number;
}
```

### 4.5 统一入口函数

```typescript
/**
 * 统一错误处理入口
 * @param error - 原始错误对象
 * @param context - 错误上下文(API/SSE/业务)
 * @param options - 处理选项
 */
handleError(error, context, options?)

// 使用示例
try {
  await apiCall();
} catch (err) {
  handleError(err, { 
    context: 'api', 
    source: 'getSessionMessages',
    onRetry: retryCallback 
  });
}

/**
 * 静默错误判断（v1.1新增）
 * 某些错误不应显示给用户
 */
const isSilentError = (error: any): boolean => {
  // 用户主动取消
  if (error.name === 'AbortError') return true;
  // 组件已卸载
  if (error.message?.includes('component')) return true;
  // 重复请求
  if (error.code === 'DUPLICATE_REQUEST') return true;
  return false;
};

/**
 * 错误去重判断（v1.1新增）
 * 相同错误在时间窗口内不重复显示
 */
const shouldShowError = (errorType: string): boolean => {
  const lastShown = recentErrors.get(errorType);
  const now = Date.now();
  if (lastShown && (now - lastShown) < ERROR_DEDUP_WINDOW) {
    return false; // 30秒内不重复显示
  }
  recentErrors.set(errorType, now);
  return true;
};
```

### 4.6 API与SSE错误处理分离（v1.1新增）

```typescript
/**
 * API错误处理 - 请求响应模式
 * 特点：失败立即提示，可手动重试
 */
const handleApiError = (error: any, context: ErrorContext) => {
  const errorType = classifyError(error);
  const config = ERROR_CONFIG_MAP[errorType];
  
  // 静默错误不处理
  if (config.silent || isSilentError(error)) {
    return;
  }
  
  // 去重检查
  if (!shouldShowError(errorType)) {
    return;
  }
  
  // 立即显示错误
  message.error({
    content: config.message,
    duration: UI_CONFIG.error.duration,
  });
  
  // 可手动重试
  if (config.retryable && context.onRetry) {
    showRetryButton(context.onRetry);
  }
};

/**
 * SSE错误处理 - 长连接模式
 * 特点：自动重连，指数退避
 */
const handleSSEError = (error: any, context: ErrorContext) => {
  const errorType = classifyError(error);
  const config = ERROR_CONFIG_MAP[errorType];
  
  // 静默错误不处理
  if (config.silent || isSilentError(error)) {
    return;
  }
  
  // 去重检查
  if (!shouldShowError(errorType)) {
    return;
  }
  
  // 显示警告（连接断开）
  message.warning(config.message);
  
  // 自动重连
  if (config.retryable) {
    handleReconnect({
      maxRetries: config.maxRetries,
      retryDelay: config.retryDelay,
      backoff: config.retryBackoff,
      onRetry: context.onRetry,
      onStop: () => {
        // 全部失败后显示错误
        message.error(config.message + '，请刷新页面重试');
      }
    });
  }
};
```

## 八、重试次数配置设计

### 8.1 重试次数分类

| 错误类型 | 重试次数 | 重试延迟 | 适用场景 |
|---------|----------|---------|---------|
| **网络类** | | | |
| CONNECTION_REFUSED | 3次 | 1000ms | 后端未运行 |
| CONNECTION_RESET | 3次 | 1000ms | 连接被重置 |
| NETWORK_ERROR | 2次 | 1500ms | 网络不稳定，2次就够了 |
| WEAK_NETWORK | 1次 | 2000ms | 弱网只重试1次，避免频繁 |
| **超时类** | | | |
| REQUEST_TIMEOUT | 3次 | 1000ms | 请求等待超时 |
| IDLE_TIMEOUT | 2次 | 500ms | 空闲超时，快速重试 |
| **服务器类** | | | |
| SERVER_500 | 0次 | - | 服务器内部错误，不重试 |
| SERVER_502 | 2次 | 2000ms | 网关错误，可重试 |
| SERVER_503 | 2次 | 2000ms | 服务不可用，可重试 |
| **认证类** | | | |
| AUTH_401 | 0次 | - | 未授权，不重试 |
| AUTH_403 | 0次 | - | 禁止访问，不重试 |
| **业务类** | | | |
| SESSION_CONFLICT | 0次 | - | 会话冲突，重试会继续冲突 |
| VERSION_CONFLICT | 0次 | - | 版本冲突，不重试 |
| DUPLICATE_REQUEST | 0次 | - | 重复请求，不重试 |
| **存储类** | | | |
| STORAGE_FULL | 0次 | - | 存储满了，重试无效 |

| **操作类** | | | |
| SAVE_FAILED | 3次 | 1000ms | 保存失败，可重试 |
| LOAD_FAILED | 2次 | 1000ms | 加载失败，可重试 |

### 8.2 重试配置速查表

```
总结：
- 需要重试的：网络类(2-3次)、超时类(2-3次)、部分服务器(502/503)、操作类(2-3次)
- 不需要重试的：服务器500、认证错误、业务冲突、存储满了、静默类
- 弱网特殊处理：只重试1次，避免频繁失败
```
