## 二、分类方法详解

### 2.1 分类类型

函数将用户消息分为两类：

| 类型 | 置信度阈值 | 说明 |
|------|-----------|------|
| **文件操作** | confidence >= 0.3 | 需要调用FileTools执行文件操作 |
| **普通对话** | confidence < 0.3 | 简单的问答、知识查询等 |

### 2.2 支持的操作类型

| 操作类型 | 关键词示例 | 说明 |
|---------|-----------|------|
| **read** | 读、查看、打开、看、显示 | 读取文件内容 |
| **write** | 写、创建、新建、编辑、保存 | 创建或修改文件 |
| **list** | 列、列出、显示目录 | 列出目录内容 |
| **delete** | 删除、移除、清空 | 删除文件 |
| **move** | 移动、重命名、改名 | 移动或重命名文件 |
| **search** | 搜索、查找、查询 | 搜索文件 |

### 2.3 置信度计算方法

```python
# 计算每个意图的匹配得分
best_intent = None
best_score = 0.0

for intent, config in intent_patterns.items():
    score = 0.0
    matched_keywords = []
    
    for keyword in config["keywords"]:
        if keyword in message_lower:
            # 完整词匹配得分更高
            if keyword in message_lower.split() or len(keyword) >= 6:
                score += 0.3
            else:
                score += 0.2
            matched_keywords.append(keyword)
    
    # 应用权重
    score *= config["weight"]
    
    # 如果有多个关键词匹配，增加置信度
    if len(matched_keywords) >= 2:
        score += 0.2
    
    # 如果包含文件路径特征，增加置信度
    if any(char in message for char in ['/', '\\', '.txt', '.md', '.py', '.json']):
        score += 0.1
    
    if score > best_score:
        best_score = score
        best_intent = intent
```

## 五、现有方法的问题

### 5.1 准确率问题

- **误报**：某些非文件操作的请求被错误识别为文件操作
- **漏报**：某些文件操作请求因缺少关键词而被遗漏
- **置信度不稳定**：相似内容的消息可能得到不同的置信度

### 5.2 关键词覆盖问题

- **新词汇**：无法识别新出现的文件操作相关词汇
- **上下文**：无法理解上下文和语义
- **歧义**：无法处理有歧义的表达

### 5.3 维护问题

- **规则复杂**：关键词和规则越来越多，难以维护
- **扩展性差**：添加新类型需要修改代码
- **可解释性差**：难以解释分类结果
