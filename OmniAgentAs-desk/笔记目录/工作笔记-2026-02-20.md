## 16 修复CRSS黑名单过度拦截问题 【小沈】2026-02-20 13:06:20

### 16.1 问题背景

用户反馈：以下命令应该返回不同风险等级，但实际都返回了10分（CRITICAL）：
- `del *.tmp`: 期望6分（MEDIUM），实际10分
- `rmdir src/`: 期望8分（HIGH），实际10分
- `rm -rf temp/`: 期望8分（HIGH），实际10分

### 16.2 问题分析

通过调试发现三个问题：

1. **黑名单子串匹配过度**：
   - 使用 `if dangerous in command_lower` 进行简单子串匹配
   - 导致 `'rm -rf /'` 匹配了 `'rm -rf /bin'`、`'rmdir /'` 匹配了 `'rmdir src/'`

2. **正则表达式模式过于宽泛**：
   - `r'rm\s+-[rf]+\s+[/\\*]+'` 匹配任何 `rm -rf /xxx`
   - `r'rmdir\s+[/\\*]+'` 匹配任何 `rmdir /xxx`

3. **SCOPE_PATTERNS 误判**：
   - `del *.tmp` 被识别为 CROSS_DIR（因为包含 `*`）
   - 影响范围系数过大导致评分偏高

### 16.3 修复方案

1. **改进黑名单匹配逻辑**：使用正则表达式单词边界 `\b` 进行精确匹配

2. **修复危险命令模式**：只匹配根目录和系统目录，不匹配普通目录

3. **修复 SCOPE_PATTERNS**：只匹配明确的全盘操作，不匹配普通通配符

4. **调整 CRSS 评分参数**：
   - TEMP 权重：1 → 3
   - PROJECT 权重：7 → 3-4
   - SINGLE_FILE 系数：1.0 → 1.1
   - DIRECTORY 系数：1.5 → 1.45
   - 使用 round() 替代 int() 进行四舍五入

### 2.2 修改2：时间格式为UTC（已完成）

**文件**：`backend/app/api/v1/sessions.py`

**修改内容**：
1. 导入 `timezone`

3. 新增 `_convert_to_utc()` 函数

**新增函数**：
```python
from datetime import datetime, timezone

def get_utc_timestamp() -> str:
    """获取UTC时间戳，ISO格式"""
    return datetime.now(timezone.utc).isoformat().replace("+00:00", "Z")

def _convert_to_utc(time_value) -> str:
    """将时间转换为UTC ISO格式"""
    if not time_value:
        return get_utc_timestamp()
    # 如果已经是ISO格式，直接返回
    if 'Z' in str(time_value) or '+' in str(time_value):
        return str(time_value)
    # 否则尝试解析并转换为UTC
    try:
        dt = datetime.fromisoformat(str(time_value).replace(' ', 'T'))
        dt_utc = dt.astimezone(timezone.utc)
        return dt_utc.isoformat().replace("+00:00", "Z")
    except:
        return get_utc_timestamp()
```
