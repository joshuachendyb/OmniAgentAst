### 4.2 问题详细说明

#### 问题1：sessions.py - session_id格式验证

**现象**: session_id路径参数未做UUID格式验证
**位置**: 所有会话相关接口
**风险**: 极低（FastAPI路径参数防护 + 404处理）
**建议**: 可选添加UUID验证
**修复代码（可选）:
```python
from uuid import UUID

def validate_session_id(session_id: str):
    try:
        UUID(session_id)
        return True
    except ValueError:
        return False
```

#### 问题2：chat.py - running_tasks无锁

**现象**: running_tasks全局字典无锁保护
**位置**: chat_stream和cancel_stream_task
**风险**: 低（Python GIL已保护）
**建议**: 可选添加asyncio.Lock
**修复代码（可选）:
```python
import asyncio
running_tasks_lock = asyncio.Lock()

# 使用:
async with running_tasks_lock:
    running_tasks[task_id] = ...
```

#### 问题3：chat.py - 调用不存在的方法 🔴 P1-高（已修复✅）

**现象**: 第300行调用AIServiceFactory.get_service_for_model，该方法不存在
**位置**: chat.py:300
**风险**: 🔴 高（运行时会崩溃）
**根因**: chat_stream接口尝试使用前端指定的provider+model，但工厂类没有这个方法
**修复方案**: 添加get_service_for_model方法到AIServiceFactory（✅ 已完成）
**修复位置**: backend/app/services/__init__.py:477-587
**方法功能**: 使用统一Fallback逻辑验证provider和model是否有效，有效则使用，无效则fallback
**状态**: ✅ 已修复
