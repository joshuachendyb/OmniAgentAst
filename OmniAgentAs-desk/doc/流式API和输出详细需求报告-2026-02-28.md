## 二、后端及API功能需求

### 2.1 流式API接口

#### API-STREAM-001：流式消息发送
- **接口路径**：`POST /api/v1/chat/stream`
- **功能描述**：发送流式消息请求，返回SSE格式响应
- **请求参数**：
  ```json
  {
    "messages": [{"role": "user", "content": "消息内容"}],
    "stream": true,
    "task_id": "可选，前端生成的任务ID"
  }
  ```
- **响应格式**：SSE格式，包含以下事件类型
  - `start`：开始事件，返回task_id
  - `thought`：思考过程
  - `action`：工具调用
  - `observation`：观察结果
  - `chunk`：内容片段
  - `final`：最终结果
  - `interrupted`：任务中断
  - `error`：错误信息
- **来源文档**：《调试笔记-2026-02-25.md》

#### API-STREAM-002：任务中断
- **接口路径**：`POST /api/v1/chat/stream/cancel/{task_id}`
- **功能描述**：取消正在执行的流式任务
- **路径参数**：
  - `task_id`：任务ID
- **响应格式**：
  ```json
  {"success": true, "message": "任务已取消"}
  ```
- **来源文档**：《调试笔记-2026-02-25.md》

## 四、交互流程图

### 4.1 流式消息发送流程

```
用户发送消息
    ↓
NewChatContainer.executeStreamSend()
    ├─ 生成 taskId = crypto.randomUUID()
    ├─ setCurrentTaskId(taskId)
    └─ setTaskId(taskId)
            ↓
       useSSE Hook
            ↓
       sendMessage()
            ↓
       fetch 请求
            └─ body: { messages, stream: true, task_id }
            ↓
       后端处理
            ├─ 返回 start 事件（包含task_id）
            ├─ 返回 thought/action/observation 事件
            └─ 返回 chunk 事件（逐token）
            ↓
       MessageItem.tsx 显示
            ├─ 逐token显示内容
            ├─ 显示光标提示
            └─ 展开思考过程
```

### 4.2 任务中断流程

```
用户点击中断按钮
    ↓
NewChatContainer.handleInterrupt()
    ├─ 获取 taskId（serverTaskId || currentTaskId）
    └─ 请求 /chat/stream/cancel/{taskId}
            ↓
       后端处理
            ├─ 设置任务取消标志
            └─ 在执行步骤中检查cancelled状态
            ↓
       前端收到 interrupted 事件
            ├─ 显示中断提示
            └─ 停止流式显示
```
