## 二、问题根本原因

### 2.1 配置文件缺少 `ai.model`

**现状**：
- 配置文件只有 `ai.provider: opencode`
- 没有 `ai.model` 字段
- 后端 `getConfig()` 返回的 `ai_model` 是空字符串

**导致的问题**：

```
用户选择 "OpenCode (kimi-k2.5-free)"
  ↓
前端更新：_currentProvider = "opencode-kimi-k2.5-free"
  ↓
前端调用 updateConfig，但只传了 ai_provider
  ↓
后端只更新 config['ai']['provider'] = "opencode"
  ↓
后端读取模型：config['ai']['opencode']['model'] = "glm-5-free"（默认）
  ↓
用户以为用了 kimi-k2.5-free，但后端不知道！
```

### 2.2 前后端数据不一致

| 位置 | 当前模型 | 格式 |
|------|---------|------|
| 前端状态 | `"opencode-kimi-k2.5-free"` | `"provider-modelname"` |
| 后端读取 | `""`（空字符串） | 无 |
| 配置文件 | 无 | 缺少 ai.model |

### 3.2 后端API设计

#### 3.2.1 getConfig - 返回 ai.model

#### 3.2.2 updateConfig - 同时更新 provider 和 model

```python
@router.put("/config")
async def update_config(config_update: ConfigUpdate):
    config_path = _get_config_path()
    
    with open(config_path, 'r+', encoding='utf-8') as f:
        config_data = yaml.safe_load(f) or {}
    
    # 更新 ai.provider
    if config_update.ai_provider:
        config_data['ai']['provider'] = config_update.ai_provider
    
    # 【新增】更新 ai.model
    if config_update.ai_model:
        config_data['ai']['model'] = config_update.ai_model
    
    # 可选：同时更新provider下的默认model
    if config_update.ai_provider and config_update.ai_model:
        provider = config_update.ai_provider
        if 'ai' in config_data and provider in config_data['ai']:
            config_data['ai'][provider]['model'] = config_update.ai_model
    
    # 写回配置文件
    with open(config_path, 'w', encoding='utf-8') as f:
        yaml.dump(config_data, f, allow_unicode=True, default_flow_style=False)
    
    # 重新加载
    config = get_config_instance()
    config.reload()
    
    return {"success": True, ...}
```

#### 3.2.3 get_model_list - 增加 current_model标记

```python
class ModelInfo(BaseModel):
    """模型信息"""
    id: int = Field(..., description="模型ID序号")
    provider: str = Field(..., description="提供商名称(小写)")
    model: str = Field(..., description="模型名称")
    display_name: str = Field(..., description="显示名称，格式: Provider (model)")
    current_model: bool = Field(default=False, description="是否为当前模型")  # 【新增】

@router.get("/config/models", response_model=ModelListResponse)
async def get_model_list():
    config = get_config_instance()
    ai_config = config.get("ai", {})
    
    # 【新增】读取当前模型
    current_provider = ai_config.get("provider", "")
    current_model = ai_config.get("model", "")  # 【新增】
    
    models = []
    model_id = 1
    
    # 按照配置文件中的provider顺序构建模型列表
    for provider in SUPPORTED_PROVIDERS:
        provider_config = ai_config.get(provider, {})
        models_list = provider_config.get("models", [])
        
        if isinstance(models_list, list) and models_list:
            for model_name in models_list:
                display_name = f"{provider_display_names.get(provider, provider)} ({model_name})"
                
                models.append(ModelInfo(
                    id=model_id,
                    provider=provider,
                    model=model_name,
                    display_name=display_name,
                    current_model=(  # 【新增】标记当前模型
                        provider == current_provider and 
                        model_name == current_model
                ))
                model_id += 1
    
    return ModelListResponse(
        models=models,
        default_provider=ai_config.get("provider", "")
    )
```

  ↓
后端更新 config.yaml：

## 八、总结

### 核心改进

| 方面 | 改进前 | 说明 |
|------|---------|------|
| **配置文件** | 缺少顶层 `ai.model` 字段，无法明确记录用户选择的模型 | 增加 `ai.model` 字段，明确记录用户选择的模型 |
| **后端API** | `getConfig()` 无法获取当前模型，`updateConfig()` 无法更新模型，`get_model_list()` 无法标记当前模型 | `getConfig()` 返回 `ai.model`，`updateConfig()` 同时更新 `ai.model`，`get_model_list()` 增加 `current_model` 标记 |
| **前端一致性** | 前端只传provider，无法知道具体模型，导致前后端使用不一致 | 前端同时传递 `ai_provider` 和 `ai_model`，确保前后端使用一致 |
| **用户体验** | 切换模型后下次启动不能保持上次选择，显示模型名可能错误 | 次换模型后立即更新配置，下次启动自动使用上次选择的模型，保持模型显示正确 |
