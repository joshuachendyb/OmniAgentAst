### 2.1 P0级问题（致命 - 必须立即修复）

#### P0-001: 删除模型时Popconfirm的onVisibleChange误触发删除

**位置**: `Settings/index.tsx` 第886-892行

**问题描述**: 
`Popconfirm` 组件的 `onVisibleChange` 在弹框关闭时会触发，导致不需要点击确认就执行删除操作。

**错误代码**:
```typescript
<Popconfirm
  title={`确定删除模型 ${model} 吗？`}
  onConfirm={(e) => {
    e?.stopPropagation();
    handleDeleteModel(selectedProvider.name, model);
  }}
  okText="确定"
  cancelText="取消"
  okButtonProps={{ danger: true }}
  onVisibleChange={(visible) => {
    if (!visible) {
      handleDeleteModel(           // ❌ 错误！弹框关闭时触发删除
        selectedProvider.name,
        model
      );
    }
  }}
>
  <Button type="text" size="small" danger icon={<DeleteOutlined />}>
    删除
  </Button>
</Popconfirm>
```

**影响程度**: 
- 用户每次点击删除按钮，弹框消失时就会误删模型
- 数据丢失风险，严重影响用户体验

**修复方案**:
```typescript
<Popconfirm
  title={`确定删除模型 ${model} 吗？`}
  onConfirm={(e) => {
    e?.stopPropagation();
    handleDeleteModel(selectedProvider.name, model);
  }}
  okText="确定"
  cancelText="取消"
  okButtonProps={{ danger: true }}
  // 删除 onVisibleChange，只用 onConfirm
>
  <Button type="text" size="small" danger icon={<DeleteOutlined />}>
    删除
  </Button>
</Popconfirm>
```

#### P0-002: 批量删除的取消按钮无效（竞态条件问题）

**位置**: `Settings/index.tsx` 第357-416行

**问题描述**: 
`deleteCancelled` 是 React 状态，在 `map` 循环中读取的是闭包捕获的旧值，设置状态是异步的，无法真正取消删除操作。

**错误代码**:
```typescript
const handleBatchDeleteModels = async (
  providerName: string,
  models: string[]
) => {
  setDeleteProgress({ current: 0, total: models.length });
  setDeleteModalVisible(true);
  setDeleteCancelled(false);  // 设置状态（异步）

  const controller = new AbortController();
  deleteControllerRef.current = controller;

  try {
    const deletePromises = models.map(async (modelName, index) => {
      if (deleteCancelled) {  // ❌ 闭包捕获的是旧值（false）
        return { success: false, model: modelName, cancelled: true };
      }
      // 删除操作继续执行...
    });

    const results = await Promise.all(deletePromises);
    // ...
  }
};
```

**影响程度**:
- 点击取消按钮后，删除操作仍会继续
- 用户无法中止批量删除操作

**修复方案**:
```typescript
// 使用 useRef 存储取消状态
const deleteCancelledRef = useRef<boolean>(false);

const handleBatchDeleteModels = async (
  providerName: string,
  models: string[]
) => {
  deleteCancelledRef.current = false;  // 同步更新
  
  const deletePromises = models.map(async (modelName, index) => {
    if (deleteCancelledRef.current) {  // ✅ 读取最新值
      return { success: false, model: modelName, cancelled: true };
    }
    // ...
  });
};

// 取消按钮
<Button onClick={() => {
  deleteCancelledRef.current = true;  // 同步设置
  setDeleteCancelled(true);
}}>
  取消
</Button>
```

### 2.2 P1级问题（重要 - 尽快修复）

#### P1-001: Provider切换时重复调用API

**位置**: `Settings/index.tsx` 第463-480行

**问题描述**: 
切换Provider时调用了两次 `getFullConfig`，造成不必要的网络请求。

**错误代码**:
```typescript
const onProviderChange = async (provider: string) => {
  const fullConfig = await configApi.getFullConfig();  // 第一次调用
  const providerData = providers.find((p) => p.name === provider);
  if (!providerData) return;
  
  setCurrentProvider(provider);
  const targetModel = fullConfig.current_provider === provider
    ? fullConfig.current_model
    : providerData.models[0];
  setCurrentModel(targetModel);
  
  await configApi.updateConfig({
    ai_provider: provider as "zhipuai" | "opencode" | "longcat",
    ai_model: targetModel,
  });
  
  loadConfig();  // 第二次调用（loadConfig内部会调用getFullConfig）
};
```

**修复方案**:
```typescript
const onProviderChange = async (provider: string) => {
  // 删除第一次调用，直接从已有的 providers 中获取
  const providerData = providers.find((p) => p.name === provider);
  if (!providerData) return;
  
  setCurrentProvider(provider);
  const targetModel = providerData.models[0];
  setCurrentModel(targetModel);
  
  await configApi.updateConfig({
    ai_provider: provider,
    ai_model: targetModel,
  });
  
  loadConfig();  // 只保留这一次调用
};
```

#### P1-003: 模型切换后UI可能不同步

**位置**: `Settings/index.tsx` 第483-490行

**问题描述**: 
切换模型后调用 `loadConfig()` 刷新，但 `selectedProvider` 可能还是旧的Provider。

**错误代码**:
```typescript
const onModelChange = async (model: string) => {
  setCurrentModel(model);
  await configApi.updateConfig({
    ai_provider: currentProvider as "zhipuai" | "opencode" | "longcat",
    ai_model: model,
  });
  loadConfig();  // 刷新providers，但selectedProvider可能未同步
};
```

**修复方案**:
```typescript
const onModelChange = async (model: string) => {
  setCurrentModel(model);
  await configApi.updateConfig({
    ai_provider: currentProvider,
    ai_model: model,
  });
  await loadConfig();
  // 同步更新 selectedProvider
  const updated = providers.find(p => p.name === currentProvider);
  if (updated) {
    setSelectedProvider(updated);
  }
};
```

#### P1-005: 后端恢复备份时删除备份文件

**位置**: `backend/app/api/v1/config.py` 第443-456行

**问题描述**: 
验证失败时恢复备份，但删除了备份文件，如果后续需要进一步回滚就没有备份了。

**错误代码**:
```python
if not is_valid:
    if not backup_restored and backup_path and backup_path.exists():
        logger.warning(f"配置验证失败，恢复备份：{backup_path}")
        shutil.copy2(str(backup_path), str(config_path))
        backup_restored = True
    backup_path.unlink(missing_ok=True)  # ❌ 删除了备份文件
    return {
        "success": False,
        "message": "配置验证失败",
        # ...
    }
```

**修复方案**:
```python
if not is_valid:
    if not backup_restored and backup_path and backup_path.exists():
        logger.warning(f"配置验证失败，恢复备份：{backup_path}")
        shutil.copy2(str(backup_path), str(config_path))
        backup_restored = True
    # 不删除备份文件，保留用于可能的进一步回滚
    return {
        "success": False,
        "message": "配置验证失败",
        "backup_path": str(backup_path)  # 返回备份路径，让前端决定是否删除
    }
```

#### P1-006: 删除当前Provider后ai.model可能不匹配

**位置**: `backend/app/api/v1/config.py` 第886-890行

**问题描述**: 
删除当前Provider后，只更新了 `ai.provider`，没有更新 `ai.model`，可能导致配置不一致。

**错误代码**:
```python
# 如果删除的是当前provider，切换到第一个
if config['ai'].get('provider') == provider_name:
    remaining = [k for k in config['ai'].keys() if k != 'provider']
    if remaining:
        config['ai']['provider'] = remaining[0]  # 只更新provider
        # ❌ 没有更新 ai.model！
```

**修复方案**:
```python
# 如果删除的是当前provider，切换到第一个有models的provider
if config['ai'].get('provider') == provider_name:
    for new_provider in config['ai'].keys():
        if new_provider == 'provider' or new_provider == 'model':
            continue
        provider_data = config['ai'].get(new_provider, {})
        if 'models' in provider_data and provider_data['models']:
            config['ai']['provider'] = new_provider
            config['ai']['model'] = provider_data['models'][0]  # 同时更新model
            break
```

    // ...

} catch (error) {
