## 一、问题现象

### 1.1 模型名称显示不一致

| 位置 | 显示内容 |
|------|---------|
| **顶部下拉框** | `OpenCode (kimi-k2.5-free)` |
| **消息气泡** | `AI助手`（无模型名称） |

**期望**：消息气泡显示 `AI助手 (OpenCode (kimi-k2.5-free))`

### 1.2 模型名称显示时机滞后

| 当前 | 期望 |
|------|------|
| 消息完成后才显示模型名称 | 消息开始时就显示模型名称 |

---

## 二、解决方案

**核心思路**：后端返回 `provider` + `model` → 前端直接组装显示

**不需要**：前端去匹配下拉框列表（这是分析的范畴，程序不需要处理）

---

## 二.1、当前实现方式（前端）

### 两种API调用方式

| 模式 | 调用的API | 说明 |
|------|-----------|------|
| **流式模式** | `/chat/stream` | 使用useSSE hook，默认开启 |
| **普通模式** | `/chat` | 非流式，一次性返回 |

### 流式开关位置

**UI位置**：消息输入框上方（带⚡图标）

**代码位置**：`frontend/src/components/Chat/NewChatContainer.tsx` 第562-572行

```tsx
<Tag.CheckableTag
  checked={useStream}
  onChange={(checked) => {
    setUseStream(checked);
    if (!checked) {
      setShowExecution(false);
    }
  }}
>
  <ThunderboltOutlined /> {useStream ? '流式开启' : '流式关闭'}
</Tag.CheckableTag>
```

**默认状态**：`useState(true)` - 默认开启流式

### 结论

**两种API都有使用**，取决于用户的`useStream`开关设置。

## 三、修改需求

### 3.1 后端流式响应增加provider字段（兜底用）

**修改文件**: `backend/app/api/v1/chat.py`

**修改内容**: 在返回 `model` 字段的事件中，同时返回 `provider` 字段

**修改原因**：
- 3.2的start事件只发一次，有丢失风险
- 在chunk/final中也带provider，作为**兜底**确保前端一定能收到

**需要修改的位置**:

| 行号 | 事件类型 | 修改内容 |
|------|---------|---------|
| 第408行 | `chunk` | 增加 `provider` 字段 |
| 第413行 | `final` | 增加 `provider` 字段 |
| 第371行 | `final`（文件操作） | 增加 `provider` 字段 |

**修改示例**:
```python
# 修改前
yield f"data: {json.dumps({'type': 'chunk', 'content': chunk.content, 'model': chunk.model})}\n\n"

# 修改后
yield f"data: {json.dumps({
    'type': 'chunk', 
    'content': chunk.content, 
    'model': chunk.model,
    'provider': ai_service.provider  # 新增
})}\n\n"
```

### 3.4 非流式API也需增加provider字段

**问题**：非流式API `/chat` 直接返回JSON，没有SSE事件

**当前状态**：
```python
class ChatResponse(BaseModel):
    success: bool
    content: str
    model: str       # ✅ 已有
    error: Optional[str]
    # ❌ 没有 provider 字段
```

**修改内容**：在 `ChatResponse` 中增加 `provider` 字段

**修改位置**: 第34-40行

```python
class ChatResponse(BaseModel):
    """聊天响应"""
    success: bool = Field(..., description="是否成功")
    content: str = Field(default="", description="回复内容")
    model: str = Field(default="", description="使用的模型")
    provider: str = Field(default="", description="使用的提供商")  # 新增
    error: Optional[str] = Field(default=None, description="错误信息")
    execution_steps: Optional[List[Dict]] = Field(default=None, description="执行步骤详情列表")
```

**返回时填充provider**（第241-246行）:
```python
return ChatResponse(
    success=response.success,
    content=response.content,
    model=response.model,
    provider=ai_service.provider  # 新增
)
```

## 四、前端配合修改（小新处理）

后端返回 `provider` + `model` 后，前端直接组装显示：

```typescript
// 收到 provider 和 model 后
const providerName = getProviderName(provider);  // opencode -> OpenCode
const displayName = `${providerName} (${model})`;  // OpenCode (kimi-k2.5-free)

// 显示：AI助手 (OpenCode (kimi-k2.5-free))
```

**需要增加的映射**:
```typescript
const getProviderName = (provider: string) => {
  const names: Record<string, string> = {
    zhipuai: '智谱GLM',
    opencode: 'OpenCode',
    longcat: 'LongCat',
  };
  return names[provider] || provider;
};
```

## 五、验收标准

| 序号 | 验收项 | 期望结果 |
