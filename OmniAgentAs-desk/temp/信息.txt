### 2.4 SSE接口所有type类型详细说明

| type值 | 发送位置(行号) | 发送场景 | 包含的字段 |
|--------|---------------|---------|-----------|

### 2.2 后端各字段详细信息

| 字段名 | 来源/值 | 数据类型 | 含义 |
|--------|---------|---------|------|
| type | 固定值 'observation' | string | 步骤类型 |
| step | i + 2 | int | 步骤序号 |
| thought | step_thought变量 | string | Agent.thought的值 |
| action | step_action变量 | string | Agent.action的值 |
| observation | step_observation变量 | object | Agent.observation的值 |
| content | step_thought变量 | string | 值为step_thought |
| tool | step_action变量 | string | 值为step_action |
| result | step_result变量 | string | 值为simplify_observation(observation) |

### 2.3 后端字段问题（对照2.2逐一分析）

| 字段名 | 可能的问题 | 前端是否需要关注 |
|--------|-----------|----------------|
| **type** | 固定为'observation'，但前端可能收到其他type（start/chunk/final等） | 需要，前端需根据type做不同处理 |
| **step** | 从2开始（i+2），可能与前端期望的序号不一致 | 需要，前端用于显示步骤序号 |
| **thought** | **可能为空字符串**，LLM不一定会返回推理过程 | 需要，前端不能依赖此字段显示 |
| **action** | 可能为空（LLM决定不调用工具时） | 需要，前端用于显示执行的工具名 |
| **observation** | **是嵌套对象**，直接显示会显示成JSON大片数据 | 需要，但需要简化处理 |
| **content** | **值等于thought**，所以也可能为空 | 前端当前用于显示content，需要理解这一点 |
| **tool** | **值等于action**，两者等价 | 前端有tool字段，与此等价 |
| **result** | 是simplify_observation()处理后的文本，但**前端没有使用此字段** | **需要修改前端使用此字段** |

- 前端确实需要根据type做不同处理
