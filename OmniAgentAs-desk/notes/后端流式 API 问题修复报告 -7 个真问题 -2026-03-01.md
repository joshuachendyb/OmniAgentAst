## 3 问题详细分析

### 3.1 问题 1：错误信息直接返回 str(e)（P1-高）

**位置**: `chat.py:520-521`

**当前代码**:
```python
except Exception as e:
    yield f"data: {json.dumps({'type': 'error', 'content': f'流式响应出错：{str(e)}'})}\n\n"
```

**问题分析**:
1. **直接返回 `str(e)`** - 可能包含敏感信息（如 API Key、内部路径、堆栈跟踪）
2. **没有错误分类** - 所有错误都用相同格式返回
3. **用户看到技术细节** - 不友好，可能泄露系统信息

**示例泄露场景**:
```python
# 如果错误是 API Key 无效
str(e) = "HTTP 401: Unauthorized - API Key: sk-abc123..."

# 前端收到
{"type": "error", "content": "流式响应出错：HTTP 401: Unauthorized - API Key: sk-abc123..."}

# ❌ API Key 泄露！
```

**修复方案**:
```python
except Exception as e:
    # ✅ 记录详细错误日志（后端）
    logger.error(f"流式响应异常：{e}", exc_info=True)
    # ✅ 返回用户友好的错误信息（前端）
    error_type = type(e).__name__
    if error_type == "TimeoutError":
        error_message = "请求超时，请重试"
    elif error_type == "ConnectionError":
        error_message = "网络连接失败，请检查网络"
    else:
        error_message = "AI 处理异常，请稍后重试"
    yield f"data: {json.dumps({'type': 'error', 'content': error_message})}\n\n"
```

### 3.2 问题 2：错误类型不统一（P1-高）

**位置**: `chat.py` 多处

**当前代码对比**:

**场景 1 - 文件操作错误** (第 473-474 行):
```python
except Exception as e:
    yield f"data: {json.dumps({'type': 'error', 'content': f'执行出错：{str(e)}'})}\n\n"
```

**场景 2 - 普通对话错误** (第 520-521 行):
```python
except Exception as e:
    yield f"data: {json.dumps({'type': 'error', 'content': f'流式响应出错：{str(e)}'})}\n\n"
```

**场景 3 - 安全检测错误** (第 408 行):
```python
yield f"data: {json.dumps({'type': 'error', 'content': f'危险操作需确认：{risk}'})}\n\n"
```

**场景 4 - Agent 执行错误** (第 468-471 行):
```python
yield f"data: {json.dumps({
    'type': 'error',
    'content': result_error,
    'model': ai_service.model,
    'display_name': display_name,
    'provider': ai_service.provider
})}\n\n"
```

**问题分析**:
1. **错误格式不统一** - 有的只返回 content，有的返回 model/provider
2. **错误前缀不一致** - "执行出错"、"流式响应出错"、"危险操作需确认"
3. **前端无法统一处理** - 需要根据不同格式写不同处理逻辑

**正确做法**:
```python
# 统一的错误响应格式
def create_error_response(error_type: str, content: str, model: str = None, provider: str = None):
    response = {
        'type': 'error',
        'error_type': error_type,  # ✅ 新增错误类型字段
        'content': content,
    }
    if model:
        response['model'] = model
    if provider:
        response['provider'] = provider
    return response

# 使用
yield f"data: {json.dumps(create_error_response('timeout', '请求超时', ai_service.model, ai_service.provider))}\n\n"
```

### 3.3 问题 3：缺少错误日志记录（P2-中）

**位置**: `chat.py:520-521`

**当前代码**:
```python
except Exception as e:
    yield f"data: {json.dumps({'type': 'error', 'content': f'流式响应出错：{str(e)}'})}\n\n"
```

**问题分析**:
1. **没有 logger.error** - 后端日志中没有错误记录
2. **无法追踪问题** - 运维无法通过日志定位问题
3. **缺少上下文** - 没有记录 task_id、用户、时间等信息

**修复方案**:
```python
except Exception as e:
    # ✅ 记录详细错误日志
    logger.error(
        f"流式响应异常：task_id={task_id}, error={e}",
        exc_info=True,  # 包含堆栈跟踪
        extra={
            'task_id': task_id,
            'user': 'current_user',  # 如果有用户系统
            'timestamp': datetime.now().isoformat()
        }
    )
    # ✅ 返回用户友好的错误信息
    yield f"data: {json.dumps({'type': 'error', 'content': 'AI 处理异常，请稍后重试'})}\n\n"
```

### 3.4 问题 4：文件操作和对话操作错误处理不一致（P2-中）

**位置**: `chat.py:473-474` vs `chat.py:520-521`

**文件操作错误** (第 473-474 行):
```python
except Exception as e:
    yield f"data: {json.dumps({'type': 'error', 'content': f'执行出错：{str(e)}'})}\n\n"
```

**对话操作错误** (第 520-521 行):
```python
except Exception as e:
    yield f"data: {json.dumps({'type': 'error', 'content': f'流式响应出错：{str(e)}'})}\n\n"
```

**问题分析**:
1. **错误前缀不同** - "执行出错" vs "流式响应出错"
2. **没有统一错误处理函数** - 代码重复
3. **维护困难** - 修改错误格式需要改多处

**修复方案**:
```python
# 提取统一的错误处理函数
async def handle_stream_error(task_id: str, error: Exception, context: str = ""):
    """统一处理流式错误"""
    # 记录日志
    logger.error(
        f"流式错误：context={context}, task_id={task_id}, error={error}",
        exc_info=True
    )
    # 返回错误
    error_message = get_user_friendly_error(error)
    yield f"data: {json.dumps({'type': 'error', 'content': error_message})}\n\n"

# 使用
# 文件操作
except Exception as e:
    await handle_stream_error(task_id, e, "file_operation")

# 对话操作
except Exception as e:
    await handle_stream_error(task_id, e, "chat")
```

### 3.5 问题 5：中断处理逻辑分散（P2-中）

**位置**: `chat.py` 多处（第 386-389、400-403、439-442、459-462、481-484、499-502 行）

**当前代码**（6 处中断检查）:
```python
# 第 386-389 行
async with running_tasks_lock:
    if running_tasks.get(task_id, {}).get("cancelled", False):
        yield f"data: {json.dumps({'type': 'interrupted', 'content': '任务已被中断'})}\n\n"
        return

# 第 400-403 行（相同逻辑）
# 第 439-442 行（相同逻辑）
# 第 459-462 行（相同逻辑）
# 第 481-484 行（相同逻辑）
# 第 499-502 行（相同逻辑）
```

**问题分析**:
1. **代码重复 6 次** - 违反 DRY 原则
2. **逻辑分散** - 难以维护和修改
3. **容易遗漏** - 新增流程可能忘记检查中断

**修复方案**:
```python
# 提取统一的中断检查函数
async def check_interrupted(task_id: str) -> bool:
    """检查任务是否被中断，如果是则返回 True"""
    async with running_tasks_lock:
        return running_tasks.get(task_id, {}).get("cancelled", False)

async def yield_interrupted(task_id: str):
    """生成中断消息"""
    yield f"data: {json.dumps({'type': 'interrupted', 'content': '任务已被中断'})}\n\n"

# 使用
if await check_interrupted(task_id):
    await yield_interrupted(task_id)
    return
```

### 3.7 问题 7：AI 服务 chat_stream 缺少错误详情（P1-高）

**位置**: `base.py:145-149`

**当前代码**:
```python
except httpx.TimeoutException:
    yield StreamChunk(content="", model=self.model, is_done=True)
except Exception as e:
    print(f"[BaseAIService] 流式调用失败：{str(e)}")
    yield StreamChunk(content="", model=self.model, is_done=True)
```

**问题分析**:
1. **错误被吞掉** - 只 print，不抛出也不返回错误信息
2. **前端不知道失败原因** - 只收到空 chunk，不知道是超时还是其他错误
3. **使用 print 而不是 logger** - 生产环境无法追踪

**影响**:
- ❌ 前端收到空响应，不知道是网络超时还是 API 错误
- ❌ 无法区分错误类型，无法给用户准确提示
- ❌ 后端日志中没有详细错误信息

**修复方案**:
```python
# 方案 1：在 StreamChunk 中添加错误字段
@dataclass
class StreamChunk:
    content: str
    model: str
    is_done: bool
    error: Optional[str] = None  # ✅ 新增错误字段
    error_type: Optional[str] = None  # ✅ 新增错误类型

# 使用
except httpx.TimeoutException:
    logger.warning(f"流式调用超时：model={self.model}")
    yield StreamChunk(content="", model=self.model, is_done=True, error="请求超时", error_type="timeout")
except Exception as e:
    logger.error(f"流式调用失败：model={self.model}, error={e}", exc_info=True)
    yield StreamChunk(content="", model=self.model, is_done=True, error=str(e), error_type="unknown")
```

```python
# 方案 2：抛出异常，让上层处理
except httpx.TimeoutException:
    logger.warning(f"流式调用超时：model={self.model}")
    raise TimeoutError("AI 服务请求超时")
except Exception as e:
    logger.error(f"流式调用失败：model={self.model}, error={e}", exc_info=True)
    raise  # 重新抛出，让 chat.py 处理
```

### 8.4 修复效果对比

#### 修复前：
```python
# 错误处理不统一

    yield f"data: {json.dumps({'type': 'error', 'content': f'执行出错：{str(e)}'})}\n\n"

async with running_tasks_lock:
    if running_tasks.get(task_id, {}).get("cancelled", False):
```

#### 修复后：
```python
# 统一错误处理
except Exception as e:

# 统一中断检查（5 处）
is_interrupted, interrupt_msg = await check_and_yield_if_interrupted(
```
