**主题**: 后端接口升级后的前端适配修复方案  
**原则**: 向后兼容、渐进式升级、类型安全、不破坏现有功能

## 一、修复策略

### 1.1 向后兼容设计

**目标**：同时支持新旧两种格式

**旧格式**（当前前端期望）：
```typescript
onError?: (error: string) => void
onComplete?: (fullResponse: string, model?: string) => void
```

**新格式**（后端返回）：
```typescript
onError?: (error: {
  type: string;
  message: string;
  rawMessage: string;
  model?: string;
  provider?: string;
}) => void

onComplete?: (fullResponse: string, metadata?: {
  model?: string;
  provider?: string;
  displayName?: string;
}) => void
```

**兼容方案**：
```typescript
// 使用函数重载支持两种格式
onError?: {
  (error: string): void;
  (error: ErrorObject): void;
}

// 在实现中自动适配
const handleError = (error: string | ErrorObject) => {
  const errorObj = typeof error === 'string' 
    ? { type: 'unknown', message: error }
    : error;
  // 统一处理
}
```

## 二、详细修复方案

### 2.1 sse.ts - 错误处理适配

**位置**：`sse.ts:477-487`

**修复前**：
```typescript
case "error": {
  const errorMsg = step.content || "未知错误";
  message.error(`流式传输错误：${errorMsg}`);
  onError?.(errorMsg);  // ❌ 只传递字符串
  break;
}
```

**修复后**：
```typescript
case "error": {
  // ✅ 处理新格式，兼容旧格式
  const errorType = rawData.error_type || "unknown_error";
  const errorMsg = rawData.content || step?.content || "未知错误";
  const model = rawData.model;
  const provider = rawData.provider;
  
  // ✅ 根据错误类型显示不同提示
  const errorMessages: Record<string, string> = {
    timeout_error: "请求超时，请检查网络连接",
    connection_error: "网络连接失败，请检查网络",
    http_error: "服务器响应异常，请稍后重试",
    value_error: "参数值错误，请检查输入",
    file_operation_error: "文件操作执行失败",
    unknown_error: errorMsg
  };
  
  const userMessage = errorMessages[errorType] || errorMsg;
  message.error(`流式传输错误：${userMessage}`);
  
  // ✅ 传递完整错误对象（兼容字符串）
  const errorObj = {
    type: errorType,
    message: userMessage,
    rawMessage: errorMsg,
    model,
    provider
  };
  onError?.(errorObj);
  
  console.error("[SSE] 错误:", errorObj);
  break;
}
```

**兼容性保证**：
- ✅ 如果后端返回旧格式（只有 content），也能正常处理
- ✅ onError 回调既支持字符串也支持对象

### 2.4 NewChatContainer.tsx - onError 回调适配

**位置**：`NewChatContainer.tsx:347-373`

**修复前**：
```typescript
// onError - 流式错误
useCallback((error: string) => {
  console.error("SSE 流式错误:", error);
  message.error({
    content: `AI 响应失败：${error}`,
    duration: 5,
  });
  setMessages((prev) => {
    // ...
    updated[updated.length - 1] = {
      ...lastMessage,
      content: lastMessage.content || `**错误**: ${error}`,
      isError: true,
      isStreaming: false,
    };
    return updated;
  });
  setLoading(false);
}, [])
```

**修复后**：
```typescript
// onError - 流式错误
useCallback((error: string | {
  type: string;
  message: string;
  rawMessage: string;
  model?: string;
  provider?: string;
}) => {
  // ✅ 支持字符串和对象两种格式
  const errorObj = typeof error === 'string' 
    ? { type: 'unknown_error', message: error, rawMessage: error }
    : error;
  
  console.error("SSE 流式错误:", errorObj);
  
  message.error({
    content: `AI 响应失败：${errorObj.message}`,
    duration: 5,
  });
  
  setMessages((prev) => {
    const lastMessage = prev[prev.length - 1];
    if (lastMessage && lastMessage.role === "assistant") {
      const updated = [...prev];
      updated[updated.length - 1] = {
        ...lastMessage,
        content: lastMessage.content || `**错误**: ${errorObj.message}`,
        isError: true,
        isStreaming: false,
        errorType: errorObj.type,  // ✅ 保存错误类型
        model: errorObj.model,  // ✅ 保存模型
        provider: errorObj.provider,  // ✅ 保存提供商
      };
      return updated;
    }
    return prev;
  });
  setLoading(false);
}, [])
```

**兼容性保证**：
- ✅ 既支持旧格式（string）也支持新格式（object）
- ✅ sse.ts 传递对象，其他调用方传递字符串都能工作

### 2.5 NewChatContainer.tsx - onComplete 回调适配

**位置**：`NewChatContainer.tsx:244-260`

**修复前**：
```typescript
// onComplete - 流式完成
useCallback(
  async (fullResponse: string, model?: string) => {
    setMessages((prev) => {
      const lastMessage = prev[prev.length - 1];
      if (lastMessage && lastMessage.role === "assistant") {
        const updated = [...prev];
        updated[updated.length - 1] = {
          ...lastMessage,
          content: fullResponse,
          isStreaming: false,
          model: model || lastMessage.model,
        };
        return updated;
      }
      return prev;
    });
    // ...
  },
  [sessionId, pendingMessage]
)
```

**修复后**：
```typescript
// onComplete - 流式完成
useCallback(
  async (fullResponse: string, metadata?: string | {
    model?: string;
    provider?: string;
    displayName?: string;
  }) => {
    // ✅ 支持旧格式（model 字符串）和新格式（metadata 对象）
    const metadataObj = typeof metadata === 'string'
      ? { model: metadata }
      : metadata || {};
    
    setMessages((prev) => {
      const lastMessage = prev[prev.length - 1];
      if (lastMessage && lastMessage.role === "assistant") {
        const updated = [...prev];
        updated[updated.length - 1] = {
          ...lastMessage,
          content: fullResponse,
          isStreaming: false,
          model: metadataObj.model || lastMessage.model,
          provider: metadataObj.provider || lastMessage.provider,
          displayName: metadataObj.displayName || lastMessage.displayName,
        };
        return updated;
      }
      return prev;
    });
    // ...
  },
  [sessionId, pendingMessage]
)
```

**兼容性保证**：
- ✅ 支持旧格式：`onComplete(response, "gpt-4")`
- ✅ 支持新格式：`onComplete(response, { model: "gpt-4", provider: "openai" })`
- ✅ sse.ts 传递对象，其他调用方传递字符串都能工作
