## 修复详情

### 修复1：P0 - 版本号递增错误 ⭐

**文件**: `backend/app/api/v1/sessions.py`
**位置**: 第656行

**修复前**:
```python
if fields_exist['version'] and should_update_title:
    update_fields.append('version = ?')
    update_values.append(1)  # ❌ 错误：设置为固定值1
```

**修复后**:
```python
if fields_exist['version'] and should_update_title:
    update_fields.append('version = version + 1')  # ✅ 正确：递增版本号
```

**修复原因**:
- 后端在保存第一条消息时自动生成标题
- 原代码错误地将版本号设置为固定值1
- 导致版本号重置，版本控制失效

**影响**:
- 修复后，版本号会正确递增
- 版本控制机制正常工作
- 409冲突检测正确

### 修复2：P1 - 删除前端不必要的标题保存 ⭐

**文件**: `frontend/src/components/Chat/NewChatContainer.tsx`
**位置**: 第274-281行

**修复前**:
```typescript
// 保存AI回复
await sessionApi.saveMessage(sessionIdToUse, {
    role: "assistant",
    content: fullResponse,
});

// 🔴 修复：确保会话标题持久化
if (
    sessionTitle &&
    sessionTitle.trim() &&
    sessionTitle !== "新会话"
) {
    debouncedSaveTitle(sessionIdToUse, sessionTitle);  // ❌ 不必要的保存
}

console.log("✅ 消息和标题保存成功");
```

**修复后**:
```typescript
// 保存AI回复
await sessionApi.saveMessage(sessionIdToUse, {
    role: "assistant",
    content: fullResponse,
});

// 【小新第二修复 2026-03-02】后端已在 save_message 中自动生成和保存标题，
// 不需要前端再次保存，避免覆盖后端的自动标题
console.log("✅ 消息保存成功");
```

**修复原因**:
- 后端在 `save_message` 中已经自动生成和保存了标题
- 前端在保存消息后再次保存会覆盖后端的自动标题
- 导致标题可能显示为前端的默认标题（如 "新会话 2026-02-26 18:32"）

**影响**:
- 修复后，标题由后端统一管理
- 避免了前端和后端标题不一致的问题
- 标题显示更加准确

### 修复3：P2 - 移除前端对 `title_source` 的依赖 ⭐

**文件**: `frontend/src/components/Chat/NewChatContainer.tsx`
**位置**: 多处

**修复内容**:

1. **删除状态定义**（第108行）:
```typescript
// 删除
// const [titleSource, setTitleSource] = useState<"user" | "auto">("auto");
```

2. **删除从后端读取 `title_source` 的代码**（5处）:
```typescript
// 删除
if (sessionData.title_source) {
    setTitleSource(sessionData.title_source);
}
```

3. **修改 UI 显示逻辑**（第1487-1509行）:
```typescript
// 修复前
style={{
    color: titleSource === "auto" ? "#666" : "#000",
    fontSize: titleSource === "auto" ? "14px" : "16px",
    fontWeight: titleSource === "user" ? "bold" : "normal",
}}
{titleSource === "auto" && (
    <Tooltip title="AI自动生成的标题">
        <InfoCircleOutlined />
    </Tooltip>
)}

// 修复后
style={{
    color: titleLocked ? "#000" : "#666",
    fontSize: titleLocked ? "16px" : "14px",
    fontWeight: titleLocked ? "bold" : "normal",
}}
{!titleLocked && (
    <Tooltip title="AI自动生成的标题">
        <InfoCircleOutlined />
    </Tooltip>
)}
```

4. **用户修改标题时更新 `titleLocked`**（第1454行）:
```typescript
// 修复前
setSessionTitle(titleInput.trim());
setTitleSource("user");

// 修复后
setSessionTitle(titleInput.trim());
setTitleLocked(true);
```

**修复原因**:
- `title_source` 是后端根据 `title_locked` 动态计算的，不是从数据库读取的
- 前端维护 `titleSource` 状态没有意义（会话刷新后会丢失）
- 直接使用 `titleLocked` 判断即可

**影响**:
- 修复后，前端状态更加简洁
- 只需要维护 `titleLocked` 一个状态
- UI 显示逻辑更加清晰
