
### 2.1 问题 1：错误消息没有标识（P0-紧急）

**位置**: `NewChatContainer.tsx:245-268`

**问题描述**:
- `onError` 回调没有设置 `isError: true` 标记
- 错误消息和正常消息样式完全一样
- 用户无法从视觉上区分错误消息

**当前代码**:
```typescript
// onError - 流式错误
useCallback((error: string) => {
  console.error("SSE 流式错误:", error);
  message.error({
    content: `AI 响应失败：${error}`,
    duration: 5,
  });

  setMessages((prev) => {
    const lastMessage = prev[prev.length - 1];
    if (lastMessage && lastMessage.role === "assistant") {
      const updated = [...prev];
      updated[updated.length - 1] = {
        ...lastMessage,
        content: lastMessage.content || "抱歉，发生了错误",
        isStreaming: false,
        // ❌ 缺少：isError: true
      };
      return updated;
    }
    return prev;
  });
  setLoading(false);
}, [])
```

**影响**:
- ❌ 用户不知道哪条消息出错了
- ❌ 错误消息没有视觉警示
- ❌ 用户体验差

**严重性**: P0-紧急

**影响**:

## 4 正确的流式对话显示流程 【理想设计】2026-03-01 00:50:28

### 4.1 理想的流程设计

```
用户发送消息
    ↓
显示用户消息
    ↓
显示"AI 思考中..."状态（可选）
    ↓
收到第一个 chunk
    ↓
创建 assistant 消息，content = 第一个 chunk
    ↓
继续接收 chunk，追加到 content
    ↓
同时接收 step，追加到 executionSteps
    ↓
收到 final 或 error
    ↓
设置 isStreaming = false
设置 isError（如果有错误）
设置 model 信息
    ↓
消息完成显示
```

### 4.2 关键改进点对比

| 改进点 | 当前做法 | 正确做法 | 优势 |
|--------|---------|---------|------|
| **消息创建时机** | 立即创建空消息 | 收到第一个 chunk 再创建 | 避免空消息显示 |
| **空消息显示** | 显示空白气泡 | 显示"AI 思考中..."提示 | 用户体验更好 |
| **错误标识** | 无 isError 标记 | 添加 isError 和红色样式 | 一眼识别错误 |
| **模型信息** | 不保存 | 保存 model 字段 | 可追溯模型 |
| **状态管理** | 只有 isStreaming | 添加 isError、isCompleted 等 | 状态完整 |
| **执行追溯** | 步骤和内容分离 | 步骤和内容关联 | 可追溯思考过程 |

### 7.2 问题 6：buffer 累积导致内存泄漏（P2-中）

**位置**: `sse.ts:210`


- ❌ **没有超时处理** - 60 秒超时是整体的，不是针对单行的

### 11.3 问题 C：start 到 chunk 之间的空白（P1-高）

**正确实现**：
```typescript
// 收到 start 事件时
case "start":
    // ✅ 创建占位消息，显示"AI 思考中..."
    const placeholderMessage: Message = {
        id: (Date.now() + 1).toString(),
        role: "assistant",
        content: "🤔 AI 正在思考...",  // ✅ 有内容
        isStreaming: true,
        isPlaceholder: true,  // ✅ 标记为占位
        model: rawData.model,  // ✅ 保存模型信息

    break;

// 收到第一个 chunk 时
case "chunk":
    // ✅ 更新占位消息为真实消息

        if (lastMessage && lastMessage.isPlaceholder) {

            updated[updated.length - 1] = {
                ...lastMessage,
                content: rawData.content,
                isPlaceholder: false,  // ✅ 移除占位标记
            };
            return updated;
        }
        // ...
    });
    break;
```

### 15.2 D2：执行步骤属于哪条消息（代码分析）

**当前代码**：

```
t4: 收到 A 的执行步骤 → 关联到 B（错误！）
```
