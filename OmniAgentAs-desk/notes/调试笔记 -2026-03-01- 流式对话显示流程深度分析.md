### 1.1 当前实现的数据流

```
用户点击发送
    ↓

    ↓
executeStreamSend(userMessage)
    ↓
【步骤 1】添加用户消息到 messages
    setMessages((prev) => [...prev, userMessage])
    ↓
【步骤 2】创建空的 assistant 消息占位
    assistantMessage = {
      content: "",
      isStreaming: true,
      executionSteps: []
    }
    setMessages((prev) => [...prev, assistantMessage])
    ↓
【步骤 3】调用 SSE Hook
    sendStreamMessage(userMessage.content)
    ↓
SSE 连接建立，开始接收数据
    ↓
onStep(执行步骤) → 更新最后一条消息的 executionSteps
onChunk(内容片段) → 追加到 content
onComplete(完成) → 设置 isStreaming=false
onError(错误) → 设置 isStreaming=false
```

## 2 发现的严重问题清单 【问题分析】2026-03-01 00:50:28

### 2.1 问题 1：错误消息没有标识（P0-紧急）

**位置**: `NewChatContainer.tsx:245-268`

**问题描述**:
- `onError` 回调没有设置 `isError: true` 标记
- 错误消息和正常消息样式完全一样
- 用户无法从视觉上区分错误消息

**当前代码**:
```typescript
// onError - 流式错误
useCallback((error: string) => {
  console.error("SSE 流式错误:", error);
  message.error({
    content: `AI 响应失败：${error}`,
    duration: 5,
  });

  setMessages((prev) => {
    const lastMessage = prev[prev.length - 1];
    if (lastMessage && lastMessage.role === "assistant") {
      const updated = [...prev];
      updated[updated.length - 1] = {
        ...lastMessage,
        content: lastMessage.content || "抱歉，发生了错误",
        isStreaming: false,
        // ❌ 缺少：isError: true
      };
      return updated;
    }
    return prev;
  });
  setLoading(false);
}, [])
```

**影响**:
- ❌ 用户不知道哪条消息出错了
- ❌ 错误消息没有视觉警示
- ❌ 用户体验差

**严重性**: P0-紧急

### 2.2 问题 2：空消息立即显示（P0-紧急）

**位置**: `NewChatContainer.tsx:826-835`

**问题描述**:
- 在发送请求前就创建空的 assistant 消息
- 空消息立即显示在界面上
- 用户看到空白气泡在等待

**当前代码**:
```typescript
const executeStreamSend = async (userMessage: Message) => {
  setLoading(true);
  clearSteps();

  // 【关键修复】先创建 assistant 消息占位，设置 isStreaming=true
  const assistantMessage: Message = {
    id: (Date.now() + 1).toString(),
    role: "assistant",
    content: "",  // ❌ 空内容
    timestamp: new Date(),
    executionSteps: [],
    isStreaming: true,
    model: undefined,
  };
  setMessages((prev) => [...prev, assistantMessage]);  // ❌ 立即添加到列表

  // 保存待发送消息，用于 onComplete 时保存到会话
  setPendingMessage(userMessage);

  // 发送流式请求
  sendStreamMessage(userMessage.content);  // ❌ 然后才发送请求
};
```

**影响**:
- ❌ 空消息立即显示，用户体验差
- ❌ 如果请求失败，空消息已经显示在列表中
- ❌ 消息列表中出现空白气泡

**严重性**: P0-紧急

### 2.3 问题 3：执行步骤和内容分离（P1-高）

**位置**: `NewChatContainer.tsx:152-189`

**问题描述**:
- `onStep` 只更新 `executionSteps`
- `onChunk` 只更新 `content`
- 用户看不到思考和回复的对应关系

**当前代码**:
```typescript
// onStep 回调 - 只更新 executionSteps
useCallback((step: ExecutionStep) => {
  setMessages((prev) => {
    const lastMessage = prev[prev.length - 1];
    if (lastMessage && lastMessage.role === "assistant" && lastMessage.isStreaming) {
      const updatedSteps = [...(lastMessage.executionSteps || []), step];
      const updated = [...prev];
      updated[updated.length - 1] = {
        ...lastMessage,
        executionSteps: updatedSteps,  // ✅ 更新执行步骤
      };
    }
  });
}, [])

// onChunk 回调 - 只更新 content
useCallback((chunk: string) => {
  setMessages((prev) => {
    const lastMessage = prev[prev.length - 1];
    if (lastMessage && lastMessage.role === "assistant" && lastMessage.isStreaming) {
      const updated = [...prev];
      updated[updated.length - 1] = {
        ...lastMessage,
        content: lastMessage.content + chunk,  // ✅ 更新内容
      };
    }
  });
}, [])
```

**影响**:
- ❌ 无法追溯某段内容是哪个思考步骤产生的
- ❌ 用户看不到 AI 决策过程的关联
- ❌ 调试困难，无法分析 AI 的思考链路

**严重性**: P1-高

## 4 正确的流式对话显示流程 【理想设计】2026-03-01 00:50:28

### 4.1 理想的流程设计

```
用户发送消息
    ↓
显示用户消息
    ↓
显示"AI 思考中..."状态（可选）
    ↓
收到第一个 chunk
    ↓
创建 assistant 消息，content = 第一个 chunk
    ↓
继续接收 chunk，追加到 content
    ↓
同时接收 step，追加到 executionSteps
    ↓
收到 final 或 error
    ↓
设置 isStreaming = false
设置 isError（如果有错误）
设置 model 信息
    ↓
消息完成显示
```

### 4.2 关键改进点对比

| 改进点 | 当前做法 | 正确做法 | 优势 |
|--------|---------|---------|------|
| **消息创建时机** | 立即创建空消息 | 收到第一个 chunk 再创建 | 避免空消息显示 |
| **空消息显示** | 显示空白气泡 | 显示"AI 思考中..."提示 | 用户体验更好 |
| **错误标识** | 无 isError 标记 | 添加 isError 和红色样式 | 一眼识别错误 |
| **模型信息** | 不保存 | 保存 model 字段 | 可追溯模型 |
| **状态管理** | 只有 isStreaming | 添加 isError、isCompleted 等 | 状态完整 |
| **执行追溯** | 步骤和内容分离 | 步骤和内容关联 | 可追溯思考过程 |
