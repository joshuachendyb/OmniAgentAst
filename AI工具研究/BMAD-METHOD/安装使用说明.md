## 一、概述

### 1.1 什么是BMAD-METHOD

**BMAD** = Breakthrough Method of Agile AI Driven Development（敏捷AI驱动开发的突破性方法）

BMAD-METHOD是一个开源的敏捷AI驱动开发框架，具有以下核心特点：

| 特点 | 说明 |
|------|------|
| **开源免费** | MIT许可证，完全免费使用 |
| **专业Agent** | 21个专业AI助手（产品经理、架构师、开发人员、测试工程师等） |
| **丰富工作流** | 50+结构化工作流程，覆盖开发全生命周期 |
| **本地运行** | 数据存储在项目目录，安全可控 |
| **多IDE集成** | 支持Claude Code、Cursor、Windsurf、OpenCode等 |
| **可扩展** | 支持自定义Agent和工作流 |

### 1.2 BMAD-METHOD能做什么

BMAD-METHOD可以帮助开发团队：

| 功能模块 | 具体能力 |
|---------|---------|
| **需求分析** | PRD文档创建、产品简报、用户故事拆分 |
| **架构设计** | 系统架构设计、技术选型、UX设计 |
| **代码开发** | 快速开发、代码生成、技术规范 |
| **测试质量** | 测试策略、测试设计、代码审查、自动化测试 |
| **项目管理** | 冲刺规划、进度跟踪、风险管理 |
| **文档生成** | API文档、架构图、数据流图 |

### 1.3 适用场景

| 场景 | 推荐使用程度 |
|------|-------------|
| **小团队（2-5人）** | ⭐⭐⭐⭐⭐ 非常推荐 |

## 二、安装方法对比分析

### 2.1 两种安装方法总览

BMAD-METHOD有两种安装方式，分别适用于不同的使用场景：

| 安装方案 | 方案A：直接使用BMAD-METHOD | 方案B：Skill方式 |
|---------|---------------------------|-----------------|
| **安装位置** | 项目目录 `.bmad-core/` | OpenCode Skills目录 |
| **依赖要求** | Node.js ≥ 20.0.0 | 无额外依赖 |
| **功能完整度** | ✅ 完整功能 | ⚠️ 需要转换 |
| **维护方式** | 独立更新 | 需手动同步 |
| **使用便捷性** | ✅ 成熟稳定 | ⚠️ 需要额外配置 |
| **推荐指数** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

### 2.2 方案A：直接使用BMAD-METHOD

#### 2.2.1 方案概述

直接在项目目录安装BMAD-METHOD框架，通过命令行和OpenCode对话调用使用。

#### 2.2.2 优点

| 优点 | 说明 |
|------|------|
| **功能完整** | 使用BMAD全部功能，无功能缺失 |
| **官方支持** | 获得官方更新和维护 |
| **稳定可靠** | 经过大量用户验证，稳定成熟 |
| **一键安装** | `npx bmad-method install` 即可完成 |
| **完整文档** | 官方提供详细文档和社区支持 |

#### 2.2.3 缺点

| 缺点 | 说明 |
|------|------|
| **Node.js依赖** | 需要Node.js环境（≥20.0.0） |
| **项目级安装** | 每个项目需要单独安装 |
| **配置较多** | 配置文件较多，学习曲线稍陡 |

#### 2.2.4 适用人群

| 适合 | 不适合 |
|------|--------|
| ✅ 需要完整功能 | ❌ 临时快速使用 |
| ✅ 长期使用 | ❌ Node.js环境不可用 |
| ✅ 愿意学习配置 | ❌ 只需要简单功能 |
| ✅ 小团队/个人开发者 | ❌ 不想安装任何依赖 |

### 2.3 方案B：Skill方式

#### 2.3.1 方案概述

### 3.4 常见安装问题解决

#### 3.4.1 Node.js版本问题

**问题**：安装失败，提示Node.js版本不足

**解决方法**：
```bash
# 检查当前版本
node -v

# 如果低于20，使用nvm切换版本
nvm install 20
nvm use 20

# 验证
node -v
```

#### 3.4.2 NPM权限问题

**问题**：安装失败，提示权限不足

**解决方法**（Windows）：
```powershell
# 以管理员身份运行PowerShell
# 或修改npm全局目录
npm config set prefix "C:\Users\<用户名>\AppData\Roaming\npm"
```

**解决方法**（Linux/macOS）：
```bash
# 创建全局目录
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'

# 添加到PATH
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

#### 3.4.3 Git配置问题

**问题**：安装过程中Git相关操作失败

**解决方法**：
```bash
# 检查Git配置
git config --list

# 配置用户信息（如果未配置）
git config --global user.name "您的名字"
git config --global user.email "your@email.com"
```

#### 3.4.4 网络问题

**问题**：NPM下载超时或失败

**解决方法**：
```bash
# 使用淘宝镜像
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g bmad-method

# 或配置npm镜像
npm config set registry https://registry.npmmirror.com
```

OpenCode Skills支持两个位置：

| 位置 | 说明 | 推荐 |
|------|------|------|
| **全局** | `~/.config/opencode/skills/` | ✅ 推荐 |
| **项目级** | `./.opencode/skills/` | 用于特定项目 |

#### 4.2.2 步骤2：克隆或创建Skill

**方式1：克隆官方BMAD-METHOD（需要转换）**

```bash

cd ~/.config/opencode/skills/

# 克隆BMAD-METHOD
git clone https://github.com/bmad-code-org/BMAD-METHOD.git bmad-method
```

**方式2：创建简化版Skill（推荐用于快速使用）**

```bash
# 进入Skills目录
cd ~/.config/opencode/skills/

# 创建Skill目录
mkdir bmad-method
cd bmad-method

# 创建核心SKILL.md文件
cat > SKILL.md << 'EOF'
---
name: BMAD-METHOD
description: 敏捷AI驱动开发框架，提供结构化工作流程和专业Agent支持
---
```

## 六、卸载方法

### 6.1 方案A卸载（直接使用BMAD-METHOD）

#### 6.1.1 完整卸载步骤

**步骤1：备份重要文件（可选）**

如果需要保留BMAD生成的内容，请先备份：

```bash
# 备份BMAD生成的重要文档
cp -r docs/prd.md backup/
cp -r docs/architecture.md backup/
cp -r docs/stories/ backup/
```

**步骤2：删除BMAD核心文件**

**Windows（PowerShell）**：

```powershell
# 进入项目目录
cd D:\2bktest\MDview

# 删除BMAD核心目录
Remove-Item -Recurse -Force .bmad-core

# 删除OpenCode配置文件
Remove-Item opencode.jsonc

# 删除node_modules（可选，如果只安装了bmad）
Remove-Item -Recurse -Force node_modules

# 删除package-lock.json（可选）
Remove-Item package-lock.json

# 清理npm缓存（可选）
npm cache clean --force
```

**Windows（CMD）**：

```cmd
cd D:\2bktest\MDview
rmdir /s /q .bmad-core
rmdir /s /q node_modules
del opencode.jsonc
del package-lock.json
```

**Linux/macOS**：

```bash
# 进入项目目录
cd D:\2bktest\MDview

# 删除BMAD核心目录
rm -rf .bmad-core

# 删除OpenCode配置文件
rm opencode.jsonc

# 删除node_modules（可选）
rm -rf node_modules

# 删除package-lock.json（可选）
rm package-lock.json

# 清理npm缓存（可选）
npm cache clean --force
```

**步骤3：验证卸载**

```bash
# 检查.bmad-core目录是否还存在
ls -la | grep bmad

# 应该没有输出表示成功删除

# 验证BMAD命令不可用
npx bmad-method --version

# 应该显示错误：command not found
```

#### 6.1.2 部分卸载

如果只想卸载BMAD但保留某些配置：

```bash
# 只删除BMAD核心配置，保留opencode.jsonc
rm -rf .bmad-core

# 手动编辑opencode.jsonc，删除BMAD相关配置
```

#### 6.1.3 重新安装

如果卸载后需要重新安装：

```bash
# 1. 清理残留文件
rm -rf .bmad-core node_modules package-lock.json

# 2. 重新安装
npx bmad-method install

# 3. 验证安装
npx bmad-method --version
```

## 七、常见问题解答（FAQ）

### 7.1 安装相关问题

**Q1: 安装时Node.js版本不足怎么办？**

A1: 请使用nvm切换到Node.js 20+版本：

```bash
# 安装nvm（如果没有）
winget install CoreyButler.NVM.Windows

# 安装并使用Node.js 20
nvm install 20
nvm use 20

# 验证
node -v
```

**Q2: 安装过程中出现权限错误怎么办？**

A2: 检查npm权限或使用管理员模式：

```bash
# 方法1：使用管理员模式运行终端
# Windows: 右键管理员运行PowerShell
# macOS/Linux: sudo

# 方法2：修改npm全局目录
npm config set prefix "~/npm-global"
echo 'export PATH=~/npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

**Q3: 安装完成后BMAD命令不可用怎么办？**

A3: 按以下步骤排查：

```bash
# 1. 检查.bmad-core目录是否存在
ls -la .bmad-core

# 2. 尝试重新安装
npx bmad-method install

# 3. 检查opencode.jsonc配置
cat opencode.jsonc

# 4. 重启OpenCode
```

**Q4: 选择了错误的IDE怎么办？**

A4: 重新运行安装命令选择正确的IDE：

```bash
npx bmad-method install
```

| **获取帮助** | `/bmad-help` | 系统引导 |

### 8.2 最小命令集（必知必会）

```
# 1. 快速开发（最常用）
quick-dev

# 2. 需求分析（第二常用）
quick-spec

# 3. 代码审查（完成开发后）
code-review

# 4. 获取帮助（不确定时）
/bmad-help

# 5. 查看状态（每日开始）
workflow-status
```

### 8.3 每日工作流程

```
每日开始：
1. workflow-status  → 查看今日任务
2. quick-dev       → 开始开发
3. dev-story       → 执行用户故事
4. code-review     → 代码审查
5. 重复步骤2-4     → 完成下一个任务
```

### 8.4 每周工作流程

```
每周一：
1. sprint-planning  → 规划本周任务

每日：
1. workflow-status  → 查看进度
2. quick-dev       → 开发功能
3. code-review     → 代码审查

每周五：
1. sprint-status   → 总结本周
2. retrospective   → 回顾改进
```
