## 3 注册机制详细对比

### 3.1 Hermes 注册机制

**实际代码路径**: `OmniAgentAs-desk/backend/app/services/tools/registry.py`
**注意**: 当前代码库中 Hermes 已合并到 OmniAgentAs-desk，注册机制统一。

**注册方式**：纯函数式 `registry.register()` + 废弃的 `@register_tool()` 装饰器

**`register()` 方法签名**（共 15 个参数）：

```python
def register(
    self,
    name: str,                                              # 必填
    description: str,                                       # 必填
    category: ToolCategory,                                 # 必填
    implementation: Callable,                               # 必填
    version: str = "1.0.0",                                 # 可选
    dependencies: Optional[List[str]] = None,               # 可选
    input_model: Optional[Type[BaseModel]] = None,          # 可选
    input_schema: Optional[Dict] = None,                    # 可选
    output_schema: Optional[Dict] = None,                   # 可选
    examples: Optional[List[Dict]] = None,                  # 可选
    expose_to_llm: bool = True,                             # 可选
    next_actions: Optional[Dict[str, Any]] = None,          # 可选
    failure_hint_fn: Optional[Callable] = None,             # 可选
    safety_level: ToolSafetyLevel = ToolSafetyLevel.SAFE,  # 可选 (v3.4 新增)
    action_safety_map: Optional[Dict[str, ToolSafetyLevel]] = None,  # 可选
) -> Dict[str, Any]:
```

**关键修正**: 15个参数（4必填+11可选），非13个。

**遗留的 `@register_tool()` 装饰器**（第 244-290 行）：仍然定义但**不再使用**，最后一批工具在 2026-05-02 重构期间已迁移至显式调用。

### 3.3 OpenCode 注册机制

**架构特征**（基于 Hathora 函数式架构）：

| 维度 | 说明 |
|------|------|
| **核心类型** | `Tool`（不可变记录）、`ToolDefinition`（序列化格式）、`PluginTool`（插件形式） |
| **注册方式** | `Tool.make()` 工厂方法（4个重载）+ `newTool()` 便捷函数 |
| **Schema** | Zod 对象 + `JsonSchema7Type` 转换 |
| **插件工具** | `PluginTool` 从 `plugin.json` 读取，通过 `pluginToolDefToDefinition()` 转换 |
| **LLM 格式化** | `lowerTool()` → OpenAI tool_choice 格式 |
| **运行时派发** | `Effect.forEach(tools, runTool)` → 并行执行 |
| **工具来源** | 内建 `built-in/` + 插件 `plugins/` + 用户自定义 |

**Tool 数据结构**：

```typescript
interface Tool {
  name: string
  description: string
  parameters: JsonSchema7Type  // Zod 编译后的 JSON Schema
  hides?: string[]  // 运行时隐藏参数（如 sessionId）
  permit?: boolean  // 是否需用户确认
}
```

**框架关键函数**：

| 函数 | 位置 | 功能 |
|------|------|------|
| `Tool.make()` | `packages/core/src/tool.ts` | 工厂，4个重载创建 Tool |
| `newTool()` | `packages/core/src/tool.ts` | 便捷函数包装 Tool.make |
| `lowerTool()` | `packages/core/src/tool.ts` | Tool → OpenAI格式 |
| `runTool()` | `packages/core/src/tool.ts` | 执行工具+截断结果 |
| `pluginToolDefToDefinition()` | `packages/core/src/tool.ts` | 插件工具 → ToolDefinition |
| `getToolDefinitions()` | `packages/core/src/tool.ts` | 运行时获取所有工具定义 |

## 5 安全体系对比

### 5.1 Hermes / OmniAgent 安全体系（ToolSafetyLevel）

**5 级安全等级**（定义于 `tool_types.py:60-70`）：

| 级别 | value | needs_confirmation | needs_safety_check | 示例 |
|------|-------|-------------------|-------------------|------|
| READ_ONLY | `"read_only"` | false | false | read_text_file |
| SAFE | `"safe"` | false | false | write_text_file |
| DESTRUCTIVE | `"destructive"` | true | true | set_env, file_operation(delete) |
| DANGEROUS_SANDBOX | `"dangerous_sandbox"` | true | true | execute_code |
| DANGEROUS | `"dangerous"` | true | true | execute_shell_command, kill_process |

**action_safety_map 机制**：工具可对不同 action 设置不同安全级别（例如 `file_operation` 的 read=SAFE, delete=DESTRUCTIVE）。

**执行检查链**：
1. `ToolExecutor.execute()` → 检查 safety_level
2. 需要确认 → 发送确认请求
3. 需要安全检查 → `command_security.py` 黑名单检查
4. 通过后执行 → 返回结果

### 5.2 OpenCode 安全体系

无显式安全等级系统。安全性通过以下方式保障：

| 机制 | 说明 |
|------|------|
| **permit 字段** | 部分工具标记 `permit: true` 需用户确认 |
| **Effect 隔离** | 函数式 Effect 确保无副作用泄漏 |
| **MCP 协议** | 通过子进程运行 MCP server，sandbox 隔离 |

## 7 工具加载时机对比

### 7.1 OmniAgent 懒加载

**流程**：
```
请求到达 → Agent.__init__() → _init_tools_and_executor() → ensure_tools_registered()
                                                                    ↓
                                                         懒加载器按需导入7个模块
                                                                    ↓
                                                         每个模块执行注册函数
                                                                    ↓
                                                         标记 _registered_categories
```

**关键特征**：
- 首次触发时全量加载（非按需加载具体工具）
- 使用 `_registered_categories: set` 防止重复注册
- 启动时不加载工具，仅加载 registry 本身

### 7.2 OpenCode 按需加载

**流程**：
```
CLI 启动 → 初始化 Effect Runtime
              ↓
   用户输入 → intent 分析
              ↓
   确定需要的工具 → 加载对应 tool module
              ↓
   构建 ToolDefinition 列表
              ↓
   发送给 LLM → 执行工具
```

**关键特征**：
- 真正的按需加载（只加载用户需要的工具）
- 插件工具在插件启用时注册

2.1 OmniAgentAs-desk — 显式调用 + Pydantic自动Schema
模式: 每个分类的 _register.py 调用 tool_registry.register()

tool_registry.register(
    name="read_text_file",
    description="读取文本文件内容",
    category=ToolCategory.FILE,
    implementation=FileTools().read_text_file,
    input_model=ReadTextFileInput,   # ← Pydantic V2 BaseModel

关键特性:

2.2 Hermes — 模块级自注册 + AST发现

def discover_builtin_tools():

2.3 OpenCode — Plugin Hooks + Effect-TS
模式: 插件通过 Hooks.tool 返回工具映射

// plugin.ts
export interface Hooks {
  tool?: { [key: string]: ToolDefinition }
  "tool.execute.before"?: (input, output) => Promise<void>
  "tool.execute.after"?: (input, output) => Promise<void>
  "tool.definition"?: (input, output) => Promise<void>

// 工具定义 — Effect-TS Schema
Tool.make({
  description: "Execute shell command",
  parameters: Schema.Struct({ command: Schema.String }),
  success: Schema.Struct({ output: Schema.String }),

纯函数风格: Effect-TS 类型安全，编译时检查

Zod兼容: 插件作者可用 Zod 定义工具参数
