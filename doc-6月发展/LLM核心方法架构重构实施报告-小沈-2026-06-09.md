# LLM核心方法架构重构实施报告

**版本**: v1.0 | **创建时间**: 2026-06-09 14:31:33 | **作者**: 小沈

---

## 一、重构目标

将旧架构（4个方法按名字分模式）重构为新架构（3个方法按mode参数分模式）。

### 1.1 旧架构（已删除）

```
Service层：
  ├─ chat() → chat_with_tools(tools=None)
  ├─ chat_with_tools() → chat_with_tools_stream()
  └─ chat_with_tools_stream() → 直接调API

SDK层：
  ├─ chat() → 非流式
  └─ chat_stream() → 流式
```

**问题**：
- 方法数量多（4个）
- 模式按方法名区分，扩展需新增方法
- 请求体构建逻辑分散
- Service层直接调API，未使用SDK层

### 1.2 新架构（当前）

```
Service层（BaseAIService）：
  ├─ request(messages, mode, tools, ...)      # 非流式
  ├─ request_stream(messages, mode, tools, ...) # 流式
  └─ chat(message, history)                   # 便捷方法

SDK层（LLMClient）：
  ├─ request(messages, mode, tools, ...)      # 非流式
  └─ request_stream(messages, mode, tools, ...) # 流式
```

**优势**：
- 方法数量少（3个）
- 模式按mode参数区分，扩展只需加mode值
- 请求体构建集中在`_build_request_body()`
- Service层调用SDK层，职责清晰

```
SDK层（LLMClient）:
  ├─ request(messages, mode, tools, ...)      # 非流式
  └─ request_stream(messages, mode, tools, ...) # 流式

Service层（BaseAIService）:
  ├─ request(messages, mode, tools, ...)      # 非流式 → 调SDK.request()
  ├─ request_stream(messages, mode, tools, ...) # 流式 → 调SDK.request_stream()
  └─ chat(message, history)                   # 便捷 → request_stream(mode="text")

Agent（UniversalAgent）:
  ├─ _call_llm_fc_stream() → request_stream(mode="tools")  # 流式FC
  ├─ _call_llm_text_stream() → request_stream(mode="text")  # 流式text
  └─ _call_llm_text_nostream() → request(mode="text")       # 降级用
```

**修改`_build_request_body()`**：
```python
def _build_request_body(
    messages: List[Dict],
    model: str,
    mode: str = "text",                    # 新增mode参数
    max_tokens: Optional[int] = None,
    temperature: Optional[float] = None,
    seed: Optional[int] = None,
    tools: Optional[List[Dict]] = None,
    tool_choice: Optional[str] = None,
    stream: bool = False,
) -> Dict:
    body = {"model": model, "messages": messages}
    if max_tokens is not None:
        body["max_tokens"] = max_tokens
    if temperature is not None:
        body["temperature"] = temperature
    if seed is not None:
        body["seed"] = seed
    if stream:
        body["stream"] = True
    if mode == "tools" and tools:          # mode控制tools
        body["tools"] = tools
        if tool_choice:
            body["tool_choice"] = tool_choice
    return body
```

### 第3步：修改Service层

**文件**: `backend/app/services/llm_core/llm_core.py`

**删除**：

from app.services.llm_core.tool_caller import ToolCallerMixin

**新增**：
```python
class BaseAIService:
    """通用AI服务 — request/request_stream/chat + mode参数"""
    
    async def request(
        self,
        messages: List[Dict],
        mode: str = "text",
        tools: Optional[List[Dict]] = None,
        tool_choice: str = "auto",
    ) -> ChatResponse:
        """非流式请求 - Agent用"""
        self._ensure_client()
        response = await self._llm_sdk.request(
            messages=messages,
            mode=mode,
            tools=tools,
            tool_choice=tool_choice,
            max_tokens=self.max_tokens,
            temperature=self.temperature,
            seed=self.seed,
        )
        # 解析响应...
        return ChatResponse(...)
    
    async def request_stream(
        self,
        messages: List[Dict],
        mode: str = "text",
        tools: Optional[List[Dict]] = None,
        tool_choice: str = "auto",
    ) -> AsyncGenerator[StreamChunk, None]:
        """流式请求 - SSE服务层/Agent用"""
        self.reset_cancel()
        self._ensure_client()
        
        async for data_str in self._llm_sdk.request_stream(...):
            if self._cancelled:
                yield self._create_cancelled_chunk()
                return
            chunk = self._parse_sse_data(data_str)
            if chunk:
                yield chunk
                if chunk.is_done:
                    return
    
    async def chat(
        self,
        message: str,
        history: Optional[List[Dict]] = None,
    ) -> AsyncGenerator[StreamChunk, None]:
        """流式对话便捷方法 - SSE服务层用"""
        messages = self._build_messages(message, history)
        async for chunk in self.request_stream(messages=messages, mode="text"):
            yield chunk
    
    def _build_messages(self, message: str, history: Optional[List[Dict]] = None) -> List[Dict]:
        """构建消息列表"""
        messages = []
        if history:
            for msg in history:
                if isinstance(msg, dict):
                    messages.append(msg)
                else:
                    messages.append({"role": msg.role, "content": msg.content})
        messages.append({"role": "user", "content": message})
        return messages
```

## 四、最终架构

### 4.1 调用关系

```
前端SSE
  ↓
chat_router.py
  ↓
run_sse_stream.py
  ↓
agent.run_react_cycle()
  ↓
agent._call_llm()
  ↓
agent._call_llm_fc_stream()
  ↓
llm_client.request_stream(mode="tools")    ← 新方法
  ↓
SDK.request_stream()
  ↓
httpx.stream()
```

### 4.2 方法清单

| 层 | 方法 | 参数 | 返回 | 说明 |
|---|------|------|------|------|
| **SDK** | `request()` | messages, mode, tools, ... | Dict | 非流式统一入口 |
| **SDK** | `request_stream()` | messages, mode, tools, ... | AsyncGenerator | 流式统一入口 |
| **Service** | `request()` | messages, mode, tools, ... | ChatResponse | 非流式，调SDK |
| **Service** | `request_stream()` | messages, mode, tools, ... | AsyncGenerator[StreamChunk] | 流式，调SDK |
| **Service** | `chat()` | message, history | AsyncGenerator[StreamChunk] | 便捷，调request_stream |
| **Agent** | `_call_llm_fc_stream()` | messages, tools | AsyncGenerator | 流式FC |
| **Agent** | `_call_llm_text_stream()` | messages | AsyncGenerator | 流式text |
| **Agent** | `_call_llm_text_nostream()` | messages | str | 降级用 |

### 4.3 mode参数

| mode值 | 说明 | 请求体 |
|--------|------|--------|
| `"text"` | 纯文本 | `{"model": ..., "messages": ...}` |
| `"tools"` | Function Calling | `{"model": ..., "messages": ..., "tools": [...]}` |

## 五、遵守的10大原则

| 原则 | 遵守情况 | 说明 |
|------|---------|------|
| **SRP** | ✅ | SDK层只管发HTTP请求，Service层只管业务逻辑，职责分离 |
| **DRY** | ✅ | 统一`_build_request_body()`和`_build_messages()`，消除重复 |
| **KISS** | ✅ | 3个方法+1个参数，比4个方法更简单 |
| **SLAP** | ✅ | SDK层和Service层在同一抽象层，不混搭 |
| **YAGNI** | ✅ | 只实现当前需要的功能，不预留未来扩展 |
| **禁止backward compatibility** | ✅ | 彻底删除旧代码，不保留任何兼容代码 |
| **OCP** | ✅ | 新增mode值即可扩展新模式，不修改现有代码 |
| **LSP** | ✅ | 新方法完全替代旧方法，行为一致 |
| **ISP** | ✅ | 接口职责单一，每个方法只做一件事 |
| **复用优先** | ✅ | 复用现有`_build_request_body()`逻辑 |

## 七、优势总结

| 维度 | 旧架构 | 新架构 |
|------|--------|--------|
| 方法数量 | 4个（含死代码） | 3个（chat + request + request_stream） |
| 模式选择 | 按方法名（chat_with_xxx） | 按mode参数 |
| 扩展新模式 | 新增方法+文件 | 加一个mode值 |
| 消息构建 | 分散在各方法 | 集中在`_build_messages()` |

| Service调SDK | ❌ 未使用 | ✅ 统一使用 |
