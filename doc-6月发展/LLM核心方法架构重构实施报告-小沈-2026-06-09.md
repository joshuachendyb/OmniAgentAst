# LLM核心方法架构重构实施报告

**版本**: v1.2 | **创建时间**: 2026-06-09 14:31:33 | **更新时间**: 2026-06-14 12:03:55 | **作者**: 小沈

---

## 一、重构目标

将旧架构（4个方法按名字分模式）重构为新架构（3个方法，FC-only，mode参数已删除）。

### 1.1 旧架构（已删除）

```
Service层：
  ├─ chat() → chat_with_tools(tools=None)
  ├─ chat_with_tools() → chat_with_tools_stream()
  └─ chat_with_tools_stream() → 直接调API

SDK层：
  ├─ chat() → 非流式
  └─ chat_stream() → 流式
```

### 1.2 当前架构（FC-only 重构后 — 小沈 2026-06-11）

**FC-only 变更**（2026-06-11）：
- `mode` 参数从 SDK/Service/Agent 全部删除
- tools 注入条件从 `mode == "tools"` 改为 `tools is not None`
- Agent 仅保留 `_call_llm_fc_stream()`，删除 text 降级路径
- `tool_calls` 以原生 dict 在 ReAct 循环中传递，不再 JSON roundtrip

| SDK 方法 | 实际 HTTP 调用 | 对应接口 |
|----------|---------------|---------|
| `LLMClient.request()` | `self._client.post(...)` + `response.json()` | HTTP整包请求 |
| `LLMClient.request_stream()` | `self._client.stream(...)` + SSE行解析 | HTTP流式请求 |

```
SDK层（LLMClient）:
  ├─ request(messages, tools, tool_choice)           # 非流式
  └─ request_stream(messages, tools, tool_choice)    # 流式 (yield SSE data str)

Service层（BaseAIService）:
  ├─ request(messages, tools, tool_choice)           # 非流式 → 调SDK.request()
  ├─ request_stream(messages, tools, tool_choice)    # 流式 → 调SDK.request_stream(), yield StreamChunk
  └─ chat(message, history)                          # 便捷 → build_llm_messages() + request_stream()

Agent（UniversalAgent）:
  └─ _call_llm_fc_stream(messages, openai_tools)     # 流式FC → request_stream(tools=openai_tools), yield ("chunk"|"response")
```

**优势**：
- 方法数量少（3个）
- FC-only 消除 JSON roundtrip，`tool_calls` 原生传递
- 请求体构建集中在`_build_request_body()`
- Service层调用SDK层，职责清晰
- `_normalize_tool_params()` 集中修复 provider 级 double-encode 问题（2026-06-14）

**关键说明**：
- `mode` 参数已删除，`tools` 参数直接决定是否注入 tools
- `request_stream()` 负责底层 HTTP SSE 流式请求，yield 原始 SSE data 字符串
- 旧方法 `chat()` / `chat_stream()` 已全部删除

**_build_request_body() 当前代码**：

```python
def _build_request_body(
    messages: List[Dict],
    model: str,
    max_tokens: Optional[int] = None,
    temperature: Optional[float] = None,
    seed: Optional[int] = None,
    tools: Optional[List[Dict]] = None,
    tool_choice: Optional[str] = None,
    stream: bool = False,
) -> Dict:
    body = {"model": model, "messages": messages}
    if max_tokens is not None:
        body["max_tokens"] = max_tokens
    if temperature is not None:
        body["temperature"] = temperature
    if seed is not None:
        body["seed"] = seed
    if stream:
        body["stream"] = True
    if tools:                                # tools is not None 时注入（无 mode 参数）
        body["tools"] = tools
        if tool_choice:
            body["tool_choice"] = tool_choice
    return body
```

### 2.3 Service层：BaseAIService

**文件**: `backend/app/services/llm_core/llm_core.py`

**当前代码**（FC-only，v1.1 重构后）：

```python
class BaseAIService:
    """通用AI服务 — request/request_stream/chat（FC-only）"""

    async def request(
        self,
        messages: List[Dict],
        tools: Optional[List[Dict]] = None,
        tool_choice: str = "auto",
    ) -> ChatResponse:
        """非流式请求 - Agent用"""
        self._ensure_client()
        response = await self._llm_sdk.request(
            messages=messages, tools=tools, tool_choice=tool_choice,
        )
        return ChatResponse(
            content=response.get("content", ""),
            tool_calls=response.get("tool_calls"),
        )

    async def request_stream(
        self,
        messages: List[Dict],
        tools: Optional[List[Dict]] = None,
        tool_choice: str = "auto",
    ) -> AsyncGenerator[StreamChunk, None]:
        """流式请求 - yield StreamChunk"""
        self.reset_cancel()
        self._ensure_client()
        async for data_str in self._llm_sdk.request_stream(
            messages=messages, tools=tools, tool_choice=tool_choice,
        ):
            if self._cancelled:
                yield self._create_cancelled_chunk()
                return
            chunk = self._parse_sse_data(data_str)
            if chunk:
                yield chunk
                if chunk.is_done:
                    return

    async def chat(
        self, message: str,
        history: Optional[List[Dict]] = None,
    ) -> AsyncGenerator[StreamChunk, None]:
        """流式对话便捷方法"""
        from app.utils.message_builder import build_llm_messages
        messages = build_llm_messages(message, history)
        async for chunk in self.request_stream(messages=messages):
            yield chunk
```

**关键说明**：
- `mode` 参数已从 `request()` / `request_stream()` 签名中删除
- `_build_messages()` 已删除，统一使用 `message_builder.build_llm_messages()`
- `_parse_sse_data()` 将 SDK yield 的 SSE data 字符串解析为 `StreamChunk`（tool_calls 以原生 dict 传递）
- `_create_cancelled_chunk()` 用于取消场景
- `_normalize_tool_params()` 修复 provider 级 tool 参数 double-encode 问题（2026-06-14 新增）
- 旧 mixin 继承（`ChatWithToolsStreamMixin`, `ToolCallerMixin`）已全部删除

### 2.4 Agent层：UniversalAgent

**文件**: `backend/app/services/agent/universal_agent.py`

**当前代码**（FC-only，仅 `_call_llm_fc_stream()`，无 text 降级路径）：

```python
class UniversalAgent(BaseAgent, ReactAgentMixin):
    async def _call_llm(self) -> AsyncGenerator[Tuple[str, Any], None]:
        """入口：FC-only，委托 _call_llm_fc_stream"""
        async for item in self._call_llm_fc_stream(...):
            yield item

    async def _call_llm_fc_stream(
        self, messages, openai_tools
    ) -> AsyncGenerator[Tuple[str, Any], None]:
        """流式FC — 唯一LLM调用方法"""
        async for chunk in self.llm_client.request_stream(
            messages=messages, tools=openai_tools, tool_choice="auto",
        ):
            if self._cancelled:
                yield "chunk", ChatCacheChunk(is_done=True, content="")
                return
            yield "chunk", ChatCacheChunk(...)
            if chunk.is_done and chunk.tool_calls:
                yield "response", ChatResponse(content=..., tool_calls=chunk.tool_calls)
```

**关键说明**：
- `_call_llm_text_stream()` 和 `_call_llm_text_nostream()` 已删除
- `_call_llm()` 为顶层入口（被 `_process_step()` 调用），直接调用 `_call_llm_fc_stream()`
- 不再传 `mode="tools"`，直接传 `tools=openai_tools`
- `tool_calls` 以原生 dict 在 ReAct 循环中传递，无 JSON roundtrip
- `llm_content`（2026-06-14 G1 修复）：`request_stream` yield 的 chunk 包含 content 字段，不再丢弃

## 三、最终架构（FC-only，2026-06-11）

### 3.1 调用关系

```
前端SSE
  ↓
chat_router.py
  ↓
run_sse_stream.py
  ↓
agent.run_react_cycle()
  ↓
agent._call_llm()                           ← 入口：FC-only
  ↓
agent._call_llm_fc_stream(messages, tools)  ← 唯一LLM调用方法
  ↓
llm_client.request_stream(tools=openai_tools, tool_choice="auto")  ← 无 mode 参数
  ↓
SDK.request_stream(messages, tools, tool_choice)
  ↓
_build_request_body() → httpx.stream()
```

### 3.2 方法清单（FC-only）

| 层 | 方法 | 参数 | 返回 | 说明 |
|---|------|------|------|------|
| **SDK** | `request()` | messages, tools, tool_choice | Dict | 非流式统一入口 |
| **SDK** | `request_stream()` | messages, tools, tool_choice | AsyncGenerator[str] | 流式，yield SSE data |
| **Service** | `request()` | messages, tools, tool_choice | ChatResponse | 非流式，调SDK |
| **Service** | `request_stream()` | messages, tools, tool_choice | AsyncGenerator[StreamChunk] | 流式，解析data为chunk |
| **Service** | `chat()` | message, history | AsyncGenerator[StreamChunk] | 便捷方法 |
| **Service** | `_normalize_tool_params()` | tools | List[Dict] | 修复provider double-encode |
| **Agent** | `_call_llm()` | 无 | AsyncGenerator | 顶层入口 |
| **Agent** | `_call_llm_fc_stream()` | messages, tools | AsyncGenerator | 唯一LLM调用（FC） |

> ✅ `_call_llm_text_stream()` 和 `_call_llm_text_nostream()` 已删除

### 3.3 mode参数（已删除）

| 状态 | 说明 |
|------|------|
| ❌ `mode` 参数 | 2026-06-11 全部删除 |
| ✅ | 改为 `tools is not None` 判断是否注入 tools |
| ✅ | 无 tools 时的纯文本请求走同样的 request_stream() 路径 |

## 四、遵守的10大原则

| 原则 | 遵守情况 | 说明 |
|------|---------|------|
| **SRP** | ✅ | SDK层只管发HTTP请求，Service层只管业务逻辑，职责分离 |
| **DRY** | ✅ | 统一`_build_request_body()`和`build_llm_messages()`，消除重复 |
| **KISS** | ✅ | FC-only 3个方法，比旧架构4个方法+mode参数更简单 |
| **SLAP** | ✅ | SDK层和Service层在同一抽象层，不混搭 |
| **YAGNI** | ✅ | 只实现当前需要的功能，不预留未来扩展（已删除 text 降级路径） |
| **禁止backward compatibility** | ✅ | 彻底删除旧代码，不保留任何兼容代码（旧文件+mode参数均已删除） |
| **OCP** | ✅ | 新增provider只需加 tool_params 配置，不修改核心逻辑 |
| **LSP** | ✅ | 新方法完全替代旧方法，行为一致 |
| **ISP** | ✅ | 接口职责单一，每个方法只做一件事 |
| **复用优先** | ✅ | 复用`_build_request_body()`、`build_llm_messages()`等公用函数 |
