### 1.2 关键发现 - 核心问题

**🔴 P0 - Agent核心ReAct循环是占位方法**

```python
# base_react/run_stream.py:16-21
async def run_stream(self, task, context, max_steps, task_id):
    """占位循环 - mixin 体系已删除,业务逻辑待重构"""
    yield {"type": "not_implemented", "step": step_count}
```

**影响**：
- ❌ 所有Agent实际执行返回"not_implemented"
- ❌ 业务逻辑已删除但未重建
- ❌ 当前系统无法正常工作

**🟡 P1 - generate()内部import 7个模块在闭包里**

```python
# chat_stream_v2.py:44-98
async def generate():
    while True:
        from app.services.tools.registry import tool_registry  # 每次循环都import
        ...
```

**每次请求都重复执行import**，浪费性能。

### 2.3 评分公式验证

**有类型分**: `final[cat] = type_score + type_score × compat × 0.3`

**示例**: 用户输入"读取文件"
- 类型分: FILE = 3.0 (中文"文件"+2.0, 英文"file"+1.0)
- 动作分: read = 1.0
- 合成: final[FILE] = 3.0 + 3.0 × 1.5 × 0.3 = 4.35
- 归一化: 1.0 - 2^(-4.35) = 0.9488 ✅ 正确
