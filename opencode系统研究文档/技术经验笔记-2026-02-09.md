## 二、核心技术场景：Windows PC远程操作Linux硬件盒子

### 2.1 问题的本质
- **场景**: Windows PC电脑需要远程操作Linux系统的硬件盒子
- **任务**: 软件安装、调试、测试、配置修改
- **痛点**: 来来回回的手动操作、重复指令、等待响应

### 2.2 主要技术方案

#### 方案一：SSH连接（最常用）
```
# 基础连接
ssh username@硬件盒子IP

# 带端口连接
ssh -p 2222 username@192.168.1.100

# 常用操作
sudo apt-get install 软件名
systemctl restart 服务名
tail -f /var/log/syslog
```

#### 方案二：远程桌面（GUI界面）
```
# Linux端安装
sudo apt-get install xrdp

# Windows端
mstsc → 输入Linux盒子IP
```

#### 方案三：USB连接（现在最常用！）
```
# USB转串口或USB直接连接
# Windows设备管理器识别为COM端口
# 使用PuTTY/MobaXterm连接COM端口

# 使用方法：
1. 硬件盒子通过USB连接到PC
2. Windows自动识别为COM设备（如COM3）
3. 打开串口工具（PuTTY、MobaXterm）
4. 选择对应的COM端口
5. 设置波特率（115200、9600等）
6. 连接后获得Linux shell

# 用户补充：以前确实是串口的，现在基本上都是USB了
```

#### 方案四：网络文件共享
```
# Windows映射网络驱动器
net use Z: \\Linux盒子IP\share

# 或者使用Samba/SFTP传输文件
```

#### 方案五：自动化脚本
```
# Windows批处理脚本
@echo off
ssh username@ip "command1 && command2"

# 或使用Bash脚本
```

### 2.3 操作流程示例
```
Windows PC端：
1. 打开PuTTY/命令行
2. ssh连接到Linux盒子
3. 执行安装命令：sudo apt install XXX
4. 检查日志：journalctl -u 服务名
5. 重启服务：systemctl restart XXX
6. 验证功能
```

### 2.4 重复操作带来的烦躁
- 每次都要重新输入连接命令
- 软件安装步骤多且容易忘记顺序
- 调试日志需要反复查看
- 错误时需要从头开始
