## 一、提交前检查

### 1.1 代码质量

- [ ] 函数签名是否合理
- [ ] 异常处理是否完善
- [ ] 边界条件是否处理
- [ ] 类型注解是否正确

### 1.2 测试覆盖

- [ ] 新功能是否有测试用例
- [ ] 测试用例是否覆盖正常情况
- [ ] 测试用例是否覆盖异常情况
- [ ] 测试是否通过

### 1.3 文档同步

- [ ] 设计文档是否更新
- [ ] 版本记录是否更新
- [ ] 注释是否完整

---

## 二、功能集成检查

### 2.1 新功能检查

- [ ] 功能是否真的被调用
- [ ] 调用位置是否正确
- [ ] 是否有遗漏的导入

### 2.2 重构检查

- [ ] 旧代码是否删除
- [ ] 向后兼容是否保持
- [ ] 测试是否更新

---

## 三、质量门禁

### 3.1 自动检查

```bash
# 运行测试
python -m pytest tests/ -v

# 代码检查
python -m pylint app/

# 类型检查
python -m mypy app/
```

### 3.2 手动检查

- [ ] 阅读修改的代码
- [ ] 检查调用链
- [ ] 验证功能是否正常

---

## 四、问题追踪

### 4.1 发现问题

- 记录问题：文件、行号、问题描述
- 分类：P0/P1/P2/P3
- 分配：自己修复

### 4.2 修复问题

- 修复代码
- 编写测试
- 验证修复
- 更新文档

### 4.3 闭环确认

- 测试通过
- 文档更新
- 版本记录

---

## 五、持续改进

### 5.1 每周回顾

- 检查是否有遗漏的功能
- 检查是否有未使用的代码
- 检查是否有质量问题

### 5.2 月度总结

- 统计代码质量指标
- 分析问题趋势
- 制定改进计划
