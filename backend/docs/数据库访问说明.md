## 二、数据库表结构

### 2.1 conversations 表（会话表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | TEXT (PK) | 会话唯一标识（UUID） |
| title | TEXT | 会话标题 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| user_id | TEXT | 用户ID |
| model | TEXT | 使用的AI模型 |
| tags | TEXT | 会话标签（JSON格式） |

**示例数据**：
```sql
INSERT INTO conversations VALUES (
  'b5ffec3c-1c18-4d78-af20-7fba42e06733',
  '记录1003',
  '2026-03-12 08:30:00',
  '2026-03-12 08:35:00',
  'user_001',
  'gpt-4',
  '["work", "file-management"]'
);
```

### 2.2 messages 表（消息表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER (PK) | 消息唯一标识（自增） |
| conversation_id | TEXT (FK) | 关联的会话ID |
| role | TEXT | 角色：user/assistant/system |
| content | TEXT | 消息内容 |
| timestamp | TIMESTAMP | 消息时间戳 |
| execution_steps | TEXT | 执行步骤（JSON格式） |
| model | TEXT | 使用的AI模型 |
| is_streaming | BOOLEAN | 是否流式传输 |
| is_error | BOOLEAN | 是否错误消息 |
| display_name | TEXT | 显示名称 |

**execution_steps 字段说明**：
- **类型**: JSON 数组
- **结构**: 每个步骤是一个对象，包含 `type`、`content` 等字段
- **常用 type 值**：
  - `thought`: 思考过程
  - `action_tool`: 工具执行
  - `observation`: 执行结果
  - `chunk`: 流式传输的中间结果
  - `final`: 最终回复内容
  - `error`: 错误信息
  - `paused/resumed/interrupted/retrying`: 状态变化

**示例数据**：
```sql
INSERT INTO messages VALUES (
  337,
  'b5ffec3c-1c18-4d78-af20-7fba42e06733',
  'assistant',
  '这是AI的完整回复内容...',
  '2026-03-12 08:31:00',
  '[{"type":"thought","content":"思考中..."},{"type":"chunk","content":"部分回复..."}]',
  'gpt-4',
  0,
  0,
  'GPT-4'
);
```

## 四、重要字段说明

### 4.1 execution_steps 字段（关键字段）

**用途**：存储AI执行过程的详细步骤

**结构示例**：
```json
[
  {
    "type": "thought",
    "content": "用户想要查找E盘文件目录..."
  },
  {
    "type": "action_tool",
    "tool_name": "list_files",
    "action_description": "列出E盘文件",
    "tool_params": {"path": "E:\\"}
  },
  {
    "type": "observation",
    "result": {"entries": [...]}
  },
  {
    "type": "chunk",
    "content": "这是流式传输的中间结果...",
    "is_reasoning": false
  },
  {
    "type": "final",
    "content": "这是AI的最终回复内容..."
  }
]
```

**type 类型说明**：
- `thought`: AI的思考过程
- `action_tool`: AI调用工具执行操作
- `observation`: 工具执行结果
- `chunk`: 流式传输的中间回复内容
- `final`: AI的最终回复内容（完整）
- `error`: 错误信息
- `paused/resumed/interrupted/retrying`: 状态变化

### 4.2 content 字段

**用途**：存储消息的完整内容

**注意**：
- 对于AI消息，`content` 字段可能包含完整回复
- 但前端渲染时优先使用 `execution_steps` 中的 `chunk` 或 `final` 步骤
- 如果 `execution_steps` 中没有 `chunk`，才会回退显示 `content`

## 五、常见问题

### 5.1 数据库文件位置错误

**问题**：前端无法获取历史消息

**原因**：数据库路径配置错误

**解决**：检查 `C:\Users\40968\.omniagent\chat_history.db` 是否存在

### 5.2 execution_steps 数据异常

**问题**：AI消息显示为空或不完整

**原因**：
1. `execution_steps` 中只有 `final` 步骤，没有 `chunk` 步骤
2. 前端渲染逻辑优先显示 `chunk`，忽略 `final`
3. `content` 字段可能为空

**解决**：修改前端渲染逻辑，支持显示 `final` 步骤

```python
def get_ai_messages_with_steps(session_id):
    """查询会话中所有AI消息及其execution_steps详情"""
    conn = get_conn()
    cursor = conn.cursor()
    cursor.execute(
        '''SELECT id, content, execution_steps FROM chat_messages 
           WHERE session_id = ? AND role = 'assistant' AND execution_steps IS NOT NULL
           ORDER BY id''',
        (session_id,)
    )
    rows = cursor.fetchall()
    
    results = []
    for row in rows:
        steps = json.loads(row[2]) if row[2] else []
        chunks = [s for s in steps if s.get('type') == 'chunk']
        finals = [s for s in steps if s.get('type') == 'final']
        
        results.append({
            'id': row[0],
            'content_length': len(row[1]) if row[1] else 0,
            'chunk_count': len(chunks),
            'chunk_total_length': sum(len(c.get('content','')) for c in chunks),
            'final_count': len(finals),
            'final_length': len(finals[0].get('content','')) if finals else 0,
            'first_chunk': chunks[0].get('content','')[:30] if chunks else None
        })
    
    conn.close()
    return results
```

#### 7.3.4 查找chunk不完整的消息

```python
def find_incomplete_chunks(session_id):
    """查找chunk数量异常少的消息（可能数据不完整）"""
    conn = get_conn()
    cursor = conn.cursor()
    cursor.execute(
        '''SELECT id, content, execution_steps FROM chat_messages 
           WHERE session_id = ? AND role = 'assistant' AND execution_steps IS NOT NULL
           ORDER BY id''',
        (session_id,)
    )
    rows = cursor.fetchall()
    
    results = []
    for row in rows:
        steps = json.loads(row[2]) if row[2] else []
        chunks = [s for s in steps if s.get('type') == 'chunk']
        finals = [s for s in steps if s.get('type') == 'final']
        
        # chunk数量少于10个，或者chunk总长度远小于final长度，视为异常
        content_len = len(row[1]) if row[1] else 0
        chunk_total = sum(len(c.get('content','')) for c in chunks)
        final_len = len(finals[0].get('content','')) if finals else 0
        
        if len(chunks) < 10 or (final_len > 0 and chunk_total < final_len * 0.5):
            results.append({
                'id': row[0],
                'content_length': content_len,
                'chunk_count': len(chunks),
                'chunk_total_length': chunk_total,
                'final_length': final_len,
                'first_chunk': repr(chunks[0].get('content','')[:50]) if chunks else None
            })
    
    conn.close()
    return results
```

```python
# -*- coding: utf-8 -*-
"""
会话消息批量分析脚本
使用方法: python analyze_session.py <session_id>
"""
import sqlite3
import json
import sys

DB_PATH = r'C:\Users\40968\.omniagent\chat_history.db'

def analyze_session(session_id):
    """分析会话中的所有消息"""
    conn = sqlite3.connect(DB_PATH)
    cursor = conn.cursor()
    
    # 获取会话信息
    cursor.execute('SELECT id, title, message_count FROM chat_sessions WHERE id = ?', (session_id,))
    session = cursor.fetchone()
    if not session:
        print(f'未找到会话: {session_id}')
        conn.close()
        return
    
    print('='*60)
    print(f'会话ID: {session[0]}')
    print(f'会话标题: {session[1]}')
    print(f'消息总数: {session[2]}')
    print('='*60)
    
    # 获取AI消息统计
    cursor.execute(
        '''SELECT id, content, execution_steps FROM chat_messages 
           WHERE session_id = ? AND role = 'assistant' AND execution_steps IS NOT NULL
           ORDER BY id''',
        (session_id,)
    )
    rows = cursor.fetchall()
    
    print(f'\n共{len(rows)}条AI消息有execution_steps:')
    print('-'*60)
    
    normal_count = 0
    abnormal_count = 0
    
    for row in rows:
        steps = json.loads(row[2]) if row[2] else []
        chunks = [s for s in steps if s.get('type') == 'chunk']
        finals = [s for s in steps if s.get('type') == 'final']
        
        content_len = len(row[1]) if row[1] else 0
        chunk_count = len(chunks)
        chunk_total = sum(len(c.get('content','')) for c in chunks)
        final_len = len(finals[0].get('content','')) if finals else 0
        
        # 判断是否正常：chunk数量>10 且 chunk总长度接近final长度
        is_normal = chunk_count > 10 and (final_len == 0 or chunk_total >= final_len * 0.5)
        
        if is_normal:
            normal_count += 1
        else:
            abnormal_count += 1
            print(f'消息ID: {row[0]}')
            print(f'  content长度: {content_len}')
            print(f'  chunk数量: {chunk_count}, chunk总长度: {chunk_total}')
            print(f'  final长度: {final_len}')
            print(f'  第一个chunk: {repr(chunks[0].get("content","")) if chunks else "无"}')
            print()
    
    print('-'*60)
    print(f'统计: 正常{normal_count}条, 异常{abnormal_count}条')
    
    conn.close()

if __name__ == '__main__':
    if len(sys.argv) > 1:
        analyze_session(sys.argv[1])
    else:
        print('使用方法: python analyze_session.py <session_id>')
```

## 八、常见问题排查

### 8.1 消息显示为空（空气泡）

**排查步骤**：
1. 查询消息的execution_steps
2. 检查chunk数量是否正常（正常应该>10）
3. 检查是否有final步骤
4. 对比content长度和chunk/final长度

**SQL命令**：
```bash
# 查询指定消息的chunk和final详情
python -c "
import sqlite3
import json

DB_PATH = r'C:\Users\40968\.omniagent\chat_history.db'
conn = sqlite3.connect(DB_PATH)
cursor = conn.cursor()
cursor.execute('SELECT id, content, execution_steps FROM chat_messages WHERE id = 417')
row = cursor.fetchone()
if row[2]:
    steps = json.loads(row[2])
    chunks = [s for s in steps if s.get('type') == 'chunk']
    finals = [s for s in steps if s.get('type') == 'final']
    print(f'消息ID: {row[0]}')
    print(f'chunk数量: {len(chunks)}, final数量: {len(finals)}')
    if chunks:
        print(f'第一个chunk: {repr(chunks[0].get(\"content\",\"\"))}')
    if finals:
        print(f'final长度: {len(finals[0].get(\"content\",\"\"))}')
conn.close()
"
```
