## 一、就两种：同步 vs 异步

| 类型 | 调用者行为 |
|------|----------|
| 同步 | 站着等结果 |
| 异步 | 不等，继续干别的，完事告诉我 |

**大白话**：
- 同步 = 顺序执行,前面的结果影响后面的执行
- 异步 = 不是顺序执行的, 不等结果,让第三方自己执行,也不要需要第三方的执行结果

**异步分两种**：

| 类型 | 比喻 | 例子 |
|------|------|------|
| **不要结果**（fire-and-forget） | 公众号发文章，发完不管 | 标题生成、写日志、发通知 |
| **要结果**（fire-and-collect） | 点外卖，骑手送过来 | `await task`、读 channel、回调里取结果 |

**要结果时的 3 种拿法**：
- `await task`（单个任务）
- `for event := range channel`（流式取）
- `on_done(result)`（回调里取）

## 二、同步的 3 种执行模式

执行模式 = 函数被调用的方式。同步有 3 种，**本质上都是"我站着等你干完"**。

| # | 模式 | 同步/异步 | 一句话 | 例子 |
|---|------|----------|--------|------|
| 1 | Inline（内联） | 同步 | 写在一起不分函数 | 大段逻辑堆在一个 while 循环里 |
| 2 | 函数调用 | 同步 | 调子函数 | `def call_llm(); call_llm()` |
| 3 | 注入（Injection） | 同步 | 从外面传进来 | `__init__(self, llm)` 构造函数注入 |

### 2.1 函数调用的 3 种写法

| 写法 | 说明 |
|------|------|
| 直接函数调用 | 最普通的 `f()` |
| 链式调用 | `a.b().c()` |
| 接口多态 | `interface.F()`，运行时决定谁执行 |

## 三、异步的 5 种执行模式

异步有 5 种，**本质上都是"我不等，完事告诉我"**。**5 种方法本质上都是"把结果异步地传给调用方"**——fire-and-forget 是省略"拿结果"那一步的特殊用法。

| # | 模式 | 同步/异步 | 要/不要结果 | 一句话 | 例子 |
|---|------|----------|------------|--------|------|
| 4 | 回调（Callback） | 异步 | **要结果**（默认） | 完事叫我 | `on_done(r)` 里 r 就是结果 |
| 5 | Hook（钩子） | 异步 | **要结果** | 到点通知 | `_fire(event, data)` 把 data 传出去 |
| 6 | 生成器（Yield） | 异步 | **要结果** | 做一点还给你 | `yield v` 必被 `for` 消费 |
| 7 | 消息/事件 | 异步 | **要结果** | 丢管道里 | `channel <- event` 把 event 传出去 |
| 8 | 远程调用（RPC） | 看实现 | **要结果** | 跨机器调 | `r = rpc.call()` 拿返回值 |

**fire-and-forget 是这 5 种的"省结果"特殊用法**：

| 方法 | 正常用法（要结果） | fire-and-forget（不要结果） |
|------|------------------|--------------------------|
| 回调 | `on_done(r)` 拿 r | `on_done=None` 或空函数 |
| Hook | `_fire("on_step", data)` | 注册空 handler |
| 生成器 | `for chunk in gen():` | 没人消费 yield |
| 消息/事件 | `for e := range chan` | 没人读 channel |
| 远程调用 | `r = http.post()` | 不接返回值 |

**为什么大部分异步是 fire-and-forget**？
- 通知类：发完用户看到就行
- 日志/统计：丢给后台自己处理
- 不知道啥时候完：第三方慢得很，主流程等不了

### 3.6 Python 实例：5 种方法 × 要结果/不要结果对比

#### 1. 回调（Callback）

```python
# ✅ 要结果
def task(on_done):
    time.sleep(1)
    on_done({"status": "ok", "data": 42})   # 传结果

task(lambda r: print(f"结果是 {r}"))        # 输出: 结果是 {'status': 'ok', 'data': 42}

# ❌ 不要结果（fire-and-forget）
def task(on_done):
    time.sleep(1)
    on_done({"status": "ok", "data": 42})   # 还是传了

task(lambda: None)                          # 不接收，丢出去就算
# 或
task(lambda r: None)                        # 拿了不用
```

#### 2. Hook（钩子）

```python
class Agent:
    def __init__(self):
        self.hooks = {"on_step": []}
    def on(self, event, handler):
        self.hooks[event].append(handler)
    def _fire(self, event, data):
        for h in self.hooks[event]:
            h(data)

agent = Agent()

# ✅ 要结果
def save_to_db(data):
    db.write(data)              # data 拿来用
agent.on("on_step", save_to_db)

# ❌ 不要结果
def do_nothing(data):
    pass                       # data 拿了不用
agent.on("on_step", do_nothing)
```

#### 3. 生成器（Yield）

```python
# ✅ 要结果
def gen():
    for i in range(3):
        yield i                 # 产出数据

for v in gen():                 # 外面在消费
    print(v)                    # 0 1 2

# ❌ 不要结果
def gen():
    for i in range(3):
        yield i

gen()                           # 调用了但没人 for-range 消费，yields 出来就被回收
```

#### 4. 消息/事件（Channel / Pub/Sub）

```python
import queue
q = queue.Queue()

# ✅ 要结果
q.put({"task": "save", "data": 42})
result = q.get()                # 拿出来用
print(result)                   # {'task': 'save', 'data': 42}

# ❌ 不要结果
q.put({"task": "save", "data": 42})   # 丢进去就完事，没人 get
```

#### 5. 远程调用（HTTP）

```python
import requests

# ✅ 要结果
r = requests.post("https://api.example.com/data", json={"q": 1})
data = r.json()                 # 拿返回数据
print(data["answer"])           # 用结果

# ❌ 不要结果
requests.post("https://api.example.com/notify", json={"msg": "hi"})
# 不接返回值，丢出去就算
```

**差异核心**：

```
要结果    = 异步干完 + 调用方拿数据用
不要结果  = 异步干完 + 调用方不管
```

**关键观察**：
- "要结果" 永远是**多一步拿结果**的代码
- 异步机制本身（coroutine/channel/HTTP）不带这个开关
- "要不要结果"是**调用方的选择**，跟异步机制无关

## 四、底层实现机制（每种模式底下用啥跑）

8 种执行模式只是"代码长什么样"，底下用啥机制跑是另一回事。**一个模式可以对应多种机制**。

**底层机制 5 种**：
- **goroutine/协程**（轻量协程）
- **goroutine + channel**（协程 + 通道）
- **Pub/Sub**（事件总线）
- **线程**（OS 线程）
- **进程**（独立进程，IPC）

| 异步模式 | 常用实现机制 |
|---------|-------------|
| 回调 | 线程 / 事件循环 + 任务队列 |
| Hook | 事件循环 + 任务队列 |
| 生成器 | 协程（async/await） |
| 消息/事件 | goroutine + channel / 消息队列 |
| 远程调用 | 阻塞调用（线程）/ 异步非阻塞（asyncio） |

**一句话**：代码模式是"你看到的写法"，实现机制是"代码底下怎么跑"。

## 五、其他模式（注入/反射/eval）

| 模式 | 同步/异步 | 说明 |
|------|----------|------|
| 依赖注入 | 同步 | 构造函数注入 interface，调用时还是同步等 |
| 反射 | 看实现 | `reflect.Call()` 本身是同步 |
| eval/exec | 看实现 | 执行字符串代码，同步阻塞 |
| Plugin 加载 | 看实现 | 加载后调用看具体实现 |

**注**：依赖注入本质上就是 §二.3 注入（Injection），反射/eval 是"用字符串/类型调用代码"的特殊形式。

## 七、OpenCode 实际案例

OpenCode 的 agent 共用了 5 种执行模式：

| 模式 | 同步/异步 | 例子 |
|------|----------|------|
| 直接函数调用 | 同步 | `a.messages.List()`、`a.sessions.Get()` |
| 接口多态调用 | 同步 | `a.provider.StreamResponse()`（OpenAI/Anthropic/Gemini 不同实现） |
| 注入 | 同步 | `NewAgent(sessions, messages, tools)` 构造函数注入 |
| goroutine+channel | 异步 | `go func() { events <- ... }()` + `for event := range eventChan` |
| Pub/Sub 事件 | 异步 | `a.Publish(pubsub.CreatedEvent, result)` |

**周边基础设施的执行模式**：

| 功能 | 同步/异步 | 执行方式 |
|------|----------|---------|
| DB 操作 | 同步 | inline 直接调 `a.messages.List(ctx, sessionID)` |
| 标题生成 | 异步 | fire-and-forget goroutine |
| Summarize 历史压缩 | 异步 | 异步 goroutine |

## 八、错误理解纠正

| 错误 | 正确 |
|------|------|
| 接口调用是异步 | 同步的，只是背后谁干活不知道 |
| 依赖注入是异步 | 同步的，调用时还是等结果 |
| 消息队列 put() 是异步 | put() 本身是同步的，但整个"丢+处理"是异步 |
| 回调是同步的 | 注册同步，**执行**异步 |
| 远程调用一定是同步 | 看实现，可以是同步阻塞也可以是异步非阻塞 |

## 九、各语言关键词速查

| 语言 | 同步（默认） | 异步方式 |
|------|------------|---------|
| Python | `f()` | `async/await`、`threading.Thread()` |
| Go | `f()` | `go func()`、`ch <-`、`<-ch` |
| JS | `f()` | `async/await`、`setTimeout(f)`、`Promise` |
| Java | `f()` | `new Thread(f)`、`CompletableFuture` |

**规则**：没关键字 = 同步；有关键字 = 异步。
