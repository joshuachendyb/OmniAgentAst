### 1.3 UI优化建议

#### 1.3.1 视觉层次优化

**当前问题**:
- 所有step类型使用相似的布局，视觉区分度不够
- 徽章颜色虽然有区分，但整体视觉层次不够清晰
- 工具参数和结果显示混在一起，缺少视觉分组

**优化建议**:
```
优化方案1: 分层视觉设计
├── 顶层: 步骤类型徽章 (最醒目)
├── 中层: 工具名称+状态 (次醒目)
└── 底层: 参数和结果详情 (最弱)

优化方案2: 状态指示器
├── 成功: 绿色徽章 + ✅图标
├── 进行中: 蓝色徽章 + 转圈动画
├── 失败: 红色徽章 + ❌图标
└── 待处理: 灰色徽章
```

#### 1.3.2 交互体验优化

**当前问题**:
- 展开/折叠功能依赖外部Map状态，逻辑复杂
- 加载更多功能实现重复
- 鼠标hover效果不够明显

**优化建议**:
```
优化方案: 统一交互组件
├── StepContainer: 统一的步骤容器组件
│   ├── 内置展开/折叠状态
│   ├── 内置加载更多逻辑
│   └── 内置hover效果
└── StepContent: 按类型区分的内容组件
    ├── ToolStepContent: 工具调用步骤
    ├── ThoughtStepContent: 思考步骤
    ├── ResultStepContent: 结果步骤
    └── ErrorStepContent: 错误步骤
```

#### 1.3.3 性能优化

**当前问题**:
- StepRow每次渲染都执行getStepBadgeStyle、getStepLabelStyle等函数
- 工具结果显示通过renderToolResult渲染大量组件
- 没有使用React.memo优化

**优化建议**:
```typescript
// 使用useMemo缓存样式
const badgeStyle = useMemo(
  () => getStepBadgeStyle(effectiveType),
  [effectiveType]
);

// 拆分为独立组件避免整体重渲染
const ActionToolResult = memo(({ step, isExpanded, ... }) => {
  // 独立渲染逻辑
});
```

## 三、UI与拆分综合建议

### 3.1 优先级排序

| 优先级 | 优化项 | 工作量 | 收益 |
|--------|--------|--------|------|
| **P0** | MessageItem拆分为目录 | 中 | 高 |
| **P0** | StepRow独立文件 | 低 | 中 |
| **P1** | 视图组件独立文件 | 中 | 中 |
| **P1** | SSE回调拆分为hook | 中 | 中 |
| **P2** | Step UI样式优化 | 低 | 中 |
| **P2** | 交互体验优化 | 中 | 低 |

### 3.2 实施风险评估

| 优化项 | 风险 | 缓解措施 |
|--------|------|----------|
| MessageItem拆分 | 高 - 改动大 | 分步实施，先拆视图 |
| StepRow独立 | 低 - 接口稳定 | 直接迁移 |
| SSE回调拆分 | 中 - 状态复杂 | 保留原逻辑，逐步抽取 |

### 3.3 不建议的操作

1. **不要在重构时同时修改业务逻辑** - 保持功能不变
2. **不要删除现有的注释** - 保留历史信息
3. **不要改变组件的props接口** - 保持向后兼容
