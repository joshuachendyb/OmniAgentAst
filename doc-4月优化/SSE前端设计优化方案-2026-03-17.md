### 1.2 核心知识点提炼

#### 1.2.1 SSE协议基础

- **单工通信**：服务器向客户端单向推送数据，基于HTTP长连接
- **EventSource API**：浏览器原生支持，自动重连，事件ID追踪
- **消息格式**：`data: 内容\n\n`，支持多行data、event、id、retry字段
- **自动重连**：浏览器自动实现，支持Last-Event-ID用于断点续传

#### 1.2.2 React中SSE实现模式

- **自定义Hook模式**：封装useSSE管理连接状态和事件处理
- **Context模式**：提供SSEProvider全局共享连接
- **useReducer vs useState**：流式数据场景推荐useReducer处理复杂状态转换
- **ReadableStream**：fetch API配合getReader()实现自定义流处理

#### 1.2.3 重试策略最佳实践

- **指数退避**：delay = baseDelay * 2^attempt，避免服务器压力
- **抖动（Jitter）**：添加随机性防止“惊群效应”，推荐Full Jitter策略
- **最大延迟限制**：设置maxDelay防止等待过长
- **可重试错误分类**：5xx服务器错误、429限流可重试，4xx客户端错误不可重试

#### 1.2.4 连接保活策略

- **心跳机制**：服务器定期发送空消息或注释维持连接
- **代理超时**：Nginx默认60秒超时，需配置proxy_read_timeout
- **云平台限制**：AWS Lambda 15秒、Vercel 10-60秒、Google Cloud Run 60秒
- **X-Accel-Buffering: no**：禁用代理缓冲保持连接活跃

#### 1.2.5 AI/LLM流式响应优化

- **TTFT（Time To First Token）**：首 token 响应时间关键指标
- **令牌级流式**：逐 token 推送，用户体验最佳
- **分块传输**：多个 token 合并为 chunk 减少网络开销
- **思考过程流式**：is_reasoning 字段区分思考和正式内容

#### 1.2.6 生产环境实践

- **连接池管理**：服务器端维护客户端连接映射
- **Redis Pub/Sub**：多实例间跨服务器广播
- **优雅关闭**：客户端断开时清理资源，服务器端通知
- **监控指标**：连接数、消息速率、错误率、断线率

### 3.2 具体差距分析

#### 3.2.1 重试策略差距

**当前实现**：
```typescript
const calculateReconnectDelay = (attempt: number, baseDelay: number, maxDelay: number): number => {
  const delay = Math.min(baseDelay * Math.pow(2, attempt), maxDelay);
  return delay + Math.random() * 1000; // 简单抖动
};
```

**最佳实践**：
- 缺少最大重试次数限制
- 抖动策略不完整（应区分Full Jitter、Equal Jitter、Decorrelated Jitter）
- 缺少对Retry-After头的响应
- 缺少对不同错误类型的重试策略区分

#### 3.2.2 心跳机制差距

**当前实现**：无心跳机制

**最佳实践**：
- 服务器应每30秒发送心跳维持连接
- 前端检测心跳超时触发重连
- 处理代理和云平台60秒超时限制

#### 3.2.3 状态管理差距

**当前实现**：useState管理5个独立状态

**最佳实践**：
- 流式数据场景推荐useReducer
- 可用useSyncExternalStore或专门流式库
- Vercel AI SDK的useChat hook是成熟方案

#### 3.2.4 性能指标差距

**当前实现**：无性能监控

**最佳实践**：

#### 4.2.1 P1：增强重试策略

**目标**：实现完整的指数退避+Full Jitter+错误分类重试策略

**实现方案**：
```typescript
// 新增重试配置接口
interface RetryConfig {
  maxRetries: number;          // 最大重试次数（默认5）
  baseDelay: number;          // 基础延迟（默认1000ms）
  maxDelay: number;           // 最大延迟（默认30000ms）
  jitter: 'full' | 'equal' | 'decorrelated';  // 抖动策略
  retryableStatuses: number[]; // 可重试的HTTP状态码
  retryableErrors: string[];   // 可重试的错误类型
}

// Full Jitter计算公式
const calculateFullJitter = (baseDelay: number, attempt: number, maxDelay: number): number => {
  const exponentialDelay = baseDelay * Math.pow(2, attempt);
  const jitter = Math.random() * exponentialDelay;
  return Math.min(jitter, maxDelay);
};

// 错误分类重试决策
const shouldRetry = (error: SSEError): boolean => {
  // 5xx服务器错误可重试
  if (error.status >= 500) return true;
  // 429限流可重试
  if (error.status === 429) return true;
  // 网络错误可重试
  if (error.error_type === 'network' || error.error_type === 'timeout') return true;
  // 4xx客户端错误不重试
  return false;
};
```

**目标**：使用useReducer优化流式数据状态管理

```typescript
// 状态和动作定义
interface SSEState {
  isConnected: boolean;
  isReceiving: boolean;
  executionSteps: ExecutionStep[];
  currentResponse: string;
  reconnectStatus: ReconnectStatus;
}

type SSEAction =
  | { type: 'SET_CONNECTED'; payload: boolean }
  | { type: 'SET_RECEIVING'; payload: boolean }
  | { type: 'ADD_STEP'; payload: ExecutionStep }
  | { type: 'APPEND_CHUNK'; payload: string }
  | { type: 'SET_RESPONSE'; payload: string }
  | { type: 'CLEAR' }
  | { type: 'SET_RECONNECT_STATUS'; payload: ReconnectStatus };

// Reducer
function sseReducer(state: SSEState, action: SSEAction): SSEState {
  switch (action.type) {
    case 'ADD_STEP':
      return {
        ...state,
        executionSteps: [...state.executionSteps, action.payload]
      };
    case 'APPEND_CHUNK':
      return {
        ...state,
        currentResponse: state.currentResponse + action.payload
      };
    // ... 其他case
    default:
      return state;
  }
}

// 使用
const [state, dispatch] = useReducer(sseReducer, initialState);

// 分发动作（自动批处理）
dispatch({ type: 'APPEND_CHUNK', payload: chunkContent });
```

#### 4.2.5 P5：消息协议标准化

**目标**：添加event、id、retry字段支持

**实现方案**：
```typescript
// 标准化消息格式
interface SSEMessage {
  event: string;           // 事件类型：start/thought/chunk/final/error/heartbeat
  id?: string;             // 消息ID（用于断点续传）
  data: any;               // 消息数据
  retry?: number;          // 重试间隔（毫秒）
}

// 服务器端发送格式
// event: start
// id: 1700000000001
// data: {"type": "start", ...}
//
// event: chunk
// id: 1700000000002
// data: {"type": "chunk", "content": "Hello"}

// 客户端解析
const parseSSEMessage = (line: string): SSEMessage | null => {
  let event: string | undefined;
  let id: string | undefined;
  let data: string | undefined;
  let retry: number | undefined;

  const parts = line.split('\n');
  for (const part of parts) {
    if (part.startsWith('event:')) event = part.slice(6).trim();
    if (part.startsWith('id:')) id = part.slice(3).trim();
    if (part.startsWith('data:')) data = part.slice(5).trim();
    if (part.startsWith('retry:')) retry = parseInt(part.slice(6), 10);
  }

  if (!data) return null;
  return { event: event || 'message', id, data: JSON.parse(data), retry };
};
```
