## 一、问题诊断

### 1.1 性能瓶颈分析

**核心问题**:
1. **Ant Design完整引入** - 961KB未压缩JS文件
2. **NewChatContainer组件过大** - 2000+行，20+个useState
3. **初始加载逻辑复杂** - 串行操作多，等待时间长
4. **缺少按需加载** - 全量引入，无代码分割

**数据指标**:
- `vendor-antd-BMsy5BsE.js`: 961KB（最大文件）
- `index-BVWirKC0.js`: 168KB（主应用）
- 总JS体积: ~1.2MB
- 预计加载时间: 2-5秒（10Mbps网络）

### 2.1 Ant Design按需加载（P0 - 最高优先级）

> **【小强修正 2026-04-21】重要说明：**
> `vite-plugin-style-import` 是为 **antd v4** 设计的插件，本项目使用 **antd v5.12**。
> antd v5 采用 CSS-in-JS 方案，**不存在** `antd/es/xxx/style/index` 路径，强行配置此插件会导致 build 报错。
> **antd v5 已原生支持 ES Module Tree-shaking，只要使用具名 import 即可自动按需加载，无需任何额外插件。**

### 2.2 优化初始加载逻辑（P0）

#### 2.2.1 简化loadSession函数

> **【小强修正 2026-04-21】两处问题已修正：**
> 1. `Promise.all` 包裹同步操作（`searchParams.get` / `sessionStorage.getItem` 均为同步）无任何并行效果，改为直接同步调用。
> 2. `display_name` 校验过严：用户刚发消息但 AI 未回复时，消息列表里没有 `display_name`，会误判为无效缓存并丢弃。改为只判断 `messages.length > 0`。

```typescript
// 在NewChatContainer.tsx中修改第1502-1719行的useEffect
useEffect(() => {
  const loadSession = async () => {
    // 1. 立即显示骨架屏，不等待任何异步操作
    setIsMessageListLoading(true);
    
    // 2. 同步读取：searchParams 和 sessionStorage 都是同步 API，
    //    用 Promise.all 包裹毫无并行效果，直接同步取值即可
    const urlSessionId = searchParams.get("session_id");
    const savedState = (() => {
      try {
        const saved = sessionStorage.getItem(STORAGE_KEY);
        if (!saved) return null;
        const state = JSON.parse(saved);
        const currentTime = Date.now();
        const savedTime = state.timestamp || 0;
        
        // 只恢复5分钟内的状态
        if (currentTime - savedTime > SESSION_EXPIRY_TIME) {
          sessionStorage.removeItem(STORAGE_KEY);
          return null;
        }
        
        // 检查缓存消息是否有效（只判断有无消息，不校验 display_name）
        // 【修正】原来判断 hasDisplayName 过严：用户发消息后 AI 未回复时，
        //         消息里没有 display_name，会误丢弃有效缓存
        if (!state.messages || state.messages.length === 0) {
          return null;
        }
        
        return state;
      } catch {
        return null;
      }
    })();
    
    // 3. 优先使用sessionStorage（最快路径）
    if (savedState && !urlSessionId) {
      console.log("🟢 从缓存恢复会话状态，消息数:", savedState.messages?.length);
      setMessages(savedState.messages || []);
      setSessionId(savedState.sessionId);
      currentSessionIdRef.current = savedState.sessionId;
      setSessionTitle(savedState.sessionTitle || "会话");
      
      // 恢复暂停状态
      if (savedState.isPaused !== undefined) {
        setIsPaused(savedState.isPaused);
        isPausedRef.current = savedState.isPaused;
      }
      
      // 立即关闭骨架屏
      setIsMessageListLoading(false);
      setIsInitialized(true);
      return; // 快速返回，不等待API
    }
    
    // 4. 如果有URL参数，加载指定会话（带超时）
    if (urlSessionId) {
      setSessionJumpLoading(true);
      message.loading({
        content: "正在加载会话...",
        key: "session-load",
        duration: 0,
      });
      
      try {
        // 设置5秒超时
        const timeoutPromise = new Promise((_, reject) => 
          setTimeout(() => reject(new Error('加载超时')), 5000)
        );
        
        const loadPromise = loadHistoryMessages(urlSessionId);
        const result = await Promise.race([loadPromise, timeoutPromise]) as any;
        
        if (result) {
          setSessionId(result.sessionId);
          currentSessionIdRef.current = result.sessionId;
          setMessages(result.messages);
          setSessionTitle(result.title);
          if (result.version !== undefined) setSessionVersion(result.version);
          if (result.title_locked !== undefined) setTitleLocked(result.title_locked);
          
          showLoadSuccess("会话加载成功");
        } else {
          // 降级：显示新会话
          setSessionTitle("新会话");
          setMessages([]);
          setSessionId(null);
        }
      } catch (error) {
        console.warn("加载URL会话失败:", error);
        // 降级：显示新会话
        setSessionTitle("新会话");
        setMessages([]);
        setSessionId(null);
        showLoadErrorWithKey("加载会话失败，请检查网络后重试", "session-load");
      } finally {
        setSessionJumpLoading(false);
        message.destroy("session-load");
        setIsMessageListLoading(false);
        setIsInitialized(true);
      }
      return;
    }
    
    // 5. 加载最近会话（延迟执行，让页面先显示）
    setTimeout(async () => {
      try {
        const result = await loadLatestHistoryMessages();
        if (result) {
          setSessionId(result.sessionId);
          currentSessionIdRef.current = result.sessionId;
          setSessionTitle(result.title);
          if (result.version !== undefined) setSessionVersion(result.version);
          if (result.title_locked !== undefined) setTitleLocked(result.title_locked);
          setMessages(result.messages);
        } else {
          // 没有会话，显示新会话
          setSessionTitle("新会话");
          setMessages([]);
          setSessionId(null);
        }
      } catch (error) {
        console.warn("加载最近会话失败:", error);
        // 降级：显示新会话
        setSessionTitle("新会话");
        setMessages([]);
        setSessionId(null);
      } finally {
        setIsMessageListLoading(false);
        setIsInitialized(true);
      }
    }, 100); // 延迟100ms，让骨架屏先显示
    
  };
  
  loadSession();
  
  // 清理函数
  return () => {
    // 清理可能的定时器
    if (waitTimerRef.current) {
      clearInterval(waitTimerRef.current);
    }
  };
}, [searchParams]); // 只依赖searchParams
```

#### 2.3.3 修改NewChatContainer使用拆分组件（正确版本）

> **【小强修正 2026-04-21】`React.lazy()` 反模式已修正：**
> `SessionHeader`（标题区）和 `MessageList`（消息列表）是**首屏必须立即渲染的核心组件**。
> 对它们用 `lazy()` 会导致：页面先显示 Suspense fallback（灰色占位），再替换为真实内容，产生明显闪烁，体验比不懒加载更差。
>
> **`React.lazy()` 正确使用场景**：历史记录弹窗、设置面板、导出功能等**用户点击后才出现**的非首屏内容。

import compression from 'vite-plugin-compression';

export default defineConfig(({ command }) => {
  const isBuild = command === "build";
  
  return {

      react(),

        disable: false,
        threshold: 10240, // 10KB以上才压缩

        ext: '.gz',
      }),
      // 添加brotli压缩（更好的压缩率）
      isBuild && compression({
        verbose: true,
        disable: false,
        threshold: 10240,
        algorithm: 'brotliCompress',
        ext: '.br',
      }),
      // ... 其他插件
    ].filter(Boolean),
    build: {
      // ... 其他配置
      // 添加资源内联阈值
      assetsInlineLimit: 4096, // 4KB以下图片转base64

});

#### 2.5.1 修改index.html添加预加载（正确版本）

> **【小强修正 2026-04-21】硬编码 preload 路径问题已修正：**
> Vite 构建后文件名带 content hash（如 `vendor-react-BMsy5BsE.js`），
> 写死 `/assets/vendor-react.js` 永远无法命中，preload 形同虚设。
>
> Vite 已自动在生成的 `index.html` 中注入 `<link rel="modulepreload">` 标签，无需手动处理。
> 正确做法是：**只内联骨架屏 CSS，消除白屏；不手写硬编码 preload 路径。**

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>OmniAgent</title>
    
    <!--
      ❌ 删除硬编码 preload：
      <link rel="preload" href="/assets/vendor-react.js" as="script" />  ← hash 文件名永远不匹配
      <link rel="preload" href="/assets/vendor-antd.js" as="script" />   ← 同上
      Vite build 后自动注入 <link rel="modulepreload">，无需手动写。
    -->
    
    <!-- 预加载字体 -->
    <link rel="preload" href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&display=swap" as="style" crossorigin />
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
    
    <!-- 预连接关键域名 -->
    <link rel="preconnect" href="https://fonts.googleapis.com" />
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
    
    <!-- DNS预解析 -->
    <link rel="dns-prefetch" href="//your-api-domain.com" />
    
    <!-- 关键CSS内联 -->
    <style>
      /* 关键CSS：骨架屏样式 */
      @keyframes skeleton-loading {
        0% { background-position: 200% 0; }
        100% { background-position: -200% 0; }
      }
      
      /* 初始加载样式 */
      #root {
        min-height: 100vh;
        background: #f5f5f5;
      }
      
      /* 骨架屏基础样式 */
      .skeleton-message {
        height: 60px;
        background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
        background-size: 200% 100%;
        animation: skeleton-loading 1.5s ease-in-out infinite;
        border-radius: 8px;
        margin-bottom: 12px;
        opacity: 0.7;
      }
    </style>
  </head>
  <body>
    <div id="root">
      <!-- 初始骨架屏 -->
      <div style="padding: 20px;">
        <div class="skeleton-message"></div>
        <div class="skeleton-message"></div>
        <div class="skeleton-message"></div>
      </div>
    </div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```
