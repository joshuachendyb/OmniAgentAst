## 一、问题分析

### 1.1 当前问题

| 问题 | 位置 | 影响 |
|------|------|------|
| **每次GET请求都reload** | config.py:734 `get_model_list()` | ⚠️ 启动慢的主要原因 |
| **reload()过于频繁** | 10+处调用 | 不必要的文件IO |

### 1.2 现有配置更新机制

```
配置更新流程：
┌─────────────────────────────────────────────────────┐
│ 1. PUT /config (update_config)                       │
│    ├─ 备份配置文件                                │
│    ├─ 更新配置到YAML                            │
│    ├─ config.reload() ← 更新后正确reload           │
│    └─ 返回成功                                │
└─────────────────────────────────────────────────────┘

配置读取流程（有问题）：
┌─────────────────────────────────────────────────────┐
│ 2. GET /config/models (get_model_list)               │
│    ├─ config.reload() ← ❌ 每次都读文件         │
│    ├─ 解析YAML                                │
│    └─ 返回模型列表                              │
└─────────────────────────────────────────────────────┘
```

### 1.3 根因

当前 `Config` 类的单例模式实现有缺陷：
- `reload()` 会重新调用 `_load_config()` 读取YAML文件
- 每次API请求都调用 `reload()`，导致每次都读文件

## 三、解决方案设计（完善版）

### 3.1 核心思路：缓存 + 变更检测

**关键原则**：
- `读取`操作（GET）：**不强制reload**，自动检测变更
- `更新`操作（PUT）：**强制reload**，清空缓存
- 变更检测：配置文件mtime比较

### 3.2 双保险保护设计

```python
def _load_config(self):
    """加载配置文件 - 带缓存优化（双保险）"""
    config_path = self._get_config_path()
    
    # ⭐ 保险1：文件不存在时处理
    if not config_path.exists():
        raise FileNotFoundError(
            f"配置文件不存在: {config_path}。"
            "请在前端创建配置文件或手动创建 config/config.yaml"
        )
    
    # ⭐ 保险2：时间戳检查
    new_mtime = config_path.stat().st_mtime
    
    # 如果缓存存在且mtime相同，使用缓存
    if self._config_data is not None and self._config_mtime == new_mtime:
        logger.debug("配置文件未变更，使用缓存")
        return
    
    # 加载配置文件
    try:
        with open(config_path, 'r', encoding='utf-8') as f:
            self._config_data = yaml.safe_load(f)
        
        # 配置文件不能为空
        if not self._config_data:
            raise ValueError("配置文件为空，请检查 config/config.yaml")
    except (yaml.YAMLError, ValueError) as e:
        raise RuntimeError(
            f"加载配置文件失败: {e}。"
        )
    
    # 更新mtime
    self._config_mtime = new_mtime
    logger.info(f"配置文件已加载，mtime={new_mtime}")
    
    # 环境变量覆盖
    self._apply_env_overrides()
```

### 3.3 reload()方法保持不变

```python
def reload(self):
    """强制重新加载配置"""
    self._config_data = None
    self._config_mtime = None
    self._load_config()
    logger.info("配置已强制刷新")
```

### 3.4 工作流程图

```
【读取流程 - 带双保险】
┌─────────────────────────────────────────────────────┐
│ GET /config/models 请求                              │
│    ↓                                              │
│ 检查文件是否存在？                                 │
│    ├─ 不存在 → FileNotFoundError                  │
│    └─ 存在 → 获取mtime                          │
│    ↓                                              │
│ 比较 mtime：                                       │
│    ├─ 相同 → 使用缓存 → 返回                      │
│    └─ 不同 → 重新加载YAML                        │
│    ↓                                              │
│ 应用环境变量覆盖                                  │
│    ↓                                              │
│ 返回配置数据                                      │
└─────────────────────────────────────────────────────┘

【更新流程 - 保持不变】
┌─────────────────────────────────────────────────────┐
│ PUT /config (update_config)                         │
│    ↓                                              │
│ 写入YAML文件                                      │
│    ↓                                              │
│ config.reload() → 强制清空缓存                    │
│    ↓                                              │
│ 下次GET时自动重新加载                            │
└─────────────────────────────────────────────────────┘
```
